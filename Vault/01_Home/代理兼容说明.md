# 代理兼容说明

> 记录 Claude Code 和 Codex 在 CyberMe-Vault 中的调用差异，确保两者行为一致。

## 公共规则

两者都读取并遵守 `AGENTS.md`。公共规则只维护一份，不复制、不分叉。

## Claude Code 特征

| 特性 | 说明 |
|------|------|
| 专属规则文件 | `CLAUDE.md`（`@AGENTS.md` 引用公共规则） |
| Plan Mode | 支持，大规模操作默认先计划 |
| Skill 目录 | `.claude/skills/` |
| Agent 目录 | `.claude/agents/` |
| 子代理 | Explore（研究）、Plan（设计）、general-purpose（执行） |
| 权限机制 | 用户通过权限弹窗逐项确认 |

## Codex 特征

| 特性 | 说明 |
|------|------|
| 专属规则文件 | `.codex/instructions.md`（引用 `@AGENTS.md`） |
| Skill 目录 | `.codex/skills/` |
| 权限机制 | 基于 `.codex/permissions.yaml` 预设权限 |

## 行为一致性要求

以下行为两者必须一致：

1. **回答课程问题**：都先读 MOC → 知识笔记 → 来源
2. **文件操作**：都遵守 AGENTS.md 中的文件操作规则
3. **批量操作**：都先预览计划，再获得确认
4. **来源标注**：都区分「资料内容」「用户确认」「模型推断」
5. **原始资料保护**：都不覆盖/删除/移动 `91_Raw-Archive` 和 `学习资料`
6. **敏感信息**：都不读取或保存密码/密钥/Token

## 已知差异

| 方面 | Claude Code | Codex |
|------|------------|-------|
| Plan Mode | 内置支持 | 需手动模拟 |
| 子代理 | Explore/Plan/Agent | 无等效功能（用脚本替代） |
| Skill 格式 | `.md` with YAML frontmatter | `.md` with YAML frontmatter（可共用） |
| 大规模文件遍历 | 使用 Explore 子代理 | 使用 grep/find 脚本 |

## Skill 共用策略

Skill 的核心流程描述（做什么、输入输出、禁止操作）在两者间保持一致。
仅存放位置不同：
- Claude Code：`.claude/skills/`
- Codex：`.codex/skills/`

如需适配，在各自目录下创建同名 skill 文件，但流程规范引用同一来源。
