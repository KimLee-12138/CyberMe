# 备份与恢复说明

## 三层保护策略

| 层级 | 位置 | 用途 | 频率 |
|---|---|---|---|
| L1 本地 | `CyberMe-Vault/` | 日常使用 | 持续 |
| L2 同步副本 | 待定（OneDrive/坚果云等） | 跨设备同步与灾难恢复 | 自动同步 |
| L3 离线备份 | 待定（移动硬盘/U盘） | 不可逆损坏的最后防线 | 每周手动 |

## 重要警告

- **同步不等于备份。** 同步工具（如 OneDrive、坚果云）会在本地删除时同步删除到云端。删除操作的传播是不可逆的。
- **备份必须离线。** 至少保留一份不自动同步的离线副本。
- **备份必须定期验证。** 每次备份后至少抽查 5 个文件能否正常打开。

## 原始资料保护规则

### `91_Raw-Archive` 是逻辑只读区

以下操作**禁止**对 `91_Raw-Archive` 执行：
- 覆盖已有归档文件
- 删除归档文件（即使有"更好版本"）
- 重命名归档文件
- 直接编辑归档文件内容

以下操作**允许**：
- 新增归档副本（复制，不移动原件）
- 读取和计算哈希
- 更新登记卡（Registry）中的元数据

### 代理权限约束

AI 代理（Codex、Claude Code 等）**不得**：
- 覆盖或删除 `91_Raw-Archive` 中的任何文件
- 移动或重命名原始工作区（`学习资料/`）中的任何文件
- 在未经用户确认的情况下批量修改知识笔记

## 备份内容

每次备份应包含：
- [ ] 全部 Markdown 文件（知识笔记、配置、模板）
- [ ] `.gitignore`、`AGENTS.md`、`CLAUDE.md`
- [ ] `.claude/` 和 `.codex/` 目录下的 Skill 和 Agent 配置
- [ ] 目录结构（保留空目录）

以下内容**不进入备份**（由独立流程管理）：
- `91_Raw-Archive/` 中的大体积原始文件（单独归档备份）
- `.git/` 目录（Git 本身就是版本历史）
- 排除规则中的敏感文件

## 恢复流程

1. 将备份压缩包解压到临时目录。
2. 使用 Obsidian 打开临时目录，确认为有效 Vault。
3. 检查首页、当前任务、备份与恢复说明可正常打开。
4. 抽查至少 5 个 Markdown 文件的 YAML frontmatter 和正文完整性。
5. 确认目录结构与第 2.1 节一致。
6. 记录恢复演练结果到本文件。
