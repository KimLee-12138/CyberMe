# CyberMe 知识复习与检索系统

> **核心定位：** 这是一套以「快速检索 + 高效复习」为目标的知识库。不是 AI 训练数据，不是个人日记——而是你和他人能随时查阅、快速定位、高效复习的数据库课程知识系统。

## 使用场景

- **考前复习**：从 MOC 出发，按章节 → 知识点 → 例题逐层深入
- **快速检索**：用关键词（如"BCNF""并发异常""SQL连接"）搜索，直达目标笔记
- **错题回顾**：查看易错点和高频错误模式
- **公式速查**：进入公式与规则目录，快速对照
- **给他人讲解**：每篇笔记都有一句话解释，适合快速建立概念框架

## 系统目标（第一版）

- 能按课程、章节、资料类型和关键词找到数据库资料。
- 任取一篇正式知识笔记，可以回到原始文件及具体页码或章节。
- 数据库课程有可用的 MOC，覆盖基础、关系模型、SQL、范式、设计、安全、完整性和事务。
- 系统能依据资料解释知识，不把网上常识或模型记忆伪装成你的资料。
- 系统知道你的部分学习偏好和写作要求，但不把 AI 推断写成事实。
- Codex 与 Claude Code 都遵守同一套公共规则。
- 有固定评测题、评分标准、错误记录和复测流程。
- 有至少一份可验证的备份，并完成一次恢复演练。

## 不可违反的原则

1. **原始资料不可变。** `91_Raw-Archive` 只接收复制件，不直接编辑原件。
2. **正式结论必须可追溯。** 知识笔记必须填写来源、页码或明确的用户说明。
3. **事实与推断分离。** 用户明确确认的内容使用 `verification: explicit`；AI 推断使用 `verification: inferred`。
4. **找不到就说明找不到。** 禁止为了让答案完整而补写貌似合理的内容。
5. **冲突以证据为准。** 记录冲突并优先核对原始证据。
6. **敏感信息不入库。** 密码、API Key、Token、身份证号、银行卡等不得进入 Vault 或 Git。
7. **质量优先于数量。** 以来源完整率、抽查准确率和实际可用性衡量。
8. **先跑通样板，再批量扩展。** 数据库课程未通过验收，不复制流程到其他课程。
9. **当前阶段不训练模型。** 先用可维护的知识库、规则与反馈闭环解决问题。
