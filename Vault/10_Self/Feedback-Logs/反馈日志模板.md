# 反馈日志

> 使用方法：每次对 AI 回答有明确「满意」或「不满意」时记录一条。
> 同类错误重复 **两次** 后才考虑升级为长期规则。

## 反馈模板

```markdown
---
id: feedback-YYYYMMDD-0001
type: feedback
status: open
created: YYYY-MM-DD
updated: YYYY-MM-DD
---

# AI 回答反馈

## 原任务
（当时让 AI 做什么？）

## 使用的代理与模型
（Claude Code / Codex / 其他）

## 原回答或结果位置
（可以链接到具体文件或对话）

## 我满意的地方
（哪些做得好？为什么好？）

## 我不满意的地方
（哪些做得不好？为什么不好？）

## 我的修改
（你自己动手改了什么？）

## 这是偶发问题还是重复模式
- [ ] 偶发——仅保留案例
- [ ] 第1次出现——标记，下次再出现时评估升级
- [ ] 第2次出现——考虑升级为长期规则

## 应更新的位置
- [ ] 普通知识笔记：
- [ ] MOC：
- [ ] 写作风格：
- [ ] AGENTS.md：
- [ ] Skill：
- [ ] 个人模型：
- [ ] 不更新，仅保留案例
```

## 升级为长期规则的条件

1. 同类错误至少出现 **两次**（避免因一次偶然就改规则）
2. 有明确的「修改前 vs 修改后」对比
3. 修改不破坏现有正常行为（需回归测试）
