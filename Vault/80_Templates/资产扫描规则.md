## 优先级定义

| 优先级 | 说明 | 适用对象 |
|---|---|---|
| **P0** | 样板课程，立即处理 | 数据库（DB）全部文件 |
| **P1** | 重要课程或项目，P0 完成后处理 | 数据结构、计组、OS、C++、Python、微积分、线性代数、概率论、离散数学 |
| **P2** | 普通资料，按需处理 | 思政类、英语类、通识类课程 |
| **P3** | 暂缓，等待用户确认 | ZIP 压缩包（未解压）、未知来源文件、疑似敏感文件 |

## 重复判断规则

1. **哈希完全一致** → 归为同一重复组（`dup-XXX`），只对一个主副本建立蒸馏任务。
2. **文件名相同但哈希不同** → 标记为「版本候选」（`ver-XXX`），分别保留，人工比较后决定。
3. **文件名相似但哈希不同** → 仅标记为「候选重复」，不自动合并。
4. 重复判断基于 **SHA-256**，不使用 MD5 或仅文件名比对。

## 敏感信息标记规则

以下情况标记为 `sensitive` 并暂停处理：
- 文件名包含「密码」「账号」「密钥」「Token」「身份证」「银行卡」「私密」
- 文件内容疑似包含 API Key、Token、密码哈希
- 个人聊天记录、邮件导出
- 含身份证号、手机号、家庭住址的文件

标记为 `unknown` 的情况：
- 受密码保护的 ZIP/DOCX/PDF
- 无法判断内容来源的文件

## 文本可提取性判断

| 类型 | 默认可提取 | 可能需要 OCR | 备注 |
|---|---|---|---|
| `.md` `.txt` `.csv` `.json` `.xml` `.yaml` `.yml` | yes | no | 直接可读 |
| `.pdf` | partial | 视情况 | 文本型 PDF → yes；扫描 PDF → OCR |
| `.docx` | yes | no | 可通过 python-docx 提取 |
| `.doc` | partial | no | 旧版可能提取失败 |
| `.pptx` | yes | no | 可通过 python-pptx 提取 |
| `.ppt` | partial | no | 旧版可能提取失败 |
| `.xlsx` `.xls` | yes | no | 表格数据 |
| `.zip` `.rar` `.7z` | no | no | 需先列清单，解压后另行判断 |
| `.png` `.jpg` `.jpeg` `.gif` `.bmp` | no | yes | 图片需 OCR |
| `.py` `.cpp` `.java` `.c` `.h` | yes | no | 源码 |
