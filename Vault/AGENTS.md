# CyberMe Knowledge System — Agent Rules

> 本文件是 Codex 和 Claude Code 的公共规则。两个代理都必须遵守。
> Claude Code 专属规则见 `CLAUDE.md`，不得与本文件冲突或复制本文件内容。

## Mission

维护一个以用户原始资料为证据的个人知识复习与检索系统。帮助用户和他人快速查找、高效复习数据库课程知识。

## Source of Truth（证据优先级）

1. **`91_Raw-Archive` 是原始证据**，不得覆盖、删除或改写其中的任何文件。
2. **正式知识必须可追溯**到原文件、页码、章节或用户明确说明。
3. **找不到依据时**必须写明「当前资料中未找到」，不得补写貌似合理的内容来让答案完整。
4. **四层信息必须区分**：
   - 原始资料（`91_Raw-Archive`）
   - 提取材料（`90_课程提取资料`，中间产物，未核对）
   - 正式知识（`02_知识点`，已蒸馏，待人工核对）
   - 用户事实（`10_Self` 中标记 `verification: explicit` 的内容）
   - AI 推断（标记 `verification: inferred` 的内容）
5. **冲突时保留冲突记录**，未经用户核对不得静默选择或合并矛盾信息。

## Knowledge Workflow（知识处理流程）

```
发现资料 → 登记原始路径+哈希 → 判断重复/敏感性/优先级
    → 复制到 91_Raw-Archive（不移动原件）
    → 生成带来源定位的提取版 → 人工核对
    → 蒸馏为正式知识笔记 → 更新 MOC 与双向链接
    → 加入测试题并记录错误
```

每步操作后更新 `01_Home/变更日志.md`。

### 课程链接完整性

1. `20_University` 下每篇课程笔记都必须直接包含一条指向本课程 `00_*_课程MOC.md` 的 `[[内部链接]]`。
2. 每个课程 MOC 都必须被 `01_Home/知识库总地图.md` 收录。
3. 新建课程笔记时，应在同一任务中更新课程 MOC；不得留下孤立节点。
4. `90_课程提取资料` 可以只连接课程 MOC，不强制与所有知识点互连；正式知识点还应添加必要的前置、对比或应用关系。

## Personal Model Rules（个人模型规则）

1. **只有用户明确确认的内容**才能写入正式个人档案（`10_Self/` 中 `verification: explicit`）。
2. **AI 推断必须进入** `10_Self/09_待确认推断.md`，不得混入正式档案。
3. **写作模仿**优先参考 `10_Self/Style-Samples/Approved/` 中的样本。
4. **Rejected 样本**只用于识别应避开的模式，不用于正面模仿。
5. **不得虚构**用户经历、实验、参数、结果、引用、观点或决定。

## File Operations（文件操作规则）

1. **批量修改超过 10 个文件前**，必须先输出计划和预览文件清单，获得用户确认。
2. **不直接删除知识文件**。废弃内容移动到 `99_Archive/` 并记录原因。
3. **不移动、覆盖或重命名** `学习资料/` 中的任何文件。
4. **不覆盖或删除** `91_Raw-Archive/` 中的任何文件。
5. **每批操作后**输出：新增、修改、归档和未处理文件清单。
6. **禁止读取或保存**密码、API Key、Token、身份证号、银行卡和无必要隐私。
7. **Git 只跟踪** Markdown、配置文件和目录占位符。原始附件不入库。

## Answering Rules（回答规则）

1. **回答课程问题时**：先读取 `20_University/DB_数据库/00_数据库课程MOC.md`，再读取相关知识点笔记和来源。
2. **明确标注内容类型**：
   - 「根据原始课件第X页……」→ 来自原始资料
   - 「根据你的笔记……」→ 来自学生笔记
   - 「以下是推断……」→ AI 推断，尚未核对
3. **技术解释格式**：一句话解释 → 核心原理 → 前置知识 → 规则/公式 → 例子 → 易错点 → 关系链接。
4. **结论必须对应实际证据**，不为完整感添加资料中不存在的内容。
5. **检索优先**：用户提问时，先定位到正确的知识笔记，再展开解释。每次回答末尾列出所依据的文件。
