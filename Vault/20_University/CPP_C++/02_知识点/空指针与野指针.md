# 空指针与野指针

## 一句话解释
nullptr(C++11)是类型安全的空指针。野指针(指向已释放/未初始化内存)是C++最危险的bug来源。现代C++用智能指针(unique_ptr独占/shared_ptr共享/weak_ptr弱引用)自动管理内存——RAII原则：资源获取即初始化，析构时自动释放。

## 核心原理
**野指针三种来源**：(1)未初始化→定义时初始化为nullptr (2)delete后未置null→delete后立即p=nullptr (3)返回局部变量地址→不要返回栈变量的地址。

**智能指针(C++11 memory)**：
- unique_ptr<T>：独占所有权，不可拷贝(可用std::move转移)。轻量，无额外开销。`auto p = make_unique<T>(args);`
- shared_ptr<T>：共享所有权，引用计数为0时自动delete。`auto p = make_shared<T>(args);` 注意循环引用(用weak_ptr打破)
- weak_ptr<T>：观察shared_ptr，不增加引用计数。lock()尝试获取shared_ptr。用于打破循环引用和缓存

**RAII(Resource Acquisition Is Initialization)**：C++资源管理的核心思想。构造器获取资源，析构器释放资源。即使发生异常也保证资源释放。智能指针、lock_guard、fstream都是RAII的体现。

## 完整例子
```cpp
auto sp = make_shared<Node>(); // 引用计数=1
{ auto sp2 = sp; } // 引用计数=2→1, sp2析构不delete
auto up = make_unique<int>(42);
auto up2 = move(up); // up变为nullptr, up2拥有资源
```

## 容易出错的地方
- delete后忘记p=nullptr→悬空指针
- 不能delete同一个指针两次(double free)
- shared_ptr循环引用→内存泄漏。用weak_ptr打破
- make_shared/make_unique比直接new更安全(异常安全)
