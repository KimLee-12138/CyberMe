# 数据结构公式与算法速查

> 考前快速对照。每个算法附时间/空间复杂度和稳定性。

## 二叉树性质

| 性质 | 公式 |
|------|------|
| 第 i 层最多节点数 | 2^{i-1} |
| 深度 k 最多节点数 | 2^k - 1 |
| 叶子与度为2节点关系 | n₀ = n₂ + 1 |
| 完全二叉树节点 i 的左孩子 | 2i |
| 完全二叉树节点 i 的父节点 | ⌊i/2⌋ |

## 排序算法速查

| 算法 | 最好 | 平均 | 最坏 | 空间 | 稳定 |
|------|------|------|------|------|------|
| 冒泡 | O(n) | O(n²) | O(n²) | O(1) | ✓ |
| 插入 | O(n) | O(n²) | O(n²) | O(1) | ✓ |
| 选择 | O(n²) | O(n²) | O(n²) | O(1) | ✗ |
| 希尔 | O(n log n) | ~O(n^1.3) | O(n²) | O(1) | ✗ |
| **归并** | O(n log n) | O(n log n) | O(n log n) | O(n) | **✓** |
| **快速** | O(n log n) | O(n log n) | O(n²) | O(log n) | ✗ |
| **堆** | O(n log n) | O(n log n) | O(n log n) | O(1) | ✗ |
| 基数 | O(d(n+r)) | O(d(n+r)) | O(d(n+r)) | O(n+r) | ✓ |

## 图算法速查

| 算法 | 用途 | 时间复杂度 | 核心 |
|------|------|-----------|------|
| DFS | 遍历/连通分量 | O(V+E) | 递归+visited |
| BFS | 遍历/最短路径(无权) | O(V+E) | 队列+visited |
| Prim | 最小生成树 | O(V²) / O(E log V) | 选点法（贪心） |
| Kruskal | 最小生成树 | O(E log E) | 选边法（并查集） |
| Dijkstra | 单源最短路径 | O(V²) / O(E log V) | 贪心（不能负权） |
| Floyd | 全源最短路径 | O(V³) | 三重循环DP |

## 查找算法速查

| 算法 | 平均时间 | 适用 |
|------|---------|------|
| 顺序查找 | O(n) | 无序、小数据 |
| 二分查找 | O(log n) | 有序顺序表 |
| BST | O(log n)~O(n) | 动态数据 |
| AVL | O(log n) | 频繁增删 |
| 哈希表 | O(1) | 空间换时间 |
| B+ 树 | O(log n) | 数据库索引 |

## 顺序表 vs 链表

| 操作 | 顺序表 | 链表 |
|------|--------|------|
| 随机访问 | O(1) | O(n) |
| 插入/删除 | O(n)（移动元素） | O(1)（改指针） |
| 空间 | 预分配（可能浪费） | 按需分配（有指针开销） |
| 缓存友好 | 是 | 否 |

## KMP next 数组

- next[0] = -1 / 0（取决于实现）
- 若 P[k] == P[j]：next[j+1] = k + 1
- 若 P[k] != P[j]：k = next[k]（递归回溯）
- 时间复杂度：**O(n+m)**

## 哈夫曼树 WPL

WPL = ∑(叶子权值 × 从根到叶子的边数)

构造：每次取两个最小权值合并，重复直到只剩一个节点。
