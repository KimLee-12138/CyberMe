# 哈夫曼树与编码

## 一句话解释

哈夫曼树是带权路径长度 WPL 最小的二叉树，哈夫曼编码用变长编码实现数据压缩——出现频率高的字符用短编码，低频用长编码。

## 核心原理

**WPL（带权路径长度）**：∑(叶子权值 × 到根的路径长度)

**哈夫曼树构造**：
1. 将所有叶子节点（权值 w₁, w₂, ..., wₙ）放入最小堆
2. 每次取出两个最小权值节点，合并为新节点（权值为两者之和）
3. 新节点放回堆中
4. 重复直到堆中只剩一个节点

**哈夫曼编码**：从根出发，左分支=0，右分支=1，到每个叶子的路径就是该字符的编码。前缀码（任何编码不是另一个编码的前缀），保证解码唯一。

## 完整例子

字符集 {A:5, B:2, C:9, D:3, E:6}
- 合并 B(2)+D(3)=5 → {A:5, (B+D):5, C:9, E:6}
- 合并 A(5)+(B+D)(5)=10 → {10, C:9, E:6}
- 合并 C(9)+E(6)=15 → {10, 15}
- 合并 10+15=25（根）

最终编码：A=00, B=010, D=011, E=10, C=11

## 容易出错的地方

- 哈夫曼树不唯一（相同权值选哪两个合并结果不同，但 WPL 相同）
- 哈夫曼树只有度为 0（叶子）和度为 2（内部节点）的节点
- 哈夫曼编码是前缀码，不是固定长度编码
