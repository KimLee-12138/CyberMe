# 线性表

## 一、顺序表

用数组实现，逻辑相邻=物理相邻。

**优点**：随机访问 $O(1)$，存储密度高。**缺点**：插入/删除需移动元素 $O(n)$。

## 二、单链表

```cpp
struct Node { int data; Node* next; };
```

| 操作 | 复杂度 |
|------|--------|
| 按位查找 | $O(n)$ |
| 头插/头删 | $O(1)$ |
| 尾插(无尾指针) | $O(n)$ |

## 三、循环链表

尾结点的 next 指向头结点。可从任意结点出发遍历所有结点。

## 四、双向链表

```cpp
struct DNode { int data; DNode *prior, *next; };
```

插入/删除需修改两个指针的 prior 和 next。时间复杂度仍为 $O(1)$（已知位置）。

## 五、顺序表 vs 链表

| 操作 | 顺序表 | 链表 |
|------|--------|------|
| 随机访问 | $O(1)$ | $O(n)$ |
| 插入/删除 | $O(n)$ | $O(1)$(已知位置) |
| 存储密度 | 高 | 低(有指针开销) |
| 动态扩容 | 需整体搬迁 | 可灵活增长 |
