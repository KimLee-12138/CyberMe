# 图结构

## 一句话解释

图由顶点集合和边集合组成。存储方式有邻接矩阵和邻接表。DFS/BFS 是图的基本遍历，最小生成树（Prim/Kruskal）和最短路径（Dijkstra/Floyd）是核心算法。

## 核心原理

**图的存储**：
| 方式 | 空间 | 适合场景 |
|------|------|---------|
| 邻接矩阵 | O(V²) | 稠密图、需要快速判断是否邻接 |
| 邻接表 | O(V+E) | 稀疏图、遍历邻接点 |

**DFS（深度优先）**：递归 + visited 标记。类似树的先序遍历。
**BFS（广度优先）**：队列 + visited 标记。类似树的层序遍历。

**最小生成树（MST）**：
- **Prim 算法**：选点法。从任意点开始，每次选与当前集合距离最小的点加入。O(V²)（邻接矩阵）或 O(E log V)（堆优化+邻接表）。
- **Kruskal 算法**：选边法。每次选权重最小且不形成环的边。用并查集判断环。O(E log E)。

**最短路径**：
- **Dijkstra**：单源最短路径，不能处理负权边。贪心思想。O(V²) 或 O(E log V) 堆优化。
- **Floyd**：所有点对最短路径。三重循环，O(V³)。dp[k][i][j] = min(dp[k-1][i][j], dp[k-1][i][k] + dp[k-1][k][j])。
- 拓扑排序：有向无环图(DAG)，每次选入度为0的顶点。可用于判断图是否有环。

## 完整例子

```cpp
// DFS - 邻接矩阵
void DFS(int v, bool visited[], int graph[][MAX], int n) {
    visited[v] = true;
    printf("%d ", v);
    for (int i = 0; i < n; i++)
        if (graph[v][i] && !visited[i])
            DFS(i, visited, graph, n);
}
```

## 容易出错的地方

- Prim 和 Kruskal 的区别：Prim 选点（适合稠密图），Kruskal 选边（适合稀疏图）
- Dijkstra **不能处理负权边**（负权用 Bellman-Ford）
- Floyd 的中间节点 k 必须放在**最外层循环**
- 无向图的邻接矩阵是对称的，邻接表每条边存两次
- BFS 求最短路径只适用于**无权图**（或每条边权值相同）
