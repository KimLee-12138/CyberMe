# 图算法 — 例题集

## 例题1：Prim 算法

**题目**：用 Prim 算法求下图最小生成树，从顶点 A 开始。

```
边集：A-B(4), A-C(2), B-C(1), B-D(5), C-D(8), C-E(10), D-E(2), D-F(6), E-F(3)
```

**解题步骤**：
1. 选 A，候选边 A-B(4), A-C(2) → 选最小 A-C(2)，集合={A,C}
2. 候选边 A-B(4), C-D(8), C-E(10) → 选最小 A-B(4)，集合={A,B,C}
3. 候选边 B-D(5), C-D(8), C-E(10) → 选最小 B-D(5)，集合={A,B,C,D}
4. 候选边 D-E(2), D-F(6), C-E(10) → 选最小 D-E(2)，集合={A,B,C,D,E}
5. 候选边 E-F(3), D-F(6) → 选最小 E-F(3)，集合={A,B,C,D,E,F}

**答案**：MST 边集 {A-C(2), A-B(4), B-D(5), D-E(2), E-F(3)}，总权 = 2+4+5+2+3 = 16

**易错点**：Prim 每步选的是**与当前集合距离最小的外部顶点**，不是全局最小边。

## 例题2：Kruskal 算法

**题目**：同例题1的图，用 Kruskal 算法求 MST。

**解题步骤**：
1. 所有边按权值排序：B-C(1), A-C(2), D-E(2), E-F(3), A-B(4), B-D(5), D-F(6), C-D(8), C-E(10)
2. 选 B-C(1)：不形成环 ✓
3. 选 A-C(2)：不形成环 ✓
4. 选 D-E(2)：不形成环 ✓
5. 选 E-F(3)：不形成环 ✓
6. 选 A-B(4)：A-B-C 会形成环 ✗ → **跳过**
7. 选 B-D(5)：不形成环 ✓
8. 已有 5 条边（6 个顶点），停止

**答案**：MST 边集 {B-C(1), A-C(2), D-E(2), E-F(3), B-D(5)}，总权 = 13

**易错点**：Kruskal 的 MST 和 Prim 结果可能不同（本例就是），但**总权值一定相同**？不对——这里 Kruskal=13, Prim=16。说明两个算法得到的都是 MST，但**按给出的边集两个图不一样**。这题实际上 Prim 和 Kruskal 应该给出相同总权——说明我在例题1中算错了。让我修正：

重新用 Prim：A-C(2), C-B(1)（不是 A-B(4)!因为 B-C 只有1）, B-D(5), D-E(2), E-F(3) = **总权 13**。✓

**关键教训**：Prim 在步骤2时候选边应包括 B-C(1)（因为 B 与已选集合 {A,C} 中的 C 相邻）。

## 例题3：Dijkstra 算法

**题目**：求 A 到各顶点的最短路径。

```
A→B:4, A→C:1, B→D:3, C→B:2, C→D:5, C→E:6, D→E:1
```

**解题步骤**：
1. dist=[A:0, B:∞, C:∞, D:∞, E:∞]，选 A
2. 更新 A 的邻接点：dist[B]=4, dist[C]=1
3. 选距离最小未处理顶点 C(1)
4. 从 C 更新：B(1+2=3 <4), D(1+5=6), E(1+6=7) → dist=[0,3,1,6,7]
5. 选 B(3)：更新 D(3+3=6，不优于6)
6. 选 D(6)：更新 E(6+1=7，不优于7)
7. 选 E(7)

**答案**：最短距离：A→A:0, A→B:3(A→C→B), A→C:1, A→D:6(A→C→D), A→E:7(A→C→E)

**易错点**：Dijkstra 每步选的是**当前未处理中距离最小的顶点**，选中后不再更新。

## 例题4：拓扑排序

**题目**：课程依赖关系——A 无先修，B 依赖 A，C 依赖 A，D 依赖 B 和 C，E 依赖 D。给出一个拓扑排序。

**解题步骤**：
1. 计算入度：A:0, B:1, C:1, D:2, E:1
2. 选入度为 0 的 A，删除 A 的所有出边 → B:0, C:0
3. 选 B（或 C），删除 → D:1
4. 选 C，删除 → D:0
5. 选 D，删除 → E:0
6. 选 E

**答案**：A → B → C → D → E（或 A → C → B → D → E）

**易错点**：拓扑排序不唯一。判断图是否有环——如果最终还有顶点入度不为 0，则有环。
