# 候选码求解

## 一句话解释

候选码求解是从给定的函数依赖集中找出能唯一标识所有属性的最小属性集。

## 核心原理

**闭包（Closure）**：属性集 X 关于函数依赖集 F 的闭包 X⁺ᵯ 是能由 X 根据 F 推导出的所有属性的集合。

**求解候选码的步骤**：
1. 将属性分为四类：L 类（仅出现在左边）、R 类（仅出现在右边）、LR 类（两边都出现）、N 类（都不出现）
2. L 类和 N 类必属于任何候选码
3. 计算 (L ∪ N)⁺：
   - 若 = U，则 (L ∪ N) 是唯一候选码
   - 若 ⊂ U，逐个添加 LR 类属性，直到闭包 = U

## 前置知识

- [[函数依赖]]

## 规则或公式

X⁺ = 从 X 出发通过 F 中的函数依赖能推导出的所有属性

## 完整例子

R(A, B, C, D)，F = {A → B, B → C, C → D}
- L 类：A（仅左边出现），R 类：D（仅右边出现），LR 类：B, C
- A⁺ = {A, B, C, D} = U
- 候选码：{A}

R(A, B, C, D)，F = {AB → C, C → D, D → A}
- LR 类：A, B, C, D
- 分别尝试计算 AB⁺, BC⁺, CD⁺...
- 候选码：{A, B}, {B, C}, {C, D}（可能有多个）

## 容易出错的地方

- 只标记 L/R/LR/N 类属性而不计算闭包验证 → 漏掉候选码
- 候选码不唯一时，需要列出全部
- 主属性 vs 非主属性：只要包含在任一候选码中就是主属性
