# 数据库设计流程

## 一句话解释

数据库设计分为需求分析、概念设计、逻辑设计、物理设计、实施和维护六个阶段。

## 核心原理

**六个阶段**：
1. **需求分析**：了解用户的数据需求和处理需求，输出数据字典和数据流图
2. **概念结构设计**：利用 E-R 图描述实体和联系，独立于具体 DBMS
3. **逻辑结构设计**：将 E-R 图转换为关系模式，应用规范化理论优化
4. **物理结构设计**：为关系模式选择存储结构和存取方法（索引等）
5. **数据库实施**：用 DDL 创建数据库，编写应用程序，载入数据
6. **数据库运行与维护**：监控性能、备份恢复、调整重构

## 规则或公式

E-R 图 → 关系模式的转换规则：
1. 实体 → 关系
2. 1:1 联系 → 可合并到任一端或独立成关系
3. 1:N 联系 → 合并到 N 端
4. M:N 联系 → 独立成关系

## 完整例子

学生选课系统设计：
1. 需求：学生信息管理、课程管理、选课与成绩
2. 概念设计：E-R 图（Student 实体，Course 实体，SC 联系）
3. 逻辑设计：Student(sno, sname, sdept), Course(cno, cname, credit), SC(sno, cno, grade)
4. 物理设计：sno 建 B+树索引，cno 建 Hash 索引

## 容易出错的地方

- 跳过需求分析直接建表 → 后期大量修改
- 概念设计与逻辑设计混淆 → E-R 图不是关系模式
- 过度设计（范式过高）可能导致性能问题
