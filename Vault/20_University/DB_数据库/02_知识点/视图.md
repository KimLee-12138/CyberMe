# 视图

## 一句话解释

视图是从一个或几个基本表导出的虚表，数据库中只存放定义而不存放数据。

## 核心原理

视图是基于 SQL 查询的虚拟表。对视图的查询最终转化为对基本表的查询。

视图的作用：
1. 简化用户操作（预定义复杂查询）
2. 提供安全保护（隐藏敏感列）
3. 提供逻辑数据独立性
4. 使用户能以多种角度看待同一数据

- [[SQL数据定义]]

## 规则或公式

```sql
CREATE VIEW 视图名 [(列名,...)]
AS 子查询
[WITH CHECK OPTION];
```

WITH CHECK OPTION：对视图进行 UPDATE/INSERT/DELETE 时，确保操作的行满足视图定义条件。

## 完整例子

```sql
-- CS 系学生视图
CREATE VIEW CS_Student AS
SELECT sno, sname, sage FROM Student WHERE sdept = 'CS';

-- 查询 CS 系 20 岁以下学生（通过视图）
SELECT * FROM CS_Student WHERE sage < 20;

-- 带 WITH CHECK OPTION 的视图
CREATE VIEW Young_CS AS
SELECT * FROM Student WHERE sdept = 'CS' AND sage < 20
WITH CHECK OPTION;
```

## 容易出错的地方

- 视图不存储数据，每次查询都重新计算（物化视图除外）
- 不是所有视图都可更新（涉及聚合、DISTINCT、多表连接的通常不可更新）
- 删除视图不影响基本表
