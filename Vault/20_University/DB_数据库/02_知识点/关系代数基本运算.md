# 关系代数基本运算

## 一句话解释

关系代数是一组以关系为运算对象的代数运算，是 SQL 查询的数学基础。

## 核心原理

**传统的集合运算**：
- 并（Union）：R ∪ S = {t | t ∈ R ∨ t ∈ S}，要求 R 和 S 属性相同
- 差（Difference）：R - S = {t | t ∈ R ∧ t ∉ S}
- 交（Intersection）：R ∩ S = {t | t ∈ R ∧ t ∈ S}
- 笛卡儿积：R × S = {tᵣtₛ | tᵣ ∈ R ∧ tₛ ∈ S}

**专门的关系运算**：
- 选择（Selection）σ：从关系中选取满足条件的元组（横向切割）
- 投影（Projection）Π：从关系中选取指定属性列（纵向切割）
- 连接（Join）⋈：从两个关系的笛卡儿积中选取满足条件的元组
- 除（Division）÷：找出与另一关系中所有值都匹配的元组

## 规则或公式

- 选择：σ_{条件}(R) → 条件可用 ∧（与）、∨（或）、¬（非）组合
- 自然连接：R ⋈ S → 在公共属性上做等值连接，自动去重

## 完整例子

R(A, B) = {(1, a), (2, b), (3, c)}
S(A, B) = {(1, a), (4, d)}
- R ∪ S = {(1, a), (2, b), (3, c), (4, d)}
- R ∩ S = {(1, a)}
- R - S = {(2, b), (3, c)}

## 容易出错的地方

- 并/交/差要求两个关系属性相同（同目/同类），笛卡儿积无此要求
- 自然连接会自动去除重复列，等值连接不会
- 选择操作不改变属性，投影可能改变
