# 关系完整性

## 一句话解释

关系模型有三类完整性约束：实体完整性、参照完整性和用户定义的完整性。

## 核心原理

**实体完整性（Entity Integrity）**：
- 规则：主码的值不能为空（NULL）
- 原因：主码用于唯一标识元组，NULL 无法完成此功能

**参照完整性（Referential Integrity）**：
- 规则：外码的值要么为空，要么等于被参照关系中某个元组的主码值
- 原因：保证关系之间的联系是有效的

**用户定义的完整性（User-defined Integrity）**：
- 规则：针对具体应用环境定义的约束条件
- 示例：年龄在 0-150 之间，性别只能是男/女

## 前置知识

- [[候选码、主码与外码]]

## 规则或公式

- 实体完整性：主码属性 ≠ NULL
- 参照完整性：外码 = NULL 或 外码 ∈ 被参照关系的主码值集合

## 完整例子

Student 表主码 sno 不能为空 → 实体完整性
SC 表外码 sno 必须存在于 Student 表中 → 参照完整性
Student 表 sage CHECK(sage >= 0 AND sage <= 150) → 用户定义完整性

## 容易出错的地方

- 外码为 NULL 是允许的（参照完整性），但主码为 NULL 不允许
- 参照完整性不要求外码与被参照主码同名
