# 封锁协议

## 一句话解释

封锁是并发控制的主要技术，通过加锁阻止其他事务访问数据，保证事务的隔离性。

## 核心原理

**锁的类型**：
- **排他锁（X 锁，写锁）**：持有期间其他事务不能读也不能写
- **共享锁（S 锁，读锁）**：持有期间其他事务可读不可写

**三级封锁协议**：
- **一级封锁协议**：修改前加 X 锁，事务结束后释放 → 解决丢失修改
- **二级封锁协议**：一级 + 读前加 S 锁，读完释放 → 解决丢失修改 + 读脏数据
- **三级封锁协议**：一级 + 读前加 S 锁，事务结束后释放 → 解决全部三类并发异常

**两段锁协议（2PL）**：事务分为加锁阶段和解锁阶段，加锁阶段不能解锁，解锁阶段不能加锁。遵守 2PL 的事务调度是可串行化的。

## 规则或公式

| 锁兼容矩阵 | S | X |
|-----------|---|---|
| S | ✓ | ✗ |
| X | ✗ | ✗ |

一次封锁法：一次性申请所有需要的锁 → 预防死锁（但降低并发度）
顺序封锁法：按固定顺序申请锁 → 预防死锁

## 完整例子

转账事务：
```sql
-- 加锁阶段
X_LOCK(A);
X_LOCK(B);
READ(A); A = A - 100; WRITE(A);
READ(B); B = B + 100; WRITE(B);
-- 解锁阶段
UNLOCK(A);
UNLOCK(B);
```

## 容易出错的地方

- 只读不加锁 → 可能读到不一致的中间状态
- 加锁但不遵守 2PL → 调度可能不可串行化
- 死锁不可避免，需要死锁检测和解除机制
- 封锁粒度（行/页/表）影响并发度和系统开销
