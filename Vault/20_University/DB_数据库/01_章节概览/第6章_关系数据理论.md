# 第6章 关系数据理论

> 本章是数据库理论的难点——范式判断和候选码求解是考试高频题。

## 本章核心概念

| 概念 | 一句话 | 详情 |
|------|--------|------|
| 函数依赖 | X 的值唯一确定 Y 的值，记 X→Y | [[02_知识点/函数依赖]] |
| 候选码求解 | 通过闭包计算找出能推导所有属性的最小属性集 | [[02_知识点/候选码求解]] |
| 2NF | 1NF + 非主属性完全依赖候选码 | [[02_知识点/第二范式]] |
| 3NF | 2NF + 非主属性不传递依赖候选码 | [[02_知识点/第三范式]] |
| BCNF | 每个非平凡FD的决定因素都是超码 | [[02_知识点/BCNF]] |
| 数据库设计 | E-R图→关系模式→规范化→物理设计 | [[02_知识点/数据库设计流程]] |

## 范式判断速查

```text
问：关系R是第几范式？

1. 先找候选码（L/R/LR/N分类法 + 闭包计算）
2. 列出所有非主属性
3. 检查有没有部分依赖 → 有 = 1NF，无 = 至少2NF
4. 检查有没有传递依赖 → 有 = 最多2NF，无 = 至少3NF
5. 检查每个FD的决定因素是否都是超码 → 是 = BCNF
```

## 范式层级

```text
1NF ⊃ 2NF ⊃ 3NF ⊃ BCNF ⊃ 4NF
       ↑消除    ↑消除    ↑消除主属性
       部分依赖  传递依赖  对码的部分/传递依赖
```

## 常见考试题型

1. 给定关系模式和FD集，求候选码
2. 判断关系模式属于第几范式
3. 将低范式分解为高范式（保持无损连接）
4. 判断分解是否保持函数依赖
