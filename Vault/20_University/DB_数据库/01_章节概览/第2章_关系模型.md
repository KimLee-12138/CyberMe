# 第2章 关系模型

> 本章回答：数据在数据库中怎么组织？表之间怎么关联？

## 本章核心概念

| 概念 | 一句话 | 详情 |
|------|--------|------|
| 关系 | 一张二维表，笛卡儿积的子集 | [[02_知识点/关系、域与元组]] |
| 候选码 | 能唯一标识元组的最小属性集 | [[02_知识点/候选码、主码与外码]] |
| 主码/外码 | 主码选定的候选码，外码建立表间联系 | [[02_知识点/候选码、主码与外码]] |
| 实体完整性 | 主码不能为空 | [[02_知识点/关系完整性]] |
| 参照完整性 | 外码要么空要么等于被参照表的主码 | [[02_知识点/关系完整性]] |
| 关系代数 | 选择σ/投影Π/连接⋈/并∪/差-/交∩ | [[02_知识点/关系代数基本运算]] |

## 复习要点

1. 关系、元组、属性、域、笛卡儿积的定义与关系
2. 候选码的两个性质（唯一性、最小性）
3. 主码 vs 候选码 vs 外码
4. 三类完整性约束的含义和例子
5. 关系代数五种基本运算 + 连接/除
6. 自然连接与等值连接的区别

## 典型考题模式

- 给一个关系模式，找出所有候选码
- 判断某个操作违反了什么完整性约束
- 用关系代数表达式写出查询需求
- 计算两个关系的并/交/差/笛卡儿积/自然连接结果
