# 算法.md：提取版

> 本文件是检索中间材料，不是已核对的正式知识结论。

这张回忆版的图片**含金量非常高**！它直接揭示了你们老师的出题风格：**严重偏向 LeetCode 经典原题**，且覆盖了算法课最核心的四大板块（分治、回溯、DP）。

根据这张图，我为你详细拆解去年的题目，并根据这个难度和风格，为你整理了四大考点的**“必刷题单”**（每类约 10 道）。

------

### 第一部分：去年真题深度还原与考点分析

这张图片里的四道题，全是 LeetCode 上的原题或变体。

#### 1. 分治法找任意 Top 数

- **对应题目：** **LeetCode 162. 寻找峰值 (Find Peak Element)**
- **题目描述：** 给你一个整数数组 `nums`，找到峰值元素并返回其索引。数组可能包含多个峰值，在这种情况下，返回任何一个峰值所在位置即可。要求时间复杂度 $O(\log n)$。
- **核心考点：** **二分查找 (Binary Search)** 的变体。
  - **难点：** 很多人只知道二分用于有序数组，但这道题利用“爬坡”性质（如果 `nums[i] < nums[i+1]`，则右侧必有峰值），展示了二分法在非有序数组中的应用。

#### 2. n个任务分给俩工人，完成时间最短

- **对应题目：** **LeetCode 1723. 完成所有工作的最短时间 (变体)**
- **题目描述：** 其实是经典的 **“多机调度问题”** 或者 **“最小化最大值”** 问题。如果是只有 2 个工人，这其实也是 **“划分为两个和相等的子集”** 的变体。
- **核心考点：** **回溯法 (Backtracking) + 剪枝** 或 **二分答案**。
  - **推测考法：** 既然说是“简化”，且只有 2 人，老师可能考的是**回溯法**的穷举，或者看作是一个变相的 0-1 背包问题（让两个人的工作量尽可能接近总工作量的一半）。

#### 3. n阶楼梯爬楼，每次1或2步

- **对应题目：** **LeetCode 70. 爬楼梯 (Climbing Stairs)**
- **题目描述：** 最经典的入门题。
- **核心考点：** **一维动态规划 (DP)**。斐波那契数列 $f(n) = f(n-1) + f(n-2)$。

#### 4. 添加运算符

- **对应题目：** **LeetCode 282. 给表达式添加运算符 (Expression Add Operators)**
- **题目描述：** 给定数字字符串 "123" 和目标值 6，在数字之间添加 `+`, `-`, `*` 使得运算结果等于 6。
- **核心考点：** **回溯法 (Backtracking)**。
  - **难点：** 这是一道 **Hard** 题！难点在于处理乘法（乘法优先级高，需要回退上一步的加减操作）。这也说明你们考试**会有难题压轴**，且很喜欢考**字符串处理的回溯**。

### 第二部分：针对性刷题推荐（各 ~10 道）

既然考试风格是 **“LeetCode 经典题 + 核心算法模板”**，我为你精选了以下题单。这些题在难度和考点上与回忆版高度对标。

#### 一、 分治策略 (Divide and Conquer)

*考点预测：主要是二分搜索的变体，以及归并排序的应用。*

1. **LC 162. 寻找峰值** (去年原题，必做)
2. **LC 33. 搜索旋转排序数组** (必考！二分法的最高频考题，比峰值更经典)
3. **LC 34. 在排序数组中查找元素的第一个和最后一个位置** (考察二分的边界处理)
4. **LC 215. 数组中的第K个最大元素** (必考！**快速选择算法 QuickSelect**，分治思想的典型应用)
5. **LC 53. 最大子数组和** (要求用分治法解决，虽然DP更简单，但分治是教材经典案例)
6. **LC 169. 多数元素** (可以使用分治法求解，左边的众数 vs 右边的众数)
7. **LC 240. 搜索二维矩阵 II** (分治或者减治思想)
8. **LC 4. 寻找两个正序数组的中位数** (Hard，二分法的终极应用，若冲满绩需掌握)
9. **LC 23. 合并K个升序链表** (归并排序思想的扩展)
10. **归并排序求逆序对** (剑指 Offer 51. 数组中的逆序对，分治经典应用)

#### 二、 动态规划 (Dynamic Programming)

*考点预测：除了爬楼梯这种简单题，大概率会考“背包问题”或“子序列问题”。*

1. **LC 70. 爬楼梯** (去年原题，热身)
2. **LC 322. 零钱兑换** (必考！**完全背包**问题的变体，求最少硬币数)
3. **LC 300. 最长递增子序列 (LIS)** (必考！经典 $O(n^2)$ 解法，甚至可能考 $O(n \log n)$)
4. **LC 1143. 最长公共子序列 (LCS)** (必考！二维 DP 的教科书题目)
5. **LC 198. 打家劫舍** (线性 DP，不能选相邻元素，与爬楼梯类似但稍难)
6. **LC 64. 最小路径和** (矩阵网格 DP，从左上走到右下)
7. **LC 62. 不同路径** (也是网格 DP，求路径数量)
8. **LC 416. 分割等和子集** (必考！**0-1 背包**问题的变体，判断能否分为两半，呼应去年第2题)
9. **LC 72. 编辑距离** (Hard，经典的两个字符串 DP)
10. **LC 5. 最长回文子串** (区间 DP 或 中心扩散)

#### 三、 回溯法 (Backtracking)

*考点预测：去年考了 LC 282 这种难题，说明对回溯要求很高。重点在于“排列、组合、子集”以及“剪枝”。*

1. **LC 282. 给表达式添加运算符** (去年原题，重点复习乘法怎么处理)
2. **LC 46. 全排列** (必考！回溯法最基本的模板)
3. **LC 78. 子集** (必考！找所有子集，经典模板)
4. **LC 51. N皇后** (必考！教材上的经典案例，必须会手写)
5. **LC 39. 组合总和** (数字可重复使用，考察如何去重和控制递归深度)
6. **LC 22. 括号生成** (卡特兰数，合法的括号组合)
7. **LC 79. 单词搜索** (在矩阵中找字符串，DFS 回溯)
8. **LC 93. 复原 IP 地址** (字符串切割问题，和 LC 282 很像)
9. **LC 1723. 完成所有工作的最短时间** (去年第2题的原型，建议掌握基本的回溯解法)
10. **LC 131. 分割回文串** (需要先判断回文，再回溯分割)

#### 四、 贪心算法 (Greedy)

*虽然图片里没明确写贪心（除了任务分配可能涉及），但你之前的作业里有很多贪心，不能掉以轻心。*

1. **LC 55. 跳跃游戏** (能否到达终点，贪心经典)
2. **LC 45. 跳跃游戏 II** (最少跳几步，比上一题难一点)
3. **LC 435. 无重叠区间** (必考！区间调度问题，按结束时间排序)
4. **LC 455. 分发饼干** (最简单的贪心，排序后一一匹配)
5. **LC 134. 加油站** (环形数组的贪心)
6. **LC 406. 根据身高重建队列** (两个维度的贪心)
7. **LC 179. 最大数** (自定义排序规则的贪心)
8. **活动安排问题** (教材经典，通常就是 LC 435 的变体)
9. **哈夫曼编码** (构建最优二叉树，贪心策略是用最小堆)
10. **Dijkstra 算法 / Prim 算法** (虽然是图论，但本质是贪心，如果你学了图算法，这两个要会手推)

------

### 三、 备考建议

1. **重点复习 LC 282 (添加运算符) 和 LC 1723 (任务分配)**：既然去年考了 Hard 题和变体，说明老师很看重**“状态空间树”**的构建能力。
2. **手写代码**：不要只在电脑上刷，拿纸笔写核心函数。去年的回忆版显示，题目要求的是“输入输出”，可能考试是**ACM模式**（自己处理 `cin/cout` 或 `input()`）或者是**核心代码模式**，要做好两手准备。
3. **关注“变体”**：比如爬楼梯，老师可能会改成“一次能爬 1, 2, 3 步”，这时候方程怎么变？($dp[i] = dp[i-1] + dp[i-2] + dp[i-3]$)。

你现在可以从**分治法**开始，先搞定 **LC 33 (搜索旋转数组)** 和 **LC 215 (第K大元素)**，这两道题性价比极高，考中概率极大。需要我详细讲讲这两道题吗？
