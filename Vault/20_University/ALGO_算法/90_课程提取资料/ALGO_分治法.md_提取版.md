## 二分查找

#### **算法核心思想**

1. **分解 (Divide)**：将数组中间的元素 $A[mid]$ 与目标值 $x$ 进行比较。
2. **治理 (Conquer)**：
   - 如果 $x < A[mid]$，则在左半部分继续递归查找。
   - 如果 $x > A[mid]$，则在右半部分继续递归查找。
   - 如果 $x = A[mid]$，则查找成功，返回索引。

#### **伪代码**

```c++
if low > high then
    return -1; // 未找到
else
    mid := (low + high) / 2;
    if x = A[mid] then
        return mid; // 找到，返回下标
    else if x < A[mid] then
        return BinarySearch(A, low, mid - 1, x); // 递归左半边
    else
        return BinarySearch(A, mid + 1, high, x); // 递归右半边
    end if
end if
```

#### **时间复杂度：**

我们来建立递归方程：

- **分解**：比较一次，计算 `mid`，耗时 $O(1)$。
- **子问题**：每次只需要解决**一个**规模为 $n/2$ 的子问题（注意：这里不是 $2T(n/2)$，因为我们只需要去一边，不用两边都去）。
- **合并**：不需要额外操作，$O(1)$。

递归方程为：

- $$T(n) = T(n/2) + O(1)$$

最终时间复杂度：

- $$T(n) = O(\log n)$$

## 归并排序（合并排序）

#### **算法核心思想**

合并排序完全对应分治法的三个步骤：

1. **分解 (Divide)**：将待排序的 $n$ 个元素分成各包含 $n/2$ 个元素的两个子序列。
2. **治理 (Conquer)**：递归地调用合并排序，对两个子序列进行排序。
3. **合并 (Combine)**：将两个已经排序的子序列“合并”成一个最终的有序序列。

#### **伪代码**

主过程：MERGESORT(A, low, high)

```c++
Algorithm MERGESORT(A, low, high)
    // 如果 low < high，说明数组至少有两个元素，需要排序
    if low < high then
        mid = (low + high) / 2       // 1. 分解：计算中间位置
        MERGESORT(A, low, mid)       // 2. 治理：递归排序左半部分
        MERGESORT(A, mid + 1, high)  // 2. 治理：递归排序右半部分
        MERGE(A, low, mid, high)     // 3. 合并：将两个有序部分合并
    end if
End Algorithm
```

合并子过程：MERGE(A, low, mid, high)

```c++
Procedure MERGE(A, low, mid, high)
    // 1. 准备工作
    // 创建一个辅助数组 B，并将 A[low...high] 的内容复制到 B 中
    for k = low to high do
        B[k] = A[k]
    end for

    // 2. 初始化指针
    i = low        // i 指向左半边 (B[low...mid]) 的开头
    j = mid + 1    // j 指向右半边 (B[mid+1...high]) 的开头
    k = low        // k 指向原数组 A 当前要填入的位置

    // 3. 归并循环 (核心比较)
    // 当左边和右边都还有元素时
    while i <= mid and j <= high do
        if B[i] <= B[j] then     // 比较 B 中的元素：如果左边的小
            A[k] = B[i]          // 把左边的放入 A
            i = i + 1            // 左边指针后移
        else                     // 如果右边的小
            A[k] = B[j]          // 把右边的放入 A
            j = j + 1            // 右边指针后移
        end if
        k = k + 1                // A 的填充位置后移
    end while

    // 4. 处理剩余元素
    // 如果左边还有剩余元素，直接把它们拷回 A
    // (右边如果剩余，因为本身就在 A 的高位，所以不需要动)
    while i <= mid do
        A[k] = B[i]
        k = k + 1
        i = i + 1
    end while

End Procedure
```

#### **时间复杂度分析**

我们需要列出递归方程来分析：

- **分解时间**：仅仅是计算中间下标，耗时 $O(1)$。
- **子问题时间**：我们需要解决**两个**规模为 $n/2$ 的子问题，所以是 $2T(n/2)$。
- **合并时间**：`MERGE` 过程需要遍历整个数组的 $n$ 个元素进行比较和复制，耗时 $O(n)$。

递归方程：

$$T(n) = \begin{cases} O(1) & \text{if } n=1 \\ 2T(n/2) + O(n) & \text{if } n > 1 \end{cases}$$

求解结果：

- 递归树每一层的工作量都是 $O(n)$，树的高度是 $\log n$。

- $$T(n) = O(n \log n)$$

## 寻找数组中的第k小元素

#### **1. 问题描述**

- **输入**：一个包含 $n$ 个互不相同元素的集合 $S$，以及一个整数 $k$ ($1 \le k \le n$)。
- **输出**：$S$ 中第 $k$ 小的元素。

#### 2. 算法核心思想

- “线性时间选择算法”
- 基本思想
- 首先，如果元素个数小于预定义的阈值44，则算法使用直接的方法计算第k小的元素，至于如何选择阂值在以后的算法分析中将会清楚。
- 下一步把n个元素划分成n/5（下取整）组，每组由5个元素组成，如果n不是5的倍数，则排除剩余的元素，这应当不影响算法的执行。每组进行排序并取出它的中项即第三个元素
- 接着将这些中项序列中的中项元素记为mm，它是通过递归计算得到的。将数组A中的元素划分成三个数组: A1, A2, A3 ，其中分别包含小于、等于和大于mm的元素。
- 最后，求出第k小的元素出现在三个数组中的哪一个，并根据测试结果，算法或者返回第k小的元素，或者在A1 or A3上递归。

#### 伪代码：

![image-20260114200309414](C:\Users\Lenovo\AppData\Roaming\Typora\typora-user-images\image-20260114200309414.png)

讲解：

第 1 步：计算当前处理的元素个数

$$p \leftarrow high - low + 1$$

$p$ 代表当前递归层级中，我们正在处理的子数组的长度。

**第 2 步：递归出口（小规模数据处理）**

> ```
> if p < 44 then 将 A 排序 return (A[k])
> ```

- 这里的 **44** 是一个阈值（不同的教材可能取值不同，如 5, 75 等，只要是常数即可）。
- **逻辑**：如果数组已经很小了（比如少于 44 个元素），没必要再切分了，直接用简单的排序（如插入排序）排好，然后返回第 $k$ 个位置的数即可。

**第 3 步：分组（关键步骤）**

> ```
> Let q = floor(p/5). 将A分成q组，每组5个元素。
> ```

- **逻辑**：将数组每 5 个元素分成一组。
- **为什么是 5？** 这是一个数学上的权衡值。每组 5 个可以保证后续选出的基准值足够“居中”，从而保证递归式收敛。

**第 4 步：找每组的中位数**

> ```
> 将q组中的每一组单独排序，找出中项。所有中项的集合为M
> ```

- **逻辑**：对这 $q$ 个小组，分别找出它们各自的中位数。
- 结果是得到一个新的数组 $M$，它包含了 $q$ 个中位数。

**第 5 步：找“中位数的中位数” (Finding the Pivot)**

> ```
> mm <- select(M, 1, q, ceil(q/2))
> ```

- **逻辑**：这是最精妙的一步。我们需要从刚才找到的 $M$ 集合中，再找出一个中位数。
- 我们递归调用 `select` 函数本身来寻找 $M$ 的中位数。
- 这个值 **$mm$ (median of medians)** 将被用作下一步划分数组的**基准值 (Pivot)**。
- **目的**：选出的 $mm$ 保证比数组中至少 30% 的元素大，同时也比至少 30% 的元素小。这避免了快速排序中选到“极值”导致划分极度不平衡的情况。

**第 6 步：划分数组**

> ```
> 将 A[low..high] 分成三组：A1(小于mm), A2(等于mm), A3(大于mm)
> ```

- 这和快速排序（QuickSort）的 Partition 过程一样。
- $A_1$：所有比基准值 $mm$ 小的数。
- $A_2$：所有等于 $mm$ 的数。
- $A_3$：所有比基准值 $mm$ 大的数。

第 7 步：判断与递归 (Case语句)

我们需要确定我们要找的第 $k$ 小的数，落在哪一个集合里：

- **情况 1：$|A_1| \ge k$**
  - 这意味着第 $k$ 小的数肯定在“小数组” $A_1$ 里。
  - **动作**：递归在 $A_1$ 中找第 $k$ 小的数。
- **情况 2：$|A_1| + |A_2| \ge k$**
  - 这意味着第 $k$ 小的数不在 $A_1$ 里，但加上 $A_2$ 后就够到了 $k$。因为 $A_2$ 里的数都等于 $mm$。
  - **动作**：直接返回 $mm$。找到了！
- **情况 3：$|A_1| + |A_2| < k$**
  - 这意味着前两部分加起来都不够 $k$ 个，目标肯定在“大数组” $A_3$ 里。
  - **动作**：递归在 $A_3$ 中找。
  - **注意**：这时候我们要找的不再是第 $k$ 小，而是第 $k - |A_1| - |A_2|$ 小。因为我们已经剔除了前两个集合的元素。

#### **复杂度证明 **

- 由于每次划分至少剔除 $30\%$ 的元素，最坏情况下递归子问题的规模为 $7n/10$。

- 递归方程：

  $$T(n) \le T(n/5) + T(7n/10) + O(n)$$

  - $T(n/5)$：寻找“中位数的中位数”的时间。
  - $T(7n/10)$：最坏情况下的递归求解时间。
  - $O(n)$：分组、寻找组中位数及划分的时间。

- **结论**：因 $1/5 + 7/10 = 0.9 < 1$，由主定理或替换法可证 **$T(n) = O(n)$**。

## 快速排序

#### **1. 核心思想**

快速排序与合并排序不同，它是 **“先处理，后递归”**。

- **合并排序**：先把数组切两半，排好序后再**合并**（重在合并）。
- **快速排序**：先把数组按大小分两边，然后再**递归**（重在划分）。

#### 伪代码

主过程：QUICKSORT(A, p, r)

```c++
Procedure QUICKSORT(A, p, r)
    // 递归基：如果 p >= r，说明子数组只有一个元素或空，不用排
    if p < r then
        // 1. 分解：核心操作，划分数组
        // 执行完这一步，A[q] 就已经在它最终应该在的位置上了
        q = PARTITION(A, p, r)
        
        // 2. 治理：递归排序左半部分
        QUICKSORT(A, p, q - 1)
        
        // 3. 治理：递归排序右半部分
        QUICKSORT(A, q + 1, r)
    end if
End Procedure
```

PARTITION (双指针交换法)

```c++
// 划分函数：返回基准值的最终下标
Function PARTITION(A, low, high)
    // 1. 选基准：通常取第一个元素
    pivot = A[low]
    
    // 2. 初始化指针
    i = low
    j = high

    // 3. 开始扫描
    // 只要 i 和 j 没相遇，就一直做
    while i < j do
        
        // 【关键点】：必须先从右边往左扫描！
        // 找一个比 pivot 小的数
        while (i < j) and (A[j] >= pivot) do
            j = j - 1
        end while
        
        // 再从左边往右扫描
        // 找一个比 pivot 大的数
        while (i < j) and (A[i] <= pivot) do
            i = i + 1
        end while

        // 如果还没相遇，说明找到了两个“不听话”的元素
        // A[i]太大该去右边，A[j]太小该去左边
        // 于是交换它俩
        if i < j then
            Swap(A[i], A[j])
        end if

    end while

    // 4. 基准归位
    // 循环结束时，i 和 j 撞在一起了
    // 把基准值 pivot (即 A[low]) 和相遇点 A[i] 交换
    Swap(A[low], A[i])
    
    // 5. 返回基准现在的坐标
    return i
End Function
```

#### 时间复杂度

##### 1. 最坏情况 (Worst Case) —— $O(n^2)$

场景：

每次划分都极其不平衡，基准值 (Pivot) 总是最大或最小的元素。

例如：数组已经有序，我们每次选第一个做基准。

- 左边子数组规模：$0$
- 右边子数组规模：$n-1$

证明推导：

代入通用公式（$k=0$）：

​						$$T(n) = T(0) + T(n-1) + cn$$

因为 $T(0) = 0$（空数组不需要排），所以：

​						$$T(n) = T(n-1) + cn$$

这是一个等差数列的递归。我们要把每一次的代价加起来：

​						$$T(n) = cn + c(n-1) + c(n-2) + \dots + c(1)$$

提取常数 $c$：

​						$$T(n) = c \cdot (n + (n-1) + \dots + 1)$$

利用等差数列求和公式：

​						$$T(n) = c \cdot \frac{n(n+1)}{2} = \frac{c}{2}n^2 + \frac{c}{2}n$$

忽略低阶项和常数：

​						$$T(n) = O(n^2)$$

#### 2. 最好情况 (Best Case) —— $O(n \log n)$

场景：

每次划分都完美平衡，基准值正好是中位数。

- 左边规模：$\approx n/2$
- 右边规模：$\approx n/2$
- 用 **递归树** 理解：
  - 每一层的总工作量都是 $cn$。
  - 因为每次减半，树的高度是 $\log_2 n$。
  - 总时间 = 层数 $\times$ 每层工作量 = $\log_2 n \cdot cn = O(n \log n)$。

#### 3.平均情况 (Average Case) —— $O(n \log n)$

## 大整数乘法

#### **1. 问题背景**

假设我们有两个非常大的整数 $X$ 和 $Y$，它们都有 $n$ 位数。

- **传统方法（小学竖式）**：我们需要让 $X$ 的每一位去乘 $Y$ 的每一位。
- **运算次数**：$n \times n = n^2$ 次乘法。

#### **2. 分治法思路 (Karatsuba 算法)**

我们尝试用分治法把大数拆小。

假设 $X$ 和 $Y$ 都是 $n$ 位数，我们将它们从中间切开，分成高位和低位：

- $X = A \cdot 10^{n/2} + B$ （$A$ 是高位，$B$ 是低位）
- $Y = C \cdot 10^{n/2} + D$ （$C$ 是高位，$D$ 是低位）

直接相乘：

​		$$X \cdot Y = (A \cdot 10^{n/2} + B)(C \cdot 10^{n/2} + D)$$$$= AC \cdot 10^n + (AD + BC) \cdot 10^{n/2} + BD$$

这里我们需要计算 4 次 $n/2$ 位数的乘法：$AC, AD, BC, BD$。

​								$$T(n) = 4T(n/2) + O(n) \Rightarrow O(n^2)$$

这并没有改进，还是 $O(n^2)$。

Karatsuba 的神来之笔：

他发现中间那一项 $(AD + BC)$ 不需要算两次乘法，可以通过简单的加减法凑出来：

​								$$(A+B)(C+D) = AC + AD + BC + BD$$

所以

​								$$AD + BC = (A+B)(C+D) - AC - BD$$

**现在的计算步骤：**

1. 计算 $U = A \cdot C$
2. 计算 $V = B \cdot D$
3. 计算 $W = (A+B)(C+D)$
4. 中间项 $Z = W - U - V$
5. 结果 $= U \cdot 10^n + Z \cdot 10^{n/2} + V$

**变化：** 我们只需要计算 **3次** 乘法 ($U, V, W$)，而不是 4 次！

#### 伪代码

```c++
// 计算 X * Y，其中 X 和 Y 的长度约为 n
Function Karatsuba(X, Y, n)
    // 1. 递归基：如果数字很小（比如 1 位数），直接乘
    if n <= 1 then
        return X * Y
    end if

    // 2. 分解 (Divide)
    // 设 m = n / 2
    // X = A * 10^m + B
    // Y = C * 10^m + D
    // (在代码中通常通过字符串切分或位移运算来实现)
    A = X / 10^m
    B = X % 10^m
    C = Y / 10^m
    D = Y % 10^m

    // 3. 治理 (Conquer) - 递归计算三次乘法
    U = Karatsuba(A, C, m)        // 算 AC
    V = Karatsuba(B, D, m)        // 算 BD
    W = Karatsuba(A+B, C+D, m)    // 算 (A+B)(C+D)

    // 4. 合并 (Combine)
    // Z = AD + BC = W - U - V
    Z = W - U - V
    
    // 结果 = U*10^2m + Z*10^m + V
    return U * 10^(2*m) + Z * 10^m + V
End Function
```

#### **4. 时间复杂度分析**

- **分解**：加减法和位移，耗时 $O(n)$。
- **子问题**：我们把原问题分解成了 **3** 个规模为 **$n/2$** 的子问题。
- **合并**：加减法和位移，耗时 $O(n)$。

递归方程：

​						$$T(n) = 3T(n/2) + O(n)$$

求解：

根据主定理 (Master Theorem)，$a=3, b=2$。

​						$$\log_b a = \log_2 3 \approx 1.585$$

因为 $1.585 > 1$，所以：

​						$$T(n) = O(n^{\log_2 3}) \approx O(n^{1.585})$$

## Strassen矩阵算法(这么恶心的算法要考出来就是让你吃屎的，我打赌肯定不考：)

#### **1. 问题背景**

假设我们要计算两个 $n \times n$ 的矩阵 $A$ 和 $B$ 的乘积 $C = A \times B$。

- 传统方法（三重循环）：

  ​                   $$C_{ij} = \sum_{k=1}^n A_{ik} \cdot B_{kj}$$

  需要三层循环（行、列、中间变量），每层 $n$ 次。

  时间复杂度：$O(n^3)$。

#### **2. 朴素的分治法 (Naive Divide & Conquer)**

我们可以把矩阵切成 4 个 $n/2 \times n/2$ 的子矩阵：

​						$$A = \begin{bmatrix} A_{11} & A_{12} \\ A_{21} & A_{22} \end{bmatrix}, \quad B = \begin{bmatrix} B_{11} & B_{12} \\ B_{21} & B_{22} \end{bmatrix}$$

根据矩阵乘法规则：

​						$$C = \begin{bmatrix} C_{11} & C_{12} \\ C_{21} & C_{22} \end{bmatrix}$$

其中：

- $C_{11} = A_{11}B_{11} + A_{12}B_{21}$
- $C_{12} = A_{11}B_{12} + A_{12}B_{22}$
- $C_{21} = A_{21}B_{11} + A_{22}B_{21}$
- $C_{22} = A_{21}B_{12} + A_{22}B_{22}$

分析：

这里总共做了 8次 子矩阵乘法（每个 $C_{ij}$ 需要 2 次乘法）和 4次 矩阵加法。

递归方程：

​						$$T(n) = 8T(n/2) + O(n^2)$$

- $8T(n/2)$：8 次递归乘法。
- $O(n^2)$：矩阵加法的时间（遍历 $n^2$ 个元素）。

结果：

利用主定理，$\log_2 8 = 3$，所以 $T(n) = O(n^3)$。

结论：这种朴素分治法完全没有改进。

#### **3. Strassen 算法的改进**

Strassen 发现了一组神奇的公式，可以通过 **7次** 乘法（而不是 8 次）算出 $C$ 的四个部分。虽然加减法次数变多了，但乘法次数减少了。

构造 7 个中间矩阵 ($M_1 \sim M_7$)：

(考试通常不需要背这个复杂的公式，但要理解其形式)

- $M_1 = (A_{11} + A_{22}) \times (B_{11} + B_{22})$
- $M_2 = (A_{21} + A_{22}) \times B_{11}$
- $M_3 = A_{11} \times (B_{12} - B_{22})$
- $M_4 = A_{22} \times (B_{21} - B_{11})$
- $M_5 = (A_{11} + A_{12}) \times B_{22}$
- $M_6 = (A_{21} - A_{11}) \times (B_{11} + B_{12})$
- $M_7 = (A_{12} - A_{22}) \times (B_{21} + B_{22})$

利用上面的 7 个 $M$，通过**加减法**拼出最终结果：

1. **$C_{11} = M_1 + M_4 - M_5 + M_7$**
2. **$C_{12} = M_3 + M_5$**
3. **$C_{21} = M_2 + M_4$**
4. **$C_{22} = M_1 - M_2 + M_3 + M_6$**

#### 伪代码：

```c++
Function STRASSEN(A, B, n)
    // 递归基
    if n == 1 then
        return A * B
    end if

    // 1. 分解 (Divide)
    // 将 A, B 分割成 n/2 x n/2 的子矩阵 (A11...B22)
    // 计算中间变量需要的加减法 (比如 S1 = B12 - B22)
    
    // 2. 治理 (Conquer) - 关键：只做 7 次递归乘法
    M1 = STRASSEN(A11 + A22, B11 + B22, n/2)
    M2 = STRASSEN(A21 + A22, B11, n/2)
    ...
    M7 = STRASSEN(A12 - A22, B21 + B22, n/2)

    // 3. 合并 (Combine)
    // 通过加减法计算 C 的四个部分
    C11 = M1 + M4 - M5 + M7
    C12 = M3 + M5
    ...
    
    return C
End Function
```

#### **5. 时间复杂度分析 (主定理应用)**

这是最精彩的部分，请拿出刚才记的 **主定理** 来套用。

- 递归方程：

  $$T(n) = 7T(n/2) + O(n^2)$$

  - **$a = 7$**：子问题个数减少到了 7 个。
  - **$b = 2$**：每次矩阵规模减半。
  - **$f(n) = O(n^2)$**：分解和合并（矩阵加减法）依然是 $n^2$ 级别的。

- 计算临界值：

  ​						$$n^{\log_b a} = n^{\log_2 7}$$

  由于 $\log_2 7 \approx 2.807$。

- 比较：

  $f(n) = n^2$ 比 $n^{2.81}$ 小（小了一个 $n^{0.81}$ 的多项式因子）。

  属于情况 1 (Case 1)：叶子占主导。

- 最终结果：

  ​						$$T(n) = \Theta(n^{\log_2 7}) \approx O(n^{2.81})$$

## 最近点对问题

#### **1. 问题背景**

- **输入**：平面上 $n$ 个点 $P = \{p_1, p_2, \dots, p_n\}$。
- **输出**：距离最近的两个点之间的距离。
- **朴素做法 (Brute Force)**：
  - 算出所有点两两之间的距离。
  - 点对数量是 $C_n^2 = \frac{n(n-1)}{2}$。
  - 时间复杂度：**$O(n^2)$**。

#### **2. 分治法思路**

我们要利用分治法把平面一分为二。

**第一步：预处理 (Preprocessing)**

- 先把所有点按照 **x 坐标** 从小到大排序，得到数组 $P_x$。
- （优化版本通常也要求按 y 坐标排序得到 $P_y$，以便后面合并时用）。

**第二步：分解 (Divide)**

- 在 x 轴上画一条垂直线 $l$（通常取中间点的 x 坐标），把所有点分成左右两半：
  - 左半边集合 $S_L$
  - 右半边集合 $S_R$

**第三步：治理 (Conquer)**

- 递归地在 $S_L$ 中找到最近距离，记为 $\delta_L$。
- 递归地在 $S_R$ 中找到最近距离，记为 $\delta_R$。
- 令 **$\delta = \min(\delta_L, \delta_R)$**。
  - 这意味着：如果不考虑跨越左右两边的点对，目前找到的最短距离就是 $\delta$。

**第四步：合并 (Combine) —— 关键难点**

- 我们还需要检查一种情况：**一个点在左边，一个点在右边**，且它俩的距离小于 $\delta$。
- **筛选范围 (The Strip)**：
  - 显然，离中线 $l$ 距离超过 $\delta$ 的点根本不用看。
  - 我们只取中线左右各 $\delta$ 宽度内的点，形成一个**宽为 $2\delta$ 的带状区域 (Strip)**。
- **检查带状区域**：
  - 将这个区域内的点按 **y 坐标** 排序。
  - **神奇的结论**：对于带状区域里的每个点 $p$，我们只需要检查它**下方紧邻的 7 个点**（有些教材证明是 5 个或 6 个，常数级）即可！
  - 如果这 7 个点里没有比 $\delta$ 更近的，那就不可能有了

#### 伪代码：

```
// 输入：按 x 坐标排好序的点集 P
Function ClosestPair(P)
    n = length(P)
    
    // 1. 递归基：如果点很少（比如 <= 3），直接暴力算
    if n <= 3 then
        return BruteForce(P)
    end if

    // 2. 分解
    mid = n / 2
    Px_Left  = P[1...mid]
    Px_Right = P[mid+1...n]
    
    // 3. 治理 (递归)
    dL = ClosestPair(Px_Left)
    dR = ClosestPair(Px_Right)
    d  = min(dL, dR)   // 当前已知的最小值

    // 4. 合并 (处理跨越中线的点对)
    // 收集所有在带状区域内的点 (x 坐标距中线 < d)
    Strip = []
    midX = P[mid].x
    for point p in P do
        if abs(p.x - midX) < d then
            Strip.append(p)
        end if
    end for

    // 必须按 Y 坐标排序 Strip 中的点
    Sort Strip by y-coordinate
    
    // 扫描 Strip
    min_dist = d
    for i = 1 to length(Strip) do
        // 只需检查后面紧邻的 7 个点
        for j = i + 1 to length(Strip) do
            // 如果 Y 轴距离都已经超过 min_dist，后面的点更不用看了，直接 break
            if (Strip[j].y - Strip[i].y) >= min_dist then
                break
            end if
            
            // 计算真实距离并更新
            real_dist = dist(Strip[i], Strip[j])
            if real_dist < min_dist then
                min_dist = real_dist
            end if
        end for
    end for

    return min_dist
End Function
```

#### **4. 时间复杂度分析**

我们需要列出递归方程。

- **分解**：寻找中点，划分数组，$O(1)$（如果是数组下标操作）。
- **子问题**：$2T(n/2)$。
- **合并**：
  - 这一步最耗时的是 **“按 Y 坐标排序”**。
  - 如果在合并里直接做快排，那就是 $O(n \log n)$，总复杂度会变成 $O(n \log^2 n)$。
  - **优化技巧**：如果我们一开始就在递归外把点按 Y 排好序，并在递归过程中把排序好的数组传进去（类似归并排序的 merge 过程），合并步就只需要扫描一遍，$O(n)$。

采用优化后的合并 ($O(n)$)：

​						$$T(n) = 2T(n/2) + O(n)$$

根据主定理（Case 2）：

​						$$T(n) = O(n \log n)$$
