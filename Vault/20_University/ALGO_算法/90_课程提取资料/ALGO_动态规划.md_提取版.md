# 动态规划

> cover：小孩不哭站起来

## 动态规划的三个基本要素

#### 1. 最优化子结构 (Optimal Substructure)

这是动态规划的基础。它意味着**问题的最优解包含其子问题的最优解**。

- **理解：** 我们可以通过组合子问题的最优解，构造出原问题的最优解。
- **例子：** 在求从 A 点到 C 点的最短路径时，如果路径经过 B 点，那么这条最优路径一定包含了从 A 到 B 的最短路径。

#### 2. 重叠子问题 (Overlapping Subproblems)

这是动态规划区别于分治算法（如归并排序）的关键特征。

- **理解：** 在递归求解过程中，相同的子问题会被反复计算多次。
- **优化：** 动态规划通过**存储（记忆化）**这些子问题的解，避免重复劳动，从而将时间复杂度从指数级降低到多项式级。

#### 3. 无后效性 (No After-effect)

这是一个相对抽象但非常重要的约束条件。

- **理解：** 某阶段状态一旦确定，就不受这个状态以后决策的影响。也就是说，“未来”不影响“过去”，且当前状态是对过去所有决策的总结。
- **重要性：** 如果一个问题具有后效性，就不能直接使用动态规划，除非通过增加维度来“消除”后效性。

## 斐波那契数列

#### 1. 问题描述与递归定义

斐波那契数列定义为：序列中的每一个数是它前面两个数的和 1。

- $f_1 = 1, f_2 = 1$
- 当 $n \ge 3$ 时，$f(n) = f(n-1) + f(n-2)$ 2222。

#### 2.伪代码 (递归版本)

```c++
1. procedure f(n)
2. if (n=1) or (n=2) then return 1
3. else return f(n-1) + f(n-2)
```

## 最长公共子序列

#### 1. 问题描述

给定两个序列 $X=\{x_1, x_2, ..., x_m\}$ 和 $Y=\{y_1, y_2, ..., y_n\}$，我们要找出它们之间最长的公共子序列 。

- **注意**：子序列不需要连续，只要保持原本的相对顺序即可 。
- **举例**：$X=\{A, B, C, B, D, A, B\}$，$Y=\{B, D, C, A, B, A\}$。它们的LCS长度为4（例如 $\{B, C, B, A\}$） 

#### 2. 状态转移方程 (核心模块)

我们需要定义一个二维数组 $c[i, j]$ (或课件中的 $L[i,j]$)，它表示 **序列 $X$ 的前 $i$ 个字符和序列 $Y$ 的前 $j$ 个字符的最长公共子序列的长度** 

根据课件第22页，递推公式如下：

$$c[i,j] =  \begin{cases}  0 & \text{if } i=0 \text{ or } j=0 \\ c[i-1, j-1] + 1 & \text{if } i,j > 0 \text{ and } x_i = y_j \\ \max(c[i, j-1], c[i-1, j]) & \text{if } i,j > 0 \text{ and } x_i \neq y_j  \end{cases}$$

```c++
Algorithm LCS-LENGTH(X, Y)
1. m = length[X], n = length[Y]
2. 初始化第一行和第一列为 0  // 对应 c[i,0]=0 和 c[0,j]=0
3. for i = 1 to m do
4.     for j = 1 to n do
5.         if x[i] == y[j] then
6.             c[i, j] = c[i-1, j-1] + 1
7.             b[i, j] = "↖"  // 记录来源，用于回溯
8.         else if c[i-1, j] >= c[i, j-1] then
9.             c[i, j] = c[i-1, j]
10.            b[i, j] = "↑"
11.        else
12.            c[i, j] = c[i, j-1]
13.            b[i, j] = "←"
14. return c and b
```

## 矩阵链相乘问题

#### 1. 问题描述

给定 $n$ 个矩阵的序列 $\{A_1, A_2, ..., A_n\}$。

- 矩阵 $A_i$ 的维度为 $p_{i-1} \times p_i$。
- 我们要计算它们的连乘积 $A_1 A_2 ... A_n$。
- **目标**：由于矩阵乘法满足结合律（即 $(AB)C = A(BC)$），我们可以通过加括号来改变计算次序。不同的加括号方式会导致完全不同的标量乘法次数（计算成本）。我们要找到一种**加括号的方式（计算次序）**，使得总的数乘次数**最少** 

#### 2. 状态转移方程 (核心模块)

我们需要定义 $m[i, j]$ 为计算矩阵子链 $A_{i...j}$ (即 $A_i A_{i+1} ... A_j$) 所需的**最少**数乘次数 

递推公式如下 ：

$$m[i,j] =  \begin{cases}  0 & \text{if } i = j \\ \min_{i \le k < j} \{ m[i,k] + m[k+1,j] + p_{i-1}p_k p_j \} & \text{if } i < j \end{cases}$$

**解释**：

1. **基准情况 ($i=j$)**：只有一个矩阵，不需要做乘法，代价为0。
2. **递归情况 ($i<j$)**：
   - 我们在 $A_k$ 和 $A_{k+1}$ 之间把链条切断（$k$ 是分割点，范围从 $i$ 到 $j-1$）。
   - 总代价 = 左半部分的最优代价 $m[i,k]$ + 右半部分的最优代价 $m[k+1,j]$ + 这两部分结果矩阵相乘的代价。
   - 这两部分结果矩阵相乘的代价是：(左边结果的行数 $p_{i-1}$) $\times$ (中间的列数 $p_k$) $\times$ (右边结果的列数 $p_j$)。
   - 我们需要遍历所有可能的切分点 $k$，取其中的**最小值**。

#### 3.伪代码

```c++
Algorithm MATRIX-CHAIN-ORDER(p)
1. n = length[p] - 1  // n个矩阵
2. for i = 1 to n do
3.     m[i, i] = 0    // 初始化对角线为0
4. end for
5. for l = 2 to n do  // l 是链的长度，从2开始一直到n
6.     for i = 1 to n - l + 1 do
7.         j = i + l - 1      // 确定链的终点 j
8.         m[i, j] = ∞        // 初始化为无穷大
9.         for k = i to j - 1 do  // 尝试所有分割点 k
10.            q = m[i, k] + m[k+1, j] + p[i-1]*p[k]*p[j]
11.            if q < m[i, j] then
12.                m[i, j] = q
13.                s[i, j] = k   // 记录最优分割点，用于后续构造解
14.            end if
15.        end for
16.    end for
17. end for
18. return m and s
```

## Floyd求最短路径

#### 1. 问题描述

给定一个有向图 $G=(V, E)$，每条边 $(i, j)$ 有一个非负的长度（权重）$l[i, j]$。如果 $i$ 到 $j$ 没有边，则长度为 $\infty$ 

目标：我们要找出图中任意两个顶点之间的最短路径距离 

不同于 Dijkstra 算法（它是求一个源点到其他所有点的距离），Floyd 算法一次性求出所有点对之间的距离。

#### 2. 状态转移方程 (核心模块)

我们需要定义 $D_k[i, j]$ 为：**从顶点 $i$ 到顶点 $j$ 的最短路径长度，且该路径上除起点和终点外，只允许经过编号 $\le k$ 的中间顶点**

递推公式如下：

$$D_{k}[i,j] = \min \left\{ \underbrace{D_{k-1}[i,j]}_{\text{不经过顶点 } k}, \quad \underbrace{D_{k-1}[i,k] + D_{k-1}[k,j]}_{\text{经过顶点 } k} \right\}$$

解释 ：

这是动态规划的典型思路。假设我们已经知道了只经过前 $k-1$ 个点时的最短路径。现在引入第 $k$ 个点，对于任意两点 $i$ 和 $j$，新的最短路径有两种可能：

1. **不经过 $k$**：路径依然是旧的路径，长度保持 $D_{k-1}[i, j]$ 不变。

2. 经过 $k$：路径变成了“从 $i$ 走到 $k$”加上“从 $k$ 走到 $j$”。长度为 $D_{k-1}[i, k] + D_{k-1}[k, j]$。

   我们取这两者中的较小值。

#### 3.伪代码

```c++
Algorithm FLOYD(l)
输入: n x n 维矩阵 l[1..n, 1..n]
输出: 矩阵 D，其中 D[i,j] 是 i 到 j 的距离

1. D = l              // 将输入矩阵复制到 D，初始化 D[i,i]=0, 无边为 ∞
2. for k = 1 to n do  // 核心循环：依次把顶点 k 作为中间跳板
3.     for i = 1 to n do
4.         for j = 1 to n do
5.             D[i, j] = min{ D[i, j], D[i, k] + D[k, j] }
6.         end for
7.     end for
8. end for
9. return D
```

## 0-1背包问题

#### 1. 问题描述

- **输入**：
  - 一个物品集合 $U=\{u_1, u_2, ..., u_n\}$。
  - 每个物品 $u_j$ 有两个属性：**体积/重量 ($s_j$)** 和 **价值 ($v_j$)**。
  - 一个背包，它的**最大容量**为 $C$ 。

- **目标**：
  - 从这些物品中选出一部分装入背包。
  - 要求：被选物品的**总体积不超过 $C$**。
  - 目的：使被选物品的**总价值最大化** 。

#### 2. 状态转移方程 (核心模块)

我们需要定义二维数组 $V[i, j]$。

**含义**：表示仅从**前 $i$ 个物品** $\{u_1, ..., u_i\}$ 中选择，放入容量为 **$j$** 的背包中，所能获得的最大价值 。

递推公式如下 ：

$$V[i, j] =  \begin{cases}  0 & \text{if } i=0 \text{ or } j=0 \\ V[i-1, j] & \text{if } j < s_i \\ \max \{ V[i-1, j], \quad V[i-1, j-s_i] + v_i \} & \text{if } j \ge s_i \end{cases}$$

详细解释：

这是典型的**“选或不选”**决策：

1. **装不下 ($j < s_i$)**：如果当前背包容量 $j$ 甚至连第 $i$ 个物品都装不下 ($s_i > j$)，那没得选，只能不装。最大价值等于“前 $i-1$ 个物品在容量 $j$ 下的价值”，即 $V[i-1, j]$。
2. **能装下 ($j \ge s_i$)**：此时有两种选择，我们取最大值：
   - **不装第 $i$ 个物品**：价值依然是 $V[i-1, j]$。
   - **装第 $i$ 个物品**：此时价值 = 第 $i$ 个物品的价值 $v_i$ + “剩余空间 ($j - s_i$) 能装的前 $i-1$ 个物品的最大价值”。即 $v_i + V[i-1, j - s_i]$。

#### 3.伪代码

```c++
Algorithm KNAPSACK(U, C)
1. for i = 0 to n do
2.     V[i, 0] = 0    // 容量为0时，价值为0
3. end for
4. for j = 0 to C do
5.     V[0, j] = 0    // 没有物品时，价值为0
6. end for
7. for i = 1 to n do  // 遍历每一个物品
8.     for j = 1 to C do  // 遍历每一个容量刻度
9.         V[i, j] = V[i-1, j]   // 先假设不选这个物品
10.        if s[i] <= j then     // 如果装得下
11.       
    // 尝试装入，看是否更划算
12.            V[i, j] = max{ V[i, j], V[i-1, j-s[i]] + v[i] }
13.        end if
14.    end for
15. end for
16. return V[n, C]
```
