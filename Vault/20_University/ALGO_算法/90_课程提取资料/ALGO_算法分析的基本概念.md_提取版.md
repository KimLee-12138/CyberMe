# 算法分析的基本概念

> cover：小孩不哭站起来

## 一、算法的定义

- 算法是定义好的能够完成一定任务的有限指令集合，通常它由一个初始化状态，终止于一个对应的可识别的结束状态。

## 二、算法的五个特性

1. 有效性
   算法必须在有限的时间能够完成，甚至用纸和笔完成
2. 确定性
   算法的每一步能够清楚的定义.
3. 有限性
   算法能够在有限的步骤完成
4. 输入
     算法有0个或者多个输入
5. 输出
     算法有一个或者多个输出

- 满足有效性，确定性，输入，输出特征的程序是一个过程而不是算法,举例：操作系统是过程而不是算法

## 三、搜索（用作时间复杂度的引入案例）

#### 问题描述：

设A[1..n] 为一个包含n个元素的数组.考虑这样的问题：判断给定元素X是否在A中。结果:找到这个元素的下标j, 1<=j <= n，使得x=A[j];如果不存在这个元素,j=0.

#### 线性搜索：

- 扫描数组A，依次将每个元素和x进行比较. 
- 如何经过j次比较，1<=j <= n，找到x, x=A[j]，则返回下标 j;
- 否则返回下标0，表示搜索失败了.

##### 伪代码：

```cpp
Input: An array A [1…n] of n elements and an element x. 
Output: j if x = A [j], 1 <= j <= n, and 0 otherwise.
    1. j = 1
    2.while (j < n) and (x != A [j])
    3.  j = j + 1
    4. end while
    5. if x = A [j] then return j else return 0.
```

#### 二分搜索：

- 设A [low…high] 为一个非递减元素排序的数组. 其中 A [mid] 表示A的中间位置的元素；假设 x>A [mid]. 
- 如何在数组A中, 那么X在A [mid]的后面 
- 我们进一步在子集A[mid+1…high]中搜索x

##### 伪代码：

```cpp
Input: 数组A [1…n]包含n个非递减次序的元，查找元素x. 
Output: j if x = A [j], 1 <= j <= n, and 0 otherwise.
    1.low=1; high = n; j = 0
    2.while (low <= high) and (j = 0)
    3.   mid= low+high/2(取小)
    4.   if x = A [mid] then j =mid
    5.   else if x< A [mid] then high = mid – 1
    6.   else low = mid + 1
    7. end while
    8.return j
```

##### 二分搜索算法分析：

- **定理:规模为n的二分搜索算法中元素比较次数最多 $\lfloor{logn}\rfloor+1$.** 

  什么时候达到最大的比较次数呢？

- 在while循环的第j次循环时，剩余元素的数目是$\lfloor{n/2^{j-1}}\rfloor$。
- 循环最大次数是满足条件$\lfloor{n/2^{j-1}}\rfloor=1$时的j值，因此， $1\leq\mathrm{n/2^{j-1}<2}$ → $j=\lfloor{logn}\rfloor+1$

## 四、时间复杂度

#### 定义：

- 对于任何计算步骤,它的代价有一个时间常量作为上界,而不管输入数据或执行的算法,我们称之为“元运算”

- 注：时间复杂度不是程序运行所花费的实际时间,而是对于输入规模n的算法元运算的次.

#### 渐进符号：

- $O(g(n))$：增长次数小于等于$g(n)$以及其常数倍的函数集合——渐进上界

  - **直观含义**：通常用于描述**最坏情况**

  - **数学定义：**存在常数 $c > 0$ 和 $n_0 > 0$，使得当 $n \ge n_0$ 时，满足：

    ​                                       $$0 \le f(n) \le c \cdot g(n)$$

  - **例子**：

    - $3n^2 + 10n + 5 = O(n^2)$ （主要看最高次项）
    - $3n^2 + 10n = O(n^3)$ 也是对的（因为 $n^2$ 确实小于 $n^3$），但我们通常习惯找最贴切的那个上界。
    - **例：证明100n+5 ∈O($n^2$)**
      - $100n+5 \le100n+n(当n \geq 5)=101n \le101n^2$
              取$c=101，n_0=5$
        $100n+5 \le100n+5n(当n \geq 1)=105n \le105n^2$
             取$c=105，n_0=1$

- $\Omega({g(n))}$：增长次数大于等于$g(n)$以及其常数倍的函数集合——渐进下界

  - **直观含义**：通常用于描述**最好情况**或者证明某个问题不可能再快了。

  - **数学定义：**存在常数 $c > 0$ 和 $n_0 > 0$，使得当 $n \ge n_0$ 时，满足：

    ​                                       $$0 \le c \cdot g(n) \le f(n)$$

  - **例子**：

    - $3n^2 + 10n = \Omega(n^2)$
    - $3n^2 + 10n = \Omega(n)$ （因为 $n^2$ 确实大于 $n$）

- $\Theta({g(n)})$：增长次数等于$g(n)$以及其常数倍的函数集合——渐进紧确界

  - **直观含义**：算法运行时间的“**真实阶数**”。当一个算法的**上界**和**下界**是同一个级别时，我们就用 $\Theta$ 来表示。这是最精确的描述。

  - **数学定义：**存在常数 $c_1, c_2 > 0$ 和 $n_0 > 0$，使得当 $n \ge n_0$ 时，满足：

    $$0 \le c_1 \cdot g(n) \le f(n) \le c_2 \cdot g(n)$$

  - **例子**：

    - $3n^2 + 10n = \Theta(n^2)$
    - **注意**：$3n^2 \ne \Theta(n^3)$，因为虽然它小于 $n^3$，但它不“等于”$n^3$ 的量级。

#### 利用极限比较增长率:

​	比较两个特定函数的增长率

​				$\lim_{n\to\infty}\frac{t(n)}{g(n)}=\begin{cases}\quad0\\c>0\\\quad\infty&\end{cases}$

- 情况①②意味着$t(n) ∈$$O(g(n))$
- 情况②③意味着$t(n) ∈$ $\Omega({g(n))}$
- 情况②意味着$t(n) ∈ $ $\Theta({g(n)})$

#### 使用递推关系

- **$n$**：表示数组或数据实例的大小（元素的个数）。
- **$C(n)$**：表示在大小为 $n$ 的实例中，执行比较操作的次数。

##### 建立递推公式 (Recurrence Relation)

这是分析的核心部分，它给出了一个分段函数：

$$C(n) \le \begin{cases} 1 & \text{if } n = 1 \\ C(\lfloor n/2 \rfloor) + 1 & \text{if } n \ge 2 \end{cases}$$

- **基准情形 ($n=1$)**：如果数组只有一个元素，只需要进行 **1** 次比较就能确定是否找到，或者确定不存在。
- **递归情形 ($n \ge 2$)**：
  - **$C(\lfloor n/2 \rfloor)$**：表示问题规模缩小了一半。因为二分查找每次都排除掉一半的元素，只在剩下的一半中继续查找。
  - **$+ 1$**：表示当前这一层递归中做了一次比较（将目标值与中间值 `mid` 进行比较）。

##### 推导过程 (Solving the Recurrence)

PPT 展示了通过 **迭代法（Substitution Method）** 来解这个递推公式的过程：

1. **第一步**：$C(n) \le C(\lfloor n/2 \rfloor) + 1$
2. **第二步**：将 $n/2$ 带入公式，再次展开 $\rightarrow C(\lfloor n/4 \rfloor) + 1 + 1$
3. **规律**：每次展开，问题规模除以 2，后面的常数项加 1。
4. **以此类推**：经过 $k$ 次查找后，规模变为 $n/2^k$，比较次数增加 $k$。

##### 最终结论

- 当问题规模缩小到 1 时停止，即 $n/2^k = 1$，解得 $k = \log_2 n$。

- 因此，总的比较次数公式为：

  $$C(n) = \lfloor \log n \rfloor + 1$$

- 用大O符号表示时间复杂度：

  $$C(n) = O(\log n)$$
