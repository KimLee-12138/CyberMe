2. 构建状态空间树  (State Space Tree)  
为了系统地搜索解空间，我们将解空间组织成一棵树，称为 状态空间树  
树的结构 ：从根节点出发，每一层代表解向量中的一个分量   的选择。
路径即状态 ：从根节点到树中任意节点的路径，确定了问题的一个状态  
答案状态 ：那些满足问题所有条件的解状态，称为答案状态  
3. 搜索策略：深度优先搜索  (Depth-First Search)  
回溯法是按照 深度优先 的策略在状态空间树中搜索解的  。
节点的状态定义 （这是理解算法运行过程的关键）：
活结点  (Live Node) ：一个已经生成，但其所有儿子结点还没有全部生成的结点  。
E- 结点  (E-Node / Expanding Node) ：当前正在扩展（正在生成其儿子）的活结点  。
死结点  (Dead Node) ：不再进一步扩展，或者其所有儿子结点都已经生成的结点  。
4. 核心技巧：剪枝函数  (Pruning Functions)  
回溯法之所以比 “ 傻搜 ” （蛮力法）快，是因为它在搜索过程中会 “ 剪枝 ” 。如果判断出某个路径不可能得
到解，就立刻停止该路径的搜索，不再生成其子树  。
1. 约束函数  (Constraint Function) ：在扩展节点处检查是否满足显式的约束条件（例如： 3 着色
中，检查相邻节点颜色是否冲突）。如果不满足，剪去该子树。
2. 限界函数  (Bounding Function) ：用于最优化问题（如求最小成本）。如果当前部分解的预估成
本已经超过了已知的一个解的成本，就剪去该子树（因为不可能得到更优解）。

1. 核心概念  
分支限界法 的基本思想是在生成当前节点（称为  E- 结点，即正在扩展的结点）的 全部儿子之后，再选择

1. 分支  (Branching) ：系统地生成子节点。
2. 限界  (Bounding) ：利用 限界函数 （Bounding Function ）计算每个节点的成本或界限。如果某个
子节点的界限表明它不可能包含最优解（或无法达到答案），则直接丢弃该节点（剪枝），不再将
其加入活结点表

这种方法的缺陷  
从图中您可以非常直观地看到：
1. 盲目性 ：算法只是机械地按照 “ 上右下左 ” 的顺序生成，完全不知道哪个节点离目标更近。比如，也
许节点  4 （向下移动）离目标最远，但算法依然会把它加入队列并将来花时间去扩展它  。
2. 状态空间爆炸 ：随着深度的增加，节点数呈指数级增长。如果目标解在很深的地方， FIFO 方法通
过广度优先搜索，内存（队列）会迅速爆满。
