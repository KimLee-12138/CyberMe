# 动态规划

> 一句话：把原问题分解为重叠子问题，存储子问题答案避免重复计算，将指数级降为多项式级。

## 一、DP 的三个基本要素

### 1.1 最优子结构

**问题的最优解包含其子问题的最优解**。

> 例如：从 A 到 C 的最短路径经过 B，则 A→B 这一段也是最短路径。

### 1.2 重叠子问题

**递归求解中相同子问题被反复计算**。DP 通过**记忆化存储**避免重复。

> 这是 DP 与分治法的关键区别：分治的子问题不相交（如归并排序切分后的子数组相互独立），DP 的子问题大量重叠。

### 1.3 无后效性

**某阶段状态一旦确定，就不受之后决策的影响**。当前状态是对过去所有决策的完整总结。

## 二、斐波那契数列 — DP入门

### 2.1 递归版本（指数级）

$$F(n) = F(n-1) + F(n-2), \quad F(0)=0, F(1)=1$$

纯递归的递归树中有大量重复计算：$F(5)$ 算了 $F(2)$ 三次。时间复杂度 $O(2^n)$。

### 2.2 DP 版本（线性级）

```c++
f[0] = 0; f[1] = 1
for i = 2 to n:
    f[i] = f[i-1] + f[i-2]
return f[n]
```

自底向上填表，每个子问题只算一次。$O(n)$。

## 三、最长公共子序列

### 3.1 问题

给定序列 $X = x_1x_2\dots x_m$ 和 $Y = y_1y_2\dots y_n$，求最长的公共子序列（不需要连续，保持相对顺序即可）。

**例**：$X = ABCBDAB$，$Y = BDCABA$，LCS 为 $BCBA$，长度 4。

### 3.2 状态定义

$L[i][j]$：$X$ 的前 $i$ 个字符和 $Y$ 的前 $j$ 个字符的 LCS 长度。

### 3.3 状态转移方程

$$L[i][j] = \begin{cases} 0 & i=0 \text{ 或 } j=0 \\ L[i-1][j-1] + 1 & X_i = Y_j \\ \max(L[i][j-1], L[i-1][j]) & X_i \neq Y_j \end{cases}$$

**理解**：末尾字符相同则匹配（左上+1）；不同则各退一格取最大值。

### 3.4 伪代码

```
Algorithm LCS(X, Y)
    m = len(X); n = len(Y)
    for i = 0 to m: L[i][0] = 0
    for j = 0 to n: L[0][j] = 0
    for i = 1 to m:
        for j = 1 to n:
            if X[i] == Y[j]: L[i][j] = L[i-1][j-1] + 1
            else: L[i][j] = max(L[i-1][j], L[i][j-1])
    return L[m][n]
```

**复杂度**：$O(mn)$，空间 $O(mn)$（可优化至 $O(\min(m,n))$）。

## 四、矩阵链相乘

### 4.1 问题

给定 $n$ 个矩阵 $A_1, A_2, \dots, A_n$，其中 $A_i$ 的维度为 $p_{i-1} \times p_i$。通过加括号改变乘法次序，使总的标量乘法次数最少。

### 4.2 状态定义

$m[i][j]$：计算子链 $A_{i\dots j}$ 所需的最少乘法次数。

### 4.3 状态转移方程

$$m[i][j] = \begin{cases} 0 & i = j \\ \min\limits_{i \le k < j} \{ m[i][k] + m[k+1][j] + p_{i-1}p_k p_j \} & i < j \end{cases}$$

**理解**：在 $A_k$ 处切分，左半最优化 + 右半最优化 + 两部分结果相乘的代价。

### 4.4 伪代码（$O(n^3)$）

```
for i = 1 to n: m[i][i] = 0
for len = 2 to n:                    // 链长递增
    for i = 1 to n - len + 1:
        j = i + len - 1
        m[i][j] = ∞
        for k = i to j - 1:          // 尝试所有切分点
            q = m[i][k] + m[k+1][j] + p[i-1] * p[k] * p[j]
            if q < m[i][j]: m[i][j] = q; s[i][j] = k
```

三层循环 → $O(n^3)$。

## 五、Floyd 全源最短路径

### 5.1 问题

求图 $G=(V,E)$ 中**任意两个顶点之间**的最短路径距离。与 Dijkstra（单源）不同，Floyd 一次性算出所有点对。

### 5.2 状态定义

$D_k[i][j]$：从 $i$ 到 $j$ 且中间顶点编号均 $\le k$ 的最短路径长度。

### 5.3 状态转移

$$D_k[i][j] = \min\big(D_{k-1}[i][j], \quad D_{k-1}[i][k] + D_{k-1}[k][j]\big)$$

**理解**：引入第 $k$ 个顶点作为中转。若经过 $k$ 更短则更新。

### 5.4 伪代码（$O(n^3)$）

```
for k = 1 to n:
    for i = 1 to n:
        for j = 1 to n:
            D[i][j] = min(D[i][j], D[i][k] + D[k][j])
```

三重循环的顺序**不可交换**：$k$ 必须在最外层（因为每个阶段所需的 $k-1$ 信息必须在进入下一阶段前完整计算）。

## 六、0-1 背包问题

### 6.1 问题

$n$ 个物品，第 $i$ 个物品体积 $s_i$、价值 $v_i$。背包容量 $C$。每件物品**要么全拿要么不拿**，求总价值最大。

### 6.2 状态定义

$V[i][j]$：仅从前 $i$ 个物品中选择，放入容量 $j$ 的背包的最大价值。

### 6.3 状态转移（选或不选）

$$V[i][j] = \begin{cases} 0 & i=0 \text{ 或 } j=0 \\ V[i-1][j] & j < s_i \quad \text{(装不下)} \\ \max(V[i-1][j], \; V[i-1][j-s_i] + v_i) & j \ge s_i \end{cases}$$

### 6.4 伪代码（$O(nC)$）

```
for i = 0 to n: V[i][0] = 0
for j = 0 to C: V[0][j] = 0
for i = 1 to n:
    for j = 1 to C:
        V[i][j] = V[i-1][j]
        if s[i] <= j:
            V[i][j] = max(V[i][j], V[i-1][j-s[i]] + v[i])
return V[n][C]
```

可优化空间至 $O(C)$（只用一行 + 逆序遍历）。

## 七、DP vs 贪心 vs 分治

| 维度 | 分治 | 动态规划 | 贪心 |
|------|------|---------|------|
| 子问题关系 | 独立不相交 | 大量重叠 | 贪心选择后只剩一个子问题 |
| 求解方向 | 自顶向下+递归 | 自底向上填表 | 自顶向下 |
| 最优性保证 | — | 全局最优 | 需证明贪心选择性质 |
| 经典问题 | 归并排序 | LCS、背包 | Dijkstra |

## 八、易错点

1. **Floyd 的 $k$ 循环必须在最外层**，否则得到的是中间结果而非最终解
2. **0-1 背包与分数背包不同**：0-1 不能用贪心，必须 DP
3. **矩阵链的 $p$ 数组长度是 $n+1$**，因为 $n$ 个矩阵有 $n+1$ 个维度
4. **DP 不是万能**：需要同时满足最优子结构和重叠子问题才高效
