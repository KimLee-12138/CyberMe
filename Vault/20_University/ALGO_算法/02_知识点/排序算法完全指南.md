# 排序算法完全指南

> 一句话：排序是算法的基石。选择排序、插入排序教你归纳思维；归并排序、快速排序教你分治思维；堆排序教你数据结构思维。

## 一、基于比较的排序下界

任何基于**元素间比较**的排序算法，最坏情况下至少需要 $\Omega(n\log n)$ 次比较。

**证明思路**：决策树模型。$n$ 个元素有 $n!$ 种排列，决策树高度 $h$ 满足 $2^h \ge n!$。由 Stirling 近似：$h \ge \log(n!) = \Theta(n\log n)$。

**推论**：归并排序和堆排序都是渐进最优的。快速排序平均也是最优。突破 $O(n\log n)$ 必须用非比较排序。

## 二、简单排序算法（归纳思想）

### 2.1 选择排序

**思想**：每次从剩余元素中选最小的放到前面。

| . | 最好 | 平均 | 最坏 | 空间 | 稳定 |
|---|------|------|------|------|------|
| | $O(n^2)$ | $O(n^2)$ | $O(n^2)$ | $O(1)$ | ✗ |

**分析**：无论输入如何，每次必须扫描剩余 $n-i$ 个元素，比较次数 $= n(n-1)/2$。

### 2.2 插入排序

**思想**：将当前元素插入到前面已排序序列的正确位置。

| . | 最好 | 平均 | 最坏 | 空间 | 稳定 |
|---|------|------|------|------|------|
| | $O(n)$ | $O(n^2)$ | $O(n^2)$ | $O(1)$ | ✓ |

**关键特性**：
- 最好情况（已排序）：$O(n)$
- 对**基本有序**的序列非常快
- 实际中小规模排序的首选（快排递归到小数组时切换为插入排序）

### 2.3 冒泡排序

| . | 最好 | 平均 | 最坏 | 空间 | 稳定 |
|---|------|------|------|------|------|
| | $O(n)$ | $O(n^2)$ | $O(n^2)$ | $O(1)$ | ✓ |

每轮将最大的"冒泡"到最后。可通过标志位检测提前结束（最好 $O(n)$）。

## 三、高效排序算法（分治思想）

### 3.1 归并排序

**核心**：递归分治 + 合并两个有序数组。

**模板**：
```
void mergeSort(A, l, r):
    if l >= r: return
    mid = (l+r)/2
    mergeSort(A, l, mid)
    mergeSort(A, mid+1, r)
    merge(A, l, mid, r)     // 合并两个有序段, O(n)
```

**复杂度**：$T(n) = 2T(n/2) + O(n) \Rightarrow O(n\log n)$

**特点**：稳定、外排序基础、需要 $O(n)$ 额外空间、链表排序首选。

### 3.2 快速排序

**核心**：partition 划分 + 递归。先处理再递归。

**partition 模板**（Lomuto 或 Hoare 分区）：
```
int partition(A, lo, hi):
    pivot = A[lo]; i = lo; j = hi
    while i < j:
        while i < j and A[j] >= pivot: j--
        while i < j and A[i] <= pivot: i++
        if i < j: swap(A[i], A[j])
    swap(A[lo], A[i])
    return i
```

**复杂度**：

| 情况 | 递归式 | 结果 |
|------|--------|------|
| 最好（每次中位数） | $T(n)=2T(n/2)+O(n)$ | $O(n\log n)$ |
| 最坏（已排序+选首元素） | $T(n)=T(n-1)+O(n)$ | $O(n^2)$ |
| 平均 | — | $O(n\log n)$ |

**优化策略**：
1. **三数取中**选 pivot：取首、尾、中三个元素的中位数
2. **小数组切插入排序**：$n<10$ 时切为插入排序
3. **三路快排**：分成 < pivot、= pivot、> pivot 三部分（适合有大量重复元素）

### 3.3 堆排序

**核心**：建最大堆 → 反复弹出堆顶（与末尾交换，下沉调整）。

**步骤**：
1. `buildMaxHeap(A)`：从最后一个非叶节点 `n/2-1` 向上做 `siftDown`，$O(n)$
2. 将堆顶（最大）与末尾交换，堆大小减 1，`siftDown(0)`，重复 $n-1$ 次

```
void heapSort(int A[], int n):
    for i = n/2-1 downto 0: heapify(A, n, i)   // 建堆 O(n)
    for i = n-1 downto 0:                        // 排序 O(n log n)
        swap(A[0], A[i])
        heapify(A, i, 0)

void heapify(int A[], int n, int i):   // siftDown
    int largest = i, l = 2*i+1, r = 2*i+2
    if l < n and A[l] > A[largest]: largest = l
    if r < n and A[r] > A[largest]: largest = r
    if largest != i:
        swap(A[i], A[largest])
        heapify(A, n, largest)
```

| . | 最好 | 平均 | 最坏 | 空间 | 稳定 |
|---|------|------|------|------|------|
| | $O(n\log n)$ | $O(n\log n)$ | $O(n\log n)$ | $O(1)$ | ✗ |

**优势**：原地排序，最坏也是 $O(n\log n)$。

## 四、线性时间排序（非比较）

### 4.1 计数排序

**适用**：整数、范围 $k$ 已知且不大。

**步骤**：统计每个值出现次数 → 前缀和确定位置 → 反向填充。

$O(n+k)$，$k$ 为值域范围。空间 $O(n+k)$。稳定。

### 4.2 基数排序

**适用**：多位关键字（如固定位数的整数、定长字符串）。

**步骤**：从最低位到最高位，每轮用稳定排序（通常计数排序）按当前位排序。

$O(d \times (n+k))$，$d$ 为位数，$k$ 为基数（通常 10）。

### 4.3 桶排序

**适用**：输入均匀分布。

**步骤**：将数据分到 $m$ 个桶 → 每个桶内排序 → 按顺序合并。

平均 $O(n)$，最坏 $O(n^2)$（所有元素落入同一个桶）。

## 五、排序算法选择指南

| 场景 | 推荐算法 |
|------|---------|
| 数据量小 | 插入排序 |
| 通用场景 | 快速排序（三数取中优化） |
| 需要稳定 | 归并排序 |
| 链表排序 | 归并排序 |
| 外排序 | 归并排序 |
| 大量重复元素 | 三路快排 |
| 最坏保证 $O(n\log n)$ | 堆排序 |
| 整数、范围小 | 计数排序 |
| 固定位数整数 | 基数排序 |

## 六、C++ STL 排序

- `std::sort`：内省排序（快排+堆排混合），$O(n\log n)$，不稳定
- `std::stable_sort`：归并排序，$O(n\log n)$，稳定
- `std::partial_sort`：堆排序选出前 $k$ 个，$O(n\log k)$

## 七、易错点

1. **快排 partition 必须先右后左**：否则 pivot 归位位置可能不对
2. **堆排序的下沉中**，左右子节点下标是 $2i+1$ 和 $2i+2$（0-indexed）
3. **归并排序需要额外空间**，无法原地
4. **基数排序每轮必须是稳定排序**，否则高位排序会破坏低位的有序性
