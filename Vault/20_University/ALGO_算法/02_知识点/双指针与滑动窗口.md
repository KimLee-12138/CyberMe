# 双指针与滑动窗口

> 一句话：用两个指针协作扫描数组，将 $O(n^2)$ 暴力优化到 $O(n)$。双指针和滑动窗口是算法竞赛中最实用的优化技巧之一。

## 一、双指针

### 1.1 对撞指针

两指针从数组两端向中间移动。**适用场景**：有序数组中的配对问题。

**模板**：
```
int lo = 0, hi = n - 1;
while (lo < hi) {
    if (condition) lo++;
    else if (other_condition) hi--;
    else { /* 找到目标 */ break; }
}
```

**经典题**：

**两数之和 II** [LC 167]：有序数组找两个数和为 target。

```
vector<int> twoSum(vector<int>& A, int target) {
    int lo = 0, hi = A.size() - 1;
    while (lo < hi) {
        int sum = A[lo] + A[hi];
        if (sum == target) return {lo+1, hi+1};
        else if (sum < target) lo++;
        else hi--;
    }
    return {};
}
```

$O(n)$，暴力 $O(n^2)$。

**三数之和** [LC 15]：先排序 → 固定第一个数 → 对剩余部分用双指针找两数之和。

**盛最多水的容器** [LC 11]：两指针从两端向中间收缩，每次移动较短的一边。

### 1.2 快慢指针

两指针同向移动，速度不同。**适用**：链表环检测、去重。

**环形链表** [LC 141]：快指针每次走 2 步，慢指针走 1 步。若相遇则有环。

**删除有序数组中的重复项** [LC 26]：慢指针指向已去重末尾，快指针扫描。

### 1.3 分离指针（归并式）

两个指针分别遍历两个有序数组。**适用**：合并有序序列。

**合并两个有序数组** [LC 88]：从后往前填，避免额外的移位。

## 二、滑动窗口

### 2.1 核心思想

维护一个可变大小的窗口 $[l, r]$，窗口右边界 $r$ 不断右移扩展，当窗口不满足条件时，左边界 $l$ 右移收缩。

**模板**：
```
int l = 0;
for (int r = 0; r < n; r++) {
    add(A[r]);           // 扩展窗口
    while (!valid()) {   // 收缩窗口直到合法
        remove(A[l]); l++;
    }
    ans = max(ans, r - l + 1);  // 更新答案
}
```

### 2.2 经典题目

**无重复字符的最长子串** [LC 3]：维护窗口内字符集合。当右边界字符重复时，收缩左边界。

```
int lengthOfLongestSubstring(string s) {
    vector<int> cnt(128, 0);
    int l = 0, ans = 0;
    for (int r = 0; r < s.size(); r++) {
        cnt[s[r]]++;
        while (cnt[s[r]] > 1) cnt[s[l++]]--;
        ans = max(ans, r - l + 1);
    }
    return ans;
}
```

$O(n)$。

**最小覆盖子串** [LC 76]：找包含目标串所有字符的最短子串。窗口扩展直到覆盖所有目标字符，然后收缩左边界找最短。

**长度最小的子数组** [LC 209]：和 ≥ target 的最短子数组。

**至多包含两个不同字符的最长子串** [LC 159]：滑动窗口 + 字符频次统计。

**替换后的最长重复字符** [LC 424]：窗口长度 - 窗口内最多字符频次 > k 时收缩。

### 2.3 固定窗口大小

窗口大小固定为 $k$，只需维护窗口内的统计信息。每次滑动：加入新元素、移除窗口外旧元素。

**字符串排列** [LC 567]：判断 s2 是否包含 s1 的排列。固定窗口大小 = s1.length()，滑动比较字符频次。

**找到字符串中所有字母异位词** [LC 438]：同上，固定窗口。

### 2.4 滑动窗口时间复杂度分析

每个元素被 `add` 一次、`remove` 一次，所以窗口内部操作总共 $O(n)$。关键是**内部 while 不是 $O(n^2)$**——虽然每次收缩可能移动多个位置，但左指针总共只移动 $n$ 次。

## 三、双指针 vs 滑动窗口

| 维度 | 对撞双指针 | 滑动窗口 |
|------|-----------|---------|
| 指针移动方向 | 相向 | 同向 |
| 前提 | 通常要求有序 | 通常处理连续子数组 |
| 目标 | 找配对 | 找满足条件的最长/最短子数组 |
| 复杂度 | $O(n)$ | $O(n)$ |

---

## 四、易错点

1. **滑动窗口的收缩循环条件**：是 `while (!valid)` 不是 `if`，可能需要连续收缩多次
2. **双指针要求数组有序**（对撞型），不满足有序性就不能用
3. **窗口更新时机**：更新答案通常在收缩循环**之后**（窗口合法时）
4. **固定窗口长度时要处理起始位置**：对于排列匹配类问题，窗口长度固定不变
