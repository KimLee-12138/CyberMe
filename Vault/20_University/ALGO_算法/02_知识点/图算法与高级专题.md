# 图算法与高级专题

> 一句话：在前述图遍历和贪心图算法的基础上，深入处理网络流、匹配、NP 难问题等高级主题。

## 一、网络流

### 1.1 基本概念

- **流网络** $G=(V,E)$：有向图，每条边有容量 $c(u,v) \ge 0$
- **源点 $s$**：流出的起点；**汇点 $t$**：流进的终点
- **流 $f(u,v)$**：满足容量限制（$0 \le f \le c$）和流量守恒（$\sum f_{in} = \sum f_{out}$）

### 1.2 最大流问题

从 $s$ 到 $t$ 能推送的最大总流量。

### 1.3 Ford-Fulkerson 方法

**核心思想**：不断寻找增广路径（从 $s$ 到 $t$ 还有剩余容量的路径），沿着路径推送流量，更新残留网络。当找不到增广路径时，当前流即为最大流。

**Edmonds-Karp 算法**：每次用 BFS 找最短增广路径。时间复杂度 $O(VE^2)$。

### 1.4 最大流最小割定理

**最大流的值 = 最小割的容量**。

割 $(S,T)$ 将顶点分成两部分（$s \in S, t \in T$），割容量 = $\sum_{u\in S, v\in T} c(u,v)$。

这一定理是网络流理论的核心，将流问题转化为割问题。

## 二、二分图匹配

### 2.1 问题

二分图 $G=(L \cup R, E)$，求一个最大匹配（一组边，没有两条边共享顶点）。

### 2.2 匈牙利算法

交替寻找增广路径。时间复杂度 $O(VE)$。

**增广路径**：从未匹配的左顶点出发，交替经过未匹配边和匹配边，到达未匹配的右顶点的路径。反转路径上边的匹配状态，匹配数 +1。

### 2.3 转化为最大流

添加超级源点 $s$ 连接所有 $L$ 中顶点（容量 1），所有 $R$ 中顶点连接超级汇点 $t$（容量 1）。跑最大流即得最大匹配。

## 三、NP 完全性

### 3.1 问题分类

| 类别 | 全称 | 特征 |
|------|------|------|
| P | Polynomial | 存在多项式时间算法 |
| NP | Nondeterministic Polynomial | 解可在多项式时间内验证 |
| NP-complete | NP 完全 | 同时满足：属于 NP + NP 中最难 |
| NP-hard | NP 困难 | 至少和 NP-complete 一样难，但不一定属于 NP |

### 3.2 P vs NP 问题

**P = NP？** 至今未解决。若 P=NP，则所有能高效验证的问题也能高效求解。

### 3.3 经典 NP 完全问题

- **SAT**（可满足性问题）：第一个被证明为 NP-complete
- **旅行商问题 (TSP)**：访问所有城市一次回到起点，总路程最短
- **团问题**：图中找最大完全子图
- **顶点覆盖**：用最少顶点覆盖所有边
- **哈密顿回路**：访问每个顶点恰好一次的回路

### 3.4 处理 NP 难问题的策略

1. **回溯 + 剪枝（小规模）**
2. **分支限界（中等规模）**
3. **近似算法**：多项式时间内给出"接近最优"的解
4. **启发式/元启发式**：遗传算法、模拟退火（不保证最优）

## 四、常用图算法复杂度速查

| 算法 | 用途 | 复杂度 |
|------|------|--------|
| BFS | 遍历/无权最短路径 | $O(V+E)$ |
| DFS | 遍历/拓扑排序/SCC | $O(V+E)$ |
| Dijkstra | 单源最短路径（非负权） | $O((V+E)\log V)$ |
| Bellman-Ford | 单源最短路径（可负权） | $O(VE)$ |
| Floyd | 全源最短路径 | $O(V^3)$ |
| Kruskal | MST（稀疏图） | $O(E\log E)$ |
| Prim | MST（稠密图） | $O((V+E)\log V)$ |
| Ford-Fulkerson | 最大流 | $O(E\cdot\max\_flow)$ |
| Edmonds-Karp | 最大流 | $O(VE^2)$ |
| 匈牙利 | 二分图最大匹配 | $O(VE)$ |
| Kosaraju | 强连通分量 | $O(V+E)$ |

## 五、易错点

1. **最大流算法要求源点只有出边、汇点只有入边**
2. **Bellman-Ford 可以处理负权边但不能处理负权环**（第 $n$ 轮还能松弛说明存在负环）
3. **Dijkstra 不能处理负权边**的根本原因：一旦顶点被标记为"已确定"，其距离不会再被更新
4. **NP-complete 问题不存在已知的多项式时间算法**，考试中若遇到应使用近似或回溯法
