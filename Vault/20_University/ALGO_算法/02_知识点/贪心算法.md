# 贪心算法

> 一句话：每一步都做当前看起来最好的选择，希望局部最优累积成全局最优。

## 一、贪心算法的特征

与动态规划不同，贪心算法做选择后**不回溯、不比较、不存储子问题解**。每步选完就走，永远不回头。

**适用前提**：问题必须满足**贪心选择性质**和**最优子结构性质**。

## 二、分数背包问题

### 2.1 问题

$n$ 个物品可**按比例切割**放入容量 $C$ 的背包。求最大总价值。

### 2.2 贪心策略

按**单位体积价值**（$v_i / s_i$）从高到低排序，依次装入。装满为止。

**为什么不能用"价值最大优先"？** 一个体积超大但价值高的物品会迅速占满背包。

**为什么不能用"体积最小优先"？** 装很多体积小的低价值物品，总价值增长太慢。

### 2.3 伪代码（$O(n\log n)$，主要时间在排序）

```
for each item: unitValue = value / weight
sort items by unitValue descending
totalValue = 0; remaining = C
for each item:
    if remaining == 0: break
    if weight <= remaining:
        totalValue += value; remaining -= weight
    else:
        totalValue += value * (remaining / weight)
        break
return totalValue
```

## 三、Dijkstra 单源最短路径

### 3.1 问题

给定非负权有向图 $G=(V,E)$ 和源点 $s$，求 $s$ 到所有其他顶点的最短距离。

### 3.2 贪心策略

每次从**未确定**顶点中选择距离源点最近的顶点，将其标记为**已确定**，然后更新其邻居的距离（松弛操作）。

**为什么是贪心**：每次选的顶点，其当前距离就是最终最短距离（因为所有边权非负，不会再被更新得更短）。

### 3.3 伪代码

```
X = {1}; Y = V - {1}; λ[1] = 0
for y = 2 to n:
    λ[y] = (y 与 1 相邻) ? w[1][y] : ∞

for j = 1 to n-1:
    选 y ∈ Y 使 λ[y] 最小        // 贪心选择
    X = X ∪ {y}; Y = Y - {y}
    for 每条边(y, w) with w ∈ Y:
        if λ[y] + w[y][w] < λ[w]:
            λ[w] = λ[y] + w[y][w]   // 松弛
```

**复杂度**：
- 朴素实现 $O(n^2)$
- 二叉堆优化 $O((V+E)\log V)$
- 斐波那契堆优化 $O(E + V\log V)$

## 四、最小生成树

### 4.1 问题

给定连通无向带权图 $G=(V,E)$，找一棵覆盖所有顶点的树 $T$，使边的总权重最小。

### 4.2 Kruskal 算法（由边构林）

**贪心策略**：按权重递增排序所有边，逐一考察。若该边连接两个不同的连通分量（不构成回路），则将其加入 MST。

**数据结构**：需要并查集 (Union-Find) 支持高效合并和查找。

```
sort edges by weight ascending
for each vertex v: MAKESET(v)
T = {}
while |T| < n-1:
    (x, y) = 取下一条边
    if FIND(x) ≠ FIND(y):
        T = T ∪ {(x, y)}
        UNION(x, y)
return T
```

**复杂度**：$O(E\log E)$（排序主导）。并查集操作近似 $O(1)$。

### 4.3 Prim 算法（由点构树）

**贪心策略**：维护已选顶点集 $X$ 和未选顶点集 $Y$。每次选取连接 $X$ 与 $Y$ 的权重最小边。

```
X = {1}; Y = V - {1}; T = {}
for y = 2 to n: C[y] = (y邻接于1) ? w[1][y] : ∞; N[y] = 1
for j = 1 to n-1:
    y = argmin_{y∈Y} C[y]    // 贪心选择：离树最近的顶点
    T = T ∪ {(y, N[y])}
    X = X ∪ {y}; Y = Y - {y}
    for 每条边(y, w) with w ∈ Y:
        if w[y][w] < C[w]: C[w] = w[y][w]; N[w] = y
return T
```

**复杂度**：朴素 $O(n^2)$，堆优化 $O((V+E)\log V)$。

### 4.4 Kruskal vs Prim 对比

| 维度 | Kruskal | Prim |
|------|---------|------|
| 构建方式 | 由边构林（并查集） | 由点构树 |
| 适合 | 稀疏图 ($E$ 少) | 稠密图 ($E$ 多) |
| 核心数据结构 | 并查集 | 优先队列 |

## 五、贪心 vs 动态规划

| 问题 | 贪心 | DP |
|------|------|-----|
| 分数背包 | ✅ 最优 | ✅ 也可但多余 |
| 0-1 背包 | ❌ 不一定最优 | ✅ 正确解 |
| 最短路径(Dijkstra) | ✅ 最优 | ✅ Floyd 也行 |
| 最小生成树 | ✅ 最优 | N/A |

**核心区别**：贪心做"不可逆"的选择（选了就不能反悔），DP 枚举所有可能的决策。

## 六、易错点

1. **贪心不是万能**：0-1 背包用贪心（选单位价值最高的）会得到错误答案
2. **Dijkstra 不能处理负权边**：负权边破坏了"一旦确定就是最终距离"的贪心前提
3. **Kruskal 排序后边的顺序在算法过程中不再改变**
4. **Prim 与 Dijkstra 伪代码结构极相似**，区别在于更新条件（Dijkstra 用累加距离，Prim 用单边权重）
