# 回溯法与分支限界

> 一句话：系统地搜索解空间树，在搜索过程中用约束条件剪枝，避免穷举所有可能。

## 一、回溯法的基本思想

**回溯 = DFS 搜索 + 剪枝**。在解空间树上深度优先遍历，走到某一步发现不可能产生有效解时，立即退回上一步（回溯），尝试其他分支。

### 1.1 解空间树

将问题的所有可能解组织成一棵树：
- **根**：初始状态（尚未做任何决策）
- **内部节点**：部分解（做了部分决策）
- **叶子**：完整解或死胡同

### 1.2 回溯框架

```
Algorithm Backtrack(depth):
    if depth == n:    // 到达叶子，找到一个完整解
        output solution
        return
    for each choice at current level:
        if choice is valid:        // 约束条件检查（剪枝！）
            make choice
            Backtrack(depth + 1)   // 深入下一层
            undo choice            // 回溯：恢复状态
```

## 二、生成全排列

### 2.1 交换回溯法

**核心思想**：维护一个已初始化的数组 $P=[1,2,\dots,n]$。函数 `perm1(m)` 确定第 $m$ 个位置放谁——把位置 $m$ 的数字依次和位置 $m \sim n$ 的数字交换。

```
for j = 1 to n: P[j] = j
perm1(1)

Procedure perm1(m):
    if m == n: output P[1..n]
    else:
        for j = m to n:
            swap(P[m], P[j])
            perm1(m+1)
            swap(P[m], P[j])    // 回溯：恢复原状
```

### 2.2 填空插入法

**核心思想**：初始化全 0 数组，然后从 $n$ 倒序填到 1。`perm2(m)` 把数字 $m$ 放入一个空位。

```
for j = 1 to n: P[j] = 0
perm2(n)

Procedure perm2(m):
    if m == 0: output P[1..n]
    else:
        for j = 1 to n:
            if P[j] == 0:        // 找到空位
                P[j] = m
                perm2(m-1)
                P[j] = 0         // 回溯
```

**复杂度**：两种方法都是 $O(n \cdot n!)$，因为输出本身有 $n!$ 个排列。

## 三、n 皇后问题

### 3.1 问题

在 $n \times n$ 棋盘上放置 $n$ 个皇后，要求任意两个不在同一行、同一列或同一对角线上。

### 3.2 回溯解法

逐行放置皇后。对第 $k$ 行，尝试每一个列位置 $j$，检查是否与已放置的皇后冲突。

**剪枝条件**：
- 同列：$j$ 已被使用
- 同对角线：$|k - i| = |j - col[i]|$（行差等于列差）

```
Procedure NQueens(k):    // 正在放第 k 行的皇后
    if k > n: output solution; return
    for j = 1 to n:
        if j is safe (not in same column/diagonal):
            col[k] = j
            NQueens(k + 1)
```

## 四、0-1 背包的回溯解法

与 DP 不同，回溯法搜索的是子集树（每个物品选或不选）。

```
Procedure KnapsackBT(i, curWeight, curValue):
    if i > n:
        best = max(best, curValue); return
    // 分支1：不选第 i 个物品
    KnapsackBT(i+1, curWeight, curValue)
    // 分支2：选第 i 个物品（能装下才选）
    if curWeight + w[i] <= C:
        KnapsackBT(i+1, curWeight + w[i], curValue + v[i])
```

**优化**：可以加入限界函数——估算"选第 i 个"后最多能达到多少价值，若还不如当前最优则剪枝。

## 五、分支限界法

### 5.1 与回溯的区别

| 维度 | 回溯 | 分支限界 |
|------|------|---------|
| 搜索策略 | DFS | BFS 或 Best-First |
| 目标 | 找出所有解 | 找最优解 |
| 剪枝依据 | 约束条件 | 限界函数（上/下界） |

### 5.2 核心概念：限界函数

对每个活节点，计算其"最好情况"能达到的目标值。若该值已不如当前最优解，则果断剪枝。

**例（0-1 背包）**：剩余容量装满单位价值最高的物品，得到价值上界。若上界 ≤ 当前最优，直接剪掉。

## 六、归纳法相关算法

### 6.1 二分幂（快速幂）

$$x^n = \begin{cases} (x^{n/2})^2 & n \text{ 为偶数} \\ x \cdot (x^{(n-1)/2})^2 & n \text{ 为奇数} \end{cases}$$

每次递归规模减半，$O(\log n)$。

### 6.2 霍纳法则

将多项式 $P(x) = a_n x^n + a_{n-1}x^{n-1} + \dots + a_0$ 改写为：
$$P(x) = ((\dots(a_n x + a_{n-1})x + \dots)x + a_1)x + a_0$$

只需 $n$ 次乘法 + $n$ 次加法 = $O(n)$。

### 6.3 多数元素投票法

给定数组，找出出现次数 > n/2 的元素。维护候选人和计数器。遇到相同计数器+1，不同则-1。计数归零则换候选人。$O(n)$ 时间，$O(1)$ 空间。

### 6.4 基数排序

不直接比较元素大小，而是按位（个位→十位→百位→...）利用稳定排序。每轮扫描一遍数组 $O(n)$，共 $d$ 轮 → $O(dn)$。当 $d$ 很小时快于 $O(n\log n)$。

## 七、易错点

1. **回溯必须恢复状态**：忘记撤销操作会导致后续分支状态错误
2. **分支限界的限界函数不能太松**：太松无法有效剪枝；太紧可能把最优解剪掉
3. **n 皇后的对角线判断**：$|行差| = |列差|$ 即在同一对角线上
4. **回溯解 0-1 背包是 $O(2^n)$**，仅当 $n$ 小或剪枝强时才可接受
