# 字符串算法

> 一句话：字符串匹配是文本处理的刚需。KMP 做到 $O(n+m)$、Rabin-Karp 用哈希加速、Trie 实现前缀高效检索。

## 一、朴素匹配

主串 $T[0..n-1]$，模式串 $P[0..m-1]$。每个起始位置逐一比较。

```
for i = 0 to n-m:
    for j = 0 to m-1:
        if T[i+j] != P[j]: break
    if j == m: return i    // 找到匹配
```

最坏：$T$ = "AAAAAB"，$P$ = "AAAB"，每次匹配到最后一个字符才失败。$O(nm)$。

## 二、KMP 算法

### 2.1 核心思想

利用匹配失败后的已知信息，主串指针**不回溯**。失败时模式串跳到 `next[j]` 位置继续匹配。

### 2.2 next 数组

`next[j]` = 模式串前 $j$ 个字符组成的子串的**最长相等前后缀长度**。

**构造方法**（递推）：

```
void getNext(string P, vector<int>& next) {
    int m = P.size();
    next[0] = 0;  // 或 -1（不同教材定义不同）
    int j = 0;    // 前缀末尾
    for (int i = 1; i < m; i++) {
        while (j > 0 && P[i] != P[j]) j = next[j-1];  // 回溯
        if (P[i] == P[j]) j++;
        next[i] = j;
    }
}
```

**手算 next 数组示例**（$P$ = "ABABC"）：

| $j$ | 前缀 | 最长公共前后缀 | next[j] |
|-----|------|---------------|---------|
| 0 | A | 无 | 0 |
| 1 | AB | 无 | 0 |
| 2 | ABA | "A" (长度1) | 1 |
| 3 | ABAB | "AB" (长度2) | 2 |
| 4 | ABABC | 无 | 0 |

### 2.3 匹配过程

```
int kmp(string T, string P) {
    vector<int> next(m); getNext(P, next);
    int j = 0;  // 模式串指针
    for (int i = 0; i < T.size(); i++) {
        while (j > 0 && T[i] != P[j]) j = next[j-1];
        if (T[i] == P[j]) j++;
        if (j == m) return i - m + 1;  // 匹配成功
    }
    return -1;
}
```

**复杂度**：$O(n+m)$，无论输入如何。

### 2.4 本质

KMP 的 next 数组构建过程 = 模式串自己匹配自己。一旦理解这点，KMP 不再神秘。

## 三、Rabin-Karp 算法

### 3.1 核心思想

用**滚动哈希**（sliding window hash）快速计算每个窗口的哈希值，$O(1)$ 滑动。

```
hash(s[l..r]) = (s[l] × B^{len-1} + s[l+1] × B^{len-2} + ... + s[r]) mod M
```

滑动时：
$$
\text{new\_hash} = (B \times (\text{old\_hash} - s[l] \times B^{m-1}) + s[r+1]) \mod M
$$

### 3.2 算法

1. 预计算 $P$ 的哈希值
2. 计算 $T$ 中每个长度为 $m$ 的窗口的哈希值
3. 哈希值匹配时，再做逐字符验证（防哈希冲突）

**复杂度**：期望 $O(n+m)$，最坏（大量哈希冲突）$O(nm)$。

### 3.3 vs KMP

| . | KMP | Rabin-Karp |
|----|-----|-----------|
| 最坏 | $O(n+m)$ | $O(nm)$ |
| 多模式匹配 | 需修改 | 自然适配 |
| 实现难度 | 中等 | 简单 |
| 额外空间 | $O(m)$ | $O(1)$ |

## 四、Trie（前缀树/字典树）

### 4.1 定义

每个节点表示一个前缀。根为空，每条边代表一个字符。

```
struct TrieNode {
    TrieNode* child[26] = {};
    bool isEnd = false;
};
```

### 4.2 操作

**插入**：$O(L)$，$L$ 为单词长度。逐字符下落，不存在则创建节点。

**查找**：$O(L)$。逐字符下落，最后检查 `isEnd`。

**前缀查找**：$O(L)$。只需走到最后一个字符即可，不要求 `isEnd`。

### 4.3 应用

- 自动补全
- 拼写检查
- 最大异或对 [LC 421]：二进制 Trie
- 单词搜索 II [LC 212]：Trie + DFS

---

## 五、易错点

1. **KMP next 数组的下标定义**：不同教材可能用 `next`（长度）或 `nextval`（优化版），做题前确认定义
2. **Rabin-Karp 需要选好基数 $B$ 和模数 $M$**：常用 $B=131$ 或 $B=13331$，$M=10^9+7$ 或取双模防冲突
3. **Trie 空间消耗大**：每个节点 26 个子指针（如果字符集是 a-z）。可用 `unordered_map` 优化稀疏子节点
