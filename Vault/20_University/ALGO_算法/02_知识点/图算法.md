# 图算法

> 一句话：图是节点和边的集合，掌握 BFS/DFS 的遍历范式即可引申出几乎所有图算法。

## 一、图的表示

### 1.1 邻接矩阵

空间 $O(V^2)$。适合**稠密图**。$O(1)$ 时间判断边是否存在。

### 1.2 邻接表

每个顶点维护一个邻居链表。空间 $O(V+E)$。适合**稀疏图**。遍历邻居快，判断边存在性慢。

## 二、广度优先搜索

### 2.1 算法思想

从源点 $s$ 开始，按"距离递增"的层次逐层访问。用**队列**实现。

### 2.2 伪代码

```
BFS(G, s):
    for each v: color[v] = WHITE; d[v] = ∞
    color[s] = GRAY; d[s] = 0
    Q = queue containing s
    while Q not empty:
        u = Q.dequeue()
        for each v in Adj[u]:
            if color[v] == WHITE:
                color[v] = GRAY; d[v] = d[u] + 1
                π[v] = u
                Q.enqueue(v)
        color[u] = BLACK
```

**复杂度**：$O(V+E)$

**应用**：
- 无权图的最短路径（边数最少）
- 二分图检测（染色法）
- 连通分量计数

## 三、深度优先搜索

### 3.1 算法思想

沿一条路径走到尽头再回溯。用**递归**（隐式栈）实现。

### 3.2 伪代码

```
DFS(G):
    for each v: color[v] = WHITE; time = 0
    for each v: if color[v] == WHITE: DFS-Visit(v)

DFS-Visit(u):
    color[u] = GRAY; time++; d[u] = time
    for each v in Adj[u]:
        if color[v] == WHITE:
            π[v] = u
            DFS-Visit(v)
    color[u] = BLACK; time++; f[u] = time
```

**复杂度**：$O(V+E)$

**边的分类**（根据 $d$ 和 $f$ 时间戳）：
- **树边**：DFS 树上的边
- **回边**：指向祖先的边
- **前向边**：指向后代的非树边
- **横跨边**：连接不同子树的边

## 四、拓扑排序

### 4.1 问题

对有向无环图 (DAG) 的顶点进行线性排序，使得对每条有向边 $(u,v)$，$u$ 都排在 $v$ 前面。

### 4.2 算法（DFS 版）

``` 
完成 DFS 后，按 f[u] 降序排列顶点。
```

或使用 BFS（Kahn 算法）：统计入度，每次取入度为 0 的节点。

### 4.3 应用

- 课程先修关系排序
- 项目任务调度
- 编译依赖分析

## 五、强连通分量

### 5.1 定义

有向图的一个极大子图，其中任意两顶点互相可达。

### 5.2 Kosaraju 算法

1. 对原图做 DFS，按完成时间 $f$ 降序排列顶点
2. 对**转置图** $G^T$（所有边反向），按步骤 1 的顺序再做一次 DFS
3. 每棵 DFS 树就是一个 SCC

**复杂度**：$O(V+E)$

## 六、BFS vs DFS 对比

| 维度 | BFS | DFS |
|------|-----|-----|
| 数据结构 | 队列 | 栈（递归） |
| 适用场景 | 最短路径、层次遍历 | 拓扑排序、SCC、连通性 |
| 空间 | $O(V)$（最坏全在队列） | $O(V)$（递归栈） |
| 记忆 | 无差别逐层 | 有"穷追到底再回溯"的特点 |

## 七、图的连通性

- **连通图**：任意两顶点间存在路径
- **连通分量**：无向图的极大连通子图
- **桥**：删除后使连通分量数量增加的边
- **关节点**：删除后使连通分量数量增加的顶点

## 八、易错点

1. DFS 的**白色/灰色/黑色**三色标记中，灰色表示"正在访问但尚未完成"，用于识别回边
2. 拓扑排序**仅适用于 DAG**，有环图不存在拓扑序
3. 邻接表 BFS 遍历邻居时判断 `color == WHITE` 即可，无需额外标记
