# 数据结构选讲

> 一句话：算法 = 数据结构 + 操作。本节聚焦算法设计中高频使用的数据结构：堆、哈希表、并查集、二叉搜索树。

## 一、堆

### 1.1 定义

堆是一棵**完全二叉树**，满足堆性质：
- **最大堆**：每个节点的值 ≥ 其子节点的值
- **最小堆**：每个节点的值 ≤ 其子节点的值

### 1.2 存储

用数组存储，0-indexed：
- 节点 $i$ 的左子：$2i+1$
- 节点 $i$ 的右子：$2i+2$
- 节点 $i$ 的父：$(i-1)/2$

### 1.3 核心操作

**插入**（$O(\log n)$）：放到数组末尾，然后向上冒泡（`siftUp` / `bubbleUp`）。

**删除堆顶**（$O(\log n)$）：将末尾元素移到堆顶，然后向下下沉（`siftDown` / `heapify`）。

**建堆**（$O(n)$）：从最后一个非叶节点 `n/2-1` 倒序到 0，依次 `siftDown`。不是 $O(n\log n)$！虽然每个节点都可能下沉，但叶子节点数量占一半且深度浅，总操作次数 $\le 2n$。

```
// siftDown: 维护堆性质
void heapify(vector<int>& heap, int n, int i) {
    int largest = i, l = 2*i + 1, r = 2*i + 2;
    if (l < n && heap[l] > heap[largest]) largest = l;
    if (r < n && heap[r] > heap[largest]) largest = r;
    if (largest != i) {
        swap(heap[i], heap[largest]);
        heapify(heap, n, largest);
    }
}
```

### 1.4 应用

| 场景 | 堆类型 |
|------|--------|
| 堆排序 | 最大堆 |
| 优先队列（Dijkstra, Prim） | 最小堆 |
| Top K 问题 | 最小堆（大小为 k） |
| 中位数维护 | 双堆（最大堆+最小堆） |

## 二、哈希表

### 2.1 核心思想

通过**哈希函数**将键映射到数组索引，期望 $O(1)$ 增删查。

### 2.2 冲突解决

| 方法 | 原理 | 优点 | 缺点 |
|------|------|------|------|
| 拉链法 | 每个槽存链表 | 简单、删除方便 | 额外指针开销 |
| 开放寻址法 | 冲突时找下一个空位 | 缓存友好 | 删除需特殊标记 |

### 2.3 哈希函数

好的哈希函数三要素：**确定性、均匀性、高效性**。

常用方法：除留余数法 `h(k) = k mod m`（$m$ 取素数）、乘法哈希法。

### 2.4 动态扩容

负载因子 $\alpha = n/m$ 超过阈值（通常 0.75）时扩容：分配更大数组 → 重新哈希所有元素。

**摊销分析**：虽然单次扩容 $O(n)$，但均摊到每次插入是 $O(1)$。

### 2.5 应用

- 两数之和：$O(n)$
- 去重
- 字符频率统计
- LRU 缓存

## 三、并查集

### 3.1 定义

维护**不相交集合**的合并和查询。支持：
- `Find(x)`：找到 $x$ 所属集合的代表元素（根）
- `Union(x,y)`：合并 $x$ 和 $y$ 所在的集合

### 3.2 两种优化

**路径压缩**：`Find` 时将所有经过的节点直接指向根。后续查询 $O(1)$。

**按秩合并**：`Union` 时把小树挂到大树下，控制树高。

两者结合：均摊 $O(\alpha(n))$，$\alpha$ 是阿克曼反函数，实际中 $\le 4$，视为常数。

```
struct DSU {
    vector<int> parent, rank;
    DSU(int n) : parent(n), rank(n, 0) {
        iota(parent.begin(), parent.end(), 0);
    }
    int find(int x) {
        return parent[x] == x ? x : parent[x] = find(parent[x]);
    }
    void unite(int x, int y) {
        x = find(x); y = find(y);
        if (x == y) return;
        if (rank[x] < rank[y]) swap(x, y);
        parent[y] = x;
        if (rank[x] == rank[y]) rank[x]++;
    }
};
```

### 3.3 应用

- Kruskal MST
- 连通分量计数
- 冗余连接检测 [LC 684]
- 朋友圈 [LC 547]

## 四、二叉搜索树

### 4.1 BST 性质

左子树所有值 < 根 < 右子树所有值。中序遍历得到有序序列。

| 操作 | 平均 | 最坏 |
|------|------|------|
| 查找 | $O(\log n)$ | $O(n)$（退化为链表） |
| 插入 | $O(\log n)$ | $O(n)$ |
| 删除 | $O(\log n)$ | $O(n)$ |

### 4.2 平衡二叉搜索树

**AVL 树**：任意节点左右子树高度差 ≤ 1。插入/删除后通过旋转恢复平衡。

**旋转操作**（考试重点）：
- **LL 失衡**（左子树的左子树较高）：右旋
- **RR 失衡**：左旋
- **LR 失衡**（左子树的右子树较高）：先左旋左子，再右旋根
- **RL 失衡**：先右旋右子，再左旋根

```
// 右旋
Node* rightRotate(Node* y) {
    Node* x = y->left;
    Node* T = x->right;
    x->right = y; y->left = T;   // 旋转
    // 更新高度...
    return x;
}
```

**红黑树**：放宽平衡条件（最长路径 ≤ 2×最短路径），插入/删除只需 $O(1)$ 次旋转。实际中比 AVL 更常用。

---

## 五、易错点

1. **建堆是 $O(n)$ 不是 $O(n\log n)$**：不要被直觉骗了
2. **并查集不用按秩合并仅路径压缩也能接近 $O(1)$**，但单用路径压缩最坏可达 $O(\log n)$
3. **BST 删除有三种情况**：叶子直接删、单子用子替代、双子用后继替代
4. **AVL 旋转次数**：插入最多 1 次旋转、删除最多 $O(\log n)$ 次旋转
