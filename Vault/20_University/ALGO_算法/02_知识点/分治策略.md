# 分治策略

> 一句话：将大问题递归地分解为若干小问题，分别求解后再合并结果。

## 一、分治法的三个步骤

1. **分解 (Divide)**：将原问题划分为若干个规模更小的子问题
2. **治理 (Conquer)**：递归求解每个子问题（直到规模足够小、可直接求解）
3. **合并 (Combine)**：将子问题的解组合成原问题的解

## 二、二分查找

### 2.1 问题

在**有序**数组 $A[1..n]$ 中查找元素 $x$。

### 2.2 算法思想

每次将查找范围缩小一半：比较 $A[mid]$ 与 $x$。若等于则返回；若小于则在右半查找；若大于则在左半查找。

### 2.3 伪代码

```
Algorithm BinarySearch(A, low, high, x)
    if low > high then return -1
    mid = (low + high) / 2
    if x == A[mid] then return mid
    else if x < A[mid] then return BinarySearch(A, low, mid-1, x)
    else return BinarySearch(A, mid+1, high, x)
```

### 2.4 复杂度分析

递归方程：$T(n) = T(n/2) + O(1)$

每次只需要解决**一个**规模为 $n/2$ 的子问题（不是两个，因为只需要去一边）。主定理 Case 2：$\log_2 1 = 0$，$f(n)=O(1)=O(n^0)$。

$$T(n) = O(\log n)$$

## 三、归并排序

### 3.1 算法思想

将数组分成两半，递归排序每个子数组，然后将两个有序子数组合并。

### 3.2 伪代码（合并过程）

```
Procedure MERGE(A, low, mid, high)
    // 复制 A[low..high] 到辅助数组 B
    for k = low to high: B[k] = A[k]
    i = low        // 左半指针
    j = mid + 1    // 右半指针
    k = low
    while i <= mid and j <= high:
        if B[i] <= B[j]: A[k++] = B[i++]
        else:           A[k++] = B[j++]
    while i <= mid: A[k++] = B[i++]    // 剩余元素
```

### 3.3 复杂度

递归方程：$T(n) = 2T(n/2) + O(n)$

递归树法：每层工作量都是 $O(n)$，共 $\log n$ 层。

$$T(n) = O(n\log n)$$

## 四、快速排序

### 4.1 算法思想

与归并排序不同，快排是"先处理，后递归"。先通过 `PARTITION` 将数组划分为"小的在左、大的在右"，再递归排序左右两部分。

### 4.2 PARTITION 过程（双指针交换法）

```
Function PARTITION(A, low, high)
    pivot = A[low]    // 选第一个元素为基准
    i = low; j = high
    while i < j:
        while i < j and A[j] >= pivot: j--    // 从右找小的
        while i < j and A[i] <= pivot: i++    // 从左找大的
        if i < j: swap(A[i], A[j])
    swap(A[low], A[i])    // 基准归位
    return i
```

**关键点**：必须先从右向左扫描，这样才能保证相遇时 $A[i] \le pivot$。

### 4.3 复杂度分析

| 情况 | 场景 | 递归式 | 结果 |
|------|------|--------|------|
| 最坏 | 每次基准都是极值（如已排序） | $T(n)=T(n-1)+O(n)$ | $O(n^2)$ |
| 最好 | 每次基准都是中位数 | $T(n)=2T(n/2)+O(n)$ | $O(n\log n)$ |
| 平均 | 随机输入 | — | $O(n\log n)$ |

**最坏情况详细推导**：每次只排除 1 个元素，剩余 $n-1$ 个：
$$T(n) = cn + c(n-1) + \dots + c(1) = c \cdot \frac{n(n+1)}{2} = O(n^2)$$

## 五、大整数乘法（Karatsuba 算法）

### 5.1 问题

两个 $n$ 位数 $X$ 和 $Y$ 相乘。传统竖式乘法需要 $O(n^2)$ 次乘法。

### 5.2 核心技巧

将 $X = A \cdot 10^{n/2} + B$，$Y = C \cdot 10^{n/2} + D$：

- 传统：$AC \cdot 10^n + (AD+BC) \cdot 10^{n/2} + BD$，共 4 次乘法
- **Karatsuba**：$AD+BC = (A+B)(C+D) - AC - BD$，只需 3 次乘法！

### 5.3 复杂度

$$T(n) = 3T(n/2) + O(n)$$

主定理：$\log_2 3 \approx 1.585$，$T(n) = O(n^{1.585}) \ll O(n^2)$

## 六、寻找第k小元素（线性时间选择）

### 6.1 算法思想

基于"中位数的中位数"(Median of Medians) 选 pivot，保证每次至少剔除 30% 的元素。

**步骤**：将数组每 5 个一组 → 每组找中位数 → 递归找中位数的中位数 $mm$ → 用 $mm$ 做 pivot 划分 → 递归进入包含第 $k$ 小元素的那一侧。

### 6.2 复杂度

最坏情况递归方程：$T(n) \le T(n/5) + T(7n/10) + O(n)$

因 $1/5 + 7/10 = 0.9 < 1$，$$T(n) = O(n)$$

## 七、最近点对问题

### 7.1 算法思想

按 x 坐标平分平面 → 递归求左半最小距 $\delta_L$ 和右半最小距 $\delta_R$ → $\delta = \min(\delta_L, \delta_R)$ → 检查中线两侧 $2\delta$ 宽带内跨左右半平面的点对。

**关键结论**：对于宽带内的任意一点，只需检查其下方紧邻的至多 7 个点。

### 7.2 复杂度

$$T(n) = 2T(n/2) + O(n) \Rightarrow O(n\log n)$$

## 八、易错点

1. **二分查找前提是有序数组**，对链表无效
2. **快速排序必须先从右向左扫描**，否则基准归位时相遇位置可能不对
3. **归并排序需要 $O(n)$ 额外空间**，非原地排序
4. **主定理不能直接套用所有递归式**，如 $T(n)=kT(n/k)+O(n)$ 可直接用递归树
