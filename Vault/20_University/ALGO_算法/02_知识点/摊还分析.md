# 摊还分析

> 一句话：单次操作可能很贵，但连续执行多次操作的平均代价很低。摊还分析正是用来证明这种"平均"的数学工具。

## 一、为什么要摊还分析

**例子**：动态数组扩容。插入元素通常是 $O(1)$，但容量满时需要分配新数组 + 拷贝所有旧元素，单次 $O(n)$。

若仅看**最坏情况单次复杂度**：每次插入都是 $O(n)$（因为可能触发扩容）。结论：$n$ 次插入 = $O(n^2)$。

但实际中 $n$ 次插入的总时间远小于 $n^2$。摊还分析证明：$n$ 次插入总共 $O(n)$，每次均摊 $O(1)$。

## 二、三种分析方法

### 2.1 聚合分析

**方法**：直接计算 $n$ 次操作的总代价 $T(n)$，摊还代价 = $T(n)/n$。

**例（动态数组）**：容量倍增策略。插入第 $1, 2, 4, 8, 16, \dots$ 个元素时触发扩容。

总拷贝次数 = $1 + 2 + 4 + 8 + \dots + n/2 \approx n$。

$T(n) = n$（$n$ 次普通插入）+ $n$（所有拷贝）= $2n = O(n)$。

每次均摊：$O(n)/n = O(1)$。

### 2.2 记账法（Accounting Method）

**方法**：对每次操作收一笔"押金"，便宜的操作用不完存起来，留给贵的操作取用。

**例（动态数组）**：每次插入收费 3 元——1 元用于本次插入，1 元用于将来扩容时拷贝自己，1 元用于为数组中的另一个旧元素付拷贝费。

**证明**：任何时候存款 ≥ 0 → 总收费是实际代价的上界 → $n$ 次插入总代价 ≤ $3n = O(n)$。

### 2.3 势能法（Potential Method）

**方法**：定义势能函数 $\Phi(D_i)$ 表示第 $i$ 次操作后数据结构的"势能"。摊还代价 = 实际代价 + 势能变化。

$$c_i' = c_i + \Phi(D_i) - \Phi(D_{i-1})$$

**如果能证明每次 $c_i'$ 都很小，则 $\sum c_i = \sum c_i' + \Phi(D_0) - \Phi(D_n)$**。

**例（动态数组）**：定义 $\Phi = 2 \times (\text{元素数量} - \text{容量}/2)$。推导可得每次插入摊还代价 ≤ 3。

## 三、经典应用

### 3.1 动态数组扩容

| 扩容策略 | 摊还插入 |
|---------|---------|
| 容量 ×2（倍增） | $O(1)$ |
| 容量 +1（固定增量） | $O(n)$（每次都要拷贝） |

**倍增策略的摊还分析**：扩容频率随 $n$ 增大指数递减，总拷贝 $O(n)$。

### 3.2 并查集

仅路径压缩：$O(\log n)$ 均摊。路径压缩 + 按秩合并：$O(\alpha(n))$ 均摊，$\alpha$ 是阿克曼反函数。

### 3.3 哈希表扩容

负载因子超阈值时扩容（通常 2 倍）。摊还分析同动态数组：$O(1)$ 均摊插入。

### 3.4 二叉堆操作

- **建堆**：看似对 $n$ 个元素各做一次 $\log n$ 的下沉 = $O(n\log n)$？错！每个元素实际下沉深度为所在层数。总和 $\le 2n$，所以 $O(n)$。
- **插入 + 删除**：均摊 $O(\log n)$

### 3.5 栈操作（Multipop）

```
Multipop(S, k):   // 弹 k 个元素
    while S not empty and k > 0: pop(S); k--
```

单次 Multipop 最坏 $O(n)$。但 $n$ 次 push + pop + Multipop 操作的总代价 = $O(n)$（因为每个元素最多被 push 一次、pop 一次）。

### 3.6 滑动窗口

每个元素被窗口加入一次、移除一次，总操作 $O(n)$。虽然某次收缩可能移动左指针多步，但左指针总共只移动 $n$ 次。

## 四、分析对照表

| 数据结构/算法 | 最坏单次 | 摊还单次 |
|-------------|---------|---------|
| 动态数组插入 | $O(n)$ | $O(1)$ |
| 建堆 | — | $O(n)$（非 $O(n\log n)$） |
| 二叉堆插入 | $O(\log n)$ | $O(\log n)$ |
| 并查集 find | $O(n)$（无优化） | $O(\alpha(n))$ |
| 栈 Multipop | $O(n)$ | $O(1)$ |
| 哈希表插入 | $O(n)$（扩容时） | $O(1)$ |
| KMP next 数组 | $O(m^2)$（朴素理解） | $O(m)$ |

## 五、考试重点

1. **聚合分析**：直接算总和。最简单的分析方法，能用就用。
2. **动态数组倍增的摊还 $O(1)$** 几乎是必考题。
3. **建堆 $O(n)$ 的证明**：$\sum_{h} (h \cdot n/2^{h+1}) \le 2n$。考试不会让推导这个级数，但要**记住结论**。

## 六、易错点

1. **摊还分析 ≠ 平均复杂度**：摊还是指最坏输入下的均摊，不是随机输入
2. **不是所有情况都能摊还**：必须有"便宜"的操作积累"信用"
3. **势能函数的选择不是唯一的**：只要满足 $\Phi(D_0)=0$ 且 $\Phi(D_i) \ge 0$ 即可
4. **动态数组 +1 扩容策略摊还是 $O(n)$**，只有倍增才是 $O(1)$
