# 算法复杂度分析

> 一句话：用数学语言精确描述算法运行时间随输入规模增长的趋势。

## 一、大O记号

### 1.1 定义

对于函数 $f(n)$ 和 $g(n)$，若存在正常数 $c$ 和 $n_0$，使得对所有 $n \ge n_0$，有：

$$0 \le f(n) \le c \cdot g(n)$$

则称 $f(n) = O(g(n))$。

**通俗理解**：大O给出了算法的**上界**，描述最坏情况下运行时间随输入规模 $n$ 增长的趋势，忽略常数和低阶项。

### 1.2 常见复杂度排序（从快到慢）

| 记号 | 名称 | 典型算法 | $n=10^6$ 时 |
|------|------|---------|-------------|
| $O(1)$ | 常数 | 数组下标访问 | 1 |
| $O(\log n)$ | 对数 | 二分查找 | ~20 |
| $O(\sqrt{n})$ | 平方根 | 质数检验 | 1000 |
| $O(n)$ | 线性 | 遍历数组 | $10^6$ |
| $O(n\log n)$ | 线性对数 | 归并排序 | $2\times 10^7$ |
| $O(n^2)$ | 平方 | 冒泡排序 | $10^{12}$（不可行） |
| $O(2^n)$ | 指数 | 暴力子集枚举 | 天文数字 |

### 1.3 计算规则

**1. 基本操作计数**：找到算法中最内层循环的执行次数。

**2. 保大弃小**：$O(n^2 + n) = O(n^2)$，只保留最高阶项。

**3. 忽略常数系数**：$O(3n^2) = O(n^2)$。

**4. 顺序结构相加，嵌套结构相乘**：

- 两个顺序循环：$O(n) + O(m) = O(n+m)$
- 嵌套循环：$O(n) \times O(m) = O(nm)$

## 二、递归方程求解

### 2.1 递归树法

以归并排序 $T(n) = 2T(n/2) + O(n)$ 为例：

```
Level 0:   cn                       ← n 个元素的工作量
Level 1:   cn/2 + cn/2 = cn        ← 两个 n/2 子问题
Level 2:   cn/4 × 4 = cn           ← 四个 n/4 子问题
...
Level log n: 1 × n = cn            ← 树的最底层
```

总时间 = 层数 × 每层工作量 = $\log_2 n \cdot cn = O(n\log n)$。

### 2.2 迭代展开法

以 $T(n) = T(n-1) + n$ 为例：

$$T(n) = n + (n-1) + (n-2) + \dots + 1 = \frac{n(n+1)}{2} = O(n^2)$$

## 三、主定理

对于形如 $T(n) = aT(n/b) + f(n)$ 的递归式，比较 $n^{\log_b a}$ 与 $f(n)$：

| 情况 | 条件 | 结论 |
|------|------|------|
| Case 1 | $f(n) = O(n^{\log_b a - \epsilon})$ | $T(n) = \Theta(n^{\log_b a})$ |
| Case 2 | $f(n) = \Theta(n^{\log_b a})$ | $T(n) = \Theta(n^{\log_b a} \log n)$ |
| Case 3 | $f(n) = \Omega(n^{\log_b a + \epsilon})$ 且 $af(n/b) \le cf(n)$ | $T(n) = \Theta(f(n))$ |

**口诀**：比较 $n^{\log_b a}$（叶子工作量）和 $f(n)$（根的工作量）谁大，谁主导复杂度。

### 3.1 经典应用

| 算法 | 递归式 | $a$,$b$ | $\log_b a$ | $f(n)$ | 情况 | 结果 |
|------|--------|---------|------------|--------|------|------|
| 二分查找 | $T(n)=T(n/2)+1$ | 1,2 | 0 | $O(1)$ | Case 2 | $O(\log n)$ |
| 归并排序 | $T(n)=2T(n/2)+n$ | 2,2 | 1 | $O(n)$ | Case 2 | $O(n\log n)$ |
| Strassen | $T(n)=7T(n/2)+n^2$ | 7,2 | 2.81 | $O(n^2)$ | Case 1 | $O(n^{2.81})$ |
| 朴素矩阵乘 | $T(n)=8T(n/2)+n^2$ | 8,2 | 3 | $O(n^2)$ | Case 1 | $O(n^3)$ |

## 四、易错点

1. **递归树每层代价不是简单的 $f(n)$**：需要根据子问题规模重新计算
2. **主定理对 Case 3 有额外条件**（正则条件 $af(n/b) \le cf(n)$），只满足 $\Omega$ 不够
3. **$\log n$ 与 $n^\epsilon$ 的比较**：对于任意 $\epsilon > 0$，$\log n = O(n^\epsilon)$
4. **空间复杂度不可忽略**：递归算法的递归栈占用空间，如归并排序需要 $O(n)$ 额外空间
