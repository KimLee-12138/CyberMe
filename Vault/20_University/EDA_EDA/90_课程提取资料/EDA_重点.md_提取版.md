# 自顶向下的设计方法

**自顶向下 (Top-Down)：** 从**系统级**开始，将整个系统划分为若干个基本模块；然后再将每个基本模块划分为下一层次的基本单元；层层向下划分，直到落实到具体的**逻辑门级**或**晶体管级**。

**区别（加分项）：** 传统的纯硬件设计多采用“自底向上”（先选芯片，再搭电路），而现代 EDA 结合高密度可编程逻辑器件（FPGA/CPLD），必须采用“自顶向下”的设计，因为这样可以**早期发现系统级错误**，且**便于团队并行开发**。

# 两个器件FPGA,CPLD

#### 1. 内部结构与掉电特性（极高频填空/判断题！）

- **CPLD：**
  - **结构：** 基于**乘积项结构** 。  
  - **特性：** 属于**非易失性器件**，意味着**断电不会丢失数据** 。  
- **FPGA：**
  - **结构：** 基于**查找表结构**（如 SRAM 静态存储器） 。  
  - **特性：** 属于**易失性器件**，意味着**断电会丢失数据** （每次上电都需要重新配置）。  

#### 2. 优势与适用场景（简答题标准答案）

- **CPLD：**
  - **特性与优势：** 开发快速、延迟确定 （因为内部连线结构相对固定）。  
  - **应用首选：** 适用于设计译码等**复杂的组合逻辑**，且成本较低，适合新手 。  
- **FPGA：**
  - **特性与优势：** 并行处理能力强、可重构、灵活性极高 。  
  - **应用首选：** 适合设计含有**大量触发器**的电路，例如**复杂的时序电路**，适合产品开发 。（这也是为什么后半学期的重点全在考触发器和状态机的原因）。  
  - **小缺点：** 开发相对复杂，功耗和成本较高 。

# 数制与码制

1. 补码在运算中的意义：

> 通过补码可以将减去一个数用加上这个数的补码来表示。同时运算结果也是补码形式

1. 8421码指的就是二进制编码，格雷码要掌握怎么写
2. ACSII码是一组八位的二进制代码，一共256个

### 1. 补码在运算中的意义：将减法转化为加法

你的理解非常准确。在计算机底层的 ALU（算术逻辑单元）中，如果同时设计加法器和减法器，会大幅增加硬件电路的复杂度与成本。

通过引入**补码（Two's Complement）**，计算机可以将所有的减法运算转换为加法运算。

- **核心逻辑**：$A - B \equiv A + (-B)$。在计算机内部，负数是以其补码形式存储的，因此运算变成了 $[A]_补 + [-B]_补 = [A - B]_补$。
- **模（Modulo）的概念**：补码的本质是利用了溢出。以 8 位系统为例，容量是 256（即 $2^8$）。在这套系统中，$-1$ 和 $+255$ 在二进制表示上是等价的（`11111111`）。当两数相加超过 255 时，最高位的进位会被硬件自动丢弃，这个“丢弃进位”的过程天然完成了减法。
- **结果闭环**：参与运算的是补码，硬件加法器得出的结果天然也是补码形式，直接存入寄存器，逻辑高度自洽。

### 1. 正数和 0 的计算规则

**口诀：正数的原码、反码、补码完全相同。**

对于正数，不需要进行任何取反或加 1 的操作，直接将其转换为二进制即可。

- **例子：计算十进制 `+5` 的补码**
  - 十进制 `5` 的二进制是 `101`。
  - 填充至 8 位，最高位符号位为 `0`。
  - **原码**：`0000 0101`
  - **反码**：`0000 0101`
  - **补码**：`0000 0101`

### 2. 负数的计算规则

**口诀：符号位不变，其余各位取反，最后加 1。**

计算负数的补码，需要经历“原码 $\rightarrow$ 反码 $\rightarrow$ 补码”三个步骤。

- **例子：计算十进制 `-5` 的补码**
  - **第一步求原码**：先写出绝对值（5）的二进制，最高位符号位置为 `1`。
    - `-5` 的原码：`1000 0101`
  - **第二步求反码**：**符号位（最高位）保持不变**，其余数值位全部按位取反（0 变 1，1 变 0）。
    - `-5` 的反码：`1111 1010`
  - **第三步求补码**：在反码的基础上，最低位加 `1`。
    - `-5` 的补码：`1111 1010` + `1` = **`1111 1011`**

### 2. 8421码：它是 BCD 码，而非纯二进制码

这里需要作一个关键的纠正：**8421码并不是指普通的纯二进制编码**，而是 **BCD码（Binary-Coded Decimal，二进码十进数）** 中最常用的一种。

- **原理**：它使用 4 位二进制数来严格表示 1 位**十进制数**（0~9），4 位二进制的权重分别是 8、4、2、1。
- **与纯二进制的区别**：
  - 十进制的 `12`，纯二进制码是：`1100`。
  - 十进制的 `12`，8421 BCD 码是：`0001 0010` （`1` 被编码为 `0001`，`2` 被编码为 `0010`）。
- 8421 码中 `1010` 到 `1111` （即 10 到 15）这 6 个状态是**非法状态**。在进行 8421 码加法运算时，如果某四位的结果大于 9，必须加 6（即 `0110`）进行修正，这也是底层硬件和汇编编程（如 8086 中的 `DAA` 指令）中必须考虑的细节。

### 3. 格雷码（Gray Code）：消除竞争冒险的利器

格雷码的核心特性是：**任意两个相邻的数值之间，只有一位二进制位发生改变**。这在硬件传感器（如光电编码器）和异步时钟域的数据传输中至关重要，可以彻底消除多位同时跳变带来的短暂错误状态（竞争冒险）。

**如何手写/推导格雷码（二进制转格雷码规则）：**

设纯二进制码为 $B_{n-1}B_{n-2}...B_0$，对应的格雷码为 $G_{n-1}G_{n-2}...G_0$。

1. **最高位保留**：格雷码的最高位与二进制的最高位完全相同（$G_{n-1} = B_{n-1}$）。

2. **异或运算（XOR）**：其余位的值，等于对应的二进制位与其更高一位进行**异或**（相同为0，相异为1）。

   公式：$G_i = B_i \oplus B_{i+1}$ （$i$ 从 $0$ 到 $n-2$）。

*示例：将纯二进制码 `1010` 转为格雷码*

- 最高位保留：$G_3 = B_3 = 1$
- 次高位：$G_2 = B_2 \oplus B_3 = 0 \oplus 1 = 1$
- 次低位：$G_1 = B_1 \oplus B_2 = 1 \oplus 0 = 1$
- 最低位：$G_0 = B_0 \oplus B_1 = 0 \oplus 1 = 1$
- 结果：`1111`。

### 4. ASCII 码：7位标准与8位扩展

- **标准 ASCII 码**：最初由美国制定，其实是一个 **7 位**的二进制代码（占用 1 个字节的低 7 位，最高位通常为 0，或者在早期通信中用作奇偶校验位）。$2^7 = 128$，所以标准 ASCII 码只有 128 个字符（包含 0-9 数字、大小写英文字母、标点符号以及控制字符如换行、回车等）。
- **扩展 ASCII 码**：随着计算机的发展，128 个字符不够用（特别是欧洲语言中的带音标字母），于是人们启用了最高位（第 8 位），将字符集扩展到了 $2^8 = 256$ 个。这就是你提到的 256 个字符的来源。
