1. 自上而下  top-down 设计方法  
题目：什么是自上而下的设计方法？
卷面答案：
自上而下的设计方法是指从数字系统的整体功能和总体结构出发，将系统逐层分解为若干子系统和功能
模块；若子系统仍然复杂，则继续向下分解，直到每个模块都便于用逻辑电路或  HDL 程序实现。该方法
层次清晰、便于模块化设计、仿真、调试和系统集成。
2. FPGA 和  CPLD  
题目：简述  FPGA 和  CPLD 的特点及区别。
卷面答案：
FPGA 和  CPLD 都属于可编程逻辑器件。 CPLD 通常基于乘积项结构，适合实现译码、控制等中小规模组
合逻辑，延时较确定，成本较低。 FPGA 通常基于查找表  LUT 结构，内部含有大量可编程逻辑单元、触
发器和布线资源，适合实现复杂时序逻辑、并行处理和大规模数字系统。总体上， CPLD 适合中小规模控
制逻辑， FPGA 适合大规模、复杂、并行的数字系统设计。
3. FPGA/CPLD 数字系统设计流程  
题目：基于  FPGA/CPLD 的数字系统设计流程包括哪些步骤？
卷面答案：
基于  FPGA/CPLD 的数字系统设计流程一般包括：设计准备、设计输入、功能仿真、综合、适配、时序仿
真、编程下载和在线测试。设计输入可以采用原理图输入或硬件描述语言输入；综合后生成电路网表文
件；适配完成器件内部布局布线并生成下载文件；下载到目标器件后进行实际测试。
4. 综合  
题目：什么是综合？
卷面答案：
综合是指将  EDA 平台中输入的  HDL 程序或原理图描述转换成与目标硬件结构相对应的底层电路网表文
件的过程。综合的结果不是最终下载文件，而是便于可编程逻辑器件实现的电路网表。
5. 适配  
题目：什么是适配？
卷面答案：
适配是指将综合得到的电路网表配置到指定目标器件中的过程，主要完成器件内部的逻辑资源分配、布
局和布线，并最终生成可下载到  FPGA/CPLD 中的配置文件，如  .sof 或 .pof 文件。

6. 补码的概念和作用  
题目：补码在运算中的意义是什么？
卷面答案：
补码的主要意义是把减法运算转化为加法运算。在计算机中，负数通常用补码形式表示，因此执行  A-B 
时可以转化为  A+(-B) 的补码运算。运算过程中参加运算的是补码，运算结果也是补码形式。补码可以
简化硬件电路，使加法和减法统一由加法器完成。
7. 8421 码与  ASCII 码  
7.1 8421 码  
题目：什么是  8421 码？
卷面答案：
8421 码是最常用的一种  BCD 码，即二进制编码的十进制数。它用  4 位二进制代码表示  1 位十进制数， 4 
位的权值分别为  8 、 4 、 2 、 1 。 8421 码只能表示十进制数  0 ～ 9 ，因此  1010 ～ 1111 是非法状态。例如十
进制数  12 的  8421 码为  0001 0010，而不是纯二进制的  1100。
7.2 ASCII 码  
题目： ASCII 码是多少位？
卷面答案：
标准  ASCII 码是  7 位二进制代码，可以表示  128 个字符。实际存储时通常占用  1 个字节，最高位可为  0 
或用于校验。扩展  ASCII 码使用  8 位，可以表示  256 个字符。

8.2 反演定理  
题目：什么是反演定理？
卷面答案：
对逻辑函数式求反函数时，应将原式中的 “ 与 ” 换成 “ 或 ” ， “ 或 ” 换成 “ 与 ” ；将常量  0 换成  1 ， 1 换成  0 ；将原
变量换成反变量，反变量换成原变量，并保持原有运算优先级不变。所得表达式即为原函数的反函数。
8.3 电路图转逻辑表达式  
题目：如何由电路图写出逻辑表达式？
卷面答案：
由电路图写表达式时，应从输入端到输出端逐级分析。先给每一级门电路的输出标注中间变量，根据与
门、或门、非门、与非门、或非门等逻辑关系写出中间表达式，最后逐级代入，得到输出端的逻辑表达
式。
8.4 最小项的重要性质  
卷面答案：
n 个变量的最小项包含  n 个变量，每个变量以原变量或反变量形式出现一次。每一个最小项只对应真值
表中的一行。两个相邻最小项只有一个变量不同，因此可以合并，并消去不同的变量。
8.5 卡诺图化简  
卷面答案：
卡诺图化简时，将函数取  1 的最小项填入卡诺图，相邻的  1 按照  1 、 2 、 4 、 8 等   个方格进行圈组，圈
越大越好，圈数越少越好。相邻方格包括上下左右相邻以及边界首尾相邻。每个圈保留不变的变量，消
去变化的变量，最后将各圈对应的乘积项相加得到最简与或式。
9. PLD、集成度、非易失性、 SRAM 、 FPGA 结构  
题目：什么是  PLD ？ CPLD 和  FPGA 有什么特点？
卷面答案：
PLD 是  Programmable Logic Device ，即可编程逻辑器件。按集成度可分为简单  PLD 和复杂  PLD ，
CPLD 和  FPGA 属于复杂可编程逻辑器件。按编程元件和工艺可分为非易失性器件和易失性器件。非易失
性器件断电后配置信息不丢失，通常基于乘积项结构； SRAM 型器件断电后信息会丢失，属于易失性器
件。 FPGA 多基于查找表  LUT 结构，通常由可编程逻辑单元、触发器、布线资源和  I/O 模块组成。

11. assign 与  always  
题目： assign 和  always 有什么区别？
卷面答案：
assign 是连续赋值语句，常用于数据流描述，其左端一般为  wire 型网络变量。 always 是过程语句，
常用于行为描述，其内部可以使用  if 、 case 等语句，左端通常定义为  reg 型变量。多个  assign 语句和
多个  always 语句在硬件意义上都是并行执行的。 always 语句块之间不能理解为软件程序中的先后顺
序执行，也不能把  always 语句嵌套在另一个  always 语句内部。

19.4 组合逻辑电路分析与设计  
题目：组合逻辑电路分析和设计步骤是什么？
卷面答案：
组合逻辑电路的特点是任意时刻输出只取决于当前输入，与过去状态无关。分析组合逻辑电路时，应从
电路图或  Verilog 代码写出逻辑表达式，再进行化简并判断功能。设计组合逻辑电路时，应先分析功能，
确定输入输出变量，列真值表，进行逻辑化简，得到表达式，最后画电路图或写  Verilog 程序。
19.5 普通编码器与优先编码器  
题目：普通编码器和优先编码器有什么区别？
卷面答案：
普通编码器要求任意时刻只能有一个输入信号有效，并把该有效输入对应的编号编码输出。优先编码器
允许多个输入同时有效，但按照预先规定的优先级，只对优先级最高的有效输入进行编码输出。普通编
码器常用  case 语句实现，优先编码器常用  if-else if-else 语句实现。
19.6 译码器和最小项  
题目：译码器的输出与最小项有什么关系？
卷面答案：
n 变量译码器有   个输出，每一个输出对应输入变量的一个最小项。例如  3-8 译码器有  3 个输入、 8 个
输出，每个输出对应一个三变量最小项。因此可以利用  3-8 译码器产生逻辑函数所需的最小项，再通过
或门实现组合逻辑函数。

19.11 时序逻辑电路的三个方程  
题目：时序逻辑电路可用哪三个方程描述？
卷面答案：
时序逻辑电路可用输出方程、驱动方程和状态方程描述。输出方程表示输出信号与当前输入、当前状态
之间的关系；驱动方程表示触发器输入信号与当前输入、当前状态之间的关系；状态方程表示下一状态
与当前输入、当前状态之间的关系。通过这三个方程可以列状态转移表、画状态转移图并分析电路功
能。

题目： Moore 型状态机和  Mealy 型状态机有什么区别？
卷面答案：
Moore 型状态机的输出只与当前状态有关，与当前输入无直接关系，输出通常写在状态节点内。 Mealy  
型状态机的输出与当前状态和当前输入都有关，输出通常写在状态转移线上。序列检测器常采用  Mealy  
型状态机设计。
19.13 同步时序电路设计步骤  
题目：同步时序电路设计步骤是什么？
卷面答案：
同步时序电路设计步骤为：第一，分析题意，确定状态数；第二，画状态转移图，必要时进行状态化
简；第三，根据状态数确定触发器个数并进行状态编码；第四，由状态转移图列状态转移表；第五，根
据状态转移表求输出方程和状态方程；第六，选择触发器类型，求驱动方程；第七，判断自启动能力；
第八，画出组合逻辑和触发器组成的完整电路图。
19.14 状态化简与自启动  
题目：什么是状态化简？如何判断自启动？
卷面答案：
若两个状态在相同输入下具有相同输出，并且转移到相同或等价的次态，则这两个状态称为等价状态，
可以合并，从而实现状态化简。判断自启动时，应检查所有无效状态能否经过有限次状态转移进入有效
状态循环。若所有无效状态都能进入有效状态，则电路可自启动；若存在无效状态陷入无效循环，则电
路不能自启动。

19.16 锁存器  
题目：锁存器和触发器有什么区别？
卷面答案：
锁存器通常受电平控制，在使能电平有效期间输出会随输入变化；触发器通常受时钟边沿控制，只在时
钟有效边沿采样输入并更新输出。锁存器属于电平触发，触发器属于边沿触发。根据回忆版，锁存器不
是考试重点，但需要能区分概念。module seq_det_1001(
    input  wire clk,
    input  wire rst,
    input  wire x,
    output reg  z
);
    reg [1:0] cs, ns;
    parameter S0=2'd0, S1=2'd1, S2=2'd2, S3=2'd3;
    always @(posedge clk or posedge rst) begin
        if (rst) cs <= S0;
        else     cs <= ns;
    end
    always @(*) begin
        case (cs)
            S0: ns = x ? S1 : S0;
            S1: ns = x ? S1 : S2;
            S2: ns = x ? S1 : S3;
            S3: ns = x ? S1 : S0;
            default: ns = S0;
        endcase
    end
    always @(*) begin
        z = (cs == S3 && x == 1'b1) ? 1'b1 : 1'b0;
    end
endmodule

题目：有限状态机  Verilog 通常由哪几个  always 块组成？
卷面答案：
有限状态机通常由三个  always 块组成：第一个  always 块为状态寄存器，在时钟边沿到来时将下一状态  
ns 赋给当前状态  cs；第二个  always 块为下一状态逻辑，根据当前状态和输入信号确定下一状态；第
三个  always 块为输出逻辑，根据当前状态或当前状态与输入信号确定输出。三段式状态机结构清晰，便
于分析和调试。module clk_div_1hz(
    input  wire clk50m,
    input  wire rst,
    output reg  clk1hz
);
    reg [24:0] cnt;
    always @(posedge clk50m or posedge rst) begin
        if (rst) begin
            cnt    <= 25'd0;
            clk1hz <= 1'b0;
        end else if (cnt == 25'd24_999_999) begin

        end else begin
            cnt <= cnt + 1'b1;
        end
    end
endmodule
