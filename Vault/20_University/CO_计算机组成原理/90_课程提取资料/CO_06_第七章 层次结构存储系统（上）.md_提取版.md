> SRAM 用“触发器”存信息，靠 6 个 MOS 管保持 0/1；DRAM 用“电容”存信息，靠电容有没有电荷表示 0/1。

### 1. 先看整体结构：记忆单元是什么？

PPT 最下面写：

```
记忆单元 Cell → 存储器芯片 Chip → 内存条/存储器模块
```

意思是：

一个最小的存储单元只能存 **1 bit**，也就是一个 0 或 1。

很多个记忆单元组成一个存储阵列，再加上地址译码、读写控制等电路，构成一个 **存储器芯片**。

多个芯片再组合起来，就可以形成我们平时说的内存条。

------

### 2. SRAM 记忆单元

左边是 **SRAM 记忆单元**。

SRAM 全称是：

```
Static Random Access Memory
静态随机存取存储器
```

它的一个基本存储单元通常由：

```
6 个 MOS 管
```

组成。

所以 PPT 写：

```
位元：6 个 MOS 管
```

------

#### 2.1 SRAM 怎么存 0 和 1？

SRAM 本质上是用一个类似“锁存器/触发器”的结构来存数据。

它内部有两个稳定状态：

```
状态 1：存 0
状态 2：存 1
```

只要不断电，这个状态就能一直保持。

所以 SRAM 叫 **静态存储器**。

这里“静态”的意思是：

> **只要供电存在，不需要刷新，数据就能保持。**

------

#### 2.2 SRAM 里面的字选择线 W 是什么？

图上有：

```
字选择线 W
```

它的作用是：

> **选中这一行存储单元。**

PPT 写：

```
字选择线 W 高电平选中
```

也就是说：

```
W = 1：这个存储单元被选中，可以读写
W = 0：这个存储单元不参与读写
```

可以把 W 理解成“开门信号”。

只有 W 选中了这个单元，外面的读写电路才能访问它。

------

#### 2.3 SRAM 里面的 S0、S1 是什么？

左图有两根位线：

```
S0
S1
```

SRAM 通常用两根互补位线来传输数据。

可以简单理解为：

```
S0 和 S1 一起表示当前存的是 0 还是 1
```

如果存储单元中保存的是 0，读的时候会让其中一根线产生变化；如果保存的是 1，会让另一根线产生变化。

------

### 3. SRAM 的写操作

PPT 写：

```
写：写 0 则拉低 S0，写 1 则拉低 S1
```

这个意思是：

当 W 选中这个单元后，外部读写控制电路通过 S0/S1 位线强制改变存储单元内部状态。

写 0：

```
W = 1
把 S0 拉低
存储单元被强制变成 0 状态
```

写 1：

```
W = 1
把 S1 拉低
存储单元被强制变成 1 状态
```

你不用纠结为什么是拉低 S0 或拉低 S1，重点是：

> **SRAM 写数据时，通过两根位线把存储单元内部的触发器状态强行改成 0 或 1。**

------

### 4. SRAM 的读操作

PPT 写：

```
读：S0 负脉冲读出为 0，S1 负脉冲读出为 1
```

意思是：

读的时候，先选中 W。

如果单元里存的是 0，那么 S0 这根位线会出现一个可检测的电平变化，读写控制电路判断为 0。

如果单元里存的是 1，那么 S1 这根位线会出现变化，读写控制电路判断为 1。

所以：

```
S0 有响应 → 读出 0
S1 有响应 → 读出 1
```

SRAM 读数据的特点是：

> **读操作不会破坏原来的数据。**

因为 SRAM 的数据是由稳定的触发器状态保存的，读一下不会把它读没。

------

### 5. SRAM 的特点

SRAM 最大特点：

```
速度快
不需要刷新
结构复杂
单位面积存储容量小
成本高
```

为什么速度快？

因为它是触发器结构，读写时不需要等待电容充放电那么久。

为什么容量小、成本高？

因为一个 bit 要用 6 个 MOS 管，占面积大。

所以 SRAM 常用在：

```
Cache 高速缓存
寄存器堆
CPU 内部高速存储结构
```

------

### 6. DRAM 记忆单元

右边是 **DRAM 记忆单元**。

DRAM 全称是：

```
Dynamic Random Access Memory
动态随机存取存储器
```

它的一个基本存储单元通常由：

```
1 个电容 + 1 个 MOS 管
```

组成。

所以 PPT 写：

```
位元：电容 + MOS
```

------

### 7. DRAM 怎么存 0 和 1？

DRAM 用电容有没有电荷表示 0 或 1。

简单说：

```
电容有电荷 → 表示 1
电容没电荷 → 表示 0
```

PPT 写的是：

```
写 1 则数据线 S 加高电平充电
写 0 加低电平放电
```

也就是：

写 1：

```
W 选中
数据线 S 加高电平
给电容充电
电容有电荷，表示 1
```

写 0：

```
W 选中
数据线 S 加低电平
电容放电
电容无电荷，表示 0
```

------

### 8. DRAM 里面的字选择线 W

和 SRAM 一样，DRAM 也有字选择线 W。

它的作用也是：

> **选中这个存储单元。**

如果：

```
W = 1
```

MOS 管导通，电容和数据线 S 连起来，可以读写。

如果：

```
W = 0
```

MOS 管关闭，电容和外界隔开，保持原来的电荷状态。

------

### 9. DRAM 的读操作

PPT 写：

```
读：S 有放电电流读出为 1，无电流读出为 0
```

通俗理解：

读的时候，先选中 W，让电容和数据线 S 接通。

如果电容里原来有电荷，说明存的是 1。它会向数据线释放一点电荷，产生微弱电流或电压变化，读写控制电路检测到后判断为 1。

如果电容里没电荷，说明存的是 0。数据线上基本没有这种放电变化，就判断为 0。

所以：

```
检测到放电/电压变化 → 读出 1
没有明显变化 → 读出 0
```

------

### 10. DRAM 读操作的一个重要特点：破坏性读出

DRAM 这个地方很重要。

因为 DRAM 的数据存在电容里，读的时候电容会和数据线连接，电荷会发生转移。

所以读完后，电容里的电荷可能被破坏或变弱。

因此 DRAM 读操作通常是：

```
破坏性读出
```

读完以后必须重新写回原来的数据。

也就是：

```
读出 1 后，要重新给电容充电
读出 0 后，维持放电状态
```

这叫：

```
读后再生
```

------

### 11. DRAM 为什么需要刷新？

电容有一个问题：

> **电容会漏电。**

即使你不读它，电容里的电荷也会慢慢消失。

比如原来存的是 1：

```
电容有电荷
```

过一段时间后，电荷漏掉了，就可能变成：

```
电容没电荷
```

那 1 就丢了。

所以 DRAM 必须定期把数据重新读出再写回，这个过程叫：

```
刷新 Refresh
```

这也是 DRAM 叫“动态存储器”的原因。

“动态”的意思是：

> **数据不能静静放着不管，必须周期性刷新。**

------

### 12. DRAM 的特点

DRAM 最大特点：

```
结构简单
容量大
成本低
速度比 SRAM 慢
需要刷新
读出可能破坏数据
```

为什么容量大？

因为一个 bit 只需要：

```
1 个电容 + 1 个 MOS 管
```

比 SRAM 的 6 个 MOS 管省很多面积。

所以 DRAM 常用在：

```
主存
内存条
显存的大容量存储部分
```

------

### 13. SRAM 和 DRAM 最核心区别

可以直接这样记：

| 对比项           | SRAM         | DRAM                   |
| ---------------- | ------------ | ---------------------- |
| 存储原理         | 触发器状态   | 电容电荷               |
| 单元结构         | 6 个 MOS 管  | 1 个 MOS 管 + 1 个电容 |
| 是否刷新         | 不需要刷新   | 需要刷新               |
| 读出是否破坏数据 | 非破坏性读出 | 通常是破坏性读出       |
| 速度             | 快           | 慢                     |
| 集成度           | 低           | 高                     |
| 成本             | 高           | 低                     |
| 常见用途         | Cache        | 主存/内存条            |

------

### 14. 为什么 SRAM 快但贵，DRAM 慢但便宜？

这个非常适合考试问。

SRAM：

```
一个 bit 用 6 个 MOS 管
```

结构复杂，占面积大，所以单位面积能放的 bit 少，成本高。

但是它用触发器保持状态，读写速度快，不需要刷新。

所以：

```
SRAM：快，但贵，容量小
```

DRAM：

```
一个 bit 用 1 个 MOS 管 + 1 个电容
```

结构简单，占面积小，所以单位面积能放很多 bit，成本低，容量大。

但是电容会漏电，要刷新，读写也更麻烦。

所以：

```
DRAM：慢，但便宜，容量大
```

## 举例：典型的4K位DRAM的组织

![img](C:\Users\22067\Desktop\计组复习\assets\cover663_20260601075938.jpg)

典型 4K 位 DRAM 芯片共有 4096 个存储单元，可组织成 64 行 × 64 列的矩阵。

访问某一位时，需要 12 位逻辑地址，其中 6 位作为行地址，6 位作为列地址。
行地址经 X 地址译码选中一行，列地址经 Y 地址译码选中一列，最终访问行列交叉点处的存储单元。

DRAM 为了减少芯片引脚数，采用地址分时复用：
同一组地址线先送行地址，再送列地址。
/RAS 用于锁存行地址，/CAS 用于锁存列地址。

每增加 1 根地址线，由于行地址和列地址都增加 1 位，行数和列数各扩大 2 倍，因此总容量扩大 4 倍。

## 举例：典型的4Mx4位DRAM（16M位）

![img](C:\Users\22067\Desktop\计组复习\assets\cover806_20260601075938.jpg)

4M×4 位 DRAM 表示芯片有 4M 个地址，每个地址一次读写 4 bit，总容量为 16M bit。

芯片内部有 4 个 4M 位平面，每个位平面组织为 2048 行 × 2048 列。
一次访问时，行地址和列地址共同选中每个位平面中的一个存储单元，4 个位平面同时输出或写入 4 bit 数据。

因为 2048 = 2^11，所以行地址需要 11 位，列地址需要 11 位。
DRAM 采用地址分时复用，外部只需 A0~A10 共 11 根地址线：
RAS 有效时锁存行地址，CAS 有效时锁存列地址。

W 控制读写，OE 控制输出使能。
读出放大器用于放大 DRAM 微弱读出信号。
刷新计数器用于周期性刷新 DRAM 行，防止电容漏电导致数据丢失。

## DRAM芯片的刷新

![img](C:\Users\22067\Desktop\计组复习\assets\cover779_20260601075938.jpg)

### 2. 什么是刷新周期？

PPT 第一行说：

```
刷新周期：相邻两次对某个特定行进行刷新的时间间隔。
```

也就是说，DRAM 通常是 **按行刷新** 的。

如果某一行数据已经刷新过一次，那么过一段时间后必须再次刷新这一行。两次刷新之间的最大时间间隔，就是刷新周期。

PPT 里说：

```
电容数据有效保存期的上限 10ms ~ 100ms，
目前普遍接受的是 64ms。
```

意思是：

> **每一行至少要在 64ms 内刷新一次。**

注意是 **每一行都要在 64ms 内刷新一次**，不是整个芯片只刷新一次。

------

### 3. 集中刷新

集中刷新就是：

> **先集中一段时间让 CPU 正常访问 DRAM，然后再集中一段时间专门刷新。**

PPT 举例：

```
64ms 中前 60ms 用于访存，
后 4ms 用于刷新。
```

也就是：

```
前 60ms：CPU 可以正常读写内存
后 4ms：DRAM 忙着刷新，CPU 不能访问内存
```

这后面的 4ms 就叫：

```
死区
```

因为这段时间内存被刷新操作占用，CPU 不能正常访问。

所以集中刷新的特点是：

```
优点：控制简单
缺点：有死区，刷新期间不能访存
```

为什么很少使用？

因为 CPU 不能接受内存突然有一大段时间不能访问。尤其是现代计算机里，内存不能长时间“停业”。

------

### 4. 分散刷新

分散刷新就是：

> **每次正常读写操作之后，顺便刷新一行。**

比如原来一次访存可能需要：

```
读写时间
```

现在变成：

```
读写时间 + 刷新一行时间
```

这样每次访问都附带一个刷新操作。

它的好处是：

```
不存在集中刷新那种明显死区
```

因为刷新被分散到了每次访存之后。

但是缺点是：

```
每次读写操作都被拉长
刷新过于频繁
```

也就是说，CPU 每访问一次内存，都要额外等一小段刷新时间，整体访存速度会下降。

所以分散刷新可以理解为：

> **没有大块死区，但每次访存都变慢。**

------

### 5. 异步刷新

异步刷新是现在常用的方式。

它的思想是：

> **每隔一小段固定时间刷新一行，把刷新操作均匀分布在整个刷新周期内。**

PPT 写：

```
每隔一段时间对某一行进行刷新，
即多个读写操作加一个刷新操作。
```

也就是说，不是每次读写后都刷新，也不是最后集中刷新，而是：

```
访问一段时间
刷新一行
再访问一段时间
再刷新一行
……
```

这样既不会出现集中刷新那种长时间死区，也不会像分散刷新那样每次访存都刷新。

所以异步刷新比较折中，也更常用。

------

### 6. 例子：2048 行，64ms 内每行刷新一次

PPT 最后一行说：

```
以 2048 行为例，在 64ms 时间内必须轮流对每一行刷新一次，
即每隔 64ms / 2048 = 31.25μs 刷新一行。
```

我们来算一下。

总共有：

```
2048 行
```

要求：

```
64ms 内每一行都刷新一次
```

所以平均每隔多久刷新一行？

```
64ms ÷ 2048
```

因为：

```
64ms = 64000μs
```

所以：

```
64000μs ÷ 2048 = 31.25μs
```

因此：

```
每隔 31.25μs 刷新一行
```

这就是异步刷新的典型做法。

可以想象成：

```
第 0 μs      刷新第 0 行
第 31.25 μs  刷新第 1 行
第 62.5 μs   刷新第 2 行
……
直到 64ms 内把 2048 行全部刷新一遍
```

然后下一轮继续。

------

### 7. 三种刷新方式对比

| 刷新方式 | 做法                   | 优点           | 缺点                         |
| -------- | ---------------------- | -------------- | ---------------------------- |
| 集中刷新 | 一段时间集中刷新所有行 | 控制简单       | 有死区，刷新时不能访存       |
| 分散刷新 | 每次读写后刷新一行     | 没有明显死区   | 每次访存都变慢，刷新过于频繁 |
| 异步刷新 | 每隔固定时间刷新一行   | 比较均衡，常用 | 控制稍复杂                   |

## CPU与主存储器之间的通信

### 一、CPU 和主存为什么需要“通信方式”？

CPU 要访问主存时，一般要做这些事：

```
CPU 给出地址
CPU 发出读/写命令
主存根据地址读出或写入数据
CPU 取得数据或确认写入完成
```

问题是：**CPU 怎么知道主存什么时候准备好了？**

因为 CPU 很快，主存相对慢。如果 CPU 一发出读命令就马上取数据，可能数据还没准备好。

所以就需要规定一种通信配合方式。

------

### 二、异步方式：靠握手信号

PPT 上写：

```
异步方式（需握手信号）
```

异步方式的特点是：

> **CPU 和主存没有严格统一的时钟节拍，而是靠“请求—应答”来配合。**

可以理解成两个人打电话确认：

```
CPU：我要读这个地址的数据。
主存：等一下，我去找。
主存：找到了，可以取了。
CPU：收到，我取走。
```

------

#### 1. 异步读操作过程

按照 PPT：

```
CPU 送地址和读命令，然后等待；
主存读数完成后回复“完成”信号给 CPU；
CPU 接收到“完成”信号，从数据线取数。
```

具体过程是：

```
1. CPU 把地址放到地址总线上。
2. CPU 发出读命令 Read。
3. CPU 等待主存回应。
4. 主存读出数据，把数据放到数据总线上。
5. 主存发出完成信号，比如 MFC。
6. CPU 收到完成信号后，从数据总线上取走数据。
```

这里的完成信号就像之前你学过的数据通路里的：

```
WMFC / MFC
```

含义是：

```
Wait for Memory Function Complete
等待主存操作完成
```

------

#### 2. 异步写操作过程

写操作也类似。

比如 CPU 要把数据写到主存：

```
1. CPU 把地址放到地址总线上。
2. CPU 把要写的数据放到数据总线上。
3. CPU 发出写命令 Write。
4. 主存完成写入后，返回“完成”信号。
5. CPU 收到完成信号，认为写操作结束。
```

所以异步方式的本质是：

```
CPU 发请求
主存完成后发应答
CPU 收到应答后继续
```

------

#### 3. 异步方式的优点

异步方式的好处是：

```
适合不同速度的设备之间通信
```

比如有的主存快，有的主存慢，有的外设更慢。异步方式不要求它们固定在某个时间完成，只要完成后发信号告诉 CPU 就行。

所以异步通信比较灵活。

------

#### 4. 异步方式的缺点

缺点是：

```
控制复杂，需要握手信号；
CPU 可能要等待，效率不一定高。
```

因为 CPU 不知道主存什么时候完成，只能等“完成”信号。

一句话：

> **异步方式像“等通知”：CPU 发出请求后等主存回话，主存说完成了，CPU 才继续。**

------

### 三、同步方式：靠统一时钟

PPT 下面讲：

```
同步方式的特点
```

同步方式的核心是：

> **CPU 和主存都受同一个时钟控制，双方提前约定好在第几个时钟周期完成什么动作。**

也就是说，不需要主存额外回复“完成”信号。

------

#### 1. 同步方式的读操作过程

PPT 写：

```
CPU 和主存由时钟控制，无需应答；
主存总是在确定的时间内准备好数据；
CPU 送出地址和读命令后，总是在确定的时间取数据。
```

可以理解成：

```
第 1 个时钟：CPU 发地址和读命令
第 2 个时钟：主存准备数据
第 3 个时钟：CPU 取数据
```

时间是事先规定好的。

所以 CPU 不用问：

```
你准备好了吗？
```

它默认：

```
到这个时钟点，你一定准备好了。
```

------

#### 2. 同步方式的关键要求

同步方式要求主存必须能在规定时间内完成读写。

比如系统规定：

```
CPU 发出读命令后，两个时钟周期后取数据。
```

那主存就必须保证：

```
两个时钟周期内一定把数据准备好。
```

如果主存太慢，就会出错，或者需要插入等待周期。

------

#### 3. 同步方式的优点

优点是：

```
控制简单；
不需要复杂握手；
时序规整；
适合高速系统。
```

因为所有动作都按照时钟节奏来。

------

#### 4. 同步方式的缺点

缺点是：

```
对时间要求严格；
主存必须跟得上时钟；
不同速度设备适配不如异步灵活。
```

如果某个存储器速度慢，就不能简单按固定时钟取数据，需要加等待周期。

一句话：

> **同步方式像“按课表上课”：大家按统一时钟行动，到点就做下一步，不需要互相确认。**

------

### 四、SDRAM 为什么叫同步存储？

PPT 最后写：

```
SDRAM 是同步存储，利用时钟上升沿与下降沿同步传送。
```

SDRAM 的全称是：

```
Synchronous DRAM
同步动态随机存储器
```

这里的 Synchronous 就是“同步”的意思。

它和普通 DRAM 的区别是：

```
普通 DRAM：更多依赖 RAS、CAS 等控制信号时序；
SDRAM：内部操作和外部总线传输受系统时钟统一控制。
```

也就是说，CPU / 内存控制器和 SDRAM 之间按照时钟节拍通信。

![img](C:\Users\22067\Desktop\计组复习\assets\cover802_20260601075939.jpg)

### 1. 主存是干什么的？

PPT 上写：

```
主存用于存放指令及数据
```

也就是说，主存里面既放程序指令，也放程序运行时用到的数据。

比如：

```
add $t0, $t1, $t2    // 指令
变量 a = 10          // 数据
数组 A[0], A[1]      // 数据
```

这些都在主存中存放。

CPU 执行程序时，需要不断从主存中：

```
取指令
读数据
写数据
```

------

### 2. CPU 访问主存需要三类总线

图里 CPU 和主存之间有三类线：

```
地址线
数据线
控制线
```

它们分别干不同的事。

------

#### 2.1 地址线：告诉主存“访问哪里”

图中地址线是：

```
32 位地址线
```

CPU 通过地址线把要访问的地址送给主存。

比如 CPU 想访问地址：

```
0x00001000
```

就把这个地址送到地址线上。

主存拿到地址后，通过地址译码器找到对应的存储单元。

所以地址线的作用是：

> **告诉主存：我要访问哪个存储单元。**

------

#### 2.2 数据线：传输读写的数据

图中数据线是：

```
64 位数据线
```

意思是 CPU 和主存之间一次可以传输 64 bit 数据。

读操作时：

```
主存 → 数据线 → CPU
```

写操作时：

```
CPU → 数据线 → 主存
```

所以数据线的作用是：

> **负责真正传输数据。**

------

#### 2.3 控制线：告诉主存“读还是写”

图中还有：

```
读/写控制信号
```

它通过控制线送到主存的读写控制电路。

比如：

```
Read = 1：读主存
Write = 1：写主存
```

所以控制线的作用是：

> **告诉主存当前要进行读操作还是写操作。**

------

### 3. MAR 是什么？

图中 CPU 里面有：

```
MAR
```

MAR 是：

```
Memory Address Register
主存地址寄存器
```

它的作用是：

> **保存 CPU 要访问的主存地址。**

比如 CPU 要读地址 `1000H` 的数据，就先把地址放入 MAR：

```
MAR ← 1000H
```

然后 MAR 的内容通过地址线送到主存。

所以 MAR 可以理解成：

> **CPU 访问主存时专门存地址的寄存器。**

------

### 4. MDR 是什么？

图中 CPU 里面还有：

```
MDR
```

MDR 是：

```
Memory Data Register
主存数据寄存器
```

它的作用是：

> **暂存 CPU 与主存之间传输的数据。**

读主存时：

```
MDR ← M[MAR]
```

意思是：主存把 MAR 指定地址中的数据读出来，送入 MDR。

写主存时：

```
M[MAR] ← MDR
```

意思是：把 MDR 中的数据写入 MAR 指定的主存地址。

所以 MDR 可以理解成：

> **CPU 和主存之间交换数据的中转站。**

------

### 5. 地址译码器是干什么的？

PPT 写：

```
使用地址译码器寻址
```

主存中有很多存储单元。CPU 给出一个地址后，主存必须知道这个地址对应哪一个单元。

这个工作由：

```
地址译码器
```

完成。

比如地址线送来：

```
00010
```

地址译码器就会选中编号为 `00010` 的那个存储单元。

图右边放大部分里有：

```
00000
00001
00010
00011
00100
...
11111
```

这些就是不同存储单元的地址。

地址译码器的作用就是：

> **把二进制地址翻译成具体选中哪一个存储单元。**

------

### 6. 寻址范围取决于地址线宽度

PPT 写：

```
寻址范围取决于地址线宽度，如图为 0 ~ 2^32 - 1
```

因为图中地址线是 32 位。

32 位地址可以表示：

```
2^32 个不同地址
```

地址编号范围是：

```
0 ~ 2^32 - 1
```

也就是：

```
0000...0000
到
1111...1111
```

如果按字节编址，那么每个地址对应 1 Byte，所以最大可寻址空间是：

```
2^32 Byte = 4GB
```

所以可以记：

> **地址线有 n 位，就能表示 2^n 个地址。**

比如：

```
16 位地址线 → 2^16 个地址
32 位地址线 → 2^32 个地址
```

------

### 7. 什么是按字节编址和按字编址？

图下方写：

```
可按字节或按字编址
```

这个很重要。

------

#### 7.1 按字节编址

按字节编址就是：

> **一个地址对应 1 个字节。**

例如：

```
地址 0 存 1 Byte
地址 1 存 1 Byte
地址 2 存 1 Byte
地址 3 存 1 Byte
```

如果是 32 位地址线：

```
2^32 个地址 × 1 Byte = 4GB
```

现代计算机一般采用按字节编址。

------

#### 7.2 按字编址

按字编址就是：

> **一个地址对应 1 个字。**

如果一个字是 32 位，也就是 4 Byte，那么：

```
地址 0 对应 1 个 32 位字
地址 1 对应下一个 32 位字
地址 2 对应再下一个 32 位字
```

这种情况下，同样 32 位地址线能寻址：

```
2^32 个字
```

如果每个字 4 Byte，则总容量是：

```
2^32 × 4 Byte
```

不过实际计算题中，一定要看题目说的是 **按字节编址** 还是 **按字编址**。

------

### 8. 读主存的过程

假设 CPU 要读主存地址 `A` 中的数据。

过程是：

```
1. CPU 把地址 A 放入 MAR
2. MAR 通过地址线把地址送到主存
3. CPU 发出读控制信号 Read
4. 主存地址译码器选中地址 A 对应的存储单元
5. 主存把该地址中的数据送到数据线
6. CPU 把数据接收到 MDR
```

可以写成：

```
MAR ← A
Read
MDR ← M[MAR]
```

通俗说：

> CPU 先告诉主存“我要读哪里”，主存找到后把数据送回来。

------

### 9. 写主存的过程

假设 CPU 要把数据 `D` 写入地址 `A`。

过程是：

```
1. CPU 把地址 A 放入 MAR
2. CPU 把要写的数据 D 放入 MDR
3. MAR 通过地址线送出地址
4. MDR 通过数据线送出数据
5. CPU 发出写控制信号 Write
6. 主存地址译码器选中地址 A
7. 主存把数据 D 写入该存储单元
```

可以写成：

```
MAR ← A
MDR ← D
Write
M[MAR] ← MDR
```

通俗说：

> CPU 先告诉主存“写到哪里”，再把“写什么”送过去。

------

### 10. 数据线 64 位和地址线 32 位分别说明什么？

图里有两个数字：

```
地址线：32 位
数据线：64 位
```

它们含义不同，别混。

地址线 32 位说明：

```
最多能表示 2^32 个地址
```

决定的是：

> **寻址范围。**

数据线 64 位说明：

```
一次能传输 64 bit 数据
```

决定的是：

> **一次传输的数据宽度。**

所以：

```
地址线宽度 → 决定能找多少个位置
数据线宽度 → 决定一次能搬多少数据
```

## 存储器芯片的拓展

![img](C:\Users\22067\Desktop\计组复习\assets\cover787_20260601075940.jpg)

### 一、先搞懂“16K×8 位”是什么意思

比如：

```
16K × 8 位
```

意思是：

```
有 16K 个存储单元；
每个存储单元宽度是 8 bit。
```

也就是：

```
字数 = 16K
位数 = 8 位
```

所以看芯片扩展时要分两件事：

```
字数够不够？
位数够不够？
```

如果字数不够，就做 **字扩展**。
 如果位数不够，就做 **位扩展**。

------

### 二、字扩展：位数不变，容量扩大

PPT 第一部分：

```
用 16K×8 位芯片扩成 64K×8 位存储器，需要几个芯片？
地址范围各是什么？
```

原芯片是：

```
16K × 8 位
```

目标存储器是：

```
64K × 8 位
```

先看位数：

```
原来 8 位
目标 8 位
```

位数一样，不需要位扩展。

再看字数：

```
64K / 16K = 4
```

所以字方向扩展 4 倍，需要：

```
4 个芯片
```

------

#### 1. 地址为什么是 16 位？

目标是：

```
64K = 2^16
```

所以整个存储器需要：

```
16 根地址线
```

地址范围是：

```
0000H ~ FFFFH
```

------

#### 2. 每个 16K 芯片需要多少地址线？

单个芯片是：

```
16K = 2^14
```

所以每个芯片内部需要：

```
14 根片内地址线
```

也就是低 14 位地址：

```
A0 ~ A13
```

------

#### 3. 高 2 位地址干什么？

总地址 16 位，片内只用 14 位，还剩：

```
16 - 14 = 2 位
```

这高 2 位用来选择 4 个芯片。

因为：

```
2^2 = 4
```

所以：

```
A15 A14 经过译码器，产生 4 个片选信号
```

分别选中 4 个芯片。

------

#### 4. 四个芯片的地址范围

| 高两位 A15A14 | 地址范围      | 选中芯片 |
| ------------- | ------------- | -------- |
| 00            | 0000H ~ 3FFFH | 第 1 片  |
| 01            | 4000H ~ 7FFFH | 第 2 片  |
| 10            | 8000H ~ BFFFH | 第 3 片  |
| 11            | C000H ~ FFFFH | 第 4 片  |

所以第一题答案是：

```
需要 4 片 16K×8 位芯片。
地址范围分别为：
0000H~3FFFH
4000H~7FFFH
8000H~BFFFH
C000H~FFFFH
```

一句话：

> **字扩展就是多个芯片轮流负责不同地址段，同一时刻只选中其中一个芯片。**

------

### 三、位扩展：字数不变，位数变宽

PPT 第二部分：

```
用 4096×1 位芯片构成 4K×8 位存储器，需要几个芯片？
地址范围各是多少？
```

原芯片是：

```
4096 × 1 位
```

其中：

```
4096 = 4K
```

目标是：

```
4K × 8 位
```

------

#### 1. 先看字数

原芯片：

```
4K 个地址
```

目标：

```
4K 个地址
```

字数一样，不需要字扩展。

------

#### 2. 再看位数

原芯片每个地址只有：

```
1 bit
```

目标每个地址需要：

```
8 bit
```

所以位方向要扩展：

```
8 / 1 = 8 倍
```

因此需要：

```
8 个芯片
```

------

#### 3. 地址范围为什么都一样？

因为这 8 个芯片不是负责不同地址段，而是一起负责同一个地址的不同数据位。

每个芯片都有 4K 个地址：

```
4K = 2^12
```

所以地址线是：

```
12 位
```

地址范围是：

```
000H ~ FFFH
```

这 8 个芯片的地址范围都一样：

```
第 1 片：000H ~ FFFH，提供 D0
第 2 片：000H ~ FFFH，提供 D1
第 3 片：000H ~ FFFH，提供 D2
...
第 8 片：000H ~ FFFH，提供 D7
```

也就是说，访问地址 `100H` 时，8 个芯片同时被访问：

```
第 1 片给出 bit0
第 2 片给出 bit1
...
第 8 片给出 bit7
```

合起来就是一个 8 位数据。

一句话：

> **位扩展就是多个芯片并排工作，每个芯片负责数据总线中的一位。**

------

### 四、字位同时扩展：字数和位数都不够

PPT 第三部分：

```
用 16K×4 位芯片构成 64K×8 位存储器，需要几个芯片？
地址范围各是多少？
```

原芯片是：

```
16K × 4 位
```

目标是：

```
64K × 8 位
```

这次字数和位数都不够。

------

#### 1. 字方向扩展

```
64K / 16K = 4
```

所以字方向扩展 4 倍。

也就是需要 4 组芯片，分别负责 4 个地址段。

------

#### 2. 位方向扩展

```
8 / 4 = 2
```

所以每组需要 2 个芯片并联，才能凑成 8 位。

一个芯片提供 4 位，两个芯片合起来提供：

```
4 位 + 4 位 = 8 位
```

比如：

```
芯片 A 接 D0~D3
芯片 B 接 D4~D7
```

------

#### 3. 总共需要几个芯片？

```
字方向 4 倍 × 位方向 2 倍 = 8 片
```

所以答案是：

```
需要 8 片 16K×4 位芯片
```

------

#### 4. 地址范围怎么分？

目标是 64K 地址，所以还是 16 位地址：

```
A0 ~ A15
```

每个芯片是 16K：

```
16K = 2^14
```

所以每片内部使用低 14 位地址：

```
A0 ~ A13
```

高 2 位：

```
A15 A14
```

用于片组选通，分成 4 个地址段。

| 高两位 A15A14 | 地址范围      | 芯片组        |
| ------------- | ------------- | ------------- |
| 00            | 0000H ~ 3FFFH | 第 1 组，两片 |
| 01            | 4000H ~ 7FFFH | 第 2 组，两片 |
| 10            | 8000H ~ BFFFH | 第 3 组，两片 |
| 11            | C000H ~ FFFFH | 第 4 组，两片 |

注意这里每一组不是 1 片，而是 **2 片同时被选中**。

因为每组要输出 8 位数据：

```
一片输出低 4 位
另一片输出高 4 位
```

一句话：

> **字位同时扩展就是先按地址分组，再在每组内部并联多片扩展数据位宽。**

------

### 五、三个扩展方式怎么区分？

| 扩展方式     | 什么时候用           | 芯片怎么接                   | 特点                     |
| ------------ | -------------------- | ---------------------------- | ------------------------ |
| 字扩展       | 地址容量不够，位宽够 | 多片负责不同地址范围         | 同一时刻只选一片         |
| 位扩展       | 地址容量够，位宽不够 | 多片负责同一地址的不同数据位 | 多片同时工作             |
| 字位同时扩展 | 地址容量和位宽都不够 | 先分地址组，每组再并联扩位   | 每组选多片，总芯片数相乘 |

## 例题：存储器芯片的拓展

![img](C:\Users\22067\Desktop\计组复习\assets\cover794_20260601075940.jpg)

### 题目信息

地址总线：

```
A15 ~ A0
```

一共 16 根地址线，所以 CPU 总寻址范围是：

```
0000H ~ FFFFH
```

题目给定：

```
ROM 区：0000H ~ 07FFH
RAM 区：0800H ~ 13FFH
```

芯片选择：

```
ROM：EPROM 芯片，2KB/片
RAM：RAM 芯片，2KB/片 和 1KB/片
```

------

### 第一步：计算 ROM 区容量

ROM 地址范围：

```
0000H ~ 07FFH
```

容量计算：

```
07FFH - 0000H + 1 = 0800H
```

而：

```
0800H = 2048B = 2KB
```

所以 ROM 区大小是：

```
2KB
```

题目说 EPROM 芯片是：

```
2KB/片
```

所以 ROM 区需要：

```
1 片 EPROM
```

地址分配：

```
EPROM：0000H ~ 07FFH
```

------

### 第二步：计算 RAM 区容量

RAM 地址范围：

```
0800H ~ 13FFH
```

容量计算：

```
13FFH - 0800H + 1 = 0C00H
```

而：

```
0C00H = 3072B = 3KB
```

所以 RAM 区大小是：

```
3KB
```

题目提供 RAM 芯片：

```
2KB/片 和 1KB/片
```

所以 RAM 区可以用：

```
1 片 2KB RAM + 1 片 1KB RAM
```

按照 PPT 规则：**先安排大容量芯片，放在低地址；再安排小容量芯片。**

所以：

```
2KB RAM：0800H ~ 0FFFH
1KB RAM：1000H ~ 13FFH
```

------

### 第三步：整理地址分配表

| 区域 | 芯片       | 容量 | 地址范围      |
| ---- | ---------- | ---- | ------------- |
| ROM  | EPROM      | 2KB  | 0000H ~ 07FFH |
| RAM  | RAM 芯片 1 | 2KB  | 0800H ~ 0FFFH |
| RAM  | RAM 芯片 2 | 1KB  | 1000H ~ 13FFH |

剩余地址：

```
1400H ~ FFFFH
```

本题没有使用。

------

### 第四步：确定每片芯片用哪些地址线

#### 1. EPROM：2KB

```
2KB = 2048B = 2^11
```

所以片内地址需要：

```
11 位地址线
```

即：

```
A0 ~ A10
```

因此 EPROM 的片内地址线接：

```
CPU A0 ~ A10
```

高位地址线：

```
A15 ~ A11
```

用于判断是否选中 EPROM。

EPROM 地址范围是：

```
0000H ~ 07FFH
```

写成二进制高位：

```
0000H = 0000 0000 0000 0000
07FFH = 0000 0111 1111 1111
```

可以看出：

```
A15 A14 A13 A12 A11 = 0 0 0 0 0
```

所以 EPROM 片选条件是：

```
A15=0, A14=0, A13=0, A12=0, A11=0
```

片选逻辑：

```
CS_ROM = /A15 · /A14 · /A13 · /A12 · /A11
```

------

#### 2. 2KB RAM：0800H ~ 0FFFH

2KB RAM 片内地址也需要：

```
A0 ~ A10
```

地址范围：

```
0800H ~ 0FFFH
```

二进制看高位：

```
0800H = 0000 1000 0000 0000
0FFFH = 0000 1111 1111 1111
```

固定不变的是：

```
A15 A14 A13 A12 A11 = 0 0 0 0 1
```

所以 2KB RAM 的片选条件是：

```
A15=0, A14=0, A13=0, A12=0, A11=1
```

片选逻辑：

```
CS_RAM_2K = /A15 · /A14 · /A13 · /A12 · A11
```

------

#### 3. 1KB RAM：1000H ~ 13FFH

1KB 是：

```
1KB = 1024B = 2^10
```

所以片内地址需要：

```
10 位地址线
```

即：

```
A0 ~ A9
```

地址范围：

```
1000H ~ 13FFH
```

二进制看高位：

```
1000H = 0001 0000 0000 0000
13FFH = 0001 0011 1111 1111
```

固定不变的是：

```
A15 A14 A13 A12 A11 A10 = 0 0 0 1 0 0
```

所以 1KB RAM 的片选条件是：

```
A15=0, A14=0, A13=0, A12=1, A11=0, A10=0
```

片选逻辑：

```
CS_RAM_1K = /A15 · /A14 · /A13 · A12 · /A11 · /A10
```

------

### 第五步：最终答案

#### 地址分配

```
EPROM 2KB：
0000H ~ 07FFH

RAM 2KB：
0800H ~ 0FFFH

RAM 1KB：
1000H ~ 13FFH
```

#### 片内地址线连接

```
EPROM 2KB：A0 ~ A10

2KB RAM：A0 ~ A10

1KB RAM：A0 ~ A9
```

#### 片选逻辑

```
CS_ROM    = /A15 · /A14 · /A13 · /A12 · /A11

CS_RAM2K  = /A15 · /A14 · /A13 · /A12 · A11

CS_RAM1K  = /A15 · /A14 · /A13 · A12 · /A11 · /A10
```

如果芯片的片选信号是 **低电平有效**，那上面的 `CS` 再取反即可。

## 提取异常

- 批量自动提取，未经人工核对。
