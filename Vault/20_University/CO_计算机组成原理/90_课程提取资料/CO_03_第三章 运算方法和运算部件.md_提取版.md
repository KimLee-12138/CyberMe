## 位拓展和位截断

核心就两种情况：

```
短类型 → 长类型：位扩展
长类型 → 短类型：位截断
```

------

### 1. 什么是“位扩展”？

**位扩展**就是：原来的数据位数不够，现在要变成长的数据类型，所以要在前面补位。

比如：

```
short → int
```

如果 `short` 是 2 字节，`int` 是 4 字节，那么就是：

```
16 位 → 32 位
```

前面要多补 16 位。

#### 无符号数扩展：前面补 0

例如：

```
unsigned short: 1000 0000 0000 0000
```

扩展成 32 位：

```
0000 0000 0000 0000 1000 0000 0000 0000
```

也就是：

```
00 00 80 00
```

这叫 **0 扩展**。

------

#### 有符号数扩展：前面补符号位

有符号数最高位是符号位：

```
0 表示正数
1 表示负数
```

如果是负数，最高位是 `1`，扩展时前面就补 `1`。

例如：

```
short: 1000 0000 0000 0000
```

这是 16 位补码中的 `-32768`。

扩展成 32 位：

```
1111 1111 1111 1111 1000 0000 0000 0000
```

也就是：

```
FF FF 80 00
```

这叫 **符号扩展**。

------

### 2. 什么是“位截断”？

**位截断**就是：原来的数据太长，现在要变成短的数据类型，所以直接丢掉高位，只保留低位。

比如：

```
int → short
```

如果 `int` 是 4 字节，`short` 是 2 字节：

```
32 位 → 16 位
```

就只保留低 16 位，高 16 位直接丢掉。

例如：

```
int: 0000 0000 0000 0000 1000 0000 0000 0000
```

截断成 short，只保留后 16 位：

```
1000 0000 0000 0000
```

这个 16 位补码表示的是：

```
-32768
```

所以数值可能发生变化

### 例 1：复习扩展

题目：

```
short si = -32768;
unsigned short usi = si;
int i = si;
unsigned ui = usi;
```

假设：

```
short = 2 字节
int = 4 字节
```

------

#### 1. `short si = -32768;`

16 位补码中，`-32768` 表示为：

```
1000 0000 0000 0000
```

十六进制是：

```
80 00
```

所以：

```
si = -32768
存储内容：80 00
```

------

#### 2. `unsigned short usi = si;`

这里是把 `short` 转成 `unsigned short`。

注意：**二进制位不变，只是解释方式变了。**

原来：

```
1000 0000 0000 0000
```

作为 signed short 看，是：

```
-32768
```

作为 unsigned short 看，是：

```
32768
```

所以：

```
usi = 32768
存储内容：80 00
```

------

#### 3. `int i = si;`

这里是：

```
short → int
```

短变长，需要扩展。

因为 `si` 是有符号数，而且是负数，最高位是 `1`，所以做**符号扩展**，前面补 `1`。

```
原来 16 位：
1000 0000 0000 0000

扩展成 32 位：
1111 1111 1111 1111 1000 0000 0000 0000
```

十六进制：

```
FF FF 80 00
```

所以：

```
i = -32768
存储内容：FF FF 80 00
```

------

#### 4. `unsigned ui = usi;`

这里是：

```
unsigned short → unsigned int
```

无符号数短变长，做 **0 扩展**，前面补 `0`。

```
原来 16 位：
1000 0000 0000 0000

扩展成 32 位：
0000 0000 0000 0000 1000 0000 0000 0000
```

十六进制：

```
00 00 80 00
```

所以：

```
ui = 32768
存储内容：00 00 80 00
```

------

#### 例 1 总结

| 变量  | 数值   | 存储内容      |
| ----- | ------ | ------------- |
| `si`  | -32768 | `80 00`       |
| `usi` | 32768  | `80 00`       |
| `i`   | -32768 | `FF FF 80 00` |
| `ui`  | 32768  | `00 00 80 00` |

重点：

```
同样是 80 00，按 signed 看是 -32768，按 unsigned 看是 32768。
```

### 例 2：截断

题目：

```
int i = 32768;
short si = (short)i;
int j = si;
```

问：

```
i 和 j 是否相等？
```

答案：

```
不相等
```

------

#### 1. `int i = 32768;`

32768 的 32 位二进制是：

```
0000 0000 0000 0000 1000 0000 0000 0000
```

十六进制：

```
00 00 80 00
```

所以：

```
i = 32768
```

------

#### 2. `short si = (short)i;`

这里是：

```
int → short
```

长变短，直接截断，只保留低 16 位。

```
原来：
0000 0000 0000 0000 1000 0000 0000 0000

保留低 16 位：
1000 0000 0000 0000
```

也就是：

```
80 00
```

但是 `short` 是有符号数，`80 00` 表示：

```
-32768
```

所以：

```
si = -32768
```

------

#### 3. `int j = si;`

这里又是：

```
short → int
```

短变长，且 `si` 是有符号负数，所以做符号扩展。

```
short:
1000 0000 0000 0000

扩展成 int:
1111 1111 1111 1111 1000 0000 0000 0000
```

十六进制：

```
FF FF 80 00
```

所以：

```
j = -32768
```

------

#### 4. 最后比较

```
i = 32768
j = -32768
```

所以：

```
i != j
```

## ALU

ALU可进行**加/减**算术运算和**逻辑**运算。其核心部件是**加法器**。串行加法器在数字逻辑课学过,下面简单回顾。

### 半加器

**半加器：**只考虑本位相加不考虑进位。

### 全加器

**全加器：**本位A和B加上进位Carryln生成一位Sum和一位进位CarryOut.

  Sum延迟两级异或门共6T;Carryout延迟一级与门和一级或门共2T（因为与门是并行处理的）

### 串行进位加法器

![image-20260624141416719](C:\Users\22067\AppData\Roaming\Typora\typora-user-images\image-20260624141416719.png)

#### 1. 基本功能

串行进位加法器可以实现两个 $n$ 位数逐位相加：

$$
X = X_nX_{n-1}\cdots X_1
$$

$$
Y = Y_nY_{n-1}\cdots Y_1
$$

得到和：

$$
F = F_nF_{n-1}\cdots F_1
$$

进位输出为：

$$
C_n
$$

#### 2. 模 $2^n$ 加法

如果最高位进位：
$$
C_n = 1
$$
但结果只保留 $n$ 位，则最高位进位被丢弃。

因此执行的是：
$$
\mod 2^n
$$
加法。

所以串行进位加法器可以实现：

- $n$ 位无符号数加法；
- $n$ 位补码加法。

#### 3. 缺点

串行进位加法器的缺点是：

```
进位按位逐级传递，速度慢。
```

也就是说：
$$
C_0 \rightarrow C_1 \rightarrow C_2 \rightarrow \cdots \rightarrow C_n
$$
高位加法必须等待低位进位传来。

#### 4. 延迟分析

每个全加器产生进位需要两级门延迟。

因此从 $C_0$ 传到 $C_n$，一共经过 $n$ 个全加器，所以进位延迟为：
$$
2n
$$
级门延迟。

若一个异或门需要 3 级门延迟，则最后一位和 $F_n$ 需要等待进位到达后再计算，因此延迟为：
$$
2n + 1
$$
级门延迟。

## 并行进位加法器（CLA）

### 1. 为什么需要 CLA？

串行进位加法器的缺点是：**进位要从低位一位一位传到高位，速度慢**。

串行进位过程：

```text
C0 → C1 → C2 → C3 → C4
```

也就是说：

```text
第 1 位算出 C1
第 2 位等 C1 算出 C2
第 3 位等 C2 算出 C3
第 4 位等 C3 算出 C4
```

位数越多，等待进位传递的时间越长。

CLA 的思路是：

```text
不等进位逐级传递，而是提前用公式直接算出各位进位。
```

------

### 2. CLA 的核心思想

CLA 可以根据输入 $X_i$、$Y_i$ 和初始进位 $C_0$，提前算出：

```text
C1、C2、C3、C4
```

因此各位进位可以并行产生，不需要逐级等待。

这就叫：

```text
先行进位
```

------

### 3. 两个重要函数：G 和 P

#### 进位生成函数

$$
G_i = X_iY_i
$$

含义：

```text
如果 Xi = 1 且 Yi = 1，则本位一定产生进位。
```

例如：

```text
1 + 1 = 10
```

所以 $G_i$ 称为**进位生成函数**。

------

#### 进位传递函数

$$
P_i = X_i + Y_i
$$

这里的 $+$ 表示逻辑或。

含义：

```text
只要 Xi 和 Yi 中有一个为 1，就能把低位传来的进位继续传到高位。
```

例如：

```text
1 + 0 + 进位 1 = 10
0 + 1 + 进位 1 = 10
```

所以 $P_i$ 称为**进位传递函数**。

------

### 4. 每一位进位的基本公式

第 $i$ 位的进位为：

$$
C_i = X_iY_i + X_iC_{i-1} + Y_iC_{i-1}
$$

也可以写成：

$$
C_i = G_i + P_iC_{i-1}
$$

含义：

```text
第 i 位产生进位有两种情况：
1. 本位自己产生进位：Gi = 1
2. 本位把低位进位传上来：PiCi-1 = 1
```

------

### 5. 4 位 CLA 的进位公式

#### 第一位进位

$$
C_1 = G_1 + P_1C_0
$$

------

#### 第二位进位

$$
C_2 = G_2 + P_2G_1 + P_2P_1C_0
$$

含义：

```text
C2 可能由第 2 位自己生成；
也可能由第 1 位生成后被第 2 位传上来；
也可能由 C0 经过第 1 位、第 2 位一路传上来。
```

------

#### 第三位进位

$$
C_3 = G_3 + P_3G_2 + P_3P_2G_1 + P_3P_2P_1C_0
$$

------

#### 第四位进位

$$
C_4 = G_4 + P_4G_3 + P_4P_3G_2 + P_4P_3P_2G_1 + P_4P_3P_2P_1C_0
$$

核心理解：

```text
某一位的进位，要么由本位生成，要么由低位生成后一路传递上来。
```

------

### 6. 为什么 CLA 快？

串行进位加法器中：

```text
C1 算完才能算 C2
C2 算完才能算 C3
C3 算完才能算 C4
```

CLA 中：

```text
C1、C2、C3、C4 可以通过公式并行算出
```

所以 CLA 不需要一级一级等待进位传递，速度更快。

------

### 7. CLA 加法器的组成

CLA 加法器主要由三部分组成：

```text
1. 进位生成 / 传递部件
2. CLA 部件
3. 求和部件
```

#### 第一步：计算 $G_i$ 和 $P_i$

$$
G_i = X_iY_i
$$

$$
P_i = X_i + Y_i
$$

延迟约为：

```text
1T
```

------

#### 第二步：CLA 部件计算进位

根据 $G_i$、$P_i$ 和 $C_0$ 计算：

```text
C1、C2、C3、C4
```

延迟约为：

```text
2T
```

------

#### 第三步：求和

求和公式为：

$$
S_i = X_i \oplus Y_i \oplus C_{i-1}
$$

也可以写成：

$$
S_i = \overline{G_i}P_i \oplus C_{i-1}
$$

延迟约为：

```text
3T
```

------

### 8. 延迟计算

#### 和 $S$ 的总延迟

```text
生成 G/P：1T
CLA 算进位：2T
求和：3T
```

所以：

$$
1T + 2T + 3T = 6T
$$

因此：

```text
和 S 的总延迟 = 6T
```

------

#### 进位 $C_4$ 的延迟

计算 $C_4$ 不需要经过求和部件，只需要：

```text
生成 G/P：1T
CLA 算进位：2T
```

所以：

$$
1T + 2T = 3T
$$

因此：

```text
进位 C4 的延迟 = 3T
```

------

### 9. CLA 的局限性

CLA 速度快，但公式会随着位数增加而迅速变复杂。

例如：

```text
4 位、8 位 CLA 较容易实现；
16 位、32 位、64 位 CLA 的进位逻辑会非常复杂。
```

因此实际中常采用：

```text
局部先行进位
```

也就是：

```text
组内并行，组间串行
```

例如：

```text
4 个 4 位 CLA 串联，组成一个 16 位加法器。
```

这样可以在速度和电路复杂度之间折中。

## 带标志加法器

### 1. 带标志加法器

普通加法器只负责计算：

$$
F = A + B
$$

但 CPU 不仅需要知道运算结果，还需要知道结果的状态，例如：

```text
结果是否为 0？
结果是否为负数？
无符号运算是否产生进位？
有符号运算是否发生溢出？
```

所以加法器通常会增加一些逻辑电路，用来产生标志位。

------

### 2. 常见标志位

#### （1）ZF：零标志 Zero Flag

ZF 用来判断结果是否为 0。

```text
ZF = 1：结果 F 全部为 0
ZF = 0：结果 F 不全为 0
```

例如：

```text
F = 0000 0000  →  ZF = 1
F = 0000 0101  →  ZF = 0
```

ZF 常用于判断两个数是否相等，例如条件跳转指令。

------

#### （2）SF：符号标志 Sign Flag

SF 表示结果的符号。

在补码中，最高位是符号位：

```text
最高位为 0：正数或 0
最高位为 1：负数
```

所以：

$$
SF = F_{n-1}
$$

也就是结果 $F$ 的最高位。

------

#### （3）CF：进位/借位标志 Carry Flag

CF 主要用于**无符号数运算**，表示最高位是否产生进位或借位。

PPT 中给出的公式是：

$$
CF = C_{out} \oplus C_{in}
$$

对于普通加法，通常：

$$
C_{in}=0
$$

所以：

$$
CF = C_{out}
$$

例如 4 位无符号加法：

```text
  1111
+ 0001
------
1 0000
```

只保留低 4 位结果：

```text
0000
```

最高位产生进位，所以：

```text
CF = 1
```

说明无符号数运算超过了可表示范围。

------

#### （4）OF：溢出标志 Overflow Flag

OF 主要用于**有符号补码运算**，判断是否发生有符号溢出。

公式：

$$
OF = C_n \oplus C_{n-1}
$$

其中：

```text
Cn：最高位的进位输出
Cn-1：进入最高位的进位
```

如果二者不同，则发生有符号溢出。

也可以用更直观的方法判断：

```text
正数 + 正数 = 负数  →  溢出
负数 + 负数 = 正数  →  溢出
正数 + 负数        →  不会溢出
```

------

### 3. CF 和 OF 的区别

| 标志位 | 主要用于       | 判断内容           |
| ------ | -------------- | ------------------ |
| CF     | 无符号数运算   | 是否产生进位或借位 |
| OF     | 有符号补码运算 | 是否发生有符号溢出 |

核心记忆：

```text
无符号数看 CF
有符号数看 OF
```

## 算术逻辑部件 ALU

ALU 是 CPU 中负责运算的核心部件，全称为：

```text
Arithmetic Logic Unit
算术逻辑部件
```

ALU 可以完成多种运算，例如：

```text
加法
减法
与运算 AND
或运算 OR
异或运算 XOR
移位
扩展
截断
比较
```

------

### 1. ALU 的基本结构

ALU 通常有两个输入：

```text
A
B
```

它们会同时送入不同的运算部件，例如：

```text
AND 门：计算 A & B
OR 门：计算 A | B
加法器：计算 A + B
```

这些运算结果会送入一个多路选择器 MUX。

MUX 根据控制信号选择最终输出哪一个结果。

------

### 2. ALUop 的作用

ALUop 是 ALU 的操作控制信号，用来决定 ALU 执行哪一种运算。

例如：

```text
ALUop = 00 → 输出加法结果
ALUop = 01 → 输出 AND 结果
ALUop = 10 → 输出 OR 结果
```

实际编码由具体 CPU 设计决定。

核心理解：

```text
ALUop 决定 ALU 当前执行什么运算。
```

------

### 3. CarryIn 和 CarryOut

ALU 中的加法器通常带有：

```text
CarryIn：输入进位
CarryOut：输出进位
```

它们不仅可以用于加法，也可以配合补码实现减法。

补码减法可以转化为：

$$
A - B = A + \overline{B} + 1
$$

其中的 `+1` 可以通过设置：

```text
CarryIn = 1
```

来实现。

------

### 4. ALU 中的标志位

ALU 输出结果后，还会产生一些标志位：

```text
ZF：结果是否为 0
SF：结果最高位，表示符号
CF：无符号运算是否产生进位/借位
OF：有符号运算是否溢出
```

这些标志位会被 CPU 用于条件判断和程序控制。

例如：

```text
判断两个数是否相等：看 ZF
判断结果是否为负：看 SF
判断无符号运算是否溢出：看 CF
判断有符号运算是否溢出：看 OF
```

## 对加减运算电路的深入认识

1. 所有算术运算都基于**加法器**实现!
2. 加法器不关心所算的是**带符号数**还是**无符号数**。
3. 加法器不判定对错,总是取低n位作为结果,并**生成标志**信息。

## 原码加/减运算

- 用于浮点数**尾数**运算
- **符号位**和**数值**部分分开处理
- 仅对**数值**部分进行加减运算,符号位起判断和控制作用
- 规则如下:
  - 加法:"**同号求和,异号求差**",减法:"**异号求和,同号求差**"
  - 求和:数值位相加,**和的符号**取被加数(被减数)的符号。若最高位产生进位,则结果溢出。
  - 求差:被加数(被减数)加上加数(减数)的**补码**。
    - a)最高数值位**产生进位**表明加法结果为**正**,所得数值位正确角。**符号位**取被加数(被减数)的符号;
    - b)最高数值位**没产生进位**表明加法结果为负,得到的是数值的**补码**,需对结果求补,还原为绝对值形式的数值位。符号位为被加数(被减数)的符号**取反**。

### 1. 原码运算的基本思想

原码由两部分组成：

```text
符号位 + 数值位
```

符号位规则：

```text
0 表示正数
1 表示负数
```

原码加减法的特点是：

```text
符号位和数值位分开处理。
```

即：

- 符号位用于判断结果正负；
- 数值位单独进行加减运算。

------

### 2. 原码加法规则

```text
同号相加：数值位相加，结果符号取相同符号。
异号相加：数值位相减，结果符号取绝对值较大的数的符号。
```

------

### 3. 原码减法规则

```text
同号相减：数值位相减。
异号相减：数值位相加。
```

求差时：

```text
若最高数值位产生进位，说明结果为正，数值位正确；
若最高数值位没有产生进位，说明结果为负，得到的是补码形式，需要再求补得到绝对值。
```

------

### 1. 先看原码乘法的基本思想

原码乘法用于浮点数的尾数乘法。

原码中：

```text
符号位 + 数值位
```

乘法时：

```text
符号位单独处理：两个符号位异或
数值位单独相乘：看作无符号小数相乘
```

也就是：

$$
z_0 = x_0 \oplus y_0
$$

如果两个数同号，结果为正；如果两个数异号，结果为负。

这页 PPT 主要讲的是数值位怎么乘。

------

### 2. 为什么乘法可以拆成加法？

假设乘数：

$$
Y = 0.y_1y_2...y_n
$$

那么它的值可以展开成：

$$
Y = y_1 \times 2^{-1} + y_2 \times 2^{-2} + ... + y_n \times 2^{-n}
$$

所以：

$$
X \times Y
$$

就可以写成：

$$
X \times Y = X \times y_1 \times 2^{-1} + X \times y_2 \times 2^{-2} + ... + X \times y_n \times 2^{-n}
$$

这里每个 $y_i$ 只能是 0 或 1。

所以：

```text
如果 yi = 0，则 X × yi = 0
如果 yi = 1，则 X × yi = X
```

因此乘法就变成了：

```text
判断乘数当前位是不是 1
是 1 就加 X
是 0 就加 0
再配合移位
```

所以乘法可以用：

```text
加法器 + 移位器
```

实现。

------

### 3. 为什么 PPT 要倒序处理？

原式是：

$$
X \times Y = Xy_1 2^{-1} + Xy_2 2^{-2} + ... + Xy_n 2^{-n}
$$

PPT 把它改写成从低位开始：

$$
= 2^{-n}Xy_n + 2^{-(n-1)}Xy_{n-1} + ... + 2^{-2}Xy_2 + 2^{-1}Xy_1
$$

然后继续整理成嵌套形式：

$$
X \times Y =
2^{-1}
\left(
2^{-1}
\left(
...
2^{-1}
\left(
2^{-1}(0+Xy_n)+Xy_{n-1}
\right)
...
+Xy_2
\right)
+Xy_1
\right)
$$

这个形式的意义是：

```text
每算一步，都乘一次 2^(-1)，也就是右移一位。
```

所以硬件中就可以按这个规律循环。

------

### 4. PPT 中的递推公式是什么意思？

PPT 设：

$$
P_0 = 0
$$

这里 $P$ 表示部分积，也就是当前已经累加出来的中间结果。

然后：

$$
P_1 = 2^{-1}(P_0 + Xy_n)
$$

$$
P_2 = 2^{-1}(P_1 + Xy_{n-1})
$$

$$
...
$$

$$
P_n = 2^{-1}(P_{n-1} + Xy_1)
$$

意思是：

```text
从乘数最低位 yn 开始判断。
如果这一位是 1，就加 X；
如果这一位是 0，就加 0；
加完后右移一位。
```

循环 n 次后，得到：

$$
P_n = X \times Y
$$

------

### 5. “判断—加法—右移”具体是什么意思？

每一轮都做三件事：

```text
1. 判断当前乘数位 yi 是 0 还是 1
2. 如果 yi = 1，则部分积加 X；如果 yi = 0，则部分积不加
3. 部分积右移一位
```

所以 PPT 最下面说：

```text
n 次“判断—加法—右移”，至求出 Pn = X × Y
```

------

$$

其中：

```text

```

从最低位开始处理，也就是顺序为：


## 原码两位乘法

### 1. 原码乘法的基本思想

原码乘法常用于浮点数尾数乘法。

原码乘法中，符号位和数值位分开处理：

```text
符号位：两个操作数符号位异或
数值位：看作无符号数相乘
```

符号位：

$$
z_0 = x_0 \oplus y_0
$$

即：

```text
同号相乘，结果为正；
异号相乘，结果为负。
```

------

### 2. 原码一位乘和两位乘

原码一位乘：

```text
每次只判断乘数的一位。
若该位为 1，则加被乘数 X；
若该位为 0，则不加。
```

缺点是速度较慢。

原码两位乘：

```text
每次判断乘数的两位。
每次处理两位，所以速度大约快一倍。
```

------

### 3. 原码两位乘的规则

每次取乘数的两位，有四种情况：

$$
00：P_{i+1}=2^{-2}P_i
$$

$$
01：P_{i+1}=2^{-2}(P_i+X)
$$

$$
10：P_{i+1}=2^{-2}(P_i+2X)
$$

$$
11：P_{i+1}=2^{-2}(P_i+3X)
$$

其中：

```text
P_i 表示当前部分积；
X 表示被乘数；
2^(-2) 表示右移两位。
```

------

### 4. 为什么 11 情况要特殊处理？

当乘数两位为 `11` 时，相当于要加：

$$
3X
$$

但是硬件中直接求 $3X$ 不方便，所以可以利用：

$$
3X = 4X - X
$$

因此：

$$
P_{i+1}=2^{-2}(P_i+3X)
$$

可以改写成：

$$
P_{i+1}=2^{-2}(P_i-X+4X)
$$

$$
P_{i+1}=2^{-2}(P_i-X)+X
$$

所以 `11` 的处理方法可以理解为：

```text
先加 -X，右移两位后，再加 X。
```

其中：

```text
-X 可以用补码加法实现，即加 [-X]补。
```

------

## 补码乘法：校正法

### 1. 基本思想

校正法的思想是：

> 乘法过程尽量按原码乘法来做，但如果乘数为负，最后需要加一个校正项。

设补码小数：

$$
[x]_{\text{补}} = x_0.x_1x_2\cdots x_n
$$

$$
[y]_{\text{补}} = y_0.y_1y_2\cdots y_n
$$

其中，$x_0$、$y_0$ 是符号位。

---

### 2. 运算规则

#### 情况一：乘数 $y$ 为正

若：

$$
y_0 = 0
$$

说明乘数 $y$ 为正数，可以直接按照原码乘法规则运算：

- 判断乘数位；
- 乘数位为 1，则加被乘数；
- 乘数位为 0，则不加；
- 然后移位。

即：

$$
[x \times y]_{\text{补}} = [x]_{\text{补}} \times [y]_{\text{补}}
$$

---

#### 情况二：乘数 $y$ 为负

若：

$$
y_0 = 1
$$

说明乘数 $y$ 为负数。

此时处理方法是：

- 先去掉乘数 $y$ 的符号位，用 $0.y_1y_2\cdots y_n$ 去乘 $x$；
- 最后再加 $[-x]_{\text{补}}$ 进行校正。

即：

$$
[x \times y]_{\text{补}}
=
[x \times (0.y_1y_2\cdots y_n)]_{\text{补}}
+
[-x]_{\text{补}}
$$

---

### 3. 校正公式的来源

对于补码小数，若 $y$ 为负数，则：

$$
y = [y]_{\text{补}} - 2
$$

又因为：

$$
[y]_{\text{补}} = 1.y_1y_2\cdots y_n
$$

所以：

$$
y = 1.y_1y_2\cdots y_n - 2
$$

而：

$$
1.y_1y_2\cdots y_n = 1 + 0.y_1y_2\cdots y_n
$$

所以：

$$
y = 0.y_1y_2\cdots y_n - 1
$$

因此：

$$
x \times y = x \times (0.y_1y_2\cdots y_n - 1)
$$

展开得：

$$
x \times y = x \times 0.y_1y_2\cdots y_n - x
$$

也就是：

$$
x \times y = x \times 0.y_1y_2\cdots y_n + (-x)
$$

所以最后要加：

$$
[-x]_{\text{补}}
$$

这就是校正项。

---

### 4. 例题

设：

$$
[x]_{\text{补}} = 0.1010
$$

$$
[y]_{\text{补}} = 1.1101
$$

求：

$$
[x \times y]_{\text{补}}
$$

---

#### 第一步：判断乘数 $y$ 的符号

$$
[y]_{\text{补}} = 1.1101
$$

符号位为 1，所以 $y$ 是负数。

因此使用校正法：

$$
[x \times y]_{\text{补}}
=
[x \times 0.1101]_{\text{补}}
+
[-x]_{\text{补}}
$$

---

#### 第二步：先计算 $x \times 0.1101$

$$
x = 0.1010
$$

去掉乘数符号位后：

$$
0.y_1y_2y_3y_4 = 0.1101
$$

数值部分相乘：

    1010 × 1101
所以：

$$
0.1010 \times 0.1101 = 0.10000010
$$

---

#### 第三步：求校正项 $[-x]_{\text{补}}$

因为：

$$
x = 0.1010
$$

扩展为 8 位小数：

$$
x = 0.10100000
$$

所以：

$$
[-x]_{\text{补}} = 1.01100000
$$

---

#### 第四步：加校正项

$$
[x \times y]_{\text{补}}
=
0.10000010 + 1.01100000
$$

计算：

    0.10000010
  + 1.01100000
------------
    1.11100010

所以：

$$
[x \times y]_{\text{补}} = 1.11100010
$$

---

#### 5. 检查结果

$$
1.11100010
$$

是负数补码。

其真值为：

$$
-0.00011110
$$

因为：

$$
0.1010 = \frac{10}{16}
$$

$$
1.1101_{\text{补}} = -0.0011 = -\frac{3}{16}
$$

所以：

$$
x \times y
=
\frac{10}{16} \times \left(-\frac{3}{16}\right)
=
-\frac{30}{256}
=
-0.00011110
$$

结果正确。

## 补码乘法：Booth 法

![image-20260625161145332](C:\Users\22067\AppData\Roaming\Typora\typora-user-images\image-20260625161145332.png)

### 1. Booth 法的基本思想

Booth 法用于补码乘法。

它的特点是：

- 被乘数和乘数可以是正数，也可以是负数；
- 不需要像校正法一样区分乘数正负；
- 每次根据乘数当前最低位和附加位判断操作；
- 每轮之后进行一次算术右移。

核心流程：

1. 设置部分积 `P = 0`
2. 设置附加位 `Y_-1 = 0`
3. 每次观察 `Y0Y_-1`
4. 根据情况决定加 `X`、加 `-X`，或者不加
5. `P、Y、Y_-1` 整体算术右移一位
6. 重复 `n` 次后，`P 和 Y` 合起来就是乘积

---

### 2. Booth 法判断规则

设：

- `P`：部分积
- `X`：被乘数
- `Y`：乘数
- `Y_-1`：附加位，初始为 0

每次看乘数最低位 `Y0` 和附加位 `Y_-1`。

| `Y0Y_-1` | 操作                    |
| -------- | ----------------------- |
| `00`     | 不加，直接算术右移      |
| `01`     | 加 `[X]补`，再算术右移  |
| `10`     | 加 `[-X]补`，再算术右移 |
| `11`     | 不加，直接算术右移      |

其中：

- `01` 表示从 0 变到 1，所以加 `X`
- `10` 表示从 1 变到 0，所以加 `-X`
- `00` 和 `11` 表示没有变化，所以不加

---

### 3. 算术右移是什么意思？

Booth 法中要做的是补码乘法，所以右移必须是**算术右移**。

算术右移时：

- 如果 `P` 的最高位是 `0`，左边补 `0`
- 如果 `P` 的最高位是 `1`，左边补 `1`

也就是右移时要保持符号位不变。

---

### 4. 例题

已知：

```text
[X]补 = 1101
[Y]补 = 0110
[-X]补 = 0011
```

求：

```text
[X × Y]补
```

先看真值：

```text
[X]补 = 1101 = -3
[Y]补 = 0110 = +6
```

所以理论结果应该是：

```text
X × Y = -3 × 6 = -18
```

8 位补码中：

```text
[-18]补 = 1110 1110
```

下面用 Booth 法计算验证。

------

### 5. 初始化

因为是 4 位补码乘法，所以循环 4 次。

初始：

```text
P = 0000
Y = 0110
Y_-1 = 0
```

写成整体：

```text
P      Y      Y_-1
0000   0110   0
```

------

### 6. 第 1 轮

当前：

```text
P      Y      Y_-1
0000   0110   0
```

观察：

```text
Y0Y_-1 = 00
```

根据规则：

```text
00：不加，直接算术右移
```

右移：

```text
0000 0110 0  →  0000 0011 0
```

所以第 1 轮后：

```text
P      Y      Y_-1
0000   0011   0
```

------

### 7. 第 2 轮

当前：

```text
P      Y      Y_-1
0000   0011   0
```

观察：

```text
Y0Y_-1 = 10
```

根据规则：

```text
10：加 [-X]补
```

因为：

```text
[-X]补 = 0011
```

所以：

```text
  0000
+ 0011
------
  0011
```

此时：

```text
P = 0011
```

然后整体算术右移：

```text
0011 0011 0  →  0001 1001 1
```

所以第 2 轮后：

```text
P      Y      Y_-1
0001   1001   1
```

------

### 8. 第 3 轮

当前：

```text
P      Y      Y_-1
0001   1001   1
```

观察：

```text
Y0Y_-1 = 11
```

根据规则：

```text
11：不加，直接算术右移
```

右移：

```text
0001 1001 1  →  0000 1100 1
```

所以第 3 轮后：

```text
P      Y      Y_-1
0000   1100   1
```

------

### 9. 第 4 轮

当前：

```text
P      Y      Y_-1
0000   1100   1
```

观察：

```text
Y0Y_-1 = 01
```

根据规则：

```text
01：加 [X]补
```

因为：

```text
[X]补 = 1101
```

所以：

```text
  0000
+ 1101
------
  1101
```

此时：

```text
P = 1101
```

然后整体算术右移。

注意：

```text
P = 1101
```

最高位是 `1`，所以算术右移时左边补 `1`：

```text
1101 1100 1  →  1110 1110 0
```

所以第 4 轮后：

```text
P      Y      Y_-1
1110   1110   0
```

------

### 10. 最终结果

循环 4 次结束后，取 `P` 和 `Y` 合起来作为乘积：

```text
P Y = 1110 1110
```

所以：

```text
[X × Y]补 = 1110 1110
```

验证：

```text
1110 1110
```

是负数补码，求真值：

```text
取反：0001 0001
加 1：0001 0010
```

所以：

```text
1110 1110 = -18
```

与：

```text
X × Y = -3 × 6 = -18
```

一致。

**注意：这里使用的是算数右移，就是补的是最高位**

## ALU中定点除法流程

- 除前预处理

  1. 若**被除数=0**且除数≠0,|被除数|<|除数|,则**商为0**,不再继续

## 恢复余数法

![img](https://changjiang-private-qn.yuketang.cn/slide/16314515/cover751_20260415095937.jpg?e=1782396958&token=IAM-gs8ue1pDIGwtR1CR0Zjdagg7Q2tn5G_1BqTmhmqa:RsSl-fwCBYfHLAWKccwKUo-tXsY=)

![img](https://changjiang-private-qn.yuketang.cn/slide/16314515/cover739_20260415095937.jpg?e=1782396958&token=IAM-gs8ue1pDIGwtR1CR0Zjdagg7Q2tn5G_1BqTmhmqa:xKPtYmVPlPU5bOskIYt2Ba1AJxY=)

### 1. 基本思想

恢复余数法是一种实现除法的方法。

它的核心思想是：

> 先用当前余数减去除数，判断够不够减。  
> 如果够减，商上 1；如果不够减，商上 0，并把除数加回来恢复余数。

---

### 2. 为什么叫“恢复余数法”？

计算机在除法过程中，通常先试着做：

$$
余数 - 除数
$$

如果结果为正，说明够减，商上 1。

如果结果为负，说明不够减，刚才不应该减除数，所以要把除数再加回来：

$$
余数 = 余数 + 除数
$$

这个“加回来”的过程就叫 **恢复余数**。

---

### 3. 运算规则

每一轮都按照下面步骤进行：

1. 当前余数减去除数；
2. 判断结果正负；
3. 若结果为正或 0，说明够减，商上 `1`；
4. 若结果为负，说明不够减，商上 `0`，并加回除数恢复余数；
5. 继续移位，求下一位商。

---

### 4. 核心记忆

```text
余数为正：够减，商上 1
余数为负：不够减，商上 0，恢复余数
```

## 不恢复余数法

![img](https://changjiang-private-qn.yuketang.cn/slide/16314515/cover740_20260415095938.jpg?e=1782396958&token=IAM-gs8ue1pDIGwtR1CR0Zjdagg7Q2tn5G_1BqTmhmqa:C5vBll-QkC4DbRgnASSCPQAu0tA=)

![img](https://changjiang-private-qn.yuketang.cn/slide/16314515/cover741_20260415095938.jpg?e=1782396958&token=IAM-gs8ue1pDIGwtR1CR0Zjdagg7Q2tn5G_1BqTmhmqa:c5D15wrBSquxhfnFwbZ6FARXfFs=)

### 1. 基本思想

不恢复余数法是除法运算的一种方法。

它和恢复余数法的区别在于：

> 如果试减后余数为负，不马上加回除数恢复余数，而是保留这个负余数，下一轮改为“加除数”。

所以叫 **不恢复余数法**。

---

### 2. 为什么可以不恢复？

恢复余数法中，如果余数为负，通常要：

```text
恢复余数 → 左移 → 再减除数
```

也就是：

$$
2(R_i + y^*) - y^*
$$

化简：

$$
2(R_i + y^*) - y^* = 2R_i + y^*
$$

所以这三步可以合并成：

```text
左移 → 加除数
```

因此，如果余数为负，就不需要马上恢复，下一轮直接加除数即可。

------

### 3. 运算规则

每一轮根据当前余数的正负来决定操作：

#### 余数为正

若：

$$
R_i > 0
$$

说明够减：

```text
上商 1
下一步：左移后减除数
```

公式：

$$
2R_i - y^*
$$

------

#### 余数为负

若：

$$
R_i < 0
$$

说明不够减：

```text
上商 0
下一步：左移后加除数
```

公式：

$$
2R_i + y^*
$$

------

### 4. 核心记忆

```text
余数为正：上商 1，左移后减除数
余数为负：上商 0，左移后加除数
```

一句话总结：

> 不恢复余数法就是：负余数不马上恢复，而是在下一轮通过“加除数”来修正。

# 浮点数加减法流程与特殊情况

## 一、浮点数加减法的基本流程

设两个浮点数为：

$$
X = X_m \times 2^{X_e}
$$

$$
Y = Y_m \times 2^{Y_e}
$$

其中：

- $X_m, Y_m$：尾数
- $X_e, Y_e$：阶码

浮点数加减法不能直接把尾数相加，必须先让两个数的阶码相同。

---

## 二、基本步骤

### 1. 求阶差

先比较两个数的阶码大小：

$$
\Delta e = Y_e - X_e
$$

结果的阶码一般先取两个阶码中较大的那个。

---

### 2. 对阶

对阶的原则是：

> 小阶向大阶对齐。

也就是：**阶码小的那个数，尾数右移**。

例如：

$$
0.123 \times 10^5 + 0.450 \times 10^2
$$

因为 $10^5$ 比 $10^2$ 阶码大，所以把第二个数对齐到 $10^5$：

$$
0.450 \times 10^2 = 0.000450 \times 10^5
$$

所以：

$$
0.123 \times 10^5 + 0.000450 \times 10^5
$$

尾数相加：

$$
(0.123 + 0.000450) \times 10^5 = 0.123450 \times 10^5
$$

---

### 3. 尾数加减

对阶后，两个数阶码相同，就可以进行尾数加减：

$$
X_m \pm Y_m
$$

加法就尾数相加，减法就尾数相减。

---

### 4. 规格化

尾数加减后，结果可能不是规格化形式，需要调整。

规格化的目标是让尾数最高有效位为 1，例如：

$$
1.xxxxx
$$

#### 情况一：尾数太大，需要右规

如果尾数加法后变成：

$$
10.xxx
$$

说明最高位产生进位，需要右规：

$$
10.xxx \times 2^e = 1.0xxx \times 2^{e+1}
$$

规则：

> 尾数右移 1 位，阶码加 1。

---

#### 情况二：尾数太小，需要左规

如果尾数减法后变成：

$$
0.00...1xxx
$$

说明最高有效位不在规定位置，需要左规：

$$
0.001xxx \times 2^e = 1.xxx \times 2^{e-3}
$$

规则：

> 尾数左移 k 位，阶码减 k。

---

## 三、浮点加减法中的特殊情况

### 1. 对阶时可能丢失精度

对阶时，阶码小的数要把尾数右移。

如果阶差很大，小数的尾数会右移很多位，低位可能被丢掉。

例如，假设机器尾数只能保留 4 位有效数字：

$$
1.234 \times 10^5 + 5.678 \times 10^1
$$

先对阶：

$$
5.678 \times 10^1 = 0.0005678 \times 10^5
$$

所以：

$$
1.234 \times 10^5 + 0.0005678 \times 10^5
$$

尾数相加：

$$
1.234 + 0.0005678 = 1.2345678
$$

但是机器只能保留 4 位有效数字，所以结果只能近似为：

$$
1.235 \times 10^5
$$

这里小数的低位信息就被丢掉了。

如果阶差特别大，小数甚至完全影响不了结果，这叫：

> 小数被大数“吃掉”。

在 IEEE 754 单精度中，有效位是 24 位，所以当阶差大于约 24 时，小阶数基本就无法影响结果。

---

### 2. 对阶右移时要保留附加位

对阶右移时，被移出去的低位不能马上全部丢掉，因为它们可能影响最后舍入。

例如，假设尾数只能保留 4 位：

$$
1.000 \times 2^5 + 1.111 \times 2^2
$$

阶差为：

$$
5 - 2 = 3
$$

所以第二个数尾数右移 3 位：

$$
1.111 \times 2^2 = 0.001111 \times 2^5
$$

如果只保留 4 位，可能只剩：

$$
0.001
$$

但后面的 $111$ 会影响最后舍入。

所以硬件会临时保留这些被移出的低位，这些位叫：

> 附加位。

附加位的作用是：

> 帮助后面进行更准确的舍入。

---

### 3. 右规时可能发生上溢

如果尾数加法后最高位产生进位，需要右规：

$$
尾数右移，阶码加 1
$$

但如果阶码已经很大，再加 1 可能超出范围。

例如：

$$
1.110 \times 2^{127} + 1.100 \times 2^{127}
$$

尾数相加：

$$
1.110 + 1.100 = 11.010
$$

所以：

$$
11.010 \times 2^{127}
$$

需要右规：

$$
11.010 \times 2^{127} = 1.1010 \times 2^{128}
$$

但是单精度规格化数的真实阶码范围约为：

$$
[-126, +127]
$$

现在阶码变成了 $128$，超过最大范围，所以发生：

> 阶码上溢。

上溢说明结果太大，机器表示不了，通常会触发异常或变成无穷大。

---

### 4. 左规时可能发生下溢

如果尾数减法后最高位为 0，需要左规：

$$
尾数左移，阶码减 1
$$

如果阶码减得太小，就会低于机器能表示的最小阶码。

例如：

$$
1.000 \times 2^{-126} - 0.111 \times 2^{-126}
$$

尾数相减：

$$
1.000 - 0.111 = 0.001
$$

所以：

$$
0.001 \times 2^{-126}
$$

需要左规 3 位：

$$
0.001 \times 2^{-126} = 1.000 \times 2^{-129}
$$

但是单精度规格化数的最小真实阶码约为：

$$
-126
$$

现在变成了 $-129$，超出范围，所以发生：

> 阶码下溢。

下溢说明结果太接近 0，机器可能无法用规格化数表示。

---

### 5. 尾数位数太长，需要舍入

尾数加减后，结果尾数可能超过规定长度。

例如，假设机器尾数只能保留 4 位：

$$
1.101101
$$

但是只能存：

$$
1.101
$$

后面的 $101$ 放不下，就需要舍入。

如果按照就近舍入，后面的部分可能会让结果进 1：

$$
1.101101 \approx 1.110
$$

所以：

> 尾数太长时，不能随便截断，要根据舍入规则处理。

---

### 6. 舍入后可能再次规格化

舍入有时会让尾数产生进位。

例如尾数只能保留 4 位：

$$
1.1111
$$

如果后面还有被舍掉的位：

$$
1.1111\ 1
$$

舍入后可能变成：

$$
10.0000
$$

这又不是规格化形式，所以还要右规：

$$
10.0000 \times 2^e = 1.0000 \times 2^{e+1}
$$

所以：

> 舍入后也可能导致阶码加 1。

如果阶码已经很大，这一步还可能引起上溢。

---

### 7. 结果尾数为 0 时，阶码也要置 0

如果两个数相减结果为 0，例如：

$$
1.101 \times 2^5 - 1.101 \times 2^5
$$

尾数相减：

$$
1.101 - 1.101 = 0.000
$$

结果就是 0。

这时不能保留原来的阶码 $2^5$，而应该把整个浮点数置为 0：

$$
阶码 = 0,\quad 尾数 = 0
$$

因为：

> 尾数为 0，说明整个结果就是 0，阶码已经没有意义。

---

### 8. 偏置常数不影响阶差计算

IEEE 754 单精度浮点数的阶码采用移码表示，偏置常数为 127。

如果真实阶码为：

$$
E_x,\ E_y
$$

那么存储的阶码为：

$$
E_x + 127,\quad E_y + 127
$$

求阶差时：

$$
(E_x + 127) - (E_y + 127)
$$

中间的 127 会抵消：

$$
= E_x - E_y
$$

所以：

> 偏置常数不会影响阶差计算。

例如：

$$
E_x = 5,\quad E_y = 2
$$

真实阶差：

$$
5 - 2 = 3
$$

存储阶码：

$$
5 + 127 = 132
$$

$$
2 + 127 = 129
$$

用存储阶码相减：

$$
132 - 129 = 3
$$

结果一样。

## 浮点加减运算中的附加位、舍入方式和溢出判断

### 一、附加位 Extra Bits 是什么？

浮点数的尾数位数是有限的。比如 IEEE 754 单精度尾数实际有效位是 24 位，但是在运算过程中，尾数右移、加减、规格化之后，可能会多出一些低位。

这些多出来的低位如果直接扔掉，结果可能不准确，所以需要临时保留一些位，叫 **附加位**。

PPT 里提到至少有两个附加位：

```text
Guard bit：保护位，紧挨着有效尾数的右边
Rounding bit：舍入位，在保护位的右边
```

可以理解成：

```text
有效尾数 | G | R
```

例如机器最后只能保留：

```text
1.1101
```

但是运算结果是：

```text
1.1101 01
```

那么：

```text
1.1101 是最终要保留的尾数
0 是 Guard bit
1 是 Rounding bit
```

这两个附加位的作用是：**帮助判断最后到底是舍掉，还是进一位。**

------

### 二、附加位为什么重要？

比如对阶时，小阶数的尾数要右移：

```text
1.111 × 2^2
```

如果要对齐到：

```text
× 2^5
```

尾数要右移 3 位：

```text
1.111 × 2^2 = 0.001111 × 2^5
```

如果机器只能保留：

```text
0.001
```

后面的 `111` 直接丢掉，会影响精度。

所以要把右移出去的低位暂时留下，作为附加位。后面舍入时，再根据这些附加位决定结果是否进位。

一句话：

> **附加位就是为了防止对阶右移时把重要低位直接丢掉。**

------

### 三、附加位在规格化中的作用

PPT 里说：

> 左规时移到尾数中；或作为舍入的依据。

什么意思？

如果尾数加减后需要左规，比如：

```text
0.001101
```

要规格化成：

```text
1.101 × 2^(-3)
```

这时原来在右边的一些附加位，可能会因为左移而进入有效尾数范围。

所以附加位不只是“等着被舍入”，有时候还会因为左规进入真正的尾数中。

------

### 四、IEEE 754 的舍入方式

舍入就是：当结果尾数太长，机器存不下时，按照规则取一个可表示的近似值。

假设真实结果是 `Z`，但机器只能表示离它最近的两个数：

```text
Z1   Z   Z2
```

其中：

```text
Z1 是 Z 左边较小的可表示数
Z2 是 Z 右边较大的可表示数
```

IEEE 754 常见舍入方式有四种。

------

#### 1. 向 +∞ 方向舍入

也叫向正无穷舍入。

意思是：结果往更大的方向取。

所以：

```text
舍入为 Z2
```

例如：

```text
1.234 → 1.24
```

对于负数，向 +∞ 是往“没那么负”的方向。

例如：

```text
-1.234 → -1.23
```

------

#### 2. 向 -∞ 方向舍入

也叫向负无穷舍入。

意思是：结果往更小的方向取。

所以：

```text
舍入为 Z1
```

例如：

```text
1.234 → 1.23
```

负数：

```text
-1.234 → -1.24
```

------

#### 3. 向 0 方向舍入

也叫截断。

意思是：直接把多余位截掉，往 0 靠近。

所以：

```text
正数：取 Z1
负数：取 Z2
```

例如：

```text
+1.234 → +1.23
-1.234 → -1.23
```

它的特点是：**不管正负，绝对值都变小。**

------

#### 4. 就近舍入

这是默认方式。

意思是：选择离真实结果最近的那个可表示数。

如果离 Z1 更近，就取 Z1；如果离 Z2 更近，就取 Z2。

如果刚好在中间，就采用“取偶”原则。

------

### 五、PPT 里的 01、11、10 是什么意思？

PPT 这里讲的是根据附加位判断舍入。

假设只保留 4 位尾数，多出来的两位是附加位。

#### 情况 1：附加位是 `01`

```text
1.110101 → 1.1101
```

保留部分：

```text
1.1101
```

附加位：

```text
01
```

`01` 比半个单位小，所以直接舍掉。

------

#### 情况 2：附加位是 `11`

```text
1.110111 → 1.1110
```

保留部分本来是：

```text
1.1101
```

附加位是：

```text
11
```

说明被舍掉的部分比较大，要进位。

所以：

```text
1.1101 + 0.0001 = 1.1110
```

------

#### 情况 3：附加位是 `10`

`10` 表示刚好在两个可表示数中间。

这时用“强迫结果为偶”的规则，也就是看保留尾数的最低位。

##### 如果最低位是 0

```text
1.111010 → 1.1110
```

保留部分：

```text
1.1110
```

最低位是 `0`，已经是偶数，所以舍掉多余位。

------

##### 如果最低位是 1

```text
1.110110 → 1.1110
```

保留部分：

```text
1.1101
```

最低位是 `1`，不是偶数，所以进 1：

```text
1.1101 + 0.0001 = 1.1110
```

所以 `10` 的规则是：

```text
如果最低位为 0：舍
如果最低位为 1：入
```

目的就是让最终结果最低位为偶数，也就是最低位为 0。

------

### 六、溢出判断

浮点数规格化时，阶码可能会改变：

```text
右规：尾数右移，阶码加 1
左规：尾数左移，阶码减 1
```

所以可能产生阶码溢出。

------

#### 1. 右规可能导致上溢

右规发生在尾数太大的时候。

例如：

```text
1.111 + 1.111 = 11.110
```

结果尾数变成：

```text
11.110
```

不是规格化形式，需要右规：

```text
11.110 × 2^E = 1.1110 × 2^(E+1)
```

也就是：

```text
尾数右移 1 位，阶码加 1
```

如果阶码本来已经很大，再加 1 就可能超出范围，这叫 **上溢**。

PPT 里的判断规则是：

```text
右规前先看阶码是否全 1；
如果已经全 1，直接认为上溢。
否则阶码加 1；
加 1 后如果变成全 1，也认为上溢。
```

因为在 IEEE 754 里，阶码全 1 通常是保留给无穷大和 NaN 的，不再表示普通规格化数。

------

#### 2. 左规可能导致下溢

左规发生在尾数太小的时候。

例如：

```text
1.001 - 1.000 = 0.001
```

结果是：

```text
0.001 × 2^E
```

不符合规格化形式，要左规：

```text
0.001 × 2^E = 1.000 × 2^(E-3)
```

也就是：

```text
尾数左移 3 位，阶码减 3
```

如果阶码本来就很小，再减就可能低于最小表示范围，这叫 **下溢**。

PPT 里的判断规则是：

```text
左规前先看阶码是否全 0；
如果已经全 0，直接认为下溢。
否则阶码减 1；
减 1 后如果变成全 0，也认为下溢。
```

因为在 IEEE 754 里，阶码全 0 通常表示 0 或非规格化数，不再是普通规格化数。
