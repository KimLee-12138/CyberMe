## 磁盘存储器（外设）

信息存储的原理：通过磁头线圈通入不同流向的电流来写1或者0在磁盘片上。读的时候磁头径向移动，磁片圆周运动来读取

![img](C:\Users\22067\Desktop\计组复习\assets\cover428_20260615083031.jpg)

![img](C:\Users\22067\Desktop\计组复习\assets\cover682_20260615083032.jpg)

1**.磁道**：

物理形态：磁盘是由多个圆形盘片组成的，每个盘片的表面被划分成一系列以盘片中心为圆心的同心圆轨道，这些轨道就称为磁道。

1. 磁道的数量取决于磁盘的存储密度，半径越小的磁道（靠近盘片中心）周长越短，**但存储的信息量与外圈磁道相同（通过调整磁信号密度实现）。**

**2.扇区**

每个磁道会被进一步划分为若干个等角度的弧段，每个弧段称为一个扇区。

**3.磁道和扇区的关系**

1. 结构层级： 磁盘盘片 → 磁道（同心圆）→ 扇区（磁道上的分段），即扇区是磁道的细分单元。
2. 数据存储逻辑： 数据按 “柱面→磁道→扇区” 的顺序组织，例如：先存满一个磁道的所有扇区，再转向下一个磁道，或同一柱面上的其他盘片磁道。
3. 实际应用： 当用户保存文件时，操作系统会将文件拆分成多个小块，按扇区地址写入对应磁道的扇区中，并通过文件系统（如 NTFS、FAT32）记录扇区的位置信息，确保数据可读取。

> 这里的第二小问要注意一下
>
> **位密度400位/cm指的都是内道位密度**，由于各道扇区数相同，又因为默认**硬磁盘中各个扇区的数据容量是相同的，所以各道的数据容量是相同的。**但是由于外道的周长会比内道要长，所以越往外，位密度会降低
>
> 第三小问中，只考虑将例题作为“单磁头工作”来简化计算传输速率，若是题目提到能够多磁头并行工作，则传输率=转速（转/秒）×单道容量（字节/转）×同时工作的磁头数
>
> 第四问中的寻址顺序：磁盘号->柱面号->盘面号->扇区号

### 1. 共有多少柱面？

先理解什么是**柱面**。

在多片磁盘中，所有盘面上**半径相同的一圈磁道**组成一个柱面。

比如所有盘面上的第 0 道合起来，就是第 0 柱面；所有盘面上的第 1 道合起来，就是第 1 柱面。

所以：

> **柱面数 = 每个盘面上的磁道数。**

不是所有盘面的磁道总数。

------

存储区是一个圆环，从内半径到外半径之间可以放磁道。

题目给的是直径，所以先算半径差：

```
外半径 = 33 / 2 = 16.5 cm
内半径 = 22 / 2 = 11 cm
有效存储宽度 = 16.5 - 11 = 5.5 cm
```

道密度是：

```
40 道/cm
```

所以一个盘面上的磁道数为：

```
磁道数 = 40 × 5.5 = 220 道
```

因此：

```
柱面数 = 220 个柱面
```

答案：

```
共有 220 个柱面。
```

------

### 2. 盘组总存储容量是多少？

容量计算要分三步：

```
每道容量
↓
每个盘面容量
↓
整个盘组容量
```

------

#### 第一步：算每条磁道容量

题目说：

```
各道扇区数相同
```

这句话很关键。

因为内圈磁道最短，外圈磁道最长。如果每道扇区数相同，就必须以内圈磁道能放下的数据量为准。

否则内圈放不下。

所以用**内圈周长**计算每道容量。

内直径是 22 cm，所以内圈周长：

```
内圈周长 = π × d = 3.14 × 22 = 69.08 cm
```

位密度是：

```
400 位/cm
```

所以每条磁道的数据量：

```
每道数据量 = 400 × 69.08 = 27632 位
```

换成字节：

```
27632 ÷ 8 = 3454 B
```

所以：

```
每条磁道容量 = 3454 B
```

------

#### 第二步：算一个盘面的容量

一个盘面有 220 条磁道。

```
一个盘面容量 = 3454 B × 220
             = 759880 B
```

------

#### 第三步：算整个盘组容量

总共有 5 片磁盘，每片 2 面：

```
总面数 = 5 × 2 = 10 面
```

但是最上、最下两个面不用：

```
可用面数 = 10 - 2 = 8 面
```

所以盘组总容量：

```
盘组容量 = 759880 B × 8
         = 6079040 B
```

换算成 MB：

如果按：

```
1 MB = 1024 × 1024 B
```

则：

```
6079040 ÷ 1024 ÷ 1024 ≈ 5.80 MB
```

所以答案：

```
盘组总容量 = 6079040 B ≈ 5.8 MB
```

------

### 3. 数据传输率是多少？

数据传输率就是：

> **磁盘每秒从磁道上转过多少数据。**

一圈读完一条磁道。

每条磁道容量刚才算过：

```
每道容量 = 3454 B
```

转速是：

```
2400 转/分
```

换成每秒：

```
2400 ÷ 60 = 40 转/秒
```

所以每秒可以读出：

```
数据传输率 = 40 × 3454 B/s
           = 138160 B/s
```

如果换成 KB/s，按 1KB = 1024B：

```
138160 ÷ 1024 ≈ 134.92 KB/s
```

答案：

```
数据传输率 ≈ 134.92 KB/s
```

------

### 4. 直接寻址的最小单位是什么？数据块如何寻址？

磁盘直接访问数据时，最小单位通常是：

```
扇区
```

也叫：

```
记录块
```

所以：

```
直接寻址的最小单位 = 一个扇区 / 一个记录块
```

------

一个数据块要定位到磁盘上的具体位置，需要说明：

```
在哪个硬盘
在哪个柱面
在哪个盘面
在哪个扇区
```

所以地址格式可以写成：

```
硬盘号 + 柱面号 + 盘面号 + 扇区号
```

如果只有一个硬盘，也可以简化为：

```
柱面号 + 盘面号 + 扇区号
```

其中：

```
柱面号：决定磁头移动到哪个半径位置
盘面号：决定使用哪一个磁头
扇区号：决定该磁道上的哪一个扇区
```

------

### 5. 文件超过一个磁道容量，应该放同一个存储面，还是同一个柱面？

答案是：

```
应该记录在同一个柱面上。
```

原因是：

> **同一个柱面上的多个磁道半径相同，只需要切换磁头，不需要移动磁臂重新寻道。**

假如文件超过了一条磁道容量，如果继续放在同一个盘面上的下一条磁道，就需要磁头移动到另一个半径位置。

这叫：

```
寻道
```

寻道比较慢。

但如果放在同一个柱面的其他盘面上，只需要从一个磁头切换到另一个磁头，不需要移动磁臂，速度更快。

所以：

```
同一柱面优先
```

### 一、硬盘读数据的完整过程

磁盘读一个扇区，不是 CPU 一说“我要这个数据”，数据马上就出来。它要经历几个动作。

#### 1. 先寻道：磁头移动到目标柱面

PPT 写：

```
所有磁头同步寻道（柱面号）
```

意思是，磁盘有多个盘面、多个磁头，但这些磁头固定在同一个机械臂上，移动时一起移动。

如果要访问某个柱面，磁臂要移动到对应半径位置。

这个过程叫：

```
寻道
```

花的时间叫：

```
寻道时间
```

比如磁头原来在第 20 柱面，现在要到第 200 柱面，就要移动磁臂。

------

#### 2. 再选择磁头：确定访问哪个盘面

到了目标柱面以后，还要确定是哪一个盘面上的数据。

因为同一个柱面包含多个盘面的同半径磁道。

所以需要选择：

```
磁头号 / 盘面号
```

------

#### 3. 等目标扇区转到磁头下面

磁头已经在目标磁道上了，但目标扇区不一定刚好在磁头下面。

磁盘还在转，所以要等它转过来。

这个等待时间叫：

```
旋转等待时间
```

平均情况下，要等半圈。

所以：

```
平均旋转等待时间 = 半圈时间
```

------

#### 4. 真正读写扇区数据

扇区转到磁头下面以后，磁头开始读数据。

这个时间叫：

```
数据传输时间
```

一般一个扇区传输时间很短，常常比寻道时间、旋转等待时间小很多。

------

### 二、平均存取时间公式

PPT 给的公式是：

```
T = 平均寻道时间 + 平均旋转等待时间 + 数据传输时间 + 开销时间
```

分别解释一下。

------

#### 1. 平均寻道时间

```
平均寻道时间：磁头找到指定磁道所需平均时间
```

题目有时候直接给你，比如 12ms。

也有时候说：

```
平均寻道时间 = 最大寻道时间的一半
```

因为磁头不一定每次都从最远处移动到最远处，平均移动距离大约按一半估算。

------

#### 2. 平均旋转等待时间

这个非常常考。

假设磁盘转速是 5400 RPM。

RPM 是：

```
Revolutions Per Minute
每分钟转数
```

先换成每秒转数：

```
5400 RPM = 5400 / 60 = 90 RPS
```

也就是每秒转 90 圈。

转一圈时间是：

```
1 / 90 s ≈ 0.0111 s = 11.1 ms
```

平均等半圈，所以：

```
平均旋转等待时间 = 11.1 / 2 = 5.56 ms
```

PPT 写成：

```
0.5 / 90 s ≈ 5.5 ms
```

这就是半圈时间。

------

#### 3. 数据传输时间

数据传输时间公式：

```
数据传输时间 = 要传输的数据量 / 数据传输率
```

例题中一个扇区是 512B，也就是：

```
512B = 0.5KB
```

数据传输率是：

```
4MB/s
```

所以：

```
数据传输时间 = 0.5KB / 4MB/s
```

如果按：

```
1MB = 1024KB
```

那么：

```
4MB/s = 4096KB/s
```

所以：

```
0.5 / 4096 s ≈ 0.000122s = 0.122ms
```

约等于：

```
0.1ms
```

------

#### 4. 开销时间

开销时间是磁盘控制器处理命令、准备传输等额外时间。

题目给：

```
磁盘控制器开销 = 1ms
```

直接加进去即可。

------

### 三、例题完整计算

题目条件：

```
每个扇区：512B
磁盘转速：5400 RPM
平均寻道时间：12ms
数据传输率：4MB/s
磁盘控制器开销：1ms
```

要求：

```
磁盘响应时间 / 平均存取时间
```

------

#### 第一步：平均寻道时间

题目直接给：

```
平均寻道时间 = 12ms
```

------

#### 第二步：平均旋转等待时间

先把转速换成每秒：

```
5400 RPM = 5400 / 60 = 90 RPS
```

一圈时间：

```
1 / 90 s = 11.1ms
```

平均等半圈：

```
平均旋转等待时间 = 0.5 / 90 s = 5.56ms ≈ 5.5ms
```

------

#### 第三步：数据传输时间

一个扇区：

```
512B = 0.5KB
```

数据传输率：

```
4MB/s = 4096KB/s
```

所以：

```
数据传输时间 = 0.5KB / 4096KB/s
             ≈ 0.000122s
             ≈ 0.122ms
             ≈ 0.1ms
```

------

#### 第四步：加上控制器开销

```
开销时间 = 1ms
```

------

#### 第五步：总平均存取时间

```
平均存取时间
= 12ms + 5.5ms + 0.1ms + 1ms
= 18.6ms
```

所以答案是：

```
磁盘平均响应时间约为 18.6ms
```

------

### 四、为什么 PPT 说“寻道时间降到 1/3 后，旋转等待时间占比增大”？

原来：

```
平均存取时间 = 12 + 5.5 + 0.1 + 1 = 18.6ms
```

其中寻道时间 12ms 占了大头。

如果技术进步，寻道时间降到原来的 1/3：

```
12 / 3 = 4ms
```

那么新的平均存取时间：

```
T = 4 + 5.5 + 0.1 + 1
  = 10.6ms
```

这时候你会发现，旋转等待时间还是 5.5ms，没有变。

也就是说：

```
寻道时间降低以后，旋转等待时间变成主要瓶颈
```

所以 PPT 说：

```
转速重要性提高
```

因为提高转速，可以降低平均旋转等待时间。

------

### 五、为什么访问局部性会让寻道时间下降？

PPT 问：

```
为什么寻道时间可能降至 1/3？
```

答案是：

```
访问局部性使得每次磁盘访问大多在局部几个磁道。
```

什么意思？

程序访问磁盘时，往往不是今天访问第 1 磁道，下一次突然访问第 10000 磁道，而是经常访问相邻区域。

比如读取一个连续文件时，数据可能就在相邻磁道或相邻柱面上。

所以磁头不用每次移动很远，平均寻道距离变短，平均寻道时间也会降低。

------

### 六、如何算每道有多少扇区？

PPT 最下面给了一个公式：

```
(4MB/s × 60s) / 5400 / 512B ≈ 91 个扇区
```

这个公式是在算：

> 一条磁道大约有多少个扇区。

因为磁盘转一圈，就读完一条磁道。

已知数据传输率是：

```
4MB/s
```

那么一分钟能传输：

```
4MB/s × 60s
```

一分钟磁盘转：

```
5400 圈
```

所以一圈能传输的数据量是：

```
(4MB/s × 60s) / 5400
```

一圈就是一条磁道，因此这是每条磁道容量。

再除以每个扇区大小 512B，就得到每道扇区数。

计算：

```
4MB/s = 4 × 1024 × 1024 B/s
```

一分钟数据量：

```
4 × 1024 × 1024 × 60 B
```

每圈数据量：

```
(4 × 1024 × 1024 × 60) / 5400 ≈ 46603B
```

每个扇区 512B：

```
46603 / 512 ≈ 91
```

所以：

```
每道大约 91 个扇区
```

## I/O接口

![img](C:\Users\22067\Desktop\计组复习\assets\cover583_20260615083045.jpg)

I/O设备：图中指的是鼠标器，键盘这类

I/O控制器：USB控制器、磁盘控制器这类的

I/O总线：图中标记的就是I/O总线

I/O 总线、I/O 控制器和 I/O 设备的关系：

1. I/O 设备是物理上独立的外部设备，如鼠标、键盘、磁盘、网卡、声卡等。

2. I/O 设备通常不直接连接 CPU，而是通过 I/O 控制器或 I/O 接口连接到 I/O 总线上。

3. I/O 控制器负责接收 CPU 命令，控制具体设备工作，并完成数据缓冲、状态报告和数据传输。

4. I/O 总线是连接多个 I/O 控制器的公共通信通路。

5. I/O 总线通过南桥、北桥等桥接芯片与 CPU 和主存连接。

6. 北桥一般连接 CPU、主存、显卡等高速部件；南桥一般连接 USB、磁盘、网卡、PCI 扩展设备等 I/O 部件。

7. 磁盘读写时，CPU 通常只发命令，真正的数据传输可由磁盘控制器通过 DMA 直接送入主存，减少 CPU 负担。

## I/O接口的一般结构

![img](C:\Users\22067\Desktop\计组复习\assets\cover716_20260615083048.jpg)

I/O 接口的一般结构：

I/O 接口位于主机和外设之间，负责连接系统总线和外部设备。

主机侧通过数据线、地址线、控制线访问 I/O 接口。
设备侧通过接口电缆与外设交换数据、状态和控制信号。

I/O 接口内部通常包括：
1. 数据缓冲寄存器
   用于暂存 CPU 与外设之间交换的数据。

2. 状态/控制寄存器
   状态寄存器用于反映外设当前状态，如忙、就绪、出错等；
   控制寄存器用于接收 CPU 发来的控制命令，如启动、停止、读写等。

3. 地址译码和 I/O 控制逻辑
   根据地址线和控制线判断 CPU 要访问哪个寄存器，以及进行读还是写操作。

4. 外设界面控制逻辑
   直接控制外部设备，与外设交换数据、状态和控制信号。

CPU 访问外设不是直接访问设备本体，而是通过读写 I/O 接口中的寄存器完成：
- 向控制寄存器写命令；
- 从状态寄存器读状态；
- 通过数据缓冲寄存器读写数据。

I/O 接口中的各类寄存器称为 I/O 端口。

## I/O设备的寻址方式

![img](C:\Users\22067\Desktop\计组复习\assets\cover717_20260615083048.jpg)

I/O 设备的寻址方式：

I/O 接口中的寄存器称为 I/O 端口。CPU 要访问外设，本质上是访问 I/O 端口，因此 I/O 端口也需要编址。一个 I/O 接口可能占用多个地址。

1. 统一编址方式（内存映射方式）
   I/O 端口和主存单元统一编址，将主存地址空间的一部分分配给 I/O 端口。
   CPU 使用普通访存指令即可访问 I/O 端口。
   地址译码逻辑根据地址范围判断访问主存还是 I/O。
   优点：无需专门 I/O 指令，编程统一。
   缺点：占用主存地址空间。

2. 独立编址方式（特殊 I/O 指令方式）
   I/O 端口和主存单元分开编号，形成独立的 I/O 地址空间。
   主存地址和 I/O 地址可以重叠，但通过不同控制信号区分。
   主存读写使用 MEMR、MEMW；
   I/O 读写使用 IOR、IOW。
   优点：不占用主存地址空间，I/O 地址线可较少。
   缺点：需要专门 I/O 指令。

核心区别：
统一编址：I/O 像内存一样访问。
独立编址：I/O 有自己的地址空间，用专门 I/O 指令访问。

## 轮询方式

![img](C:\Users\22067\Desktop\计组复习\assets\cover732_20260615083052.jpg)

程序控制 I/O（查询方式）：

查询方式是指 CPU 启动外设后，通过不断读取外设状态寄存器来判断外设是否完成工作。若未完成，则继续查询；若完成，则进行数据读写。

基本过程：
1. CPU 向外设发出启动命令；
2. 外设开始工作；
3. CPU 反复执行查询程序，读取并测试外设状态；
4. 外设完成后，CPU 进行数据传输。

典型查询循环：
IN：读取外设状态；
TEST：测试状态位；
JE：若未完成则跳回继续查询。

特点：
1. 实现简单，容易控制；
2. I/O 接口控制逻辑少；
3. CPU 与外设串行工作；
4. CPU 在等待期间不断查询，产生“踏步”现象；
5. 查询开销大，CPU 利用率低；
6. 适合慢速、简单外设。

缺点：
CPU 在 I/O 操作期间几乎全部时间都用于等待和查询，无法有效执行其他程序。

## 中断I/O方式

![img](C:\Users\22067\Desktop\计组复习\assets\cover733_20260615083052.jpg)

中断 I/O 方式：

基本思想：
CPU 启动外设后，不再反复查询外设状态，而是继续执行原程序。
当外设准备好或完成工作后，主动向 CPU 发出中断请求。
CPU 响应中断后，暂停当前程序，转去执行中断服务程序。
中断服务程序完成输入/输出操作后，CPU 返回原程序断点继续执行。

基本过程：
1. CPU 启动外设；
2. 外设开始工作；
3. CPU 与外设并行工作，CPU 继续执行原程序；
4. 外设完成后发出中断请求；
5. CPU 响应中断，保存断点和现场；
6. 执行中断服务程序；
7. 中断服务程序执行完毕后返回原程序继续执行。

优点：
CPU 不必一直等待外设，利用率高；
CPU 和外设可以并行工作；
适合低速或随机事件型外设，如键盘、鼠标、打印机等。

缺点：
硬件和控制过程比查询方式复杂；
中断处理需要保存和恢复现场，有一定开销；
大量数据传输时频繁中断效率不高，通常要配合 DMA。

## 中断处理过程

一个中断处理过程包含:中断请求、中断排队、中断响应、中断服务、中断返回。

DMA 方式：

DMA 即 Direct Memory Access，直接存储器存取，是一种外设和主存之间直接进行高速数据传输的方式。CPU 只负责设置 DMA 参数和启动 DMA，真正的数据传输由 DMA 控制器硬件自动完成，不需要 CPU 逐字节搬运数据。

DMA 适用场景：
1. 高速外设与主存之间的数据交换；
2. 成批数据传输；
3. 数据传输间隔短、连续读写的场景。

DMA 工作过程：
1. 外设准备好数据后，向 DMA 控制器发 DMA 请求；
2. DMA 控制器向 CPU 请求总线控制权；
3. CPU 发出总线响应，让出总线；
4. DMA 控制器控制外设和主存直接传输数据；
5. 传输过程中自动修改地址和计数；
6. 传输结束后，通过 DMA 结束中断通知 CPU。

DMA 与中断结合：
磁盘读写中，寻道、找扇区、结束校验等阶段适合用中断方式；
连续读写大量数据阶段适合用 DMA 方式；
DMA 结束后通过中断通知 CPU。

DMA 与中断的区别：
1. DMA 方式下，数据传送由 DMA 控制器完成；
   中断方式下，数据传送由中断服务程序完成。

2. DMA 请求的是总线控制权；
   中断请求的是 CPU 转去执行中断服务程序。

3. 中断方式可以处理异常事件；
   DMA 本身主要负责数据传输，不能单独处理异常。

4. DMA 适合高速、大批量数据传输；
   中断适合低速、随机事件型外设。
