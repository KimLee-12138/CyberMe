缺失率和关联度  
关联度是指一个主存块调入  Cache 时，可以存放的  Cache 位置个数。
三种映射方式的关联度：
1. 直接映射：关联度为  1 。每个主存块只能放到固定的一行。
2. 全相联映射：关联度为  Cache 行数。每个主存块可以放到任意一行。
3. N 路组相联映射：关联度为  N 。每个主存块只能进入固定组，但可放在组内任意一行。
关联度与性能的关系：
关联度越高，主存块可选择的位置越多，冲突减少，缺失率降低，命中率提高。
但关联度越高，需要比较的  Tag 越多，硬件复杂度增加，命中时间可能变长。
因此：
直接映射：命中时间最短，但命中率较低；
全相联映射：命中率较高，但命中时间较长；
组相联映射：在命中率和命中时间之间折中，实际使用较多。

## 第3页

替换算法 —— 先进先出（ FIFO- 简）  
FIFO 替换算法又称先进先出算法。
基本思想：
当 Cache 某组已满且需要调入新块时，淘汰最早进入该组的块。
FIFO 只根据块进入  Cache 的先后顺序进行替换，不考虑该块最近是否被访问过。
命中时， FIFO 一般不改变块的先后顺序。
优点：
实现简单，硬件开销较小。
缺点：
最早进入的块可能仍然被频繁使用，但仍会被淘汰；
因此命中率不一定高，甚至可能出现  Cache 容量增大但缺失次数反而增加的情况，即  Belady 异常。
例：
地址序列为  1 2 3 4 1 2 5 1 2 3 4 5 。
使用  FIFO 时， 3 行 / 组可能比  4 行 / 组缺失更少，说明  FIFO 不一定随容量增大而变好。

## 第4页

替换算法 —— 最近最少用  
LRU 是最近最少使用替换算法。
基本思想：
当 Cache 某组已满且需要调入新块时，淘汰最近最久没有被访问的那一块。
LRU 与  FIFO 的区别：
FIFO 按进入  Cache 的先后顺序淘汰；
LRU 按最近使用情况淘汰。
LRU 更符合程序的时间局部性，因此通常命中率更高。
LRU 是一种栈算法，命中率一般随组容量增大而提高。
当程序集中访问的块数超过  Cache 组容量时，会出现频繁替换，命中率很低，这种现象称为颠簸或  
Thrashing 。
例如  Cache 每组只有  3 行，但访问序列为  1,2,3,4,1,2,3,4,... ，由于需要反复访问  4 个块，而  Cache 只能
保存  3 个块，因此每次都可能缺失。

Cache、 Block 大小和缺失率的关系  
Cache 性能与缺失率密切相关，而缺失率受  Cache 大小和  Block 大小影响。
1. Cache 大小的影响：
Cache 越大，可以保存的主存块越多，数据被替换出去的概率越小，因此缺失率一般越低。
但 Cache 越大，硬件成本、面积和功耗也越高。
2. Block 大小的影响：
Block 是主存和  Cache 之间传送数据的单位。
适当增大  Block 可以把更多相邻数据调入  Cache ，利用空间局部性，降低缺失率。
但 Block 不能太大。
在 Cache 容量固定时， Block 越大， Cache 行数越少，能够同时保存的块数越少，容易产生替换和冲
突，缺失率反而可能升高。
因此：
Block 太小，空间局部性利用不足；
Block 太大， Cache 行数减少、替换频繁；
Block 大小应结合  Cache 大小选择适中值。性能好

分段式虚拟存储器  
分段式虚拟存储器：
分页式虚拟存储虽然页面大小固定、实现简单，但页不是程序逻辑上的独立实体，一条指令或一个数据
可能跨越两个页面。
分段式虚拟存储按照程序的逻辑结构划分地址空间，将程序划分为多个相对独立的段，如代码段、只读
数据段、可读写数据段、堆段、栈段等。各段长度可以不同。
每个段通常具有段名、基地址和长度，便于编写程序、编译器优化以及操作系统管理。
分段系统中，每个进程都有自己的段表，段表由段表项组成。段表项中通常记录段基地址、段长、访问
权限等信息。
逻辑地址格式：
段号  + 段内偏移
地址转换过程：
1. 根据段号查段表；
2. 取出段基地址和段长；
3. 判断段内偏移是否小于段长；
4. 若合法，则物理地址  = 段基地址  + 段内偏移；
5. 若越界或权限不符，则产生异常。
分段式优点：
符合程序逻辑结构，便于共享和保护。
分段式缺点：
段长不固定，容易产生外部碎片，内存管理较复杂。

## 第55页

示例：访问段号 1 偏移量 13K 的虚地址  
分段式虚拟存储器的地址映射：
虚拟地址由段号和段内偏移组成。
地址转换过程：
1. 根据段号查段表；
2. 检查装入位，若装入位为  0 ，则发生缺段；
3. 检查段内偏移是否小于段长，若偏移量超过段长，则地址出界；
4. 检查访问方式是否匹配，若不匹配，则保护违例；
5. 若以上检查都通过，则：
物理地址  = 段基地址  + 段内偏移。
例：
访问段号  1 、偏移量  13K 的虚地址。
查段表得：
段 1 长度为  40K ，基地址为  200K ，装入位为  1 。
因为  13K < 40K ，所以没有越界；
装入位为  1 ，说明该段已经在主存中。
因此：
物理地址  = 200K + 13K = 213K 。

## 第56页

存储保护的概念  
存储保护是指系统限制每个进程只能访问属于自己或被授权的存储区域，防止程序非法访问其他程序、
操作系统或重要数据。
存储区域可以设置不同权限：
1. 自己的存储区：可读可写、只读或只可执行；
2. 共享区：获得授权后可访问，通常可读不可写；
3. 未授权区域，如  OS 内核、页表等：不可访问。
发生存储保护错误的情况：
1. 地址越界：访问地址超出允许范围；
2. 访问越权：访问操作与权限不符，比如对只读页执行写操作。
存储保护通常通过页表或段表中的访问权限位实现。
