## 指令系统的基本概念

**指令系统是软件和硬件之间的接口**

- 指令系统 Instruction Set 是程序员和编译器看到的机器功能。
- 软件通过指令使用硬件。
- 硬件根据指令完成运算、访存、跳转等操作。
- 指令系统影响 CPU 设计、编译器设计、计算机性能和成本。

你要理解一句话：

> 指令系统就是 CPU 能“听懂”和“执行”的命令集合。

## 一条指令需包含的信息

冯诺依曼机中指令用二进制表示,和数据一起存放在主存中,

**操作码**和**操作数**(或其地址码)。

**操作码：**操作类型

- (长度:固定/可变)

**源操作数：**一个或多个源操作数的地址

- (来源：指令本身/寄存器/主存/1/0端口)

**目的操作数：**结果存放何处

- (结果地址：寄存器/主存/1/0端口)

**下一条指令**地址:

- (正常隐含在PC中,跳转时由指令给出)

## 指令格式的基本原则

- 应尽量**短**
- 操作码位数要**足够**
- 有**唯一**的解释
- 指令长度应是字节的**整数倍**
- **地址字段**的个数应合理
- 指令尽量**规整**

## 操作数基本类型

**地址(指针)**

- 无符号整数,用来确定主(虚)存地址

**数值数据**

- 定点数(整数):用二进制补码表示
- 浮点数(实数):IEEE754

**位、位串、字符和字符串**

- 用来表示文本、声音和图像等

**逻辑(布尔)数据**

- 按位操作(0-假/1-真)

这些操作数都存放在**寄存器**或**内存单元**中

## 寻址方式总览

**寻址方式：**根据地址找到指令或操作数的方法。

**两种类别：**指令的寻址、操作数的寻址

**指令寻址：**CPU 如何找到下一条要执行的指令

**操作数寻址：**CPU 如何找到参与运算的数据

- 来源：寄存器、主存、外设、端口、堆栈

### **地址编码原则**

#### 第一，地址码尽量短

地址码越短，指令就越短。

指令短的好处是：

```
目标代码短，省存储空间
```

比如一条指令如果只占 2 字节，比占 4 字节更省空间。

但是地址码太短也有问题：

```
能表示的地址范围变小
```

所以这是一个平衡。

------

#### 第二，操作数存放位置要灵活

操作数可以放在：

```
寄存器
内存
外设端口
堆栈
```

这样编译器就可以根据情况选择更合适的位置。

PPT 里说：

> 操作数存放位置灵活，空间大，利于编译产生高效代码。

意思是：

如果指令系统支持多种寻址方式，编译器就更容易生成高效程序。

例如：

```
经常用的数据放寄存器
大数组放内存
临时数据放栈中
```

这样程序运行效率更高。

------

#### 第三，地址计算尽量简单

地址计算越简单，CPU 执行越快。

比如：

```
EA = A
```

这种直接寻址就很简单。

但如果是：

```
EA = (R) + A
```

CPU 就要先取寄存器内容，再和地址字段相加。

如果更复杂，比如多级间接寻址，就可能要多次访问内存，速度更慢。

所以 PPT 说：

> 地址计算尽量简单，指令执行快。

## 常用操作数寻址方式总览

![img](https://changjiang-private-qn.yuketang.cn/slide/16314515/cover445_20260506095949.jpg?e=1782413544&token=IAM-gs8ue1pDIGwtR1CR0Zjdagg7Q2tn5G_1BqTmhmqa:qFWlxFLWDdiXuEPcGz4es1Q5XRI=)

A：指令中的地址字段
R：寄存器
EA：有效地址，也就是真正访问内存的地址
(X)：表示取 X 里面存放的内容

### 立即寻址

立即寻址的特点是：

```
操作数就在指令里面
```

比如：

```
MOV R1, #5
```

这里的 `5` 不是地址，而是数据本身。

所以 CPU 不需要再去内存找数据，直接拿 `5` 参与运算。

核心记忆：

```
立即寻址：A 本身就是操作数
```

优点是快，缺点是操作数大小受指令长度限制。

------

### 直接寻址

直接寻址的特点是：

```
指令中的 A 是操作数的内存地址
```

公式：

```
EA = A
```

比如：

```
LOAD R1, 100
```

意思是去内存地址 `100` 中取数据。

如果：

```
Memory[100] = 25
```

那么：

```
R1 = 25
```

核心记忆：

```
直接寻址：A 是地址
```

------

### 间接寻址

间接寻址比直接寻址多绕一层。

特点是：

```
指令中的 A 不是操作数地址，而是“存放操作数地址的地址”
```

公式：

```
EA = (A)
```

比如：

```
LOAD R1, (100)
```

如果：

```
Memory[100] = 300
Memory[300] = 25
```

那么真正的有效地址是：

```
EA = Memory[100] = 300
```

最后取到的操作数是：

```
Memory[300] = 25
```

核心记忆：

```
间接寻址：A 里面存的内容才是地址
```

------

### 寄存器寻址

寄存器寻址的特点是：

```
操作数在寄存器中
```

比如：

```
ADD R1, R2
```

意思是直接用寄存器 `R1` 和 `R2` 中的数据运算。

核心记忆：

```
寄存器寻址：操作数在寄存器里
```

它速度很快，因为访问寄存器比访问内存快。

------

### 寄存器间接寻址

寄存器间接寻址和寄存器寻址很像，但意义不同。

寄存器寻址是：

```
寄存器里放的是操作数
```

寄存器间接寻址是：

```
寄存器里放的是操作数的地址
```

公式：

```
EA = (R)
```

比如：

```
LOAD R1, (R2)
```

如果：

```
R2 = 300
Memory[300] = 25
```

那么：

```
R1 = 25
```

核心记忆：

```
寄存器间接寻址：寄存器里存的是地址
```

------

### 偏移寻址

偏移寻址的统一形式是：

```
EA = A + (R)
```

也就是：

```
有效地址 = 指令中的偏移量 + 某个寄存器的内容
```

它下面又可以分成：

```
相对寻址
基址寻址
变址寻址
```

这几个后面会单独讲。

先简单理解：

```
偏移寻址：一个固定值 + 一个寄存器值，合起来算地址
```

比如数组访问就很适合用偏移寻址：

```
数组首地址 + 下标偏移
```

------

### 堆栈寻址

堆栈寻址的特点是：

```
操作数默认在栈顶
```

比如：

```
PUSH
POP
CALL
RET
```

这些指令经常隐含使用堆栈。

堆栈寻址中，指令里不一定明确写操作数地址，而是默认通过栈指针 `SP` 找到栈顶数据。

核心记忆：

```
堆栈寻址：操作数在栈顶，由 SP 隐含指出
```

------

## 相对寻址、基址寻址、变址寻址

| 寻址方式 | 公式            | 寄存器是谁    | A 是什么              | 主要用途         |
| -------- | --------------- | ------------- | --------------------- | ---------------- |
| 相对寻址 | `EA = (PC) + A` | PC 程序计数器 | 相对当前指令的偏移量  | 循环、分支跳转   |
| 基址寻址 | `EA = (B) + A`  | 基址寄存器 B  | 相对程序基址的偏移量  | 程序重定位       |
| 变址寻址 | `EA = A + (I)`  | 变址寄存器 I  | 数组首地址 / 基准地址 | 数组、线性表访问 |

## 相对寻址实现循环与分支的跳转

## 基址寻址如何实现程序重定位

核心公式是：

```
EA = 基址值 + 偏移量
```

也就是：

```
EA = (B) + A
```

其中：

基址寄存器：告诉 CPU 这个程序从内存哪里开始
指令地址码：告诉 CPU 在这个程序内部偏移多少

### 基址寻址解决了什么问题？

它解决的是：

```
程序装入内存的位置不同，但程序内部地址不用改
```

如果没有基址寻址，程序 1 放在 100 开始，程序 2 放在 200 开始，那么指令里的地址可能都要重新修改。

有了基址寻址以后，程序内部只写偏移量：

```
42
```

真正运行时，CPU 自动加上基址：

```
100 + 42
200 + 42
```

这样程序就可以灵活地放到内存不同位置。

------

### 和相对寻址的区别

你可以这样区分：

```
相对寻址：
EA = (PC) + A
以 PC 为基准，主要用于跳转、循环、分支。
基址寻址：
EA = (B) + A
以程序基址为基准，主要用于程序重定位。
```

相对寻址关注的是：

```
从当前指令位置跳多远
```

基址寻址关注的是：

```
程序被放到内存哪里
```

## 指令格式

- **操作码**的编码有两种方式

  - --定长操作码
  - --扩展操作码

- **指令字**也可定长或变长

  - --定长指令字
  - --变长指令字

- **指令长度**与**操作码长度**是否可变**没有绝对关系**。

  操作码定长不一定指令字定长;操作码变长指令字一般变长。

- **代码长度**更重要时:变长指令字、变长操作码,存储更**紧凑**

- **性能**更重要时:定长指令字、定长操作码,访问和译码更**快速**。

## 定长操作码编码

特点

- 指令的操作码部分**固定长度**。若为**8位**,则最多可表示**256种指令**。
- 译码方便,但有信息**冗余**。

举例

- IBM360/370采用:
- **8位定长**操作码,最多可有256条指令
- 只提供了183条指令,有73种编码为冗余信息
- 机器字长32位,按字节编址
- 有16个32位通用寄存器,基址器B和变址器X可用其中任意一个。

## 拓展（变长）操作码编码

![img](https://changjiang-private-qn.yuketang.cn/slide/16314515/cover460_20260506095954.jpg?e=1782465724&token=IAM-gs8ue1pDIGwtR1CR0Zjdagg7Q2tn5G_1BqTmhmqa:nASyzzFjrq8uBSMrI0T7PEDRdyE=)

### 1. 先理解“短操作码”和“长操作码”

一条指令总共只有 16 位。

如果一条指令要带两个地址，那么两个地址已经占了：

```
6 + 6 = 12 位
```

剩下给操作码 OP 的只有：

```
16 - 12 = 4 位
```

所以二地址指令只能用 **4 位操作码**。

这就是“短操作码”。

```
二地址指令格式：

OP(4位) | 地址1(6位) | 地址2(6位)
```

但是如果是一地址指令，只需要一个 6 位地址，那么剩下：

```
16 - 6 = 10 位
```

都可以作为操作码。

```
一地址指令格式：

OP(10位) | 地址(6位)
```

所以一地址指令的操作码比二地址指令长。

如果是零地址指令，一个地址都不需要，那么 16 位都可以作为操作码。

```
零地址指令格式：

OP(16位)
```

所以这句话的意思就是：

> 地址字段越多，留给操作码的位数越少；地址字段越少，空出来的地址位就可以拿来扩展操作码。

------

### 2. 为什么先安排二地址指令？

因为二地址指令需要的地址字段最多。

它必须保留两个 6 位地址：

```
地址1：6 位
地址2：6 位
```

所以操作码只能占 4 位。

4 位操作码最多能表示：

```
2^4 = 16 种
```

题目要求二地址指令有 15 条，所以我们可以把：

```
0000 ~ 1110
```

这 15 个编码分配给二地址指令。

也就是：

```
0000 地址1 地址2
0001 地址1 地址2
0010 地址1 地址2
...
1110 地址1 地址2
```

一共 15 条二地址指令。

------

### 3. 为什么要留下 1111？

4 位操作码一共有 16 种：

```
0000 ~ 1111
```

前面用了：

```
0000 ~ 1110
```

还剩下：

```
1111
```

这个 `1111` 不分给二地址指令，而是作为一个信号：

```
如果前 4 位是 1111，说明这不是普通二地址指令，要继续往后看。
```

也就是说，`1111` 是一个 **扩展标志**。

这就是“扩展操作码”的关键。

------

### 4. 一地址指令怎么扩展？

一地址指令只需要一个地址字段，所以可以保留最后 6 位作为地址：

```
前 10 位操作码 | 后 6 位地址
```

因为要和二地址指令区分，前 4 位必须是：

```
1111
```

表示“进入扩展”。

所以一地址指令的前 10 位操作码可以长这样：

```
1111 xxxxxx
```

但是 PPT 不是把所有 `1111 xxxxxx` 都给一地址指令，因为还要给零地址指令留空间。

PPT 的分配方式是：

```
11110 + (00000 ~ 11111)
```

这一部分有：

```
2^5 = 32 条
```

也就是：

```
11110 00000 地址
11110 00001 地址
...
11110 11111 地址
```

这 32 条是一地址指令。

但是题目要求一地址指令有 34 条，还差 2 条。

所以继续用：

```
11111 + (00000 ~ 00001)
```

也就是：

```
11111 00000 地址
11111 00001 地址
```

这 2 条也是一地址指令。

所以一地址指令总共：

```
32 + 2 = 34 条
```

------

### 5. 零地址指令从哪里来？

刚才一地址指令用了：

```
11110 00000 ~ 11110 11111
```

以及：

```
11111 00000
11111 00001
```

那么在 `11111` 这一组里，剩下的还有：

```
11111 00010
11111 00011
...
11111 11111
```

也就是：

```
00010 ~ 11111
```

一共有：

```
32 - 2 = 30 种
```

这些就可以继续作为零地址指令的前缀。

零地址指令不需要地址字段，所以原来最后的 6 位地址字段也可以拿来当操作码。

最后 6 位有：

```
2^6 = 64 种
```

所以零地址指令最多有：

```
30 × 2^6 = 1920 条
```

这就是 PPT 最后一行：

```
故零地址指令最多有 30 × 2^6 种
```

## 指令与寻址例题

![img](https://changjiang-private-qn.yuketang.cn/slide/16314515/cover514_20260506095954.jpg?e=1782465724&token=IAM-gs8ue1pDIGwtR1CR0Zjdagg7Q2tn5G_1BqTmhmqa:EA5XSvzjOleWbTLlXFWbUyYNk8c=)

### 1. 先看题目要求什么

一条指令总共只有：

```
16 位
```

现在要在这 16 位里面放三类信息：

```
OP：操作码，表示是哪一条指令
X：寻址方式字段，表示用哪种寻址方式
A：形式地址 / 偏移量
```

所以指令大体格式是：

```
OP | X | A
```

问题就是：**OP、X、A 各占几位？**

------

### 2. 第一步：算 OP 操作码需要几位

题目说：

```
共有 80 条指令
```

操作码 OP 的作用是区分不同指令。

所以 OP 至少要能表示 80 种不同情况。

如果 OP 是 6 位，最多表示：

```
2^6 = 64 条指令
```

不够。

如果 OP 是 7 位，最多表示：

```
2^7 = 128 条指令
```

够了。

所以：

```
OP 采用 7 位
```

PPT 写的是：

```
log₂[80] = 7
```

严格说应该是：

```
⌈log₂80⌉ = 7
```

也就是向上取整。

------

### 3. 第二步：算寻址方式字段 X 需要几位

题目说有 4 种寻址方式：

```
直接寻址
间接寻址
相对寻址
变址寻址
```

所以要有一个字段 X 来明确表示这 4 种方式。

如果 X 是 1 位，最多表示：

```
2^1 = 2 种
```

不够。

如果 X 是 2 位，最多表示：

```
2^2 = 4 种
```

刚好够。

所以：

```
X 采用 2 位
```

可以这样编码：

```
00：直接寻址
01：间接寻址
10：相对寻址
11：变址寻址
```

具体怎么对应不唯一，只要能区分 4 种寻址方式即可。

------

### 4. 第三步：算 A 字段剩多少位

整条指令一共：

```
16 位
```

已经分给 OP：

```
7 位
```

又分给 X：

```
2 位
```

所以剩下给 A 的位数是：

```
16 - 7 - 2 = 7 位
```

所以：

```
A 采用 7 位
```

A 是形式地址，也可以在相对寻址、变址寻址中作为偏移量使用。

------

### 5. 最终指令格式

所以这条指令格式设计为：

```
| OP | X | A |
```

具体位数是：

```
OP：7 位
X ：2 位
A ：7 位
```

也就是：

```
| OP(7位) | X(2位) | A(7位) |
```

对应 PPT 下面的位号：

```
15      9 8 7 6      0
|  OP   | X |   A    |
```

解释一下：

```
OP：第 15 位到第 9 位，共 7 位
X ：第 8 位到第 7 位，共 2 位
A ：第 6 位到第 0 位，共 7 位
```

## 指令设计的复杂度

- 复杂指令集计算机CISC (Complex Instruction Set Computer)
- 精简指令集计算机RISC (Reduced Instruction Set Computter)

## CISC指令系统的主要特点

- (1)指令系统复杂
  - 变长操作码/变长指令字/指令多/寻址方式多/指令格式多
- (2)指令周期长
  - 绝大多数指令用多个时钟周期
- (3)各种指令都能访存
  - 除存储器读写指令外,运算指令也能访存
- (4)采用微程序控制
- (5)有专用寄存器
- (6)编译优化难度大

CISC的缺陷

- -研制周期变长,难以保证设计的正确性,难以调试和维护。因指令复杂而增加机器周期,从而降低了系统性能。

## RISC设计风格的主要特点

-  (1)指令系统简化
  - 指令少/寻址方式少/格式少/长度一致
- (2)以RR方式工作
  - 除Load/Store访存外,其余指令都只访问寄存器。
- (3)指令周期短
  - 以流水线方式工作,除Load/Store指令外,其他简单指令都一个时钟周期完成。
- (4)通用寄存器多,以减少访存次数
- (5)采用组合逻辑电路控制(更快),不用或少用微程序控制
- (6)采用优化的编译系统

## MIPS特点概述

- 所有指令都是**32位;**
- 有些指令有**26位目标地址**;有些只有**16位**。
- 所有动作可在**1个时钟周期**内完成;
- 所有运算都是**32位**的,没有字节和半字的运算;
- MIPS指令**固定**4个字节,x86平均长度~3个字节,所以一般二进制文件比x86大;
- **有栈顶寄存器SP,但没有栈指令PUSH和POP**,对栈的操作用SW/LW实现。
- 子函数的返回地址不存到栈中,而是存到**$31寄存器**。
