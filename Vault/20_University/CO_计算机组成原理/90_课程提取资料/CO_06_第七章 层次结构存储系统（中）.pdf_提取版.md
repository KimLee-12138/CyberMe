## 第15页

低位多体交叉存储器通过低位地址选择存储模块，使连续地址轮流分布到不同模块中，从而实现流
水线式访存。
每个存储模块有自己的地址寄存器  AR 和数据寄存器  DR ，可以分时使用公共地址总线  ABUS 和数据
总线  DBUS 。
一次读操作可分为三个阶段：
1. 送地址  ta ：通过地址总线把地址送入模块的  AR ；
2. 准备数据  tm ：模块内部根据地址读取数据；
3. 取数  td：数据进入  DR 后通过数据总线送出。
 
一个模块完成一次读操作的总时间为：
T = ta + tm + td
由于地址总线和数据总线不能被多个模块同时占用，因此相邻两个模块启动的时间间隔至少为：
τ = max(ta, td)
若交叉模块数为  m ，并能每隔  1/m 个存取周期启动一个模块，则：
τ = T / m
连续读取  n 个字时：
顺序方式耗时： nT
交叉方式耗时： T + (n - 1) τ
当 n 很大且  τ  = T/m 时，交叉方式的速度提升趋近于  m 倍。

## 第17页

程序的局部性与高速缓存  
Cache 是设置在  CPU 和主存之间的快速小容量存储器，用于存放当前活跃的程序和数据，以减少  CPU  
访问主存的次数。
Cache 加速的基础是程序局部性。
时间局部性：
刚被访问过的存储单元很可能不久后再次被访问。
做法是将最近访问过的数据保留在靠近  CPU 的  Cache 中。
空间局部性：
刚被访问过的存储单元附近的单元很可能不久后被访问。
做法是将相邻的一块数据一起调入  Cache 。
局部性的原因：
指令通常连续存放，且循环程序段会重复执行；
数据通常连续存放，数组元素常按顺序访问。
由于程序具有局部性， CPU 很可能直接从  Cache 中取得指令和数据，而不必每次都访问主存，从而提高
访存速度。

## 第20页

Cache的概念设计  
Cache 是位于  CPU 和主存之间的小容量高速缓冲存储器，通常由  SRAM 构成，速度接近  CPU ，但
容量较小。
程序运行时， Cache 保存主存中当前活跃的一部分内容，因此  Cache 可以看作主存部分内容的映像
或副本。
Cache 与主存之间通常按块  Block 传送数据，而不是只传送单个数据。这样可以利用程序的空间局
部性，提高后续访问命中率。
CPU 读写内存时，首先检查  Cache ：
若所需数据在  Cache 中，称为  Cache 命中， CPU 直接访问  Cache ，不必访问主存；
若所需数据不在  Cache 中，称为  Cache 未命中，需要访问主存，并将相关数据块调入  Cache 。
Cache 能加速的原因是：程序具有局部性， CPU 很可能多次访问最近使用过的数据，或者访问其附近的
数据。

Cache 映射是指主存块调入  Cache 时，应该放到  Cache 哪一行的规则。
主存块和  Cache 行之间有三种映射方式：
1. 直接映射：
每个主存块只能映射到  Cache 的固定行。
映射公式：
Cache 行号  = 主存块号  mod Cache 行数
优点是实现简单、查找速度快；
缺点是容易发生冲突失效。
2. 全相联映射：
每个主存块可以映射到  Cache 的任意一行。
优点是灵活、冲突少；
缺点是查找复杂、硬件成本高。
3. 组相联映射：
Cache 被分成若干组，每个主存块只能映射到固定组，但可以放在该组的任意一行。
它是直接映射和全相联映射的折中。
直接映射例子：
若 Cache 有  16 行，主存第  100 块映射到：
100 mod 16 = 4
所以应放入  Cache 第  4 行。
