寄存器组由多个寄存器组成，例如 32 个 32 位寄存器。

- 读操作：

  RA、RB 给出两个源寄存器编号，寄存器组分别把对应内容送到 busA、busB。

- 写操作：

  RW 给出目的寄存器编号，busW 给出要写入的数据。

  当写使能 WE=1 且时钟到来时，把 busW 的值写入 RW 指定的寄存器。

  写操作属于时序逻辑，需要时钟控制。

## 数据通路与时序控制

- 同步系统：所有动作有专门的时序信号来定时，由时序信号规定何时发出什么动作
- 时序信号:同步系统中用于进行同步控制的定时信号，如时钟信号
- 指令周期：取并执行一条指令的时间
- 机器周期：又称 CPU 周期，是指 CPU 完成一个基本操作（如取指、存取数据等）所需的时间，通常以存取存储器一次所需的时间来衡量。
- 两者的区别：指令周期相对较长，它是取并执行指令的完整过程的时间。机器周期是执行指令过程中完成某一细小功能如存放数据的时间，相对较短，包含在指令周期当中
- 不过现在计算机已经不采用三级时序系统，机器周期的概念也渐渐消失了

这四个寄存器都是 CPU 执行指令时的“临时中转站”，但它们管的东西不一样。你可以先这样记：

```text
MAR 管地址
MBR 管数据
IR 管当前指令
IBR 管备用/暂存指令
```

------

### 1. MAR：存储器地址寄存器

**MAR = Memory Address Register**

它的作用是：

> **保存 CPU 要访问的主存地址。**

通俗说，MAR 就像“快递地址栏”。

CPU 想去内存里取东西，必须先告诉内存：

```text
我要访问哪个地址？
```

这个地址就先放到 MAR 里。

比如：

```text
PC → MAR
```

意思是把 PC 中的指令地址送到 MAR，然后主存根据 MAR 中的地址去找指令。

再比如：

```text
操作数地址 → MAR
```

意思是把操作数所在的地址送到 MAR，然后主存根据这个地址去取数据。

一句话：

> **MAR 只管地址，不管数据内容。**

------

### 2. MBR：存储器数据寄存器

**MBR = Memory Buffer Register**
有些教材也叫 MDR：Memory Data Register。

它的作用是：

> **保存从主存读出来的数据，或者准备写入主存的数据。**

通俗说，MBR 就像“快递包裹中转箱”。

读内存时：

```text
M → MBR
```

意思是主存读出的内容先放到 MBR。

写内存时：

```text
MBR → M
```

意思是要写入主存的数据先放到 MBR，再写进内存。

所以 MBR 管的是：

```text
主存和 CPU 之间传输的数据/指令内容
```

一句话：

> **MBR 管内容，MAR 管地址。**

------

### 3. IR：指令寄存器

**IR = Instruction Register**

它的作用是：

> **保存当前正在执行的指令。**

CPU 从主存取出指令后，不能马上丢掉，要放到 IR 里，让控制器分析：

```text
这是什么指令？
要做加法还是减法？
要访问哪个寄存器？
操作数地址是多少？
```

比如：

```text
MBR → IR
```

意思是从内存取来的指令进入 IR。

然后控制器根据 IR 里的内容进行译码。

一句话：

> **IR 保存“当前正在执行的那条指令”。**

------

### 4. IBR：指令缓冲寄存器

**IBR = Instruction Buffer Register**

它的作用是：

> **暂存还没执行、但已经取出来的指令。**

这个在早期计算机结构中比较常见。

有些机器一次从主存取出一个较大的存储字，而这个存储字里可能包含两条短指令。

比如一次取出来：

```text
左半部分：指令 1
右半部分：指令 2
```

CPU 可能先把指令 1 放到 IR 里执行，把指令 2 暂存在 IBR 中。

等指令 1 执行完，再把 IBR 里的指令 2 送到 IR 执行。

所以 IBR 就像：

```text
备用指令暂存区
```

一句话：

> **IBR 保存“已经取出来但暂时还没执行的指令”。**

------

### 5. 四个放一起怎么区分？

| 名称 | 中文             | 作用                          | 通俗理解   |
| ---- | ---------------- | ----------------------------- | ---------- |
| MAR  | 存储器地址寄存器 | 保存要访问的内存地址          | 地址栏     |
| MBR  | 存储器数据寄存器 | 保存从内存读出/写入内存的数据 | 中转箱     |
| IR   | 指令寄存器       | 保存当前正在执行的指令        | 当前任务单 |
| IBR  | 指令缓冲寄存器   | 暂存已经取出但还没执行的指令  | 备用任务单 |

------

### 6. 用取指过程串起来

PPT 里的取指路径是：

```text
PC → MAR
Read M
M → MBR → IBR → IR
```

可以这样理解：

```text
PC：告诉 CPU 下一条指令在哪里
MAR：把这个地址拿去访问主存
主存 M：根据地址把指令读出来
MBR：暂时接收主存读出来的内容
IBR：暂存可能还没马上执行的指令
IR：保存当前马上要执行的指令
```

最核心一句话：

> **MAR 是“去哪儿取”，MBR 是“取出来什么”，IR 是“现在执行什么”，IBR 是“先放着等会儿执行什么”。**

## 单总线数据通路

![img](https://changjiang-private-qn.yuketang.cn/slide/16314515/cover467_20260511075939.jpg?e=1782481609&token=IAM-gs8ue1pDIGwtR1CR0Zjdagg7Q2tn5G_1BqTmhmqa:qTdPs_Pg4z_9uji8RBDCpQ3Hxb4=)

### 1. 什么是单总线？

图右边中间那条粗竖线就是 **内部总线**。

它连接了：

```
通用寄存器 R0 ~ R(n-1)
PC
IR
MAR
MDR
Y
Z
ALU
```

这些部件之间传数据，都要经过这条总线。

比如：

```
R0 → Y
```

实际过程不是 R0 直接连到 Y，而是：

```
R0 → 内部总线 → Y
```

所以写成控制信号就是：

```
R0out → Yin
```

意思是：

```
R0out：让 R0 的内容送到总线上
Yin：让 Y 从总线上接收数据
```

------

### 2. 单总线的特点

单总线最大的特点是：

```
结构简单，但一次只能传一个数据
```

因为所有部件共用一条总线，所以同一时刻只能有一个部件输出数据到总线上。

例如不能同时：

```
R1out 和 R2out
```

否则两个寄存器同时往总线上放数据，会冲突。

所以如果 ALU 要做加法，需要借助临时寄存器 Y 和 Z。

------

### 3. 图中的 Y 和 Z 是干什么的？

PPT 最下面写：

```
暂存器 Y、Z 用于某些指令临时存储数据，程序员透明。
```

意思是：

> **Y 和 Z 是 CPU 内部临时寄存器，程序员看不见，也不能直接使用。**

它们主要是为了配合单总线完成运算。

因为 ALU 加法需要两个输入，但单总线一次只能传一个数。

所以要这样做：

```
先把第一个数放入 Y
再把第二个数送到 ALU
ALU 计算后把结果放入 Z
最后 Z 再送回目标寄存器
```

所以：

```
Y：暂存 ALU 的一个输入
Z：暂存 ALU 的运算结果
```

------

### 4. 第一类操作：寄存器之间传送数据

PPT 写：

```
在寄存器间传送数据
R0out → Yin
```

意思是：

```
R0 的内容送到总线
Y 从总线接收
```

也就是：

```
Y ← R0
```

过程是：

```
R0 → 内部总线 → Y
```

这里的 `out` 表示“输出到总线”，`in` 表示“从总线写入”。

------

### 5. 第二类操作：算术、逻辑运算

PPT 给的过程是：

```
R1out → Yin
R2out, Add → Zin
Zout → R3in
```

它表示执行：

```
R3 ← R1 + R2
```

为什么要分三步？因为单总线一次只能传一个数据。

------

#### 第一步：把 R1 放进 Y

```
R1out → Yin
```

意思是：

```
Y ← R1
```

R1 先暂存在 Y 中，作为 ALU 的第一个输入。

------

#### 第二步：R2 送入 ALU，和 Y 相加

```
R2out, Add → Zin
```

意思是：

```
Z ← Y + R2
```

这里：

```
R2out：R2 输出到总线
Add：控制 ALU 做加法
Zin：把 ALU 结果写入 Z
```

因为 Y 里已经有 R1，所以 ALU 实际做的是：

```
R1 + R2
```

结果先进入 Z。

------

#### 第三步：结果写回 R3

```
Zout → R3in
```

意思是：

```
R3 ← Z
```

也就是：

```
R3 ← R1 + R2
```

所以完整过程是：

```
R1 → Y
Y + R2 → Z
Z → R3
```

------

### 6. 第三类操作：读主存

PPT 写：

```
读主存    R[R2] ← M[R[R1]]

R1out → MARin
Read, WMFC
MDRout → R2in
```

这条的意思是：

> **用 R1 中的内容作为内存地址，从主存中读出数据，放入 R2。**

也就是：

```
R2 ← M[R1]
```

注意 PPT 写得更正式：

```
R[R2] ← M[R[R1]]
```

意思是：

```
R2 寄存器 ← 主存中地址为 R1 内容的那个单元
```

------

#### 第一步：R1 的内容送入 MAR

```
R1out → MARin
```

意思是：

```
MAR ← R1
```

MAR 是存储器地址寄存器，负责告诉主存访问哪个地址。

------

#### 第二步：读主存

```
Read, WMFC
```

其中：

```
Read：发出读主存信号
WMFC：等待存储器完成
```

MFC 是 Memory Function Complete，表示主存访问完成。

`WMFC` 就是 Wait for MFC，意思是：

```
等主存把数据读出来
```

读出的数据会进入 MDR。

------

#### 第三步：MDR 的内容送入 R2

```
MDRout → R2in
```

意思是：

```
R2 ← MDR
```

也就是把主存读出的数据写入 R2。

完整流程：

```
R1 → MAR
Read M
M[MAR] → MDR
MDR → R2
```

------

### 7. 第四类操作：写主存

PPT 写：

```
写主存    M[R[R1]] ← R[R2]

R1out → MARin
R2out → MDRin
Write, WMFC
```

这条的意思是：

> **把 R2 中的数据写入主存，写入地址由 R1 给出。**

也就是：

```
M[R1] ← R2
```

------

#### 第一步：R1 的内容送入 MAR

```
R1out → MARin
```

意思是：

```
MAR ← R1
```

这一步确定写入主存的地址。

------

#### 第二步：R2 的内容送入 MDR

```
R2out → MDRin
```

意思是：

```
MDR ← R2
```

MDR 保存准备写入主存的数据。

------

#### 第三步：写主存

```
Write, WMFC
```

意思是：

```
把 MDR 中的数据写入 MAR 指定的主存单元
并等待主存写入完成
```

完整流程：

```
R1 → MAR
R2 → MDR
Write M
M[MAR] ← MDR
```

------

### 8. MAR 和 MDR 在这里的作用

这张图里，访问主存一定绕不开 MAR 和 MDR。

```
MAR：保存主存地址
MDR：保存主存数据
```

读主存：

```
地址：R1 → MAR
数据：M → MDR → R2
```

写主存：

```
地址：R1 → MAR
数据：R2 → MDR → M
```

一句话：

> **MAR 管地址，MDR 管数据。**

------

### 9. 内总线和外总线的区别

图里有两个总线概念。

#### 内总线

CPU 内部连接寄存器、ALU、MAR、MDR 等部件。

例如：

```
R1 → Y
Z → R3
R1 → MAR
MDR → R2
```

这些都走内总线。

------

#### 外总线

CPU 和主存/I/O 设备之间的连接。

图中右边写了：

```
地址线
数据线
```

访问主存时：

```
MAR 通过地址线把地址送到主存
MDR 通过数据线和主存交换数据
```

## 设计处理器

![img](https://changjiang-private-qn.yuketang.cn/slide/16314515/cover341_20260511075940.jpg?e=1782481609&token=IAM-gs8ue1pDIGwtR1CR0Zjdagg7Q2tn5G_1BqTmhmqa:yeCjjLfWpi9J4-nQ_Cu91NDD9vA=)

### 一、先理解什么是 ISA

PPT 第一行说：

```
确定 ISA 后首先要明确设计步骤
```

ISA 就是 **指令集体系结构**，也就是 CPU 要支持哪些指令。

比如：

```
add
sub
lw
sw
beq
jal
```

ISA 决定了 CPU 必须能完成哪些功能。

所以设计处理器的第一步不是先画电路，而是先问：

```
我的 CPU 要执行哪些指令？
每条指令到底做什么？
```

------

### 二、处理器设计分两大部分

PPT 左边把过程分成两块：

```
数据通路的设计
控制器的设计
```

#### 1. 数据通路

数据通路就是 CPU 里数据流动和加工的路径。

比如：

```
寄存器组
ALU
PC
指令存储器
数据存储器
立即数生成器
多路选择器
```

这些部件连接起来，就形成了数据通路。

可以理解成：

> **数据通路是 CPU 的“身体”。**

------

#### 2. 控制器

控制器负责产生控制信号。

比如：

```
RegWrite：是否写寄存器
MemRead：是否读存储器
MemWrite：是否写存储器
ALUSrc：ALU 第二个操作数来自寄存器还是立即数
MemtoReg：写回寄存器的数据来自 ALU 还是内存
Branch：是否是分支指令
```

可以理解成：

> **控制器是 CPU 的“大脑”，负责指挥数据通路怎么工作。**

------

### 三、第一步：分析每条指令的功能，并用 RTL 表示

PPT 第一条写：

```
第一步：分析每条指令的功能，并用 RTL 来表示。
```

RTL 是 **寄存器传送语言**。

它用类似下面的形式描述数据怎么流动：

```
R3 ← R1 + R2
```

意思是：

```
把 R1 和 R2 的内容相加，结果写入 R3。
```

------

#### 举例 1：add 指令

假设有一条指令：

```
add rd, rs1, rs2
```

它的功能是：

```
rd ← rs1 + rs2
```

用 RTL 表示就是：

```
R[rd] ← R[rs1] + R[rs2]
```

这句话告诉我们：

```
1. 要读寄存器 rs1
2. 要读寄存器 rs2
3. 要用 ALU 做加法
4. 要把结果写回 rd
```

------

#### 举例 2：lw 指令

比如：

```
lw rd, offset(rs1)
```

意思是从内存读数据到寄存器。

RTL 可以写成：

```
R[rd] ← M[R[rs1] + offset]
```

这句话说明：

```
1. 读 rs1
2. 立即数 offset 要扩展
3. ALU 计算地址 R[rs1] + offset
4. 访问数据存储器
5. 把内存读出的数据写回 rd
```

所以 RTL 的作用就是：

> **把一条指令拆成 CPU 内部必须完成的微操作。**

------

### 四、第二步：根据指令功能给出所需元件，并考虑如何互连

PPT 第二条写：

```
第二步：根据指令功能给出所需元件，并考虑如何互连。
```

这一步就是开始设计数据通路。

你要根据前面 RTL 分析，判断 CPU 需要哪些硬件部件。

------

#### 以 add 为例

`add` 需要完成：

```
R[rd] ← R[rs1] + R[rs2]
```

所以至少需要：

```
寄存器组：读 rs1、rs2，写 rd
ALU：执行加法
连线/总线：把寄存器数据送到 ALU，再把结果送回寄存器组
```

所以数据流是：

```
寄存器组读出 rs1、rs2
        ↓
       ALU 加法
        ↓
结果写回寄存器 rd
```

------

#### 以 lw 为例

`lw` 需要完成：

```
R[rd] ← M[R[rs1] + offset]
```

所以需要：

```
寄存器组：读 rs1，写 rd
立即数生成器：生成 offset
ALU：计算有效地址
数据存储器：根据地址读数据
多路选择器：选择写回寄存器的数据来自内存
```

数据流是：

```
rs1 → ALU
offset → ALU
ALU 计算地址
地址送入数据存储器
存储器读出数据
数据写回 rd
```

------

#### 为什么要考虑“互连”？

因为有了元件还不够，还要知道这些元件怎么连。

比如 ALU 的第二个输入有时候来自寄存器，有时候来自立即数。

```
add：ALU 第二个输入来自 rs2
lw：ALU 第二个输入来自 offset
```

那怎么办？

就需要一个多路选择器 MUX。

```
MUX 选择：
0 → rs2
1 → immediate
```

所以处理器数据通路里经常有很多 MUX，本质就是为了让不同指令共用同一套硬件。

------

### 五、第三步：确定每个元件所需控制信号的取值

PPT 第三条写：

```
第三步：确定每个元件所需控制信号的取值。
```

数据通路搭好了以后，每条指令执行时，要给各个部件不同的控制信号。

------

#### 以 add 为例

```
add rd, rs1, rs2
```

它要：

```
读寄存器
ALU 做加法
写寄存器
不读内存
不写内存
写回数据来自 ALU
```

所以控制信号大概是：

```
RegWrite = 1
ALUSrc = 0
ALUOp = add
MemRead = 0
MemWrite = 0
MemtoReg = 0
Branch = 0
```

解释一下：

```
RegWrite = 1：要写回 rd
ALUSrc = 0：ALU 第二个输入来自寄存器 rs2
ALUOp = add：ALU 做加法
MemRead = 0：不读内存
MemWrite = 0：不写内存
MemtoReg = 0：写回数据来自 ALU
```

------

#### 以 lw 为例

```
lw rd, offset(rs1)
```

它要：

```
读寄存器 rs1
立即数参与 ALU 运算
ALU 做加法算地址
读内存
把内存数据写回 rd
```

所以控制信号大概是：

```
RegWrite = 1
ALUSrc = 1
ALUOp = add
MemRead = 1
MemWrite = 0
MemtoReg = 1
Branch = 0
```

解释：

```
RegWrite = 1：要写 rd
ALUSrc = 1：ALU 第二个输入来自立即数
ALUOp = add：算地址用加法
MemRead = 1：要读数据存储器
MemWrite = 0：不写存储器
MemtoReg = 1：写回数据来自内存
```

------

#### 以 sw 为例

```
sw rs2, offset(rs1)
```

功能是：

```
M[R[rs1] + offset] ← R[rs2]
```

它要：

```
读 rs1
读 rs2
ALU 算地址
写内存
不写寄存器
```

所以控制信号大概是：

```
RegWrite = 0
ALUSrc = 1
ALUOp = add
MemRead = 0
MemWrite = 1
MemtoReg = x
Branch = 0
```

这里 `x` 表示无所谓，因为 sw 不写寄存器，写回来源不重要。

------

### 六、第四步：汇总控制信号，生成指令与控制信号间关系表

PPT 第四条写：

```
第四步：汇总控制信号，生成指令与控制信号间关系表。
```

这一步就是把不同指令对应的控制信号整理成表格。

比如：

| 指令 | RegWrite | ALUSrc | ALUOp | MemRead | MemWrite | MemtoReg | Branch |
| ---- | -------- | ------ | ----- | ------- | -------- | -------- | ------ |
| add  | 1        | 0      | add   | 0       | 0        | 0        | 0      |
| lw   | 1        | 1      | add   | 1       | 0        | 1        | 0      |
| sw   | 0        | 1      | add   | 0       | 1        | x        | 0      |
| beq  | 0        | 0      | sub   | 0       | 0        | x        | 1      |

这个表非常重要。

因为控制器本质上就是根据当前指令的 opcode、funct3、funct7 等字段，输出这些控制信号。

也就是说：

> **指令是什么，决定控制信号怎么取值。**

------

### 七、第五步：写出控制信号的逻辑表达式，据此设计控制器电路

PPT 第五条写：

```
第五步：写出控制信号的逻辑表达式，据此设计控制器电路。
```

有了上一步的控制信号表，就可以写逻辑表达式。

比如：

```
RegWrite = R型指令 OR I型运算指令 OR lw
```

意思是：

```
只要当前指令需要写寄存器，RegWrite 就为 1。
```

再比如：

```
MemWrite = sw
```

意思是：

```
只有 store 指令需要写存储器。
```

再比如：

```
MemRead = lw
```

意思是：

```
只有 load 指令需要读数据存储器。
```

然后根据这些表达式，就可以用逻辑门电路设计控制器。

------

### 八、处理器设计用到两种元件

PPT 最下面写：

```
处理器设计用到两种元件：
操作元件：由组合逻辑电路实现
存储（状态）元件：由时序逻辑电路实现
```

这句话非常关键。

------

#### 1. 操作元件：组合逻辑

操作元件就是负责计算、选择、转换的部件。

比如：

```
ALU
加法器
多路选择器 MUX
译码器
比较器
立即数生成器
控制逻辑
```

它们的特点是：

```
输入一变化，经过一定延迟，输出就变化
不需要等时钟
```

比如 ALU：

```
输入 A 和 B
控制信号是 add
经过延迟后输出 A+B
```

所以操作元件属于组合逻辑。

------

#### 2. 存储状态元件：时序逻辑

状态元件就是负责保存信息的部件。

比如：

```
PC
寄存器组
指令寄存器 IR
数据寄存器
存储器
流水线寄存器
```

它们的特点是：

```
需要时钟控制
在时钟边沿写入新值
可以保存上一个周期的状态
```

比如 PC：

```
当前 PC = 100
下一个 PC = 104
```

只有时钟到来时，PC 才会更新成 104。

所以状态元件属于时序逻辑。

------

### 九、用一条 add 指令串完整流程

假设 ISA 中有：

```
add rd, rs1, rs2
```

处理器设计过程如下。

#### 第一步：写 RTL

```
R[rd] ← R[rs1] + R[rs2]
```

#### 第二步：确定数据通路元件

需要：

```
寄存器组
ALU
写回通路
```

#### 第三步：确定控制信号

```
RegWrite = 1
ALUSrc = 0
ALUOp = add
MemRead = 0
MemWrite = 0
MemtoReg = 0
```

#### 第四步：放入控制信号表

把 add 和其他指令一起整理成表格。

#### 第五步：设计控制器

控制器识别出当前是 add 指令后，输出上面的控制信号。

最后 CPU 执行时，数据流是：

```
rs1、rs2 从寄存器组读出
→ 送入 ALU

→ 结果写回 rd
```

#### 第一步：RTL

## 单周期中典型指令的数据通路

### 三类指令（R、I、J）

![image-20260626165458606](C:\Users\22067\AppData\Roaming\Typora\typora-user-images\image-20260626165458606.png)

**R型指令：**

R型指令是只和寄存器有关的指令。R型指令的op字段为000000.区分不同的R型指令是通过func字段来区分的。

通常是将rs和rt的结果存入rd当中。但是也有特殊情况。

**（1）典型 R 型指令的操作流程**

对于算术、逻辑运算类 R 型指令（如`ADD`、`SUB`、`AND`等）：

1. 读取操作数：从`rs`和`rt`寄存器中读取源操作数。
2. ALU 运算：根据`funct`字段执行相应运算（如加法、减法）。
3. 写入结果：将运算结果写入`rd`寄存器。

示例：

- 指令格式：`ADD $rd, $rs, $rt`
- 功能：`$rd = $rs + $rt`
- 汇编示例：`add $t0, $t1, $t2` → 将`$t1`和`$t2`相加，结果存入`$t0`。

1. 特殊 R 型指令的例外情况

**(2) 移位指令（SLL/SRL/SRA）**

- 操作数：
  - `rs`字段无效（不使用）。
  - `rt`为源操作数寄存器。
  - `shamt`（移位量）指定移位位数。
- 结果：移位结果存入`rd`。

示例：

- 指令格式：`SLL $rd, $rt, shamt`
- 功能：`$rd = $rt << shamt`
- 汇编示例：`sll $t0, $t1, 2` → 将`$t1`左移 2 位，结果存入`$t0`。

**(3) 跳转指令（JR）**

- 操作数：
  - `rs`存储目标地址（如`$ra`寄存器）。
  - `rt`、`rd`、`shamt`字段均无效。
- 操作：跳转到`rs`指定的地址，不涉及`rd`。

示例：

- 指令格式：`JR $rs`
- 功能：跳转到`$rs`存储的地址（常用于子程序返回）。
- 汇编示例：`jr $ra` → 跳转到`$ra`（返回地址寄存器）。

这里稍微注意一下，RTL的指令格式的寄存器顺序和CPU中寄存器的字段顺序是不一样的

**常见指令：**

| 指令 | 功能                     | 示例              | 说明                      |
| ---- | ------------------------ | ----------------- | ------------------------- |
| ADD  | 加法                     | add $t0, $t1, $t2 | $t0 = $t1 + $t2           |
| SUB  | 减法                     | sub $t0, $t1, $t2 | $t0 = $t1 - $t2           |
| AND  | 按位与                   | and $t0, $t1, $t2 | $t0 = $t1 & $t2           |
| OR   | 按位或                   | or $t0, $t1, $t2  | $t0 = $t1 | $t2           |
| XOR  | 按位异或                 | xor $t0, $t1, $t2 | $t0 = $t1 ^ $t2           |
| NOR  | 按位或非                 | nor $t0, $t1, $t2 | $t0 = ~($t1 | $t2)        |
| SLL  | 逻辑左移                 | sll $t0, $t1, 4   | $t0 = $t1 << 4（shamt=4） |
| SRL  | 逻辑右移                 | srl $t0, $t1, 2   | $t0 = $t1 >> 2（shamt=2） |
| SRA  | 算术右移                 | sra $t0, $t1, 2   | 带符号右移                |
| SLT  | 小于则置位               | slt $t0, $t1, $t2 | 若$t1 < $t2，则$t0=1      |
| JR   | 寄存器跳转（子程序返回） | jr $ra            | 跳转到$ra（返回地址）     |

**I型指令：**

I型指令通常和立即数（也可以是内存地址偏移量、指令条数偏移量）有关

常见指令：

#### 算术与逻辑指令（立即数

| ANDI | 立即数按位与     | andi $t0, $t1, 0xFF | $t0 = $t1 & 0xFF     |
| ORI  | 立即数按位或     | ori $t0, $t1, 0x0F  | $t0 = $t1 \| 0x0F    |

## R-type指令的数据通路

![img](https://changjiang-private-qn.yuketang.cn/slide/16314515/cover350_20260511075943.jpg?e=1782481609&token=IAM-gs8ue1pDIGwtR1CR0Zjdagg7Q2tn5G_1BqTmhmqa:zB1x9_CDA1ylGz6EmxYUe71bmVY=)

### 1. 先看 R-type 指令格式

图上方是 R-type 指令的 32 位格式：

```
op | rs | rt | rd | shamt | func
```

各字段含义：

```
op：操作码，R-type 通常为 000000
rs：第一个源寄存器
rt：第二个源寄存器
rd：目的寄存器
shamt：移位位数，移位指令用
func：功能码，用来具体确定 add/sub/slt 等操作
```

对于普通加法：

```
add rd, rs, rt
```

含义是：

```
R[rd] ← R[rs] + R[rt]
```

所以：

```
rs、rt 是读寄存器编号
rd 是写寄存器编号
```

------

### 2. 图里的寄存器组在干什么？

中间这个模块是：

```
32 32-bit Registers
```

表示：

```
32 个 32 位通用寄存器
```

它有三个重要端口：

```
Ra：读端口 A，对应 rs
Rb：读端口 B，对应 rt
Rw：写端口，对应 rd
```

所以图中写：

```
Ra, Rb, Rw 对应指令的 rs, rt, rd
```

意思就是：

```
Ra = rs
Rb = rt
Rw = rd
```

------

### 3. R-type 指令的数据流过程

以：

```
add rd, rs, rt
```

为例，完整执行过程是：

------

#### 第一步：从寄存器组读出 rs 和 rt

指令中的 `rs` 字段送到 `Ra`：

```
Ra = rs
```

寄存器组从 `rs` 指定的寄存器中读出数据，送到：

```
busA
```

指令中的 `rt` 字段送到 `Rb`：

```
Rb = rt
```

寄存器组从 `rt` 指定的寄存器中读出数据，送到：

```
busB
```

所以：

```
busA = R[rs]
busB = R[rt]
```

------

#### 第二步：busA 和 busB 进入 ALU

ALU 有两个输入：

```
busA
busB
```

也就是：

```
R[rs]
R[rt]
```

然后控制器根据指令译码产生 ALU 控制信号：

```
ALUctr
```

如果是 `add` 指令：

```
ALUctr = add
```

那么 ALU 执行：

```
Result = R[rs] + R[rt]
```

如果是 `sub` 指令：

```
ALUctr = sub
```

那么 ALU 执行：

```
Result = R[rs] - R[rt]
```

所以 `ALUctr` 决定 ALU 到底做什么运算。

------

#### 第三步：ALU 结果写回 rd

ALU 的输出是：

```
Result
```

这条结果通过 `busW` 回到寄存器组的写数据端。

同时：

```
Rw = rd
```

表示要写入 `rd` 指定的寄存器。

如果控制信号：

```
RegWr = 1
```

那么时钟到来时，结果写入寄存器 `rd`：

```
R[rd] ← Result
```

也就是：

```
R[rd] ← R[rs] op R[rt]
```

------

### 4. RegWr 是什么？

图左边有一个控制信号：

```
RegWr
```

它的意思是：

> **是否允许写寄存器组。**

对于 R-type 运算指令，比如：

```
add
sub
slt
and
or
```

它们最后都要把 ALU 结果写回寄存器，所以：

```
RegWr = 1
```

如果是 `sw`、`beq` 这种不写通用寄存器的指令，那么：

```
RegWr = 0
```

------

### 5. ALUctr 是什么？

图中 ALU 上方写：

```
ALUctr: add/sub/subu/slt/sltu
```

意思是：

> **ALUctr 是控制 ALU 运算类型的信号。**

不同 R-type 指令对应不同 ALUctr：

```
add  → ALUctr = add
sub  → ALUctr = sub
slt  → ALUctr = slt
sltu → ALUctr = sltu
```

这些控制信号是由指令译码产生的。

也就是说：

```
指令的 op、func 字段
        ↓
      译码器
        ↓
  产生 ALUctr 和 RegWr
```

------

### 6. add rd, rs, rt 的控制信号是什么？

图右下角问：

```
“add rd, rs, rt” 控制信号为？
```

答案是：

```
ALUctr = add
RegWr = 1
```

解释：

```
ALUctr = add：让 ALU 执行加法
RegWr = 1：允许把结果写回 rd
```

所以执行流程就是：

```
R[rs] → busA
R[rt] → busB
ALU 做加法
Result → busW
写入 R[rd]
```

最终完成：

```
R[rd] ← R[rs] + R[rt]
```

------

### 7. 图里“溢出”是什么意思？

右边标了一个：

```
溢出
```

这是 ALU 在执行有符号加减法时可能产生的异常信号。

比如执行：

```
add
```

如果两个有符号数相加结果超出了可表示范围，就会产生溢出。

PPT 旁边写：

```
溢出时不写结果并转异常处理程序
```

意思是：

如果发生溢出：

```
1. 不把错误结果写入 rd
2. 转去异常处理程序
```

所以有溢出时，虽然 ALU 算出了 Result，但不能正常写回寄存器。

注意：

```
add 会检查溢出
addu 不检查溢出
sub 会检查溢出
subu 不检查溢出
```

## 带立即数的逻辑指令的数据通路



![img](https://changjiang-private-qn.yuketang.cn/slide/16314515/cover352_20260511075944.jpg?e=1782481609&token=IAM-gs8ue1pDIGwtR1CR0Zjdagg7Q2tn5G_1BqTmhmqa:R4n9vhAT9kLJcbD7KyFGCLkoBL8=)

![img](https://changjiang-private-qn.yuketang.cn/slide/16314515/cover353_20260511075945.jpg?e=1782481609&token=IAM-gs8ue1pDIGwtR1CR0Zjdagg7Q2tn5G_1BqTmhmqa:ZDrpjqu6mH_inBw9CiB2UFyffrA=)

### 1. 先看 ori 指令格式

图上方给出的是 I 型指令格式：

```
op | rs | rt | imm16
```

各字段含义：

```
op：操作码，说明这是 ori 这种立即数逻辑指令
rs：源寄存器
rt：目的寄存器
imm16：16 位立即数
```

和 R-type 不同，I-type 没有 `rd` 字段。

所以对于：

```
ori rt, rs, imm16
```

结果写入的是：

```
rt
```

不是 `rd`。

------

### 2. 这条指令到底做什么？

比如：

```
ori $t0, $t1, 0x00FF
```

意思是：

```
$t0 ← $t1 OR 0x000000FF
```

注意 `imm16 = 0x00FF` 是 16 位，要先扩展成 32 位：

```
0x00FF → 0x000000FF
```

然后再和 `$t1` 做按位或运算。

所以 ori 的核心数据流是：

```
R[rs] → ALU 输入 A
imm16 零扩展 → ALU 输入 B
ALU 做 or 运算
结果写入 R[rt]
```

------

### 3. 为什么需要 Ext 扩展器？

图中有一个模块：

```
Ext
```

它负责把 16 位立即数扩展成 32 位。

因为 ALU 是 32 位运算器，它的两个输入都应该是 32 位。

但是指令里的立即数只有：

```
imm16：16 位
```

所以必须扩展。

------

#### ori 为什么用零扩展？

图里写的是：

```
ZeroExt[imm16]
```

也就是零扩展。

零扩展的意思是：

```
高 16 位补 0
```

比如：

```
imm16 = 1111 1111 0000 0000
```

零扩展后：

```
0000 0000 0000 0000 1111 1111 0000 0000
```

也就是：

```
0xFF00 → 0x0000FF00
```

为什么不是符号扩展？

因为 `ori` 是逻辑运算指令，它把立即数当成一串二进制位模式，而不是有符号整数，所以要用 **零扩展**。

因此：

```
ExtOp = 0
```

在这张图里表示零扩展。

------

### 4. ALUSrc 是干什么的？

图中 ALU 第二个输入前面有一个 MUX。

这个 MUX 有两个输入：

```
0：busB，也就是寄存器 rt 读出的值
1：Ext 扩展后的立即数
```

它由控制信号：

```
ALUSrc
```

控制。

对于 R-type 指令，比如：

```
add rd, rs, rt
```

ALU 第二个操作数来自寄存器 `rt`：

```
ALUSrc = 0
```

对于 ori 指令：

```
ori rt, rs, imm16
```

ALU 第二个操作数来自立即数：

```
ALUSrc = 1
```

所以 ori 要让 MUX 选择扩展后的 `imm16`。

------

### 5. RegDst 是干什么的？

图中寄存器组上方还有一个 MUX，用来选择写入哪个寄存器。

因为不同类型指令的目的寄存器不一样：

```
R-type：结果写入 rd
I-type：结果写入 rt
```

所以需要一个 MUX 在 `rd` 和 `rt` 之间选择。

图里标的是：

```
Rd 输入编号 0
Rt 输入编号 1
```

对于 ori 是 I-type，结果写入 `rt`，所以按这张图的约定：

```
RegDst = 1
```

注意：有些教材或图中可能把 RegDst=0 设为选择 rt，RegDst=1 设为选择 rd。这里一定要按你这张 PPT 的标号来记：**这张图里 RegDst=1 选择 Rt**。

------

### 6. RegWr 是干什么的？

`RegWr` 是寄存器写使能信号。

ori 指令最后要把 ALU 运算结果写回寄存器 `rt`，所以必须打开写寄存器功能：

```
RegWr = 1
```

如果 RegWr=0，即使 ALU 算出了结果，也不会写入寄存器。

------

### 7. ALUctr 是干什么的？

`ALUctr` 是 ALU 控制信号，用来决定 ALU 做什么运算。

ori 指令要做的是按位或：

```
R[rs] OR ZeroExt[imm16]
```

所以：

```
ALUctr = or
```

如果是 addi，则 ALUctr=add；如果是 andi，则 ALUctr=and。

------

### 8. ori 指令完整执行流程

以：

```
ori rt, rs, imm16
```

为例，数据通路执行过程是：

```
1. 指令中的 rs 送到寄存器组 Ra 端口。
2. 寄存器组读出 R[rs]，送到 busA。
3. 指令中的 imm16 送入 Ext 扩展器。
4. ExtOp=0，imm16 进行零扩展，变成 32 位。
5. ALUSrc=1，MUX 选择扩展后的立即数作为 ALU 第二个输入。
6. ALUctr=or，ALU 执行按位或运算。
7. RegDst=1，写寄存器编号选择 rt。
8. RegWr=1，ALU 结果通过 busW 写入 R[rt]。
```

最终完成：

```
R[rt] ← R[rs] or ZeroExt[imm16]
```

## 带立即数的逻辑指令（lw指令）

![img](https://changjiang-private-qn.yuketang.cn/slide/16314515/cover355_20260511075945.jpg?e=1782481609&token=IAM-gs8ue1pDIGwtR1CR0Zjdagg7Q2tn5G_1BqTmhmqa:r1Of6KDyDoHr7UWcUBuQETT8g5o=)

![img](https://changjiang-private-qn.yuketang.cn/slide/16314515/cover356_20260511075946.jpg?e=1782481609&token=IAM-gs8ue1pDIGwtR1CR0Zjdagg7Q2tn5G_1BqTmhmqa:05yuLvAyyxCCwZisqYuPR1iBvwI=)

### 1. lw 指令是什么意思？

比如：

```
lw $t0, 8($t1)
```

意思是：

```
$t0 ← M[$t1 + 8]
```

也就是：

```
先看 $t1 里面存的地址
再加上偏移量 8
得到真正的内存地址
从这个地址中读出数据
写入 $t0
```

所以 lw 是 **从内存装入数据到寄存器**。

------

### 2. 指令格式

lw 是 I 型指令：

```
op | rs | rt | imm16
```

其中：

```
rs：基址寄存器
rt：目的寄存器
imm16：16 位偏移量
```

所以：

```
R[rt] ← M[ R[rs] + SignExt[imm16] ]
```

这里要注意：

```
rs 用来提供基地址
rt 用来接收内存读出的数据
imm16 是偏移量
```

------

### 3. 为什么 imm16 要符号扩展？

图里写：

```
ExtOp：0 表示零扩展，1 表示符号扩展
```

lw 中：

```
ExtOp = 1
```

因为 lw 的偏移量可以是正数，也可以是负数。

比如：

```
lw $t0, -4($sp)
```

这里的 `-4` 就是负偏移。

所以 imm16 要做 **符号扩展**，从 16 位扩展成 32 位。

例如：

```
imm16 = 0000 0000 0000 1000
符号扩展后 = 0000 0000 0000 0000 0000 0000 0000 1000
```

如果是负数：

```
imm16 = 1111 1111 1111 1100
符号扩展后 = 1111 1111 1111 1111 1111 1111 1111 1100
```

这样 ALU 才能正确计算地址。

------

### 4. lw 的执行过程

#### 第一步：读出 R[rs]

指令中的 `rs` 送到寄存器组的 `Ra` 端口。

寄存器组读出：

```
busA = R[rs]
```

这就是基地址。

------

#### 第二步：扩展 imm16

指令中的 `imm16` 送入 Ext 扩展器。

因为 lw 要符号扩展，所以：

```
ExtOp = 1
```

得到：

```
SignExt[imm16]
```

------

#### 第三步：ALU 计算有效地址

lw 的地址计算公式是：

```
有效地址 = R[rs] + SignExt[imm16]
```

所以 ALU 的两个输入分别是：

```
输入 A：busA = R[rs]
输入 B：SignExt[imm16]
```

图中 ALU 第二个输入前有一个 MUX，由 `ALUSrc` 控制。

对于 lw：

```
ALUSrc = 1
```

表示 ALU 第二个输入选择立即数扩展结果，而不是 busB。

然后 ALU 做加法：

```
ALUctr = addu
```

这里用 `addu` 是因为计算地址时一般不考虑算术溢出异常。

所以：

```
ALU Result = R[rs] + SignExt[imm16]
```

这个 Result 就是要访问的内存地址。

------

### 5. 访问数据存储器

ALU 计算出的地址送到 Memory 的地址端：

```
Addr = ALU Result
```

lw 是读内存，不是写内存，所以：

```
MemWr = 0
```

表示不写存储器。

Memory 根据地址读出数据：

```
Memory Data Out = M[ R[rs] + SignExt[imm16] ]
```

------

### 6. 内存数据写回 rt

图右边有一个 MUX，控制写回寄存器的数据来自哪里。

它由：

```
MemtoReg
```

控制。

对于普通 R-type 运算指令，写回寄存器的数据来自 ALU 结果。

但是 lw 不一样，lw 写回寄存器的数据来自内存。

所以：

```
MemtoReg = 1
```

表示选择 Memory 输出的数据，送回寄存器组的 busW。

------

### 7. 写入哪个寄存器？

R-type 指令结果写入 `rd`。

但是 lw 是 I-type 指令，结果写入 `rt`。

所以图上方有一个 MUX，用 `RegDst` 选择写寄存器编号来自 `rd` 还是 `rt`。

这张 PPT 中：

```
RegDst = 1
```

表示选择 `rt` 作为写入寄存器。

于是：

```
Rw = rt
```

并且 lw 要写寄存器，所以：

```
RegWr = 1
```

最终时钟到来时：

```
R[rt] ← Memory Data Out
```

也就是：

```
R[rt] ← M[ R[rs] + SignExt[imm16] ]
```

## 带立即数的逻辑指令（sw指令）

![img](https://changjiang-private-qn.yuketang.cn/slide/16314515/cover358_20260511075946.jpg?e=1782481609&token=IAM-gs8ue1pDIGwtR1CR0Zjdagg7Q2tn5G_1BqTmhmqa:CpUmy617m4eNEjJltrB_GWYymk0=)

![img](https://changjiang-private-qn.yuketang.cn/slide/16314515/cover359_20260511075946.jpg?e=1782481609&token=IAM-gs8ue1pDIGwtR1CR0Zjdagg7Q2tn5G_1BqTmhmqa:prIPIXkl1tP8Lj1ZV2P9hvvRjG0=)

### 1. sw 指令是什么意思？

比如：

```
sw $t0, 8($t1)
```

意思是：

```
M[$t1 + 8] ← $t0
```

也就是：

```
先读出 $t1，作为基地址；
把立即数 8 符号扩展；
ALU 计算地址 $t1 + 8；
再把 $t0 中的数据写入这个内存地址。
```

所以 `sw` 和 `lw` 的方向刚好相反：

```
lw：内存 → 寄存器
sw：寄存器 → 内存
```

------

### 2. sw 的核心公式

PPT 顶部给出：

```
M[ R[rs] + SignExt[imm16] ] ← R[rt]
```

拆开看：

```
R[rs]：基址寄存器中的值
SignExt[imm16]：16 位立即数符号扩展后的偏移量
R[rs] + SignExt[imm16]：最终内存地址
R[rt]：要写入内存的数据
```

所以 `sw` 需要完成两件事：

```
1. 算出内存地址
2. 把 rt 中的数据写入这个地址
```

------

### 3. 第一步：读寄存器 rs 和 rt

图中寄存器组有两个读口：

```
Ra ← rs
Rb ← rt
```

于是寄存器组输出：

```
busA = R[rs]
busB = R[rt]
```

这里注意：

```
busA 用来参与地址计算
busB 用来作为写入内存的数据
```

也就是说：

```
R[rs] 是地址基址
R[rt] 是要存入内存的数据
```

------

### 4. 第二步：imm16 符号扩展

`imm16` 是 16 位立即数，但是 ALU 是 32 位运算器，所以要先扩展成 32 位。

PPT 中：

```
ExtOp = 1
```

表示：

```
符号扩展
```

为什么要符号扩展？

因为偏移量可能是正数，也可能是负数。

例如：

```
sw $t0, -4($sp)
```

这里 `-4` 就必须按负数处理，所以要符号扩展。

------

### 5. 第三步：ALU 计算内存地址

ALU 要计算：

```
R[rs] + SignExt[imm16]
```

所以 ALU 的两个输入是：

```
输入 1：busA = R[rs]
输入 2：SignExt[imm16]
```

图中 ALU 第二个输入前面有一个 MUX，由 `ALUSrc` 控制。

对于 `sw`：

```
ALUSrc = 1
```

表示 ALU 第二个输入选择立即数扩展结果，而不是 busB。

然后：

```
ALUctr = addu
```

表示 ALU 做加法，用来计算地址。

所以：

```
ALU Result = R[rs] + SignExt[imm16]
```

这个 Result 就是数据存储器的地址。

------

### 6. 第四步：把 R[rt] 写入内存

这一步是 `sw` 和 `lw` 最大的区别。

`lw` 是从内存读数据，所以 Memory 输出数据。

`sw` 是向内存写数据，所以 Memory 需要两个东西：

```
Addr：写入地址
DataIn：写入数据
```

图中：

```
Addr 来自 ALU Result
DataIn 来自 busB
```

也就是：

```
Addr = R[rs] + SignExt[imm16]
DataIn = R[rt]
```

然后控制信号：

```
MemWr = 1
```

表示允许写内存。

所以最终完成：

```
Memory[Addr] ← DataIn
```

也就是：

```
M[ R[rs] + SignExt[imm16] ] ← R[rt]
```

------

### 7. 蓝色部分是什么意思？

图中标注了：

```
加蓝色部分。
```

蓝色线主要表示为了支持 `sw`，需要增加从：

```
busB → DataIn
```

的通路。

因为 `sw` 要把 `rt` 中的数据写入内存，而 `rt` 读出来后在 `busB` 上。

所以必须把 `busB` 接到数据存储器的 `DataIn` 端。

这就是 `sw` 相比 `lw` 需要重点增加的地方。

------

### 8. 为什么 RegWr = 0？

`sw` 是存数指令，它不需要写寄存器。

它的结果是写入内存，不是写回寄存器。

所以：

```
RegWr = 0
```

也就是说，寄存器组不会写入任何结果。

因此：

```
RegDst = x
MemtoReg = x
```

这里的 `x` 表示 **don’t care，无所谓**。

因为既然不写寄存器，那么：

```
写哪个寄存器不重要；
写回数据来自 ALU 还是内存也不重要。
```

## 条件转移指令的数据通路

![img](https://changjiang-private-qn.yuketang.cn/slide/16314515/cover361_20260511075947.jpg?e=1782481609&token=IAM-gs8ue1pDIGwtR1CR0Zjdagg7Q2tn5G_1BqTmhmqa:TXx9nD3KoZKq5Yo9wNwm2pBA0l8=)

![img](https://changjiang-private-qn.yuketang.cn/slide/16314515/cover362_20260511075947.jpg?e=1782481609&token=IAM-gs8ue1pDIGwtR1CR0Zjdagg7Q2tn5G_1BqTmhmqa:3iSLt50XwgthBRDbnt3hvUkznVA=)

### 1. beq 指令要完成什么？

`beq` 是条件分支指令，全称可以理解为：

```
branch if equal
```

也就是：

```
如果相等就跳转
```

例如：

```
beq $t0, $t1, label
```

含义是：

```
如果 $t0 == $t1，就跳到 label
如果 $t0 != $t1，就继续执行下一条指令
```

在硬件里，不会直接比较“等不等”，而是用 ALU 做减法：

```
R[rs] - R[rt]
```

如果结果为 0，说明：

```
R[rs] == R[rt]
```

如果结果不为 0，说明：

```
R[rs] != R[rt]
```

所以 PPT 里写：

```
Cond ← R[rs] - R[rt]
```

------

### 2. 第一步：读出 rs 和 rt

图中寄存器组有两个读端口：

```
Ra
Rb
```

对于 `beq rs, rt, imm16`：

```
Ra = rs
Rb = rt
```

所以寄存器组读出：

```
busA = R[rs]
busB = R[rt]
```

也就是把两个要比较的寄存器值读出来。

------

### 3. 第二步：ALU 做减法比较

`beq` 要判断两个数是否相等，所以 ALU 执行：

```
R[rs] - R[rt]
```

因此控制信号是：

```
ALUctr = subu
```

图中底部也写了：

```
ALUctr = subu
```

为什么是 `subu`？

因为这里做减法不是为了得到一个有符号减法结果，而是为了判断是否相等，所以一般不关心溢出，只关心结果是不是 0。

------

### 4. ALUSrc 为什么等于 0？

图中 ALU 第二个输入前有一个 MUX。

这个 MUX 可以选择：

```
0：busB，也就是 R[rt]
1：扩展后的立即数 imm16
```

对于 `beq` 来说，ALU 要做的是：

```
R[rs] - R[rt]
```

第二个操作数来自寄存器 `rt`，不是立即数。

所以：

```
ALUSrc = 0
```

意思是：

```
ALU 第二个输入选择 busB
```

------

### 5. Zero 信号是什么？

ALU 做完：

```
R[rs] - R[rt]
```

之后，会产生一个 `Zero` 信号。

如果减法结果为 0：

```
Zero = 1
```

说明：

```
R[rs] == R[rt]
```

如果减法结果不为 0：

```
Zero = 0
```

说明：

```
R[rs] != R[rt]
```

所以 `Zero` 是判断 beq 是否跳转的关键条件。

------

### 6. Branch 信号是什么？

PPT 底部写：

```
Branch = 1
```

意思是当前指令是条件分支指令。

但是注意：

```
Branch = 1 不代表一定跳转
```

真正是否跳转，要看：

```
Branch 和 Zero
```

对于 `beq`：

```
Branch = 1 且 Zero = 1 时，跳转
Branch = 1 但 Zero = 0 时，不跳转
```

可以理解成：

```
是否跳转 = Branch AND Zero
```

------

### 7. Next Addr Logic 是干什么的？

右边有一个模块：

```
Next Addr Logic
```

它负责计算下一条指令地址。

平时不跳转时：

```
PC ← PC + 4
```

如果 beq 条件成立，就跳转到目标地址：

```
PC ← PC + 4 + SignExt[imm16] << 2
```

PPT 这页没有展开左移 2 位，但 MIPS 分支指令通常是这样计算目标地址的。

可以先按这页图理解成：

```
Next Addr Logic 根据 Branch、Zero、imm16 决定下一个 PC
```

如果不跳转：

```
下一个 PC = 顺序下一条指令地址
```

如果跳转：

```
下一个 PC = 分支目标地址
```

------

### 8. imm16 在 beq 中干什么？

`imm16` 是分支偏移量。

它不是拿去和寄存器做运算的，而是用来计算跳转目标地址。

也就是说：

```
rs、rt：用于比较
imm16：用于算跳转地址
```

所以图中 `imm16` 是送到右边的 `Next Addr Logic`，而不是送入 ALU 参与减法。

------

### 9. 为什么 RegWr = 0？

`beq` 只是判断是否跳转，不会把结果写回寄存器。

它不会执行：

```
R[某个寄存器] ← ...
```

所以：

```
RegWr = 0
```

也就是不写寄存器组。

因此：

```
RegDst = x
MemtoReg = x
```

这些信号都无所谓。

因为不写寄存器时：

```
写哪个寄存器无所谓
写回数据来自哪里也无所谓
```

这里的 `x` 就是 don't care。

------

### 10. 为什么 MemWr = 0？

`beq` 不访问数据存储器，不会写内存。

所以：

```
MemWr = 0
```

它只是可能改变 PC。

------

### 11. ExtOp 为什么是 x？

PPT 底部写：

```
ExtOp = x
```

这里要按这张数据通路图理解。

因为图中的 Ext 扩展器输出是连到 ALU 第二输入 MUX 的，但是 `beq` 的 ALU 第二输入选择的是 `busB`，不是立即数。

也就是说：

```
ALUSrc = 0
```

所以 Ext 的输出不会进入 ALU。

因此在这个图中：

```
ExtOp = x
```

表示扩展方式无所谓。

不过从完整 MIPS 分支地址计算来看，`imm16` 用于计算分支目标地址时通常需要符号扩展。但这张 PPT 把分支地址计算放到了 `Next Addr Logic` 中，没有展开，所以底部控制信号这里写成 `x`。

------

### 12. beq 的完整执行过程

以：

```
beq rs, rt, imm16
```

为例：

```
1. 寄存器组读出 R[rs] 和 R[rt]
2. busA = R[rs]，busB = R[rt]
3. ALUSrc = 0，ALU 第二输入选择 busB
4. ALUctr = subu，ALU 执行 R[rs] - R[rt]
5. 如果结果为 0，Zero = 1
6. Branch = 1，Next Addr Logic 根据 Zero 判断是否修改 PC
7. 若 Zero = 1，PC 跳到目标地址
8. 若 Zero = 0，PC 继续指向下一条指令
```

## 下地址计算逻辑的设计

![img](https://changjiang-private-qn.yuketang.cn/slide/16314515/cover363_20260511075948.jpg?e=1782481609&token=IAM-gs8ue1pDIGwtR1CR0Zjdagg7Q2tn5G_1BqTmhmqa:W_qBYAqzyNJbfm832eDCifIS1oQ=)

### 1. 为什么顺序执行时 PC 要加 4？

MIPS 每条指令固定 **32 位**，也就是：

```
32 位 = 4 字节
```

所以指令在内存里通常这样排：

```
地址 1000：第 1 条指令
地址 1004：第 2 条指令
地址 1008：第 3 条指令
地址 1012：第 4 条指令
```

你会发现，每条指令之间差 4 个字节。

所以如果 CPU 当前正在取地址 `1000` 的指令，下一条顺序执行的指令地址就是：

```
1000 + 4 = 1004
```

因此顺序执行时：

```
PC = PC + 4
```

这就是 PPT 第一条：

```
顺序时：PC<31:0> = PC<31:0> + 4
```

意思就是：

> **不跳转的时候，PC 正常往后走一条指令。**

------

### 2. 转移时为什么是 PC + 4 + imm16 × 4？

条件转移，比如：

```
beq $t0, $t1, imm16
```

意思是：

```
如果 $t0 == $t1，就跳转
```

跳转目标不是简单的 `PC + 4`，而是要在 `PC + 4` 的基础上，再加一个偏移量。

所以公式是：

```
转移目标地址 = PC + 4 + SignExt[imm16] × 4
```

你可以理解为：

```
PC + 4：先走到下一条指令的位置
imm16 × 4：再根据偏移量决定往前/往后跳几条指令
```

------

### 3. 为什么 imm16 要乘 4？

这是最容易卡住的地方。

因为 `imm16` 表示的不是“字节偏移”，而是“指令条数偏移”。

比如：

```
imm16 = 1
```

不是跳 1 个字节，而是跳 1 条指令。

但一条指令占 4 字节，所以：

```
跳 1 条指令 = 地址偏移 4 字节
```

所以：

```
imm16 = 1   → 实际地址偏移 1 × 4 = 4
imm16 = 2   → 实际地址偏移 2 × 4 = 8
imm16 = -1  → 实际地址偏移 -1 × 4 = -4
```

因此公式里有：

```
SignExt[imm16] × 4
```

硬件里不会真的用乘法器去乘 4，而是用：

```
左移 2 位
```

因为二进制左移 2 位就等于乘 4。

------

### 4. 举个最简单的跳转例子

假设当前 PC 是：

```
PC = 1000
```

当前指令是：

```
beq $t0, $t1, 2
```

并且条件成立，也就是 `$t0 == $t1`。

那么转移地址是：

```
PC + 4 + imm16 × 4
= 1000 + 4 + 2 × 4
= 1000 + 4 + 8
= 1012
```

所以 CPU 下一次取指令的位置是：

```
PC = 1012
```

也就是跳到了地址 1012 的指令。

你可以画成：

```
1000：beq $t0, $t1, 2
1004：下一条
1008：再下一条
1012：跳到这里
```

因为 `imm16 = 2`，表示从 `PC+4` 开始，再往后跳 2 条指令。

------

### 5. 再举一个向前跳的例子

循环经常要向前跳。

假设：

```
PC = 1008
imm16 = -3
```

转移地址：

```
PC + 4 + imm16 × 4
= 1008 + 4 + (-3) × 4
= 1012 - 12
= 1000
```

所以跳回：

```
PC = 1000
```

这就可以实现循环。

对应内存：

```
1000：循环开始
1004：循环体
1008：beq / bne 之类的条件转移
1012：顺序下一条
```

如果条件成立，跳回 1000。

------

### 6. 为什么 PPT 后面又说只用 PC<31:2>？

这句话是这页最抽象的地方。

先记住一个事实：

> **MIPS 指令地址一定是 4 的倍数。**

因为每条指令 4 字节，所以指令地址一定长这样：

```
0
4
8
12
16
20
...
```

换成二进制，这些数的最后两位一定是：

```
00
```

比如：

```
4  = 00000100
8  = 00001000
12 = 00001100
16 = 00010000
```

最后两位都是 `00`。

所以完整 PC 是 32 位：

```
PC<31:0>
```

但是最后两位 `PC<1:0>` 永远是：

```
00
```

那既然永远是 00，计算时就可以不管它，只计算前 30 位：

```
PC<31:2>
```

最后取指令时，再把 `00` 拼回去。

------

### 7. PC<31:2> 到底是什么？

你可以把它理解成：

> **PC<31:2> 不是字节地址，而是“第几条指令”的编号。**

举例：

```
真实地址 PC = 1000
```

因为每条指令占 4 字节，所以它对应的“指令编号”可以看成：

```
1000 ÷ 4 = 250
```

这个 `250` 就类似于：

```
PC<31:2>
```

当然硬件里不是除法，而是去掉最后两个 0，本质上等价于除以 4。

------

### 8. 为什么 PC + 4 可以变成 PC<31:2> + 1？

真实地址：

```
PC = 1000
```

顺序下一条：

```
PC + 4 = 1004
```

如果用“指令编号”看：

```
1000 ÷ 4 = 250
1004 ÷ 4 = 251
```

所以：

```
真实地址 +4
```

等价于：

```
指令编号 +1
```

因此 PPT 写：

```
顺序时：PC<31:2> = PC<31:2> + 1
```

意思是：

> **如果只看前 30 位，那么顺序执行不是加 4，而是加 1。**

因为你已经把“每条指令 4 字节”这个因素隐藏掉了。

------

### 9. 转移公式为什么也能简化？

原公式是：

```
PC<31:0> = PC<31:0> + 4 + SignExt[imm16] × 4
```

如果都除以 4，也就是只看 `PC<31:2>`，就变成：

```
PC<31:2> = PC<31:2> + 1 + SignExt[imm16]
```

你可以这样对应：

```
PC + 4                 → PC<31:2> + 1
SignExt[imm16] × 4      → SignExt[imm16]
```

因为 `×4` 的影响已经通过“最后拼接 00”体现了。

------

### 10. 用完整例子对比两种算法

假设：

```
PC = 1000
imm16 = 2
```

#### 方法一：用完整 32 位地址算

```
目标地址 = PC + 4 + imm16 × 4
        = 1000 + 4 + 2 × 4
        = 1012
```

------

#### 方法二：只用 PC<31:2> 算

先把地址看成“指令编号”：

```
PC<31:2> = 1000 ÷ 4 = 250
```

转移时：

```
新 PC<31:2> = 250 + 1 + 2
            = 253
```

最后拼回低两位 `00`，也就是乘回 4：

```
真实地址 = 253 × 4 = 1012
```

结果一样。

所以 PPT 的简化方法其实就是：

> **先按“指令编号”算，最后再补上两个 0 变回真实字节地址。**

## 下址逻辑设计图

![img](https://changjiang-private-qn.yuketang.cn/slide/16314515/cover364_20260511075948.jpg?e=1782500170&token=IAM-gs8ue1pDIGwtR1CR0Zjdagg7Q2tn5G_1BqTmhmqa:btrdKpCMcLdpjwK-XhG3IVA70Tk=)

### 1. 先看这页 PPT 的核心公式

PPT 顶部写了三句话：

```
顺序：PC<31:2> = PC<31:2> + 1
转移：PC<31:2> = PC<31:2> + 1 + SignExt[imm16]
取指：指令地址 = PC<31:2> concat "00"
```

这三句话其实就是整张图的核心。

先翻译成人话：

```
顺序执行：
下一条指令就是当前指令的下一条。

条件转移：
如果分支成立，就跳到 PC+1+偏移量 对应的位置。

取指令：
真正访问指令存储器时，把 PC<31:2> 后面补两个 0，变成完整 32 位地址。
```

------

### 2. 为什么这里只算 PC<31:2>？

MIPS 每条指令占 4 字节。

所以指令地址一定是 4 的倍数：

```
0
4
8
12
16
20
...
```

这些数的二进制最低两位一定是：

```
00
```

比如：

```
4  = 00000100
8  = 00001000
12 = 00001100
16 = 00010000
```

最低两位都是 `00`。

所以完整 PC 是：

```
PC<31:0>
```

但最低两位：

```
PC<1:0>
```

永远是：

```
00
```

因此硬件只需要真正计算高 30 位：

```
PC<31:2>
```

最后取指令的时候，再把低两位 `00` 拼回去。

所以 PPT 写：

```
指令地址 = PC<31:2> concat "00"
```

意思就是：

> 用 PC 的高 30 位拼接固定的低两位 00，得到完整指令地址。

------

### 3. 左边的 PC 是干什么的？

图左边这个竖着的小框是：

```
PC
```

PC 保存的是当前要取的指令地址。

不过这张图为了简化，只保存和计算：

```
PC<31:2>
```

也就是 30 位。

PC 受时钟控制：

```
Clk
```

意思是：

> 每个时钟到来时，PC 才会更新成新的地址。

------

### 4. 第一个 Adder：计算顺序下一条指令地址

图中左边第一个加法器有两个输入：

```
PC<31:2>
常数 "1"
```

它计算：

```
PC<31:2> + 1
```

为什么加 1？

因为这里算的是高 30 位，等价于按“第几条指令”来数。

完整地址中：

```
PC + 4
```

等价于高 30 位中：

```
PC<31:2> + 1
```

举个例子：

```
当前 PC = 1000
下一条 PC = 1004
```

如果除以 4 看成指令编号：

```
1000 / 4 = 250
1004 / 4 = 251
```

所以：

```
PC + 4
```

就等价于：

```
PC<31:2> + 1
```

因此第一个 Adder 的作用是：

> 算出顺序执行时的下一条指令地址。

------

5. SignExt 是干什么的？

图下方有：

```
Instruction<15:0>
imm16
SignExt
```

`imm16` 是条件转移指令中的 16 位偏移量。

比如：

```
beq rs, rt, imm16
```

其中 `imm16` 表示跳转偏移量。

这个偏移量可能是正数，也可能是负数：

```
正数：向后跳
负数：向前跳，比如循环跳回前面
```

所以它要做：

```
符号扩展 SignExt
```

把 16 位扩展成 30 位，方便和 PC<31:2> 相加。

例如：

```
imm16 = 0000 0000 0000 0011
```

符号扩展后还是正数 3。

如果：

```
imm16 = 1111 1111 1111 1101
```

这是 -3，符号扩展后仍然表示 -3。

------

### 6. 第二个 Adder：计算分支目标地址

中间第二个加法器的输入是：

```
PC<31:2> + 1
SignExt[imm16]
```

它计算：

```
PC<31:2> + 1 + SignExt[imm16]
```

这就是分支成立时的目标地址。

为什么是 `PC + 1 + imm16`，不是直接 `PC + imm16`？

因为 MIPS 分支地址通常是相对于 **下一条指令** 来计算的。

完整地址公式是：

```
PC + 4 + SignExt[imm16] × 4
```

对应到高 30 位就是：

```
PC<31:2> + 1 + SignExt[imm16]
```

所以第二个 Adder 的作用是：

> 算出条件转移成立时要跳到的地址。

------

### 7. MUX 是干什么的？

图右边有一个 MUX，多路选择器。

它有两个输入：

```
顺序地址：PC<31:2> + 1
转移地址：PC<31:2> + 1 + SignExt[imm16]
```

MUX 要决定：

```
下一次 PC 选哪个？
```

如果不跳转，就选顺序地址。

```
PC ← PC<31:2> + 1
```

如果跳转，就选分支目标地址。

```
PC ← PC<31:2> + 1 + SignExt[imm16]
```

所以 MUX 是这张图里最关键的“选择开关”。

------

### 8. Branch 和 ZF 是怎么控制 MUX 的？

图中 MUX 下面有一个与门，输入是：

```
Branch
ZF
```

它们共同决定是否分支成立。

#### Branch 是什么？

```
Branch = 1
```

表示当前指令是分支指令，比如 `beq`。

如果当前不是分支指令，比如 `add`、`lw`、`sw`，那么：

```
Branch = 0
```

------

#### ZF 是什么？

ZF 是 Zero Flag，零标志位。

前面 beq 指令会让 ALU 做：

```
R[rs] - R[rt]
```

如果结果为 0，说明：

```
R[rs] == R[rt]
```

那么：

```
ZF = 1
```

如果结果不为 0，说明两个寄存器不相等：

```
ZF = 0
```

------

### 二者合起来是什么意思？

对于 `beq`：

```
Branch = 1 且 ZF = 1
```

才说明：

```
当前是 beq 指令，并且两个寄存器相等，分支成立。
```

所以 PPT 下面写：

```
Branch=1 ZF=1时，分支成立
```

也就是：

```
PCSrc = Branch AND ZF
```

如果：

```
Branch AND ZF = 1
```

MUX 选择转移地址。

如果：

```
Branch AND ZF = 0
```

MUX 选择顺序地址。

------

### 9. 右边 Instruction Memory 怎么取指？

右边是指令存储器：

```
Instruction Memory
```

它的输入地址分成两部分：

```
Addr<31:2>
Addr<1:0>
```

图中 `Addr<31:2>` 来自 PC 的高 30 位。

低两位 `Addr<1:0>` 固定接：

```
"00"
```

所以完整指令地址是：

```
Addr<31:0> = PC<31:2> concat "00"
```

指令存储器根据这个地址读出：

```
Instruction<31:0>
```

也就是当前取出的 32 位指令。

## 无条件跳转指令的数据通路

![img](https://changjiang-private-qn.yuketang.cn/slide/16314515/cover367_20260511075948.jpg?e=1782500170&token=IAM-gs8ue1pDIGwtR1CR0Zjdagg7Q2tn5G_1BqTmhmqa:76341gInk03XfKtMTF1ezT48DTs=)

![img](https://changjiang-private-qn.yuketang.cn/slide/16314515/cover510_20260511075949.jpg?e=1782500170&token=IAM-gs8ue1pDIGwtR1CR0Zjdagg7Q2tn5G_1BqTmhmqa:CBFDEdn0A3nl6XElg-QJoIOoHoA=)

### 1. 这页比上一页多了什么？

上一页主要处理两种情况：

```
1. 顺序执行：PC<31:2> ← PC<31:2> + 1
2. 条件分支：PC<31:2> ← PC<31:2> + 1 + SignExt[imm16]
```

也就是处理：

```
普通指令
beq 条件转移指令
```

这一页又加了第三种情况：

```
3. 无条件跳转：j target
```

所以现在 PC 的来源有三种：

```
顺序地址
条件分支地址
无条件跳转地址
```

这就是“取指部件完整设计”的意思。

------

### 2. j target 指令是什么意思？

MIPS 里有一条跳转指令：

```
j target
```

它的作用是：

```
无条件跳转到 target 指定的位置
```

和 `beq` 不同：

```
beq：要比较 rs 和 rt，相等才跳
j：不用比较，直接跳
```

所以 `j` 指令不需要 ALU 判断 Zero，也不需要读寄存器。

------

### 3. j 指令的目标地址怎么算？

PPT 顶部写：

```
PC<31:2> ← PC<31:28> concat target<25:0>
```

这句话很关键。

意思是：

> **新的 PC 高 30 位，由当前 PC 的高 4 位 + 指令中的 26 位 target 拼接而成。**

因为这页仍然只计算 `PC<31:2>`，也就是 30 位指令地址。

完整指令地址最后还要拼上：

```
00
```

所以完整地址其实可以理解为：

```
Jump Address = PC<31:28> concat target<25:0> concat 00
```

------

### 4. 为什么 target 只有 26 位？

MIPS 的 `j` 指令格式大概是：

```
op | target
```

其中：

```
op：6 位
target：26 位
```

一条指令总共 32 位，所以：

```
32 - 6 = 26
```

因此 `j` 指令里只能放 26 位 target。

但是完整地址是 32 位，怎么办？

MIPS 利用了两个事实：

```
1. 指令地址最低两位永远是 00
2. 高 4 位从当前 PC 中取
```

所以最终拼成：

```
PC高4位 + target的26位 + 低2位00
```

刚好：

```
4 + 26 + 2 = 32 位
```

------

### 5. 为什么低两位是 00？

因为 MIPS 每条指令占 4 字节，指令地址一定是 4 的倍数。

比如：

```
1000
1004
1008
1012
```

这些地址的二进制最低两位都一定是：

```
00
```

所以 `j` 指令中不用保存低两位，取指时自动拼上 `00`。

------

### 6. 为什么要用 PC<31:28>？

因为 `target` 只有 26 位，再加低两位 `00`，只能形成 28 位地址。

```
target<25:0> concat 00 = 28 位
```

还差高 4 位。

所以高 4 位直接取当前 PC 的高 4 位：

```
PC<31:28>
```

这样就形成完整 32 位跳转地址：

```
PC<31:28> + target<25:0> + 00
```

这类跳转叫 **伪直接寻址**，因为它不是把完整 32 位地址都写进指令，而是用 PC 的高 4 位补齐。

------

### 7. 举个例子

假设当前 PC 是：

```
PC = 0x00400020
```

那么 PC 的高 4 位是：

```
PC<31:28> = 0x0
```

假设 `j` 指令中的 target 是：

```
target = 0x0010008
```

那么跳转地址高 30 位是：

```
PC<31:28> concat target<25:0>
```

完整地址再拼接低两位 `00`：

```
Jump Address = PC<31:28> concat target<25:0> concat 00
```

可以简单理解为：

```
target 左移 2 位，再补上 PC 的高 4 位
```

所以 `j` 指令本质上是：

```
跳到 target 对应的指令地址
```

------

### 8. 图里下面蓝色椭圆：Branch 分支逻辑

下面蓝色圈出来的是上一页讲过的条件分支逻辑。

它负责处理：

```
beq rs, rt, imm16
```

这部分包括：

```
PC<31:2> + 1
SignExt[imm16]
第二个 Adder
Branch 和 ZF
下面那个 MUX
```

它的逻辑是：

```
如果 Branch=1 且 ZF=1，则选择分支地址；
否则选择顺序地址。
```

也就是：

```
Branch成立：PC<31:2> + 1 + SignExt[imm16]
Branch不成立：PC<31:2> + 1
```

其中：

```
ZF = 1 表示 ALU 比较结果为 0，也就是 rs == rt
```

------

### 9. 图里上面蓝色椭圆：Jump 跳转逻辑

上面蓝色圈出来的是新增的 `j target` 跳转逻辑。

它取：

```
PC<31:28>
```

再取指令中的：

```
Instruction<25:0>
```

然后拼接成：

```
PC<31:28> concat Instruction<25:0>
```

这就是 `PC<31:2>` 的新值。

最后真正访问指令存储器时，再拼：

```
Addr<1:0> = 00
```

------

### 10. 两个 MUX 分别干什么？

图里有两个关键 MUX。

#### 第一个 MUX：选择顺序地址还是分支地址

下面那个 MUX 处理：

```
顺序执行 vs 条件分支
```

输入大概是：

```
0：PC<31:2> + 1
1：PC<31:2> + 1 + SignExt[imm16]
```

控制信号来自：

```
Branch AND ZF
```

如果分支成立，就选分支地址；否则选顺序地址。

------

#### 第二个 MUX：选择普通下一地址还是 Jump 地址

右边那个竖着的 MUX 处理：

```
普通下一地址 vs j 跳转地址
```

输入大概是：

```
0：上一层选出来的顺序/分支地址
1：j target 形成的跳转地址
```

控制信号是：

```
Jump
```

如果：

```
Jump = 1
```

说明当前指令是 `j target`，于是直接选择跳转地址。

如果：

```
Jump = 0
```

就走正常的顺序/分支逻辑。
