# 指令系统

> 一句话：指令系统(ISA)是软硬件的分界线。它定义了CPU能执行的所有指令及其编码格式。

## 一、指令的基本格式

一条机器指令包含：

| 字段 | 含义 | 说明 |
|------|------|------|
| **操作码(OP)** | 做什么操作 | ADD/SUB/LW/SW/BEQ等 |
| **地址码(A)** | 操作数在哪里 | 寄存器编号或内存地址 |

**指令长度**：定长(如MIPS固定32位)或变长(如x86从1到15字节)。

## 二、寻址方式

### 2.1 基本寻址方式

| 寻址方式 | 含义 | 例(MIPS) |
|---------|------|---------|
| 立即寻址 | 操作数在指令中 | `addi $t0, $zero, 100` |
| 寄存器寻址 | 操作数在寄存器中 | `add $t0, $t1, $t2` |
| 基址寻址 | EA = 基址寄存器 + 偏移量 | `lw $t0, 32($s3)` |
| PC相对寻址 | EA = PC + 偏移量 | `beq $t0, $t1, label` |

### 2.2 寻址方式对比

| . | 优点 | 缺点 |
|----|------|------|
| 立即寻址 | 快(不需访存) | 立即数范围受限 |
| 寄存器寻址 | 快、地址短 | 寄存器数量有限 |
| 基址寻址 | 灵活(数组/结构体) | 需计算有效地址 |
| PC相对寻址 | 位置无关代码 | 跳转范围受限 |

## 三、MIPS 指令集

### 3.1 MIPS 是典型的 RISC

- 32 个 32 位通用寄存器
- 指令定长 32 位
- Load/Store 架构：只有 lw/sw 访问内存
- 三种指令格式：R型、I型、J型

### 3.2 R型指令(寄存器-寄存器)

| op(6) | rs(5) | rt(5) | rd(5) | shamt(5) | funct(6) |
|-------|-------|-------|-------|----------|----------|

例：`add $t0, $t1, $t2` → op=0, funct=0x20

### 3.3 I型指令(立即数)

| op(6) | rs(5) | rt(5) | immediate(16) |
|-------|-------|-------|---------------|

例：`lw $t0, 32($s3)` → op=0x23, rs=$s3, rt=$t0, imm=32

### 3.4 J型指令(跳转)

| op(6) | address(26) |
|-------|-------------|

`j target` → op=0x02

## 四、RISC vs CISC

| 特征 | RISC | CISC |
|------|------|------|
| 指令长度 | **定长**(32位) | **变长**(1-15字节) |
| 指令数量 | 少(几十到一百多) | 多(几百) |
| 寻址方式 | 简单(几种) | 复杂(十几种) |
| 访存指令 | 仅Load/Store | 算术指令也可访存 |
| 代表 | MIPS、ARM、RISC-V | x86 |
| 实现 | 硬连线为主、易流水线 | 微程序为主 |

**现代趋势**：x86 内部将 CISC 指令"翻译"成 RISC 类微操作(μops)。

## 五、常用 MIPS 指令

| 指令 | 格式 | 功能 |
|------|------|------|
| `add $d, $s, $t` | R | $d = $s + $t |
| `sub $d, $s, $t` | R | $d = $s - $t |
| `lw $t, offset($s)` | I | $t = Mem[$s+offset] |
| `sw $t, offset($s)` | I | Mem[$s+offset] = $t |
| `beq $s, $t, label` | I | if $s==$t: PC+=offset<<2 |
| `j target` | J | PC = target |
| `addi $t, $s, imm` | I | $t = $s + imm |

## 六、过程调用约定

| 寄存器 | 用途 |
|--------|------|
| `$a0-$a3` | 传递参数 |
| `$v0-$v1` | 返回值 |
| `$ra` | 返回地址 |
| `$sp` | 栈指针 |
| `$fp` | 帧指针 |
| `$s0-$s7` | 被调用者保存 |
| `$t0-$t9` | 调用者保存 |

## 七、易错点

1. MIPS 的 `offset<<2`：16位立即数左移2位再符号扩展到32位
2. `beq/bne` 使用 PC 相对寻址，范围 ±128KB
3. RISC 必须 Load/Store 才能访存，不能 `add $t0, 100($s0)`
