# CPU 数据通路

> 一句话：数据通路是 CPU 中数据流动的"高速公路"。控制器产生信号，数据通路执行操作。

## 一、CPU 的基本组成

```
CPU = 数据通路(执行部件) + 控制器(控制部件)

数据通路：ALU、寄存器组、多路选择器(MUX)、总线
控制器：PC、IR(指令寄存器)、指令译码器、控制信号生成逻辑
```

## 二、数据通路的元件

### 2.1 组合逻辑元件(无状态)

- ALU：算数/逻辑运算
- MUX(多路选择器)：从多个输入中选一个
- 加法器：加法运算

### 2.2 时序逻辑元件(有状态，需时钟)

| 元件 | 说明 |
|------|------|
| **寄存器** | 在时钟沿采样输入并保持到下一个时钟沿 |
| **寄存器组** | 多个寄存器组成的阵列(如 32×32位) |
| **PC** | 程序计数器，指向当前指令地址 |

**寄存器组读写**：
- 读：组合逻辑，给出编号→输出数据(不需时钟)
- 写：时序逻辑，WE=1 且时钟沿→写入

## 三、单周期数据通路

### 3.1 设计原则

**一条指令在一个时钟周期内完成**。时钟周期 = 最慢指令的执行时间。

### 3.2 MIPS 单周期数据通路的关键部件

```
取指(IF)：PC→指令存储器→IR
译码(ID)：IR→控制单元→控制信号；寄存器组读出操作数
执行(EXE)：ALU 运算
访存(MEM)：数据存储器读写
写回(WB)：结果写入寄存器组
```

### 3.3 各类指令的数据通路

**R型指令(add/sub/and/or)**：
```
PC→取指→译码(读rs,rt)→ALU(运算)→写回rd
```

**I型访存指令(lw)**：
```
PC→取指→译码(读rs)→ALU(计算地址:rs+imm)→访存(读)→写回rt
```

**I型分支指令(beq)**：
```
PC→取指→译码(读rs,rt)→ALU(比较)→若相等:PC=PC+4+offset<<2
```

## 四、控制器设计

### 4.1 控制信号

| 信号 | 作用 |
|------|------|
| RegDst | 选择写回寄存器(rd/rt) |
| RegWrite | 写使能 |
| ALUSrc | ALU的第二个操作数来源(寄存器/立即数) |
| MemRead | 读存储器 |
| MemWrite | 写存储器 |
| MemtoReg | 写回数据来源(ALU结果/存储器) |
| Branch | 分支指令标志 |
| ALUOp | ALU操作类型 |

### 4.2 硬连线控制器

组合逻辑电路 → 指令操作码输入 → 控制信号输出。**速度快但不易修改**。

### 4.3 微程序控制器

控制信号存在 ROM 中(微程序)，逐条取出执行。**灵活性好但慢**。

## 五、异常与中断

### 5.1 概念
- **异常**：CPU内部产生的(如溢出、未定义指令)→同步
- **中断**：外部设备产生的(如键盘输入)→异步

### 5.2 处理流程
1. 保存当前 PC(存入 EPC 寄存器)
2. 记录异常原因(Cause 寄存器)
3. 跳转到异常处理程序入口
4. 处理完毕用 `eret` 返回

## 六、RTL 描述

RTL(Register Transfer Level)用类似编程的语言描述数据通路行为：

```
// add $t0, $t1, $t2 的 RTL 描述
取指: IR ← Mem[PC]; PC ← PC + 4
译码: A ← R[rs]; B ← R[rt]
执行: ALUOut ← A + B
写回: R[rd] ← ALUOut
```

## 七、易错点

1. **寄存器组读是组合逻辑**(不需要时钟)，**写是时序逻辑**(需要时钟沿)
2. **单周期：时钟周期 = 最慢指令的时间**，浪费资源
3. **lw 指令使用 ALU 计算地址**：$EA = R[rs] + sign\_extend(imm)$
4. **微程序慢但灵活，硬连线快但不灵活**
