# 运算方法与运算部件

> 一句话：ALU 是计算机的"计算心脏"。从加法器到乘法器，从定点运算到浮点运算，都基于最基本的逻辑门。

## 一、基本运算概述

C 语言中涉及的运算：
1. **算术运算**：+-*/ (无符号/带符号/浮点数)
2. **按位运算**：& | ~ ^
3. **逻辑运算**：&& || !
4. **移位运算**：<< >>

## 二、移位运算

| 类型 | 左移(<<) | 右移(>>) |
|------|---------|---------|
| 无符号数 | 低位补 0 | 高位补 0(逻辑右移) |
| 有符号数 | 低位补 0 | **高位补符号位**(算术右移) |

**算术右移 vs 逻辑右移**：
- 算术右移：高位补符号位 → 保持数值的正负(相当于 ÷2)
- 逻辑右移：高位补 0 → 针对无符号数

**例**：`1000 0000`(-128) 算术右移 1 位 = `1100 0000`(-64) ✓

## 三、加法器设计

### 3.1 半加器(Half Adder)
输入：A, B。输出：Sum = A⊕B, Carry = A·B

### 3.2 全加器(Full Adder)
输入：A, B, Cin。输出：Sum = A⊕B⊕Cin, Cout = AB + (A⊕B)Cin

### 3.3 行波进位加法器(Ripple Carry Adder)
n 个全加器串接。简单，但进位传播延迟大 → $O(n)$ 延迟。

### 3.4 超前进位加法器(Carry Look-ahead Adder)
并行计算进位。$G_i = A_i B_i$(生成), $P_i = A_i \oplus B_i$(传播)
$$C_{i+1} = G_i + P_i C_i$$

延迟 $O(\log n)$。

## 四、定点乘法

### 4.1 无符号乘法(移位加)

```
乘积初始 = 0
for i = n-1 downto 0:       // 从高位到低位
    乘积 <<= 1
    if 乘数第 i 位 = 1:
        乘积 += 被乘数
```

n 位 × n 位 → 2n 位结果。

### 4.2 Booth 算法(有符号乘法)

对连续的 1 进行"打包"处理：
- 从低位到高位扫描乘数
- `10`(1→0)：减被乘数
- `01`(0→1)：加被乘数
- `00` 或 `11`：不移

减少加/减法次数。

## 五、定点除法

### 5.1 恢复余数法

试商→若余数为负(不够减)→恢复余数→商0→移位→继续。

### 5.2 不恢复余数法(加减交替法)

余数为负时不恢复，直接移位后加(而不是减)。减少一次加法。

## 六、溢出判断

### 6.1 无符号数溢出
最高位产生进位(Cout=1)→溢出。

### 6.2 有符号数溢出(补码)

**三种判断方法**：

1. **进位判断**：符号位进位 Cf != 最高数值位进位 C → 溢出
   $$OF = C_f \oplus C$$

2. **操作数符号判断**：同号相加得异号 → 溢出

3. **双符号位法**：用 2 位符号位(00正, 11负)，结果两位不同→溢出

### 6.3 例题

8 位补码：`0111 1111`(127) + `0000 0001`(1)
= `1000 0000`(-128) → 正+正=负，OF=1 ✓(溢出！)

## 七、浮点运算

### 7.1 浮点加法步骤

1. **对阶**：小阶向大阶对齐(尾数右移)
2. **尾数相加减**
3. **规格化**：调整尾数和阶码使尾数满足 $1.xxx...$
4. **舍入**：超出精度位按舍入规则处理
5. **溢出检查**

### 7.2 舍入方式

| 方式 | 规则 |
|------|------|
| 截断 | 直接丢弃多余位 |
| 就近舍入(默认) | 舍入到最近的偶数 |
| 向上舍入 | 总是向 +∞ 方向 |
| 向下舍入 | 总是向 -∞ 方向 |

### 7.3 浮点乘法

$N_1 \times N_2 = (-1)^{S_1\oplus S_2} \times (M_1 \times M_2) \times 2^{E_1+E_2}$

- 阶码相加(注意减去多余的 Bias)
- 尾数相乘
- 规格化+舍入

## 八、ALU 的结构

```
A ──┐
    ├── MUX ──→ ALU(+-*/&|^) ──→ 结果
B ──┘          ↑
            控制信号(由指令译码产生)
```

ALU 接收两个操作数和控制信号，输出结果和标志位(ZF/CF/OF/SF)。

## 九、易错点

1. **算术右移 ≠ 逻辑右移**：有符号数必须用算术右移
2. **溢出判断**：OF 针对有符号数，CF 针对无符号数
3. **浮点加法先对阶再运算**，不能直接加尾数
4. **Booth 算法扫描是"从低到高"**，不是从高到低
