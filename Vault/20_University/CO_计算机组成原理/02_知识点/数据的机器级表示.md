# 数据的机器级表示

> 一句话：所有数据都是二进制。原码直观但加减麻烦；补码用模运算统一加减；IEEE 754 用科学计数法表示实数。

## 一、进位计数制与 BCD

- **基数 r**：每位可取 r 个不同数码
- **位权**：第 i 位的权重 = $r^i$
- **BCD 码**：用 4 位二进制表示 1 位十进制。如 15 = 0001 0101(BCD)，而不是 1111(纯二进制)

## 二、原码

**规则**：最高位符号位(0正1负)，其余绝对值。

n 位原码范围：$-(2^{n-1}-1) \sim +(2^{n-1}-1)$

**致命缺陷**：
1. 0 有两种表示(+0=00...0, -0=10...0)，不利于编程中的相等比较
2. 加减法必须区分符号位单独处理，硬件复杂

## 三、补码(核心中的核心)

### 3.1 模运算思想

时钟模型：模 12 系统中，-4 ≡ +8 (mod 12)。把减法变成加法的补数。

n 位运算器相当于模 $2^n$ 的算盘：**一个负数的补码 = 模 - |该负数|**

### 3.2 补码定义

$$[X]_{\text{补}} = 2^n + X \pmod{2^n} \quad (-2^n \le X < 2^n)$$

8 位补码范围：-128 ~ +127

### 3.3 重要特性

| 特性 | 说明 |
|------|------|
| **0 的唯一性** | 0 的补码只有 0000 0000(没有负零) |
| **不对称范围** | 负数比正数多一个：-128 有补码(1000 0000)，+128 没有 |
| **符号扩展** | 高位补符号位，真值不变：1111 1111(-1) → 1111 1111 1111 1111(-1) |

### 3.4 补码与真值的互转

**正数**：补码 = 真值的二进制(符号位为 0)
**负数**：真值 → 补码 = 绝对值按位取反+1

**补码转真值**：
- 符号位为 0 → 直接作为正数读
- 符号位为 1 → 按位取反+1，再加负号

**例**：`1111 0000` → 取反=`0000 1111` → +1=`0001 0000`=16 → 真值=-16

**求相反数**：连同符号位，**按位取反+1**。如 5(0000 0101)→-5(1111 1011)

### 3.5 补码加减法

**加法**：$[X+Y]_{\text{补}} = [X]_{\text{补}} + [Y]_{\text{补}} \pmod{2^n}$

**减法**：$[X-Y]_{\text{补}} = [X]_{\text{补}} + [-Y]_{\text{补}}$

**减法变加法**：减 Y = 加 Y 的相反数的补码！

这是补码的最大优点——**加减法用同一套硬件实现**。硬件只管按位加，不管参与的数原来是正是负。

例：计算 5-3
- 5 的补码：0000 0101
- -3 的补码：1111 1101
- 相加：0000 0101 + 1111 1101 = 1 0000 0010 → 丢弃进位 → 0000 0010 = 2 ✓

## 四、移码

在真值上加偏移量(通常 $2^{n-1}$)，将所有值映射为非负数。

**与补码的关系**：**移码 = 补码的符号位取反**。

**用途**：IEEE 754 浮点数的阶码。因为移码后，指数的自然顺序就是二进制数的自然顺序，便于硬件比较大小。

## 五、无符号整数与带符号整数

| 类型 | n 位范围 | C 类型 |
|------|---------|--------|
| 无符号 | $0 \sim 2^n-1$ | `unsigned char/short/int` |
| 带符号(补码) | $-2^{n-1} \sim 2^{n-1}-1$ | `char/short/int` |

**C 语言的隐式类型转换陷阱**：
- 有符号与无符号混合运算 → 隐式转为无符号
- `-1 < 0U` 在 C 中为假！因为 -1 转为无符号后 = 4294967295 > 0

## 六、位扩展与位截断

### 6.1 零扩展(无符号数)
短→长：高位补 0。`0x80`(8位) → `0x0080`(16位)，值不变。

### 6.2 符号扩展(有符号数)
短→长：高位补符号位。`0x80`(8位，=-128) → `0xFF80`(16位，=-128)，值不变。

### 6.3 位截断(长→短)
丢弃高位保留低位。**可能改变值**。

`int` 0x12345678 截为 `short` → 0x5678(高位丢弃)。

## 七、大小端

| 模式 | 低地址存 | 例：0x12345678 @ 地址 0x100 |
|------|---------|------|
| **小端(Little Endian)** | **低字节** | 0x100:78, 0x101:56, 0x102:34, 0x103:12 |
| **大端(Big Endian)** | **高字节** | 0x100:12, 0x101:34, 0x102:56, 0x103:78 |

**口诀**：小端→"小的在低地址"，大端→像人正常读数字(高位在前)。

x86/ARM(默认)是小端；网络字节序(如 IP 地址)是大端。

## 八、IEEE 754 浮点数

### 8.1 浮点数的科学计数法表示

$$N = (-1)^S \times 1.M \times 2^{E - \text{Bias}}$$

- S：符号位(1位)
- M：尾数(隐含了最高位的 1，称为"隐藏位")
- E：阶码(使用移码)

### 8.2 单精度(32位)

| S(1位) | E 阶码(8位) | M 尾数(23位) |
|--------|------------|-------------|

Bias = 127，$E = e + 127$

**例**：10.25 → 1010.01 → $1.01001 \times 2^3$

S=0, E=3+127=130=1000 0010, M=0100 1000 0000 0000 0000 000

IEEE 754 二进制：`0 10000010 01001000000000000000000`

### 8.3 双精度(64位)

| S(1位) | E 阶码(11位) | M 尾数(52位) |
|--------|-------------|-------------|

Bias = 1023

### 8.4 IEEE 754 特殊值

| 阶码 E | 尾数 M | 表示 |
|--------|--------|------|
| 全 0 | 全 0 | ±0 |
| 全 0 | 非全 0 | 非规格化数(极小值) |
| 全 1 | 全 0 | ±∞ |
| 全 1 | 非全 0 | NaN(Not a Number) |

### 8.5 浮点数的精度

| 精度 | 有效位数 | 范围 |
|------|---------|------|
| 单精度 | ~7 位十进制 | $1.17\times 10^{-38} \sim 3.40\times 10^{38}$ |
| 双精度 | ~16 位十进制 | $2.23\times 10^{-308} \sim 1.80\times 10^{308}$ |

## 九、数据对齐

| 类型 | 大小 | 对齐要求 |
|------|------|---------|
| char | 1 | 任意地址 |
| short | 2 | 2 的倍数 |
| int/float | 4 | 4 的倍数 |
| double/long long | 8 | 8 的倍数 |

**不对齐的代价**：可能需要两次访存 + 拼接，MIPS 直接报异常。

## 十、易错点

1. **补码转真值**：符号位=1 时按位取反+1，再加负号
2. **-128 有补码 1000 0000，但 +128 没有**(8 位)
3. **补码符号扩展后真值不变**，位截断可能改变值
4. **IEEE 754 阶码 = 真值+Bias**，不是真值本身
5. **隐藏位**：规格化尾数隐含一个 1，不存储
