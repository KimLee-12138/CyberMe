# 计算机系统概述

> 一句话：冯·诺依曼结构五大部件 + 性能评价体系（CPI/MIPS/Amdahl）。

## 一、冯·诺依曼结构

### 1.1 五大部件

| 部件 | 功能 | 硬件实例 |
|------|------|---------|
| 运算器 | 算术/逻辑运算 | ALU、通用寄存器、状态寄存器 |
| 控制器 | 取指、译码、产生控制信号 | PC、IR、控制单元(CU) |
| 存储器 | 存放程序和数据 | 主存(DRAM)、辅存(SSD/HDD) |
| 输入设备 | 将外部数据送入计算机 | 键盘、鼠标、网卡 |
| 输出设备 | 将结果送出计算机 | 显示器、打印机、网卡 |

**核心思想（存储程序原理）**：程序和数据以二进制形式存放在同一存储器中，CPU 逐条取出指令并执行。这是冯·诺依曼结构的精髓。

### 1.2 冯·诺依曼 vs 哈佛结构

| 维度 | 冯·诺依曼 | 哈佛结构 |
|------|----------|---------|
| 指令和数据存储 | 同一存储器 + 同一总线 | 分开的指令存储器/数据存储器 + 分开的总线 |
| 取指与取数 | 不能同时进行（总线冲突） | 可同时取指和取数 |
| 瓶颈 | "冯·诺依曼瓶颈"（总线争用） | 硬件复杂度高 |
| 典型应用 | PC、服务器 | DSP、单片机(如 ARM Cortex-M) |

**现代计算机**：外部统一总线(冯·诺依曼)，CPU 内部有独立的指令 Cache 和数据 Cache(类似哈佛)。

## 二、计算机性能指标

### 2.1 核心公式

$$\text{CPU执行时间} = \text{指令数} \times \text{CPI} \times \text{时钟周期} = \frac{\text{指令数} \times \text{CPI}}{f}$$

| 符号 | 含义 | 说明 |
|------|------|------|
| $f$ | 时钟频率 | $f = 1/T$，单位 Hz |
| $T$ | 时钟周期 | 一次时钟脉冲的时间 |
| CPI | Cycles Per Instruction | 每条指令平均所需时钟周期数 |
| IC | Instruction Count | 程序的总指令条数 |

### 2.2 CPI 的加权平均

$$\text{CPI} = \sum_{i=1}^{n} (\text{CPI}_i \times \frac{\text{指令数}_i}{\text{总指令数}})$$

**例**：某程序有 A 类指令(CPI=1)占 40%，B 类(CPI=2)占 40%，C 类(CPI=4)占 20%：
$$\text{CPI} = 1 \times 0.4 + 2 \times 0.4 + 4 \times 0.2 = 2.0$$

### 2.3 MIPS 和 MFLOPS

$$\text{MIPS} = \frac{f}{\text{CPI} \times 10^6}$$

$$\text{MFLOPS} = \frac{\text{浮点运算次数}}{\text{时间(秒)} \times 10^6}$$

**MIPS 的局限性**：
- 不能跨 ISA 比较（RISC vs CISC 完成相同工作需要不同数量的指令）
- 同一程序在不同编译器优化下 MIPS 可能不同
- 浮点密集型程序可能 MIPS 低但实际性能好

## 三、Amdahl 定律

### 3.1 公式

若改进部分占比为 $f$，该部分加速 $S$ 倍，则系统总加速比：

$$\text{Speedup}_{\text{overall}} = \frac{1}{(1-f) + \frac{f}{S}}$$

### 3.2 重要推论

**即使对 90% 的部分无限加速($S\to\infty$)，整体最多快 10 倍。**

**实践教训**：优化必须针对**瓶颈**——占执行时间最大的部分。

### 3.3 例

某程序 80% 可并行化（$f$=0.8），在 4 核上并行（$S$=4）：
$$\text{Speedup} = \frac{1}{(1-0.8) + 0.8/4} = \frac{1}{0.4} = 2.5$$

## 四、计算机系统的层次结构

```
第5层：高级语言 (C/Java/Python)
         ↓ 编译
第4层：汇编语言
         ↓ 汇编
第3层：操作系统
         ↓
第2层：指令集体系结构(ISA) ← 软硬件分界线
         ↓ 微程序/硬连线
第1层：数字逻辑电路
```

**ISA(指令集体系结构)** 是软硬件之间的"合同"。软件不需要知道下层如何实现。这就是为什么 Intel CPU 的底层微架构代代不同，但所有 x86 软件都能运行。

## 五、易错点

1. **CPI 是加权平均，不是简单平均**
2. **MIPS 不能跨 ISA 比较**（CISC 一条指令可能顶 RISC 好几条）
3. **Amdahl 定律：只改善非瓶颈部分，整体几乎不提升**
4. **时钟频率高 ≠ 性能好**：无法忽视 CPI(Pentium 4 的高频低能就是教训)
