# 输入输出系统

> 一句话：I/O 系统是计算机与外部世界交互的桥梁。三种控制方式——程序查询、中断、DMA——速度递增、CPU参与递减。

## 一、I/O 接口

### 1.1 功能
- 数据缓冲(速度匹配)
- 信号转换(电平/格式)
- 地址译码(选中设备)
- 提供状态信息

### 1.2 I/O 端口编址

| 方式 | 说明 | 代表 |
|------|------|------|
| **独立编址** | I/O 端口有独立地址空间 | x86(in/out指令) |
| **统一编址** | I/O 端口映射到内存地址空间 | ARM、MIPS(Load/Store访问) |

## 二、三种 I/O 控制方式

### 2.1 程序查询(轮询)

```
while (设备状态 != 就绪):
    等待
传送数据
```

**特点**：CPU 全程参与，**CPU 利用率极低**("忙等"或称自旋)。适合简单低速设备。

### 2.2 中断方式

CPU 启动 I/O 后继续执行其他程序。I/O 完成时设备发出中断信号。

**流程**：
1. 设备完成 I/O → 发中断
2. CPU 响应 → 保存现场 → 执行中断服务程序
3. 中断返回 → 继续原程序

**优势**：CPU 与 I/O 可并行工作。

### 2.3 DMA(直接存储器访问)

**硬件(DMA控制器)直接完成设备与主存的数据传送，CPU 完全不参与数据搬运。**

**DMA 工作流程**：
1. CPU 初始化 DMA 控制器(源/目的地址、传送字节数)
2. DMA 控制器接管总线，直接传送数据
3. 传送完毕，DMA 发中断通知 CPU

**DMA 传送模式**：
- **突发模式**：连续传送一块数据(CPU 等待)
- **周期窃取**：在 CPU 不使用总线时插空传送

## 三、中断的详细处理

### 3.1 中断类型

| 类型 | 来源 | 例子 |
|------|------|------|
| **可屏蔽中断** | 外部设备 | 键盘、鼠标 |
| **非屏蔽中断** | 严重硬件故障 | 电源掉电 |
| **异常** | CPU内部 | 除法溢出、缺页 |
| **软中断** | 程序主动 | `int 0x80` |

### 3.2 中断向量表

存放每个中断号对应的处理程序入口地址。

### 3.3 中断优先级

多个中断同时到达时，CPU 先响应优先级高的。高优先级中断可以**嵌套**打断低优先级的中断处理。

## 四、总线

### 4.1 总线类型

| 总线 | 功能 |
|------|------|
| 数据总线 | 传送数据(双向) |
| 地址总线 | 发出内存/I/O地址(单向) |
| 控制总线 | 发送控制信号 |

### 4.2 总线仲裁

| 方式 | 说明 |
|------|------|
| 菊花链 | 设备串行连接，优先级从近到远 |
| 集中式仲裁 | 仲裁器决定谁使用总线 |
| 分布式仲裁 | 各设备自行判断 |

### 4.3 现代总线标准

- **PCIe**：串行点对点，取代并行 PCI
- **USB**：通用串行总线
- **SATA**：硬盘接口

## 五、易错点

1. **DMA 接管总线时 CPU 不能使用总线**(周期窃取时交替)
2. **中断响应过程保存的是 PC 和状态寄存器**(不是所有寄存器，那由中断服务程序来做)
3. **独立编址 vs 统一编址**：x86用in/out，ARM/MIPS用Load/Store
