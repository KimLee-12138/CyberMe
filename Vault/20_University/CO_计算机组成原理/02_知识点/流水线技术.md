# 流水线技术

> 一句话：流水线把指令执行分成多个阶段并行处理，像工厂流水线一样提高吞吐率。

## 一、流水线的基本概念

### 1.1 单周期 vs 流水线

| . | 单周期 | 流水线 |
|----|--------|--------|
| 完成一条指令 | 1 个长周期 | 5 个短周期 |
| CPI | 1 | 理想：1(实际>1) |
| 时钟周期 | 由最慢指令决定 | 由最慢阶段决定(更短) |
| 吞吐率 | 低 | 高 |

### 1.2 MIPS 五级流水

```
IF(取指) → ID(译码) → EXE(执行) → MEM(访存) → WB(写回)
```

各阶段之间用**流水线寄存器**隔开，存储中间结果。

## 二、流水线冒险

### 2.1 结构冒险

多个指令同时使用同一硬件资源。

**解决**：增加硬件(如分离指令Cache和数据Cache)。

### 2.2 数据冒险

后一条指令依赖前一条指令的结果。

**三种情况**：
```
add $t0, $t1, $t2    // $t0 = $t1+$t2
sub $t3, $t0, $t4    // 需要 $t0！但 add 还没写回
```

**解决**：
1. **转发(Forwarding/Bypassing)**：ALU 结果产生后直接转发给下一条指令的 ALU 输入，不等写回寄存器
2. **阻塞(Stall/气泡)**：插入 NOP，等待数据就绪

### 2.3 控制冒险

分支指令跳转时，后续已经进入流水线的指令可能需要被"冲刷"。

**解决**：
1. **假设分支不跳转**：继续取 PC+4
2. **延迟分支**：分支后的指令无论如何都执行
3. **分支预测**：动态预测跳转方向

## 三、流水线性能

### 3.1 加速比

理想加速比 = 流水线段数

$$\text{Speedup} = \frac{\text{非流水线时间}}{\text{流水线时间}}$$

**实际加速比 < 级数**：流水线寄存器开销、冒险导致的阻塞。

### 3.2 吞吐率

$$\text{吞吐率} = \frac{\text{指令数}}{\text{总时间}}$$

理想流水线：每个时钟周期完成一条指令。

## 四、流水线寄存器

| 寄存器 | 位置 | 存储内容 |
|--------|------|---------|
| IF/ID | IF→ID | 指令、PC+4 |
| ID/EXE | ID→EXE | 控制信号、操作数、立即数 |
| EXE/MEM | EXE→MEM | ALU结果、写数据、分支目标 |
| MEM/WB | MEM→WB | 读出的数据、ALU结果 |

## 五、数据转发的详细分析

```
add $1, $2, $3     IF ID EXE MEM WB
sub $4, $1, $5        IF ID  EXE MEM WB
```
sub 在 EXE 阶段需要 $1，但 add 在 EXE 阶段就产生了 $1 的结果(比 WB 早两个阶段)。

**转发的关键**：结果一旦在 EXE/MEM 或 MEM/WB 流水线寄存器中可用，立即转发，不等 WB。

**仍需阻塞的情况**：lw 后立即使用。

```
lw $1, 0($2)      IF ID EXE MEM WB
add $3, $1, $4       IF ID  EXE MEM WB
```
lw 的结果在 MEM 阶段才可用，而 add 在 EXE 就需要。必须**阻塞一个周期**。

## 六、易错点

1. **转发不能解决所有数据冒险**：lw 后立即使用仍需阻塞
2. **流水线提高了吞吐率，不是单条指令的执行时间**：单条指令时间不变甚至更慢(流水线寄存器延迟)
3. **流水线级数越多，冒险越复杂**：Pentium 4 的 20+ 级深流水线在分支预测失败时代价巨大
