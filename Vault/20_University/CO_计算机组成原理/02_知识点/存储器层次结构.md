# 存储器层次结构

> 一句话：利用局部性原理，用多层存储结构实现"速度接近最快、容量接近最大、价格接近最便宜"的理想存储器。

## 一、存储器的层次

```
CPU寄存器(L0) ← 最快/最小/最贵
    ↓
Cache(L1/L2/L3) ← SRAM
    ↓
主存(DRAM) ← 中等
    ↓
磁盘/SSD ← 最慢/最大/最便宜
```

**局部性原理**：
- **时间局部性**：刚访问的数据很可能再次访问(循环)
- **空间局部性**：刚访问的数据附近的数据很可能被访问(顺序执行)

## 二、SRAM vs DRAM

| . | SRAM | DRAM |
|----|------|------|
| 存储单元 | 6 个晶体管 | 1 个晶体管+1 个电容 |
| 速度 | 快(几ns) | 慢(几十ns) |
| 容量 | 小 | 大 |
| 功耗 | 低(静态) | 需刷新 |
| 用途 | Cache | 主存 |

## 三、Cache 基本原理

### 3.1 映射方式

| 方式 | 规则 | 优点 | 缺点 |
|------|------|------|------|
| **直接映射** | 每个主存块只能放在唯一 Cache 行 | 简单、快 | 冲突多 |
| **全相联** | 可放任意 Cache 行 | 无冲突 | 比较电路复杂 |
| **组相联** | 主存块可放在特定组内任意行 | 折中 | 折中 |

**组相联**最常用。标记(Tag)用于识别块的身份。

### 3.2 Cache 地址分解

$$\text{地址} = \text{Tag} | \text{Index} | \text{Offset}$$

- **Offset**：块内偏移(Bytes)
- **Index**：行(组)号
- **Tag**：剩余高位 = 标记

### 3.3 替换策略

| 策略 | 规则 |
|------|------|
| LRU | 淘汰最久未使用的(最优) |
| FIFO | 淘汰最早进入的 |
| Random | 随机淘汰 |

### 3.4 写策略

| 策略 | 写操作 | 优点 | 缺点 |
|------|--------|------|------|
| **写直达(Write-through)** | 同时写 Cache 和主存 | 一致性好 | 慢 |
| **写回(Write-back)** | 只写 Cache，替换时写回 | 快 | 需脏位 |

**写不命中时**：
- **写分配(Write-allocate)**：先读入 Cache 再写
- **写不分配(No-write-allocate)**：直接写主存

典型组合：写回+写分配；写直达+写不分配

## 四、Cache 性能

$$\text{平均访存时间} = \text{命中时间} + \text{缺失率} \times \text{缺失代价}$$

$$\text{CPU时间} = \text{执行周期数} + \text{访存阻塞周期数}$$

**缺失分类**（3C）：
1. **Compulsory(必然缺失)**：首次访问任何块都缺失
2. **Capacity(容量缺失)**：Cache 装不下所有需要的数据
3. **Conflict(冲突缺失)**：不同块映射到同一 Cache 行

## 五、虚拟存储器

### 5.1 概念
给每个程序一个独立的、连续的虚拟地址空间。通过页表映射到物理地址。

### 5.2 分页

- **虚拟页号(VPN)** → **物理页帧号(PPN)**
- **页大小**：通常 4KB
- **页表**：存在主存中；每个进程一张页表

### 5.3 TLB(快表)

页表的 Cache。避免每次访存都查页表(查两次：页表+实际数据)。

TLB 命中 → 直接得到物理地址 → 一次访存
TLB 缺失 → 查页表 → 可能需要两次访存

### 5.4 虚实地址转换

$$\text{虚拟地址} = \text{VPN} | \text{offset}$$

$$\downarrow \text{TLB/Page Table}$$

$$\text{物理地址} = \text{PPN} | \text{offset}$$

## 六、存储层次总结

| 层次 | 典型大小 | 延迟 |
|------|---------|------|
| L1 Cache | 32KB | ~1ns |
| L2 Cache | 256KB | ~5ns |
| L3 Cache | 8MB | ~15ns |
| 主存(DRAM) | 8GB | ~50ns |
| SSD | 512GB | ~100μs |
| HDD | 2TB | ~10ms |

## 七、易错点

1. **Cache 映射中 Index 决定行号，Tag 决定是否命中**
2. **写回**：替换脏块时必须先写回主存
3. **TLB 本质是"页表的 Cache"**
4. **虚拟地址到物理地址的转换发生在 Cache 访问之前**(物理Cache)或之后(虚拟Cache)
