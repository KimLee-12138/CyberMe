# 集合框架

## 一句话解释
Java集合框架提供统一的数据结构接口。Collection(单列)→List/Set/Queue，Map(双列)→HashMap/TreeMap。ArrayList数组实现(查快)，LinkedList链表实现(增删快)，HashMap(数组+链表+红黑树, JDK8+)O(1)查找。

## 核心原理
**Collection接口体系**：
- List(有序可重复)：ArrayList(O(1)随机访问/增删O(n))、LinkedList(双向链表/增删O(1))、Vector(线程安全版ArrayList，已过时)
- Set(无序不重复)：HashSet(基于HashMap，O(1))、LinkedHashSet(保持插入序)、TreeSet(红黑树，O(log n)，元素排序)
- Queue/Deque：LinkedList可作队列、PriorityQueue(堆实现，O(log n))

**HashMap原理(JDK 8+)**：hash(key)→桶位置→链表(≤8个节点)/红黑树(>8个节点)。负载因子默认0.75，超过则扩容2倍(rehash)。**key必须正确实现hashCode()和equals()**——hashCode决定桶位置，equals判断是否相同。

**fail-fast机制**：迭代中修改集合(非迭代器方式)→ConcurrentModificationException。解决：用迭代器的remove()或并发集合(ConcurrentHashMap)。

## 完整例子
```java
Map<String, Integer> map = new HashMap<>(); map.put("A",1);
List<String> list = new ArrayList<>(Arrays.asList("A","B","C"));
// 遍历删除的正确方式
Iterator<String> it = list.iterator();
while(it.hasNext()){ if(it.next().equals("B")) it.remove(); }
// Java 8+ removeIf
list.removeIf(s -> s.equals("B"));
```

## 容易出错的地方
- 不要在for-each中直接remove→ConcurrentModificationException
- HashMap的key必须重写equals()和hashCode()且两者一致
- TreeSet/TreeMap要求元素实现Comparable或提供Comparator

- ArrayList默认容量10，1.5倍扩容
