# 面向对象编程

## 一句话解释
Java OOP三大特性：封装(private+getter/setter隐藏内部状态)、继承(extends单继承)、多态(父类引用指向子类对象，运行时动态绑定方法)。抽象类vs接口：抽象类可有构造器和实现，接口支持多实现(Java 8+可有default方法)。

## 核心原理
**封装**：访问修饰符控制可见性。private(仅本类)→default(同包)→protected(同包+子类)→public(所有类)。封装的好处：隐藏实现细节、提高安全性、方便修改内部实现而不影响调用者。

**继承**：子类extends父类，获得所有非private成员。Java单继承(一个类只能有一个直接父类)，但可实现多个接口。super()调用父类构造器，必须在子类构造器第一行。

**多态=动态绑定**：编译时类型vs运行时类型。Animal a=new Dog(); a.sound()调用Dog.sound()——JVM在运行时根据实际类型决定调用哪个方法。静态方法、字段访问由编译时类型决定(不是多态)。

**抽象类vs接口**：抽象类可有状态(字段)和构造器，适合"is-a"关系。接口定义能力(如Comparable, Runnable)，支持多实现。Java 8+接口可有default方法和static方法。

## 完整例子
```java
class Animal { void sound(){ System.out.println("..."); } }
class Dog extends Animal { @Override void sound(){ System.out.println("Woof"); } }
Animal a = new Dog(); a.sound(); // "Woof"—运行时类型决定
// 工厂模式中多态的应用
Animal create(String type) { return type.equals("dog")?new Dog():new Cat(); }
```

## 容易出错的地方
- Java是值传递(对象引用也是值传递——传引用的副本)
- static方法不能被重写(只能隐藏)。@Override可帮助编译器检查
- 子类构造器默认调用super()且必须在第一行
- final类不能被继承，final方法不能被重写
