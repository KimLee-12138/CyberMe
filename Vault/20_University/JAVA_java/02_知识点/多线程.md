# 多线程

## 一句话解释
Java线程是独立执行路径。创建方式：继承Thread(不推荐，占用单继承名额)、实现Runnable(推荐)、Callable+Future(有返回值)。synchronized保证互斥访问，wait/notify实现线程通信。volatile保证可见性但不保证原子性。

## 核心原理
**线程生命周期**：NEW→RUNNABLE→BLOCKED/WAITING/TIMED_WAITING→TERMINATED。start()启动线程(run()只是普通方法调用)。

**synchronized**：获取对象的内置锁(monitor)。同步方法锁this(实例)或Class对象(静态)。同步块更灵活：synchronized(lock){...}。**是可重入锁**(同一线程可多次获取)。

**wait/notify**：必须在synchronized块内调用。wait()释放锁进入等待；notify()唤醒一个；notifyAll()唤醒所有。**用while循环检查条件**(防止虚假唤醒)。

**volatile**：保证可见性(一个线程修改后其他线程立即可见)+禁止指令重排。但不保证原子性(i++不是原子操作)。

**JUC(java.util.concurrent)**：线程池(Executors)、并发集合(ConcurrentHashMap)、原子类(AtomicInteger使用CAS)、锁(Lock/ReentrantLock支持tryLock超时)。

## 完整例子
```java
class Counter { private int c=0; public synchronized void inc(){ c++; } }
// 生产者-消费者
synchronized(lock){ while(buffer.isEmpty()) lock.wait(); buffer.take(); lock.notifyAll(); }
// 线程池
ExecutorService pool = Executors.newFixedThreadPool(10);
pool.submit(() -> { /* task */ });
pool.shutdown();
```

## 容易出错的地方
- 死锁：两线程互相等待对方的锁→避免嵌套锁、用tryLock(timeout)
- wait()用while而非if检查条件(防止虚假唤醒)
- sleep()不释放锁，wait()释放锁
- stop()/suspend()已废弃(不安全)。用interrupt()+isInterrupted()配合
