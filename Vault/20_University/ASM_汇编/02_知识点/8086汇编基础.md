# 8086 汇编基础

> 一句话：理解 CPU 如何通过总线与内存交互、如何用段地址+偏移地址定位 1MB 空间。

## 一、总线

### 1.1 三类总线

| 总线类型 | 作用 | 8086 规格 | 决定因素 |
|---------|------|----------|---------|
| **地址总线** | CPU 通过它发出要访问的内存单元地址 | 20 根 | 寻址能力 = $2^{20}$ = 1MB |
| **数据总线** | CPU 与内存/外设之间的数据传输通道 | 16 根 | 一次传 16 位数据 |
| **控制总线** | CPU 对外部器件的控制信号 | 若干根 | 控制能力 |

**关键公式**：$n$ 根地址总线 → 寻址能力 = $2^n$ 个存储单元。

### 1.2 内存地址空间

8086 的内存地址空间分配（1MB = 00000H ~ FFFFFH）：

| 地址范围 | 用途 |
|---------|------|
| 00000H ~ 9FFFFH | 主存储器（RAM） |
| A0000H ~ BFFFFH | 显存 |
| C0000H ~ FFFFFH | ROM（BIOS） |

## 二、物理地址计算

### 2.1 核心问题

8086 内部寄存器只有 16 位（寻址 64KB），但需要输出 20 位地址（寻址 1MB）。解决方案：

**物理地址 = 段地址 × 16 + 偏移地址**

即：**段地址左移 4 位 + 偏移地址**。

```
例：段地址 = 1230H，偏移地址 = 00C8H
物理地址 = 1230H × 10H + 00C8H = 12300H + 00C8H = 123C8H
```

### 2.2 段的概念

- **段是逻辑概念**，不是物理分区。内存没有"隔板"。
- 段地址 ×16 必然是 16 的倍数 → 每个段起始地址对齐到 16 字节
- 偏移地址为 16 位 → 一个段最大 64KB
- 同一个物理地址可用多种段:偏移表示（如 `0000:0010` = `0001:0000`）

### 2.3 四个段寄存器

| 段寄存器 | 用途 | 默认配合的偏移寄存器 |
|---------|------|-------------------|
| **CS** | 代码段 | IP（指令指针） |
| **DS** | 数据段 | BX, SI, DI, 立即数 |
| **SS** | 堆栈段 | SP, BP |
| **ES** | 附加段 | DI（串操作目的地址） |

## 三、CPU 执行指令过程

1. **取指令**：CPU 从 CS:IP 指向的内存单元读取指令 → 进入指令缓冲器
2. **IP 更新**：IP = IP + 当前指令长度（**发生在读取后、执行前**）
3. **执行指令**：执行指令缓冲器中的指令
4. **重复**：回到步骤 1

**注意**：执行指令时 IP 已经指向下一条指令的地址了。

## 四、修改 CS:IP

**不能用 `mov` 指令修改 CS 或 IP！** 必须通过转移指令：

| 指令 | 修改对象 |
|------|---------|
| `jmp 段地址:偏移地址` | 同时修改 CS 和 IP |
| `jmp 寄存器` | 只修改 IP |
| `jmp short 标号` | 段内短转移，修改 IP |
| `call` / `ret` | 修改 CS:IP + 栈操作 |
| `int` / `iret` | 修改 CS:IP + 标志寄存器 |

## 五、易错点

1. **物理地址计算**：段地址 × 16（十进制） = 段地址 × 10H（十六进制） = 段地址 << 4
2. **不能用 mov 改 CS**：CS 是代码段的"方向盘"，必须由转移指令控制
3. **IP 更新的时机**：在取指令后、执行前，不是执行后
4. **一个物理地址可以对应多个段:偏移**：如 `0000:0010` 和 `0001:0000` 指向同一位置
