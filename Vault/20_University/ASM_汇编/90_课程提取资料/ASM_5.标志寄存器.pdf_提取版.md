## 第1页

5.标志寄存器  
常用标志位  
第一阵营：状态标志（ ALU 的运算汇报报告）  
这 6 个标志位是被动的。每当  CPU 内部的算术逻辑单元（ ALU ）执行完一次算术运算（如  ADD, SUB）
或逻辑运算（如  AND, OR）后，硬件会自动根据计算结果修改这些位。
1. ZF (Zero Flag - 零标志位，第  6 位 ) —— “ 结果是不是  0 ？ ”
这是最常用的标志位。如果刚刚执行的指令结果为  0，那么  ZF = 1（指示灯亮起，表示 “ 是的，
结果是零 ” ）。
如果结果不为  0，则  ZF = 0。
用途：  经常配合  CMP（比较）指令使用。比如判断两个数是否相等，就是让它们相减，如果结果
为 0（ZF=1），就说明它们相等。
2. SF (Sign Flag - 符号标志位，第  7 位 ) —— “ 结果是负数吗？ ”
它直接记录了运算结果的最高位（对于  8 位数据就是第  7 位）。
如果最高位是  1，说明结果是个负数，此时  SF = 1。如果最高位是  0（正数），则  SF = 0。
3. PF (Parity Flag - 奇偶标志位，第  2 位 ) —— “ 底层校验员 ”
它统计运算结果的 最低  8 位中，数字  1 的个数。
如果  1 的个数是偶数， PF = 1。如果是奇数， PF = 0。
用途：  早期常用于底层通信的数据校验，现在的高级编程中很少直接用到。
4. CF (Carry Flag - 进位标志位，第  0 位 ) —— “ 无符号数溢出报警 ”
当进行无符号数 运算时，如果最高位向更高位产生了进位（加法），或者向更高位借了位（减
法），CF = 1。
例子：  8 位寄存器最大能装  255 （FFH）。如果你算  255 + 1，结果变成了  0，此时进位丢失
了。 CPU 就会把  CF 置为  1，提醒你： “ 大哥，无符号数装不下了，溢出啦！ ”
5. OF (Overflow Flag - 溢出标志位，第  11 位 ) —— “ 有符号数溢出报警 ”

## 第2页

标志位 功能
ZF（ zero ）零标志位 结果为 0是 zf 为 1 ；结果为 1 时 zf 为 0
PF（ parity ）奇偶标志位结果中所有二进制位中 1 的个数为偶数时， pf 为 1 ；为
奇数时， pf 为 0
SF（ symbol ）符号标志位（对 有符号数
来说有意义）结果为负时， sf 为 1 ；结果为正时， sf 为 0
CF（ cin）进位标志位（对 无符号数运算
有意义）运算结果的最高有效位向更高位的进位，或从更高位
的借位时， CF 为 1
8位有符号数的范围为 0-255
16位为 0~65535 （数据最好记住）
OF（ overflow ）溢出标志位（对 有符号
数有意义）当运算结果上溢或者下溢时， OF 为 1
8位有符号数的范围为 -128-127
16位为 -32768~32767这是最容易和  CF 搞混的！ OF 专门针对 有符号数 （带有正负号的数）。
例子：  8 位有符号数的范围是  -128 到  +127 。如果你算  100 + 100（正数加正数），按理说是  
200，但  8 位有符号数装不下  200 ，结果会变成一个负数！这种 “ 正 + 正 = 负 ” 或 “ 负 + 负 = 正 ” 的荒谬现
象，就是有符号数溢出，此时  OF = 1。
(注：第  4 位的  AF 是辅助进位标志，主要针对  BCD 码运算，现代编程中极少涉及，作为了解即可。 )
第二阵营：控制标志（程序员发出的指令）  
这 3 个标志位是主动的。它们不会因为普通的数学运算而改变，而是需要程序员用特定的指令去设置它
们，从而改变  CPU 的工作模式。
1. DF (Direction Flag - 方向标志位，第  10 位 ) —— “ 内存搬运方向盘 ” 还记得我们上一节讨论  SI 和 
DI 双指针搬运字符串的代码吗？当时我们每搬一次，都要手动写  add si, 2 和 add di, 2。
如果结合串传送指令（如  MOVSB）， CPU 会自动帮我们增加或减少  SI 和 DI。到底增还是减，就
是 DF 决定的！
通过  CLD 指令让  DF = 0：指针自动 递增（从低地址往高地址搬运，最常用）。
通过  STD 指令让  DF = 1：指针自动 递减（从高地址往低地址倒着搬运）。
2. IF (Interrupt Enable Flag - 中断允许标志位，第  9 位 )
当 IF = 1 时， CPU 允许响应外部的 “ 可屏蔽中断 ” （比如你敲击了键盘）。
当 IF = 0 时， CPU 就会 “ 屏蔽 ” 这些外部打扰，专心执行当前代码。可以使用  STI 和 CLI 指令来
开关它。
3. TF (Trap Flag - 陷阱标志位，第  8 位 )
专门为调试程序（ Debug ）设计的。如果  TF = 1，CPU 每执行完一条指令，就会自动停下来，触
发一个单步中断。这就是你用  Debug 工具可以 “ 单步执行（ Trace ） ” 查看寄存器状态的底层硬件原
理。

## 第4页

DF=0:每次操作后 si,di 递增 ;
DF=1:每次操作后 si,di 递减。
1.串传送字节
格式 1:movsb
功能 :(以字节为单位传送 )
(1)((es) × 16 +(di)) =  ((ds) × 16 + ((si))
(2)如果 DF = 0 则 :(si)=(si) +1
    (di) = (di) + 1
如果 DF=0 则 :(si)=(si)-1
      (di) = (di) - 1
2.串传送字 movsw
格式 2:movsw
功能 :(以字为单位传送 )
将ds:si指向的内存字单元中 word 送入 es:di 中 , 然后根据标志寄存器 DF 位的值 , 将 si 和 di 递增 2 或递减 2 。
movsb和 movsw 进行的是串传送操作中的个步骤 , 一般来说 ,movsb 和 movsw 都和 rep 配合使用 , 格式如下 :
rep movsb
用汇编语法来描述 repmovsb 的功能就是 :
s:movsb
loop s
rep movsw
用汇编语法来描述 repmovsw 的功能就是 :
s: movsw
loop s
可见 ,rep 的作用是根据 cx 的值 , 重复执行后面的串传送指令。
由于每执行一次 movsb 指令 si 和 di 都会递增或递减指向后一个单元或前个单元 , 则 repmovsb 就可以
循环实现 (cx) 个字符的传送。书
使用串传送指令进行数据的传送 , 需要给它提供一些必要的信息 , 它们是 :
1. 传送的原始位置： ds:si
2. 传送的目的位置： es:di
3. 传送的长度 :cx
4. 传送的方向 :DF 。
