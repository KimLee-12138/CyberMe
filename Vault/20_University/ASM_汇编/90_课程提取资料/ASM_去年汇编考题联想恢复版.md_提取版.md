## 第 1 题：8086 寻址方式与物理地址计算

### 原题可能表述

8086 CPU 有哪些常见的数据寻址方式？各有什么特点？请结合指令说明。若给定 `DS`、`SS`、`BP` 的值，求下面指令访问内存的物理地址：

```asm
mov ax,[bp]
```

也可能问：

```text
1. 8086 的寻址方式有哪些？
2. 寻址能力与地址总线有什么关系？
3. mov ax,[bp] 实际使用哪个段寄存器？物理地址是多少？
```

### 答题要点

常见寻址方式：

| 寻址方式 | 示例 | 含义 |
|---|---|---|
| 立即寻址 | `mov ax,1000h` | 操作数直接在指令中 |
| 寄存器寻址 | `mov ax,bx` | 操作数在寄存器中 |
| 直接寻址 | `mov ax,[1000h]` | 偏移地址直接给出 |
| 寄存器间接寻址 | `mov ax,[bx]` | 偏移地址在寄存器中 |
| 基址寻址 | `mov ax,[bp]` | 用 BP 形成偏移地址 |
| 变址寻址 | `mov ax,[si]` | 用 SI/DI 形成偏移地址 |
| 基址变址寻址 | `mov ax,[bx+si]` | 基址寄存器 + 变址寄存器 |
| 相对基址变址寻址 | `mov ax,[bx+si+idata]` | 基址 + 变址 + 位移量 |

关键陷阱：

```asm
mov ax,[bp]
```

默认段寄存器是 `SS`，不是 `DS`。

所以物理地址：

```text
物理地址 = SS × 16 + BP
```

如果题目给出具体值，直接代入即可。

## 第 2 题：总线的概念、分类及作用

### 原题可能表述

什么是总线？总线按照功能可以分为哪几类？各自分别影响 CPU 的什么能力？

### 答题要点

总线：CPU 与内存、I/O 设备之间传送信息的公共通道。

按照功能分类：

| 类型 | 传送内容 | 决定或影响 |
|---|---|---|
| 地址总线 | 地址信息 | 决定 CPU 寻址能力 |
| 数据总线 | 数据信息 | 决定一次数据传送位数，影响传输速度 |
| 控制总线 | 控制信息 | 决定 CPU 对外部器件的控制能力 |

常见公式：

```text
n 根地址总线可寻址 2^n 个存储单元
```

例如 8086 有 20 根地址总线：

```text
2^20 Byte = 1MB
```

## 第 3 题：CS、IP 的作用与指令执行序列

### 原题可能表述

说明 `CS` 和 `IP` 的作用。给出一段包含 `jmp` 的程序，要求写出程序执行指令的先后顺序，或者写出每次转移后 `CS:IP` 的变化。

### 题目形式可能类似

```asm
assume cs:code
code segment
start:
    mov ax,2000h
    jmp short s
    mov ax,1000h
s:
    mov bx,ax
    jmp far ptr far_label
    ; ……
far_label:
    mov cx,bx
code ends
end start
```

### 答题要点

- `CS` 保存代码段的段地址。
- `IP` 保存下一条要执行指令的偏移地址。
- CPU 从 `CS:IP` 指向的地址取指令。
- 普通顺序执行时，`IP` 自动增加。
- `jmp` 会修改 `IP` 或同时修改 `CS:IP`。

不同转移：

| 指令类型 | 改变内容 |
|---|---|
| 段内短转移 | 只改变 IP |
| 段内近转移 | 只改变 IP |
| 段间远转移 | 同时改变 CS 和 IP |

## 第 4 题：栈、push 指令、设置栈空间

### 原题可能表述

1. 说明 `push ax` 的执行过程。  
2. 编程：将 `10000H ~ 1000FH` 这段内存空间作为栈，初始状态栈为空。  
3. 将 `AX`、`BX`、`DS` 中的数据依次入栈。  
4. 将标志寄存器入栈。

### 答题要点

`push ax` 的过程：

```text
SP = SP - 2
将 AX 的内容写入 SS:SP 指向的内存单元
低字节放低地址，高字节放高地址
```

设置 `10000H ~ 1000FH` 为栈：

```text
段地址 SS = 1000H
偏移范围 = 0000H ~ 000FH
空栈初始 SP = 0010H
```

参考代码：

```asm
mov ax,1000h
mov ss,ax
mov sp,0010h

push ax
push bx
push ds
```

标志寄存器入栈：

```asm
pushf
```

如果要求出栈到标志寄存器：

```asm
popf
```

易错点：

```asm
mov ss,1000h     ; 错误，段寄存器不能直接接收立即数
```

## 第 5 题：段内转移、段间转移，以及循环程序功能分析

### 原题可能表述 A：转移分类

说明段内短转移、段内近转移、段间远转移的特点和区别。

### 答题要点 A

| 类型 | 修改内容 | 特点 |
|---|---|---|
| 段内短转移 short | IP | 8 位位移，范围较小 |
| 段内近转移 near | IP | 16 位位移，范围较大 |
| 段间远转移 far | CS 和 IP | 可跳转到其他代码段 |

---

### 原题可能表述 B：分析程序功能

分析下面程序的功能：

```asm
mov ax,datasg
mov ds,ax
mov bx,0
mov cx,4

s0:
    mov dx,cx
    mov si,0
    mov cx,3

s:
    mov al,[bx+si]
    and al,11011111b
    mov [bx+si],al
    inc si
    loop s

    add bx,16
    mov cx,dx
    loop s0
```

### 答题要点 B

这段程序的核心功能：

```text
将 datasg 段中 4 行数据里，每行前 3 个小写字母转换为大写字母。
```

原因：

```asm
and al,11011111b
```

这条指令会清除 ASCII 字符的第 5 位，使小写字母变成大写字母。

变量作用：

| 寄存器 | 作用 |
|---|---|
| BX | 每一行字符串的起始偏移，每次加 16，说明每行间隔 16 字节 |
| SI | 行内字符下标，从 0 到 2 |
| CX | loop 计数器，内外层循环都用它 |
| DX | 保存外层循环的 CX |

循环结构：

```text
外层循环 4 次：处理 4 行
内层循环 3 次：处理每行前 3 个字符
```

## 第 6 题：标志寄存器分类与标志位变化

### 原题可能表述

1. 标志寄存器可以分为哪几类？  
2. 说明 `ZF`、`CF`、`PF`、`AF`、`SF`、`OF` 的含义。  
3. 给出若干指令，判断执行后各标志位如何变化。  
4. 如果要比较两个有符号数，应如何利用标志位判断大小？

### 答题要点

状态标志：

| 标志位 | 含义 |
|---|---|
| ZF | 结果为 0 时置 1 |
| PF | 结果低 8 位中 1 的个数为偶数时置 1 |
| SF | 结果最高位为 1 时置 1 |
| CF | 无符号运算有进位或借位时置 1 |
| OF | 有符号运算溢出时置 1 |
| AF | bit3 向 bit4 有进位或借位时置 1 |

控制标志：

| 标志位 | 含义 |
|---|---|
| DF | 串操作方向标志 |
| IF | 中断允许标志 |
| TF | 陷阱标志，单步执行 |

常考规律：

```text
cmp a,b 等价于计算 a-b，只影响标志位，不保存结果。
```

无符号数比较：

```text
A = B：ZF = 1
A < B：CF = 1
A > B：CF = 0 且 ZF = 0
```

有符号数比较：

```text
A = B：ZF = 1
A < B：SF != OF
A > B：ZF = 0 且 SF = OF
```

常见指令影响：

| 指令 | 影响说明 |
|---|---|
| `add`、`sub`、`cmp` | 影响 CF、PF、AF、ZF、SF、OF |
| `inc`、`dec` | 不影响 CF，影响其他常用状态标志 |
| `and`、`or` | 一般使 CF=0、OF=0，影响 ZF、SF、PF |
| `mov` | 不影响标志位 |
| `loop` | 不影响标志位 |

### 1. 轰轰烈烈的算术组：`add`、`sub`、`cmp`

**【特点】：全量更新。** 它们会根据运算的真实物理结果，毫无保留地刷新全部 6 个状态标志位（CF, PF, AF, ZF, SF, OF）。

#### 🌰 例子：`add al, bl` (制造大爆炸)

假设 `AL = 0FFH` (无符号 255，有符号 -1)， `BL = 01H` (1) 执行 `add al, bl`，物理加法：`1111 1111 + 0000 0001 = (1) 0000 0000`。 结果 `AL = 00H`。

- **CF = 1**（最高位进位了，无符号数 255+1 爆了）
- **ZF = 1**（结果变成 0 了）
- **SF = 0**（最高位是 0，是正数）
- **OF = 0**（有符号数角度看：-1 + 1 = 0，完全合理，没溢出）
- **PF = 1**（结果 0000 0000 里有 0 个 1，0 是偶数）
- **AF = 1**（低 4 位 `1111 + 0001` 产生了向高 4 位的进位）

#### 🌰 例子：`cmp al, bl` (光打雷不下雨)

假设 `AL = 05H`， `BL = 08H` 执行 `cmp al, bl`。CPU 内部悄悄执行 `5 - 8`，结果是 `-3`（底层补码为 `FDH`：`1111 1101`），但**结果不会存回 AL，AL 还是 05H**。

- **CF = 1**（5 减 8 减不够，向高位借位了）
- **SF = 1**（结果是负数）
- **ZF = 0**（结果不是 0，说明两数不相等） *(后面写条件跳转代码时，常常就是看 `cmp` 留下的 `ZF` 和 `CF` 来决定跳不跳)*

### 2. 精打细算的计数组：`inc`、`dec`

**【特点】：不影响 CF，影响其他 5 个。**

**🧠 为什么偏偏放过 CF？** 这是一个极其天才的设计。`inc` (加1) 和 `dec` (减1) 最常用的场景是做**循环计数器**或**指针移动**。 假设你正在做两个 128 位超大数字的加法（需要带进位加法 `adc` 配合 `CF`），在加法循环里，你需要 `inc si` 来移动指针。如果 `inc si` 也会改变 `CF`，那你的超大数加法进位状态就被破坏了！所以 CPU 规定：`inc/dec` 随便用，绝不碰 `CF` 进位标志。

#### 🌰 例子：`inc al` (触发陷阱)

假设 `AL = 0FFH` (255) 执行 `inc al`。结果 `AL = 00H`。

- **CF = 原来的值！**（即使 255+1 在物理上产生了进位，CF 也绝对不会变，保持上一条指令留下的状态）
- **ZF = 1**（结果变 0 了）
- **OF = 0** 等等其他标志位正常更新。

### 3. 理性无情的逻辑组：`and`、`or`、`xor`

**【特点】：直接把 CF 和 OF 清零！影响 ZF、SF、PF。**

**🧠 为什么清零 CF 和 OF？** 逻辑运算（与、或、非、异或）是对每个 bit 独立操作的，根本不存在什么“进位”或者“数学溢出”的概念。所以 CPU 一旦执行逻辑指令，就会直接把 `CF` 和 `OF` 强行按在地上变成 0，以防干扰后面的判断。

#### 🌰 例子：`and al, bl`

假设 `AL = 0F0H` (`1111 0000`)，`BL = 00FH` (`0000 1111`) 执行 `and al, bl`。上下对着“与”一下，结果全军覆没，`AL = 00H`。

- **CF = 0**（铁律，直接清零）
- **OF = 0**（铁律，直接清零）
- **ZF = 1**（因为结果是 00H）
- **SF = 0**（最高位是 0）
- **PF = 1**（偶数个 1）

## 第 7 题：寄存器间接寻址、基址变址寻址

### 原题可能表述

寄存器间接寻址和基址变址寻址可以使用哪些寄存器？各有什么特点？可以有哪些合法组合方式？

### 答题要点

可用于形成偏移地址的寄存器：

```text
BX、BP、SI、DI
```

常见形式：

```asm
[bx]
[bp]
[si]
[di]
```

特点：

**1. 基址派：BX 和 BP —— “稳如泰山的地基”**

- **BX (Base Register，基址寄存器)：**
  - **特点：** 它是数据段（DS）的绝对主角。通常用来存放一个数组、一个字符串或一个数据块的**起始基准地址**。你可以把它理解为“定位锚点”。
- **BP (Base Pointer，基址指针)：**
  - **特点：** 它是栈段（SS）的专属管家。只要它一出场，CPU 就知道你要去**栈**里找东西。高级语言（如 C 语言）里的局部变量和函数参数，在底层全都是靠 BP 在栈里精确定位的。

**2. 变址派：SI 和 DI —— “灵活游走的游标”**

- **SI (Source Index，源变址) & DI (Destination Index，目的变址)：**
  - **特点：** 它们默认属于数据段（DS）。它们的作用就相当于高级语言循环里的**变量 `i` 和 `j`**。它们很少用来做固定的起点，而是配合基址寄存器，在循环中不断自增/自减（`inc si` / `dec di`），用来挨个遍历数据。
  - *(注：在字符串批量搬运指令 `movsb` 中，它们分工明确：SI 负责指明数据从哪来，DI 负责指明数据到哪去。)*

基址变址合法组合：

```asm
[bx+si]
[bx+di]
[bp+si]
[bp+di]
```

也可以加位移量：

```asm
[bx+si+idata]
[bx+di+idata]
[bp+si+idata]
[bp+di+idata]
```

默认段寄存器：

| 地址表达式 | 默认段寄存器 |
|---|---|
| 含 `bx`、`si`、`di`，不含 `bp` | DS |
| 含 `bp` | SS |

非法组合：

```asm
[bx+bp]
[si+di]
```

## 程序设计题：编写并安装 0 号中断处理程序

### 原题高度可能表述

编写程序，完成以下功能：

1. 将一个自定义的 0 号中断处理程序安装到内存 `0000:0200` 处；
2. 修改中断向量表，使 0 号中断指向该处理程序；
3. 当发生除法溢出时，在屏幕中间显示字符串：

```text
overflow!
```

4. 编写测试代码，触发除法溢出，验证中断处理程序是否执行。

题目也可能先问：

```text
请说明中断处理程序的设计思路和中断执行过程。
```

然后再要求写程序。

---

## 设计思路

### 1. 为什么要修改中断向量表？

0 号中断是除法溢出中断。CPU 发生除法错误时，会自动执行 0 号中断。

0 号中断向量地址：

```text
0000:0000
```

因为：

```text
中断类型码 × 4 = 0 × 4 = 0
```

中断向量表中：

```text
0000:0000 存放 IP
0000:0002 存放 CS
```

如果希望 0 号中断执行我们自己的程序，需要写：

```asm
mov word ptr es:[0*4],200h
mov word ptr es:[0*4+2],0
```

含义：

```text
IP = 0200H
CS = 0000H
```

---

### 2. 为什么要复制中断处理程序？

中断处理程序最开始在当前代码段中。为了让中断向量指向固定位置，需要把它复制到 `0000:0200`。

复制时使用：

```asm
cld
rep movsb
```

其中：

```text
DS:SI -> 源地址
ES:DI -> 目标地址
CX    -> 复制字节数
```

---

### 3. 为什么中断处理程序开头要跳过字符串？

常见写法：

```asm
do0:
    jmp short do0start
    db 'overflow!'
do0start:
    ; 正式代码
```

原因：字符串 `overflow!` 放在代码段中，CPU 执行到 `do0` 后如果不跳过字符串，就会把字符串字节当成机器指令执行，导致错误。

## 中断的设计思路以及过程

这是一份为您提炼的“期末考试标准答案”版总结，去掉了所有通俗的比喻，完全采用严谨、规范的计算机体系结构术语。您可以直接背诵并写在试卷上以获取高分。

------

### **简答题：请简述微机系统中中断机制的设计思路，并说明 CPU 响应中断的具体执行过程。**

**【标准答案】**

**一、 中断机制的设计思路**

中断机制是微机系统中处理突发事件和实现异步控制的核心机制，其设计思路主要体现在以下三个方面：

1. **异步响应与提高并发效率** 
2. **统一寻址与系统解耦（中断向量表）** 
3. **现场保护与无缝恢复** 

**二、 CPU 响应中断的执行过程（以 8086 为例）**

当 CPU 收到中断请求并决定响应时，硬件会自动按顺序执行以下 6 个步骤（即中断的内部微操作）：

1. **获取中断类型码：** CPU 从硬件总线或内部指令获取中断源对应的中断类型码 $N$。
2. **标志寄存器入栈（`pushf`）：** 将当前标志寄存器（Flags）的内容压入栈中，以保护被打断程序的状态。
3. **清除标志位（`TF=0`, `IF=0`）：** 硬件自动将陷阱标志位 TF 和中断允许标志位 IF 清零，以屏蔽其他的可屏蔽中断和单步中断，防止在此期间被意外打断。
4. **断点地址入栈（`push CS`, `push IP`）：** 依次将当前的代码段寄存器（CS）和指令指针寄存器（IP）压入栈中，保存主程序的返回地址。
5. **获取中断入口地址：** 根据中断类型码 $N$，计算中断向量在内存中的物理地址（$N \times 4$），从中读取中断处理程序的偏移地址送入 `IP`，读取段地址送入 `CS`。
6. **执行中断服务程序：** `CS:IP` 指向新的入口地址，CPU 跃迁至中断服务程序（ISR）开始执行。

**【附加要点（中断返回）】**

中断服务程序执行完毕后，必须执行 **`IRET` (中断返回指令)**。该指令会依次从栈中弹出数据至 `IP`、`CS` 和 `Flags` 寄存器，从而恢复现场，使 CPU 继续执行被中断的主程序。

## 参考答案代码

```asm
assume cs:code

code segment

start:
    ; 1. 将 do0 中断处理程序复制到 0000:0200
    mov ax,cs
    mov ds,ax
    mov si,offset do0

    mov ax,0
    mov es,ax
    mov di,200h

    mov cx,offset do0end - offset do0
    cld
    rep movsb

    ; 2. 修改 0 号中断向量表
    mov ax,0
    mov es,ax
    mov word ptr es:[0*4],200h
    mov word ptr es:[0*4+2],0

    ; 3. 程序返回 DOS
    mov ax,4c00h
    int 21h

; -----------------------------------
; 0 号中断处理程序
; 功能：显示 overflow!
; -----------------------------------
do0:
    jmp short do0start
    db 'overflow!'

do0start:
    ; 此时中断处理程序已被复制到 0000:0200
    ; 字符串位于 0000:0202
    mov ax,cs
    mov ds,ax
    mov si,202h

    mov ax,0b800h
    mov es,ax
    mov di,12*160+36*2

    mov cx,9
s:
    mov al,[si]
    mov es:[di],al
    inc si
    add di,2
    loop s

    mov ax,4c00h
    int 21h

do0end:
    nop

code ends
end start
```

## 测试除法溢出的代码

试卷截图中出现的测试方式大概率是：

```asm
mov ax,8
mov bh,0
div bh
```

原因：

```text
div bh 表示用 AX 除以 BH。
BH = 0，会触发除法错误，从而产生 0 号中断。
```

如果 0 号中断向量已经被修改，就会执行我们安装的 `do0` 处理程序，在屏幕中间显示：

```text
overflow!
```

## 中断过程答题模板

如果程序设计题前面问“中断过程”，可以这样写：

```text
执行 int n 或 CPU 检测到内部中断后：

1. 取得中断类型码 n；
2. FLAGS 入栈；
3. TF 和 IF 置 0；
4. CS 入栈；
5. IP 入栈；
6. 从中断向量表 n×4 处取出新的 IP；
7. 从中断向量表 n×4+2 处取出新的 CS；
8. 转去执行中断处理程序。
```

如果中断处理程序最后用 `iret` 返回，则：

```text
iret 依次弹出 IP、CS、FLAGS，使 CPU 回到中断前的执行位置。
```

不过王爽教材中除法溢出显示 `overflow!` 的例程，常用：

```asm
mov ax,4c00h
int 21h
```

直接返回 DOS。
