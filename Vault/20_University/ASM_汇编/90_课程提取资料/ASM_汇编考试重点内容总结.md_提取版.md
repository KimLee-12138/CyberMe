# 汇编语言考试重点内容总结

> 依据：你上传的 2024 年考试回忆截图、手写复习提纲，以及王爽《汇编语言》第 4 版的章节体系。  
> 判断：这套考试明显围绕 **8086 + 王爽教材前 13 章** 出题，题型大概率是 **问答题 7 题 × 10 分 + 程序设计题 30 分**。

---

## 一、总体命题规律

这门课不是单纯考“背概念”，而是重点考三类能力：

1. **能不能把 CPU 执行过程讲清楚**  
   例如 CS:IP 如何取指令、总线如何传数据、物理地址怎么算、jmp/call/int 如何改变执行流程。

2. **能不能看懂 8086 代码的实际含义**  
   例如 `loop` 嵌套循环、`[bx+si]` 寻址、大小写字符转换、栈操作、标志位变化。

3. **能不能写出王爽教材风格的完整程序**  
   尤其是：设置段寄存器、复制中断处理程序、修改中断向量表、显示字符串、调用 DOS 退出。

---

## 二、最高优先级考点：必须会

### 1. CPU 执行指令过程：CS:IP

重点内容：

- `CS`：代码段寄存器，保存当前代码段的段地址。
- `IP`：指令指针寄存器，保存下一条要执行指令的偏移地址。
- CPU 取指令地址：

```text
物理地址 = CS × 16 + IP
```

必须会回答：

- CPU 执行一条指令前，会从 `CS:IP` 指向的内存单元取指令。
- 取完指令后，`IP` 通常自动增加，指向下一条指令。
- `jmp`、`call`、`ret`、`iret`、`int` 等指令会改变 `CS`、`IP`，从而改变程序执行流程。
- 不能用普通 `mov` 指令直接修改 `CS`，必须通过转移类指令修改。

高频题型：

```asm
给出一段代码，要求写出指令执行顺序，或者写出 CS、IP 的变化。
```

---

### 2. 总线：地址总线、数据总线、控制总线

总线是 CPU 与内存、I/O 设备之间传送信息的公共通道。

按照功能分类，重点是三类：

| 总线类型 | 作用 | 决定或影响 |
|---|---|---|
| 地址总线 | 传送地址信息 | 决定 CPU 的寻址能力 |
| 数据总线 | 传送数据信息 | 决定一次能传送的数据位数，影响数据传输效率 |
| 控制总线 | 传送控制信号 | 决定 CPU 对外部器件的控制能力 |

常考表述：

```text
如果 CPU 有 n 根地址总线，则最多可以寻址 2^n 个存储单元。
```

例如：20 根地址总线可以寻址：

```text
2^20 Byte = 1MB
```

---

### 3. 8086 寄存器体系

必须熟悉的寄存器：

| 类型 | 寄存器 | 重点用途 |
|---|---|---|
| 通用寄存器 | AX、BX、CX、DX | 存放数据；AX 常作累加器，CX 常作循环计数器 |
| 段寄存器 | CS、DS、SS、ES | 分别用于代码段、数据段、栈段、附加段 |
| 指针寄存器 | SP、BP | SP 指向栈顶；BP 常用于访问栈中数据 |
| 变址寄存器 | SI、DI | 常用于数组、字符串、批量数据处理 |
| 指令指针 | IP | 指向下一条指令 |
| 标志寄存器 | FLAGS | 保存运算结果状态和控制状态 |

易错点：

- `CS` 不能用 `mov` 直接赋值。
- `SS`、`DS`、`ES` 不能直接装立即数，必须通过通用寄存器中转。

错误写法：

```asm
mov ss,1000h
```

正确写法：

```asm
mov ax,1000h
mov ss,ax
```

---

### 4. 物理地址与段地址、偏移地址

8086 形成物理地址的公式：

```text
物理地址 = 段地址 × 16 + 偏移地址
```

例如：

```text
DS = 2000H，BX = 0010H
mov ax,[bx]
访问物理地址 = 2000H × 10H + 0010H = 20010H
```

重点陷阱：

```asm
mov ax,[bp]
```

`[bp]` 默认使用的段寄存器不是 `DS`，而是 `SS`。

所以：

```text
mov ax,[bp] 的物理地址 = SS × 16 + BP
```

---

### 5. 寻址方式

常见寻址方式必须会识别、举例、计算地址。

| 寻址方式 | 示例 | 含义 |
|---|---|---|
| 立即寻址 | `mov ax,1000h` | 操作数直接写在指令中 |
| 寄存器寻址 | `mov ax,bx` | 操作数在寄存器中 |
| 直接寻址 | `mov ax,[1000h]` | 偏移地址直接给出 |
| 寄存器间接寻址 | `mov ax,[bx]` | 偏移地址在寄存器中 |
| 基址寻址 | `mov ax,[bp]` | 常用于访问栈 |
| 变址寻址 | `mov ax,[si]` | 常用于数组或字符串 |
| 基址变址寻址 | `mov ax,[bx+si]` | 基址寄存器 + 变址寄存器 |
| 相对基址变址寻址 | `mov ax,[bx+si+idata]` | 基址 + 变址 + 位移量 |

合法组合必须记住：

```asm
[bx+si]
[bx+di]
[bp+si]
[bp+di]
```

非法组合：

```asm
[bx+bp]
[si+di]
```

默认段寄存器：

| 表达式中包含 | 默认段寄存器 |
|---|---|
| `bx`、`si`、`di` | DS |
| `bp` | SS |

---

### 6. 栈：SS:SP、push、pop、pushf

8086 的栈由 `SS:SP` 指向。

`push ax` 的过程：

```text
SP = SP - 2
将 AX 的内容写入 SS:SP 指向的内存单元
低字节存低地址，高字节存高地址
```

`pop ax` 的过程：

```text
从 SS:SP 指向的内存单元取出一个字送入 AX
SP = SP + 2
```

如果把 `10000H ~ 1000FH` 作为栈空间：

```text
段地址 SS = 1000H
偏移范围 = 0000H ~ 000FH
初始为空时，SP 应设为 0010H
```

典型代码：

```asm
mov ax,1000h
mov ss,ax
mov sp,0010h

push ax
push bx
push ds
```

标志寄存器入栈：

```asm
pushf
```

标志寄存器出栈：

```asm
popf
```

---

### 7. 转移指令：段内短转移、段内近转移、段间远转移

重点区别：

| 类型 | 改变内容 | 特点 |
|---|---|---|
| 段内短转移 short | 只改 IP | 位移为 8 位，范围较小 |
| 段内近转移 near | 只改 IP | 位移为 16 位，范围较大 |
| 段间远转移 far | 改 CS 和 IP | 可以跳到另一个代码段 |

常考指令：

```asm
jmp short 标号
jmp near ptr 标号
jmp far ptr 标号
```

必须会说明：

- 段内转移不改变 `CS`。
- 段间转移同时改变 `CS` 和 `IP`。
- `call` 会保存返回地址，`ret` 会返回。
- `retf` 是远返回，会同时恢复 `CS` 和 `IP`。

---

### 8. loop 指令与 CX

`loop 标号` 的实际过程：

```text
CX = CX - 1
如果 CX != 0，则转移到标号处继续执行
如果 CX == 0，则继续执行下一条指令
```

高频陷阱：

- `loop` 固定使用 `CX` 作为计数器。
- 嵌套循环时，内外层都要用 `CX`，必须保存外层的 `CX`。

典型写法：

```asm
mov cx,4
s0:
    mov dx,cx      ; 保存外层 CX
    mov cx,3       ; 内层循环次数
s:
    ; 内层循环内容
    loop s
    mov cx,dx      ; 恢复外层 CX
    loop s0
```

---

### 9. 字符处理：大小写转换

ASCII 中，小写字母和对应大写字母相差第 5 位。

小写转大写：清除第 5 位。

```asm
and al,11011111b
```

大写转小写：设置第 5 位。

```asm
or al,00100000b
```

考试中出现过类似代码：

```asm
mov al,[bx+si]
and al,11011111b
mov [bx+si],al
```

功能：把内存中的小写字母转换成大写字母。

---

### 10. FLAGS 标志寄存器

FLAGS 分为两类：

#### 1）状态标志位

| 标志位 | 名称 | 含义 |
|---|---|---|
| CF | 进位标志 | 无符号运算产生进位或借位 |
| PF | 奇偶标志 | 结果低 8 位中 1 的个数为偶数时置 1 |
| AF | 辅助进位标志 | bit3 向 bit4 产生进位或借位 |
| ZF | 零标志 | 结果为 0 时置 1 |
| SF | 符号标志 | 结果最高位为 1 时置 1 |
| OF | 溢出标志 | 有符号运算产生溢出时置 1 |

#### 2）控制标志位

| 标志位 | 名称 | 含义 |
|---|---|---|
| DF | 方向标志 | 控制串操作中 SI、DI 增减方向 |
| IF | 中断允许标志 | 控制是否响应可屏蔽中断 |
| TF | 陷阱标志 | 单步执行控制 |

重点指令影响：

| 指令 | 是否影响标志位 |
|---|---|
| `add`、`sub`、`cmp` | 影响大部分状态标志 |
| `inc`、`dec` | 影响大部分状态标志，但不影响 CF |
| `and`、`or` | 通常使 CF=0、OF=0，并影响 ZF/SF/PF |
| `mov` | 不影响标志位 |
| `loop` | 不影响标志位 |

`cmp a,b` 的本质：

```text
执行 a - b，只影响标志位，不保存结果。
```

有符号数比较：

```text
相等：ZF = 1
大于：ZF = 0 且 SF = OF
小于：SF != OF
```

无符号数比较：

```text
相等：ZF = 1
低于：CF = 1
高于：CF = 0 且 ZF = 0
```

---

### 11. 串传送指令：movsb、rep、cld、std

重点指令：

```asm
cld
rep movsb
```

含义：

- `movsb`：把 `DS:SI` 指向的一个字节传送到 `ES:DI`。
- `rep`：重复执行，重复次数由 `CX` 决定。
- `cld`：清除 DF，使 `SI`、`DI` 每次递增。
- `std`：设置 DF，使 `SI`、`DI` 每次递减。

典型用途：复制一段中断处理程序到指定内存区域。

---

### 12. 中断：内中断、中断向量表、int、iret

这是程序设计题的核心。

#### 1）中断向量表

8086 的中断向量表在：

```text
0000:0000 ~ 0000:03FF
```

每个中断向量占 4 个字节：

```text
N 号中断向量地址 = 0000:(N × 4)
```

存放格式：

```text
低 2 字节：中断处理程序的 IP
高 2 字节：中断处理程序的 CS
```

例如 0 号中断：

```asm
mov ax,0
mov es,ax
mov word ptr es:[0*4],200h      ; IP = 0200H
mov word ptr es:[0*4+2],0       ; CS = 0000H
```

#### 2）int n 的执行过程

```text
1. 取得中断类型码 n
2. FLAGS 入栈
3. TF 和 IF 置 0
4. CS 入栈
5. IP 入栈
6. 从中断向量表中取出新的 IP、CS
7. 转去执行中断处理程序
```

#### 3）iret 的执行过程

```text
1. IP 出栈
2. CS 出栈
3. FLAGS 出栈
```

## 三、程序设计题最可能考的套路

### 1. 安装中断处理程序

基本步骤：

```text
1. 将中断处理程序复制到安全内存区域，比如 0000:0200。
2. 修改中断向量表，让某个中断号指向 0000:0200。
3. 触发中断，测试处理程序。
```

典型代码骨架：

```asm
assume cs:code
code segment
start:
    mov ax,cs
    mov ds,ax
    mov si,offset do0

    mov ax,0
    mov es,ax
    mov di,200h
    mov cx,offset do0end - offset do0
    cld
    rep movsb

    mov ax,0
    mov es,ax
    mov word ptr es:[0*4],200h
    mov word ptr es:[0*4+2],0

    mov ax,4c00h
    int 21h

do0:
    ; 中断处理程序

do0end:
    nop
code ends
end start
```

---

### 2. 显示字符串到显存

文本模式显存段地址：

```text
B800H
```

一行 80 个字符，每个字符占 2 个字节：

```text
字符字节 + 属性字节
```

第 12 行、第 36 列的偏移地址：

```text
12 × 160 + 36 × 2
```

典型代码：

```asm
mov ax,0b800h
mov es,ax
mov di,12*160+36*2
mov cx,9
s:
    mov al,[si]
    mov es:[di],al
    inc si
    add di,2
    loop s
```

## 五、临考背诵清单

### 必背公式

```text
物理地址 = 段地址 × 16 + 偏移地址
n 根地址总线寻址能力 = 2^n Byte
中断向量地址 = 中断类型码 × 4
显存某行某列偏移 = 行号 × 160 + 列号 × 2
```

### 必背代码

#### 1）设置 DS

```asm
mov ax,datasg
mov ds,ax
```

#### 2）设置栈

```asm
mov ax,1000h
mov ss,ax
mov sp,0010h
```

#### 3）复制代码

```asm
mov ax,cs
mov ds,ax
mov si,offset do0

mov ax,0
mov es,ax
mov di,200h
mov cx,offset do0end-offset do0
cld
rep movsb
```

#### 4）修改中断向量表

```asm
mov ax,0
mov es,ax
mov word ptr es:[0*4],200h
mov word ptr es:[0*4+2],0
```

#### 5）DOS 返回

```asm
mov ax,4c00h
int 21h
```

## 六、最容易丢分的点

1. 把 `[bp]` 默认段寄存器写成 `DS`。正确是 `SS`。
2. 写成 `mov ss,1000h`。段寄存器不能直接装立即数。
3. 栈初始化时把 `SP` 设成 `000FH`。空栈应设成栈空间末尾的下一字节，比如 `0010H`。
4. 忘记 `push` 是先 `SP-2`，再写入数据。
5. 认为 `loop` 会影响标志位。实际上 `loop` 不影响标志位。
6. 嵌套 `loop` 时忘记保存外层 `CX`。
7. 中断向量表中把 `CS` 和 `IP` 顺序写反。正确是低地址放 IP，高地址放 CS。
8. `rep movsb` 前忘记设置 `DS:SI`、`ES:DI`、`CX`，以及 `cld`。
9. 有符号比较和无符号比较混淆：无符号看 `CF/ZF`，有符号看 `SF/OF/ZF`。
10. 程序设计题只写中断处理程序，不写“安装程序”和“修改中断向量表”。
