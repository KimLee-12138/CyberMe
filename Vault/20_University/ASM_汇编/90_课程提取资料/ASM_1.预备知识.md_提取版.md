# 1.预备知识 

1. 汇编语言：汇编指令是机器指令的助记符，和机器指令一一对应。汇编语言由汇编指令，伪指令，其他符号组成

> 程序员->汇编指令->编译器->机器指令->计算机

1. 寄存器（如ax，bx）：CPU中可以存储数据的器件，一个CPU有多个寄存器
2. 存储器：向CPU提供指令和数据。指令和数据都存放在存储器中，也就是内存之中
3. CPU对存储器的读写：
   - 存储单元的地址（地址信息）
   - 器件的选择（控制信息）
   - 读或写的数据（数据信息）

***总线（CPU通过总线将这三类信息传送到存储芯片中）：***

**种类：**地址总线、数据总线、控制总线

**作用：**

- CPU通过**地址总线指定存储单元的地址**
- CPU通过数据总线和内存或其他器件之间进行数据传送
- CPU通过控制总线传送对外部器件的控制信息

**不同的总线如何影响CPU的性能：**

- 地址总线能传送多少不同信息，CPU就能对多少内存单元进行寻址，地址总线宽度为N时，CPU最多可以寻找2的N次方个内存单元。地址总线宽度越宽，CPU寻址性能越好（8086CPU寻址能力为1M）
- 数据总线的宽度决定了CPU和外界的数据传送的速度。有多少根数据总线，CPU就能一次性传送多少位的数据。
- 控制总线的宽度决定了CPU对外部器件的控制能力。控制总线越多，CPU对其他器件的控制能力越强

1. 一个存储单元可以存储8bit即一个字节1B
2. 主板和接口卡：主板相当于一个容器，主板上放有核心器件以及其他器件（如CPU、存储器等等）

内存地址空间

- 最终运行程序的是CPU。我们用汇编编程的时候，必须从CPU角度考虑问题。对CPU来讲，系统中的所有存储器的存储单元都处于一个统一的逻辑存储器中，它的容量受CPU寻址能力的限制。这个逻辑存储器即是我们所说的内存地址空间

### 🛠️ CPU 执行指令的完整三步曲

#### 第 1 步：`CS + IP 取指令`

- **底层逻辑：** CPU 要干活，首先得知道“活”在哪。在 8086 CPU 中，**CS（代码段寄存器）** 记录了代码所在内存的基地址，**IP（指令指针寄存器）** 记录了当前要执行的那条指令的偏移量。
- **物理动作：** CPU 内部的地址加法器将 `CS` 和 `IP` 拼接计算得出绝对物理地址（`CS * 16 + IP`），然后通过地址总线去内存里敲门，把这条指令的数据读出来。

#### 第 2 步：`指令放`（完整意思是：指令放入指令缓冲器）

- **底层逻辑：** 内存里的指令（机器码）被找出来了，得搬进 CPU 内部才能处理。
- **物理动作：** 读出来的指令数据通过数据总线，被送入 CPU 内部的指令缓冲器（或指令队列）中等待被解码和执行。

#### 第 3 步：`修改 IP`（极其关键的一步！）

- **底层逻辑：** CPU 是怎么知道下一条指令在哪的？全靠这一步！
- **物理动作：** 刚把指令拿进来，CPU 还没开始真正执行它呢，**IP 寄存器的值就会立刻自动增加**。增加多少？取决于刚刚读进来的那条指令占了几个字节。
- **效果：** 修改 IP 后，`CS:IP` 就已经悄悄指向了**下一条**要执行的指令的地址了。
