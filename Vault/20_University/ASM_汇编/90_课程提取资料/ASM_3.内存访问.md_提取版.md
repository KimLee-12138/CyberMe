# 3.内存访问

内存是从上到下，从低位到高位的存储，所以存储2000H时，是按如下顺序存储

## DS和[address]

DS:[address]指向数据的内存单元

**mov指令的作用：**

（1）将数据直接送入寄存器中mov ax,1000h

（2）将一个寄存器中的内容送入另一个寄存器中mov ax,bx

（3）将一个内存单元中的内容送入一个寄存器中mov ax,[0]

（注意：不能将一个内存单元中的内容送入另一个内存单元，CPU 与内存之间的沟通依赖于**系统总线**（数据总线、地址总线、控制总线）。在一次标准的总线周期内，CPU 只能向总线发出一个地址，并且只能进行一次“读”或一次“写”操作。 如果要实现“内存到内存”的直接传送，CPU 就必须在同一个周期内同时定位两个内存地址，一边读一边写，8086 的硬件根本不支持这种“一心二用”。理解）

**如何修改DS，实现数据跳转？**

```asm
mov bx,1000H
mov ds,bx
mov ax,[0];1000:0处的字型数据送入ax
mov [0],cx;cx中的16位数据送到1000:0处
```

**字的传送**

因为8086 CPU是16位结构，有16根数据线，所以可以一次性传送16位的数据，也就是一次性传送一个字

## 栈

push ax

pop ax

8086CPU的入栈和出栈操作都是以**字**为单位（占两个存储单元）进行的

**CPU如何知道一段内存空间被当作栈使用？push和pop的时候如何知道哪个单元是栈顶单元的？**

段寄存器SS指向存放栈顶的段地址，SP指向栈顶的偏移地址

任何时刻，SS：SP指向栈顶元素

**push指令和pop指令是怎么执行的？**

#### PUSH 指令的底层动作（以 `push ax` 为例，假设 `AX = 1234H`）

当 CPU 遇到 `push ax` 时，它会严格按照以下两步执行：

1. **先移指针腾出空间：** $SP = SP - 2$。

   因为 8086 压栈是以“字（2个字节）”为单位的，为了装下 `AX` 的 16 位数据，`SP` 必须向低地址方向移动 2 个字节，指向新的空闲位置。

2. **后存入数据（遵循小端法）：**

   CPU 将 `AX` 中的数据写入以当前 `SS:SP` 为起点的两个内存单元中。

   - 低位字节 `AL`（`34H`）被存入 `SS:SP` 指向的低地址单元。

   - 高位字节 `AH`（`12H`）被存入 `SS:(SP+1)` 指向的高地址单元。

     *(这与你笔记右下角的画法完全一致：如果把低地址画在上面，高地址画在下面，那么 `34` 在上，`12` 在下。)*

#### POP 指令的底层动作（以 `pop bx` 为例）

当 CPU 遇到 `pop bx` 时，动作顺序正好与 `PUSH` 相反：

1. **先读取数据：**

   CPU 从当前 `SS:SP` 指向的内存单元中读取一个字（2个字节）的数据。

   - 位于 `SS:SP`（低地址）的字节送入 `BL`。
   - 位于 `SS:(SP+1)`（高地址）的字节送入 `BH`。

2. **后移指针回收空间：** $SP = SP + 2$。

   读完数据后，`SP` 向高地址方向移动 2 个字节，指向上一个栈顶元素。

   **注意：** 出栈后，原来内存里的数据并没有被清空或抹除，它们依然静静地躺在那里，只是因为 `SP` 已经移走了，它们变得“不可见”且随时会被下一次 `PUSH` 覆盖。
