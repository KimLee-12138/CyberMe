# 附：乘除法规则.md：提取版

> 本文件是检索中间材料，不是已核对的正式知识结论。

在 8086 汇编语言中，算术运算指令的设计充满了底层硬件的智慧。为了适应有限的寄存器位宽（16位），CPU 采用了一些非常巧妙的规则，尤其是**标志位接力**和**隐式寄存器**。

下面为你系统总结这四大核心运算指令的使用方法。

------

### 一、 加减法：普通运算与“接力”运算

在处理不超过 16 位的数字时，直接使用 `ADD` 和 `SUB` 即可。但如果数字超出了 16 位（例如 32 位甚至 64 位大数运算），就需要用到 `ADC` 和 `SBB` 配合进位标志位（**CF**）来完成高位运算。

#### 1. ADD 与 ADC（加法组合）

- **`ADD dest, src` (普通加法)：** 执行 `dest = dest + src`。如果最高位产生进位，CF 会自动变成 1。
- **`ADC dest, src` (带进位加法)：** 执行 `dest = dest + src + CF`。它在做加法时，会自动把上一步留下的 CF 值（0 或 1）加进去。

**🔥 实战场景：32位大数加法**

假设我们要计算 `(DX:AX) + (CX:BX)`，结果存入 `DX:AX`。由于一个寄存器装不下 32 位，必须劈成两半算：先算低 16 位，再算高 16 位。

代码段

```
; 计算 0001FFFFh + 00020001h
mov dx, 0001h
mov ax, 0FFFFh  ; 被加数 (高位DX, 低位AX)
mov cx, 0002h
mov bx, 0001h   ; 加数   (高位CX, 低位BX)

add ax, bx      ; 【第一步】低位对加。FFFFh + 0001h = 0000h。溢出，CF=1！
adc dx, cx      ; 【第二步】高位对加。0001h + 0002h + CF(1) = 0004h。
                ; 最终结果 DX:AX = 0004 0000h
```

#### 2. SUB 与 SBB（减法组合）

- **`SUB dest, src` (普通减法)：** 执行 `dest = dest - src`。如果不够减产生了借位，CF 会自动变成 1。
- **`SBB dest, src` (带借位减法)：** 执行 `dest = dest - src - CF`。它会自动扣除上一步欠下的“借位”。32位减法的逻辑与加法完全一致：低位用 `SUB`，高位用 `SBB`。

------

### 二、 乘法与除法：隐式寄存器的魔法

乘除法最大的特点是“强买强卖”。你只能在指令里写一个操作数（乘数/除数），另一个操作数（被乘数/被除数）以及结果存放的位置，是 CPU **硬性规定**的。

它们严格区分 **8位运算** 和 **16位运算**。

#### 1. MUL (无符号乘法)

指令格式只有一种：`MUL 寄存器或内存单元`

| **运算类型** | **隐式被乘数** | **你的指令** | **运算逻辑** | **隐式结果存放** | **结果说明**                                           |
| ------------ | -------------- | ------------ | ------------ | ---------------- | ------------------------------------------------------ |
| **8位乘法**  | **AL**         | `MUL BL`     | `AL * BL`    | **AX**           | 两个8位相乘，结果最多16位，正好装满 AX。               |
| **16位乘法** | **AX**         | `MUL BX`     | `AX * BX`    | **DX : AX**      | 两个16位相乘，结果最多32位。高16位放 DX，低16位放 AX。 |

*注意：被乘数是你不用写的，CPU 会自动去 AL 或 AX 里找。*

#### 2. DIV (无符号除法)

除法是乘法的逆运算，指令格式同样为：`DIV 寄存器或内存单元`。

**核心口诀：被除数的位数，必须是除数的两倍！**

| **运算类型**   | **隐式被除数 (大)** | **你的指令 (小)** | **运算逻辑**   | **隐式结果存放**           |
| -------------- | ------------------- | ----------------- | -------------- | -------------------------- |
| **除以 8 位**  | **AX** (16位)       | `DIV BL`          | `AX / BL`      | 商在 **AL**，余数在 **AH** |
| **除以 16 位** | **DX:AX** (32位)    | `DIV CX`          | `(DX:AX) / CX` | 商在 **AX**，余数在 **DX** |

**🔥 易错点提示：**

在进行“除以 16 位”的除法前，如果你的被除数本来只有 16 位（比如放在 AX 里），你**必须手动将 DX 清零 (`mov dx, 0`)**。否则 DX 里的垃圾数据会和 AX 拼成一个巨大的 32 位数，导致除法溢出报错（Divide Overflow）。
