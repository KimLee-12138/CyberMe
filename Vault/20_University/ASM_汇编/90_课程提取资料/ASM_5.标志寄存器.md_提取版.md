# 5.标志寄存器

## 常用标志位

![image-20260502201130786](C:\Users\22067\AppData\Roaming\Typora\typora-user-images\image-20260502201130786.png)

### 第一阵营：状态标志（ALU 的运算汇报报告）

这 6 个标志位是被动的。每当 CPU 内部的算术逻辑单元（ALU）执行完一次算术运算（如 `ADD`, `SUB`）或逻辑运算（如 `AND`, `OR`）后，硬件会自动根据计算结果修改这些位。

**1. ZF (Zero Flag - 零标志位，第 6 位) —— “结果是不是 0？”**

- 这是最常用的标志位。如果刚刚执行的指令结果为 `0`，那么 `ZF = 1`（指示灯亮起，表示“是的，结果是零”）。
- 如果结果不为 `0`，则 `ZF = 0`。
- **用途：** 经常配合 `CMP`（比较）指令使用。比如判断两个数是否相等，就是让它们相减，如果结果为 0（`ZF=1`），就说明它们相等。

**2. SF (Sign Flag - 符号标志位，第 7 位) —— “结果是负数吗？”**

- 它直接记录了运算结果的最高位（对于 8 位数据就是第 7 位）。
- 如果最高位是 `1`，说明结果是个负数，此时 `SF = 1`。如果最高位是 `0`（正数），则 `SF = 0`。

**3. PF (Parity Flag - 奇偶标志位，第 2 位) —— “底层校验员”**

- 它统计运算结果的**最低 8 位**中，数字 `1` 的个数。
- 如果 `1` 的个数是偶数，`PF = 1`。如果是奇数，`PF = 0`。
- **用途：** 早期常用于底层通信的数据校验，现在的高级编程中很少直接用到。

**4. CF (Carry Flag - 进位标志位，第 0 位) —— “无符号数溢出报警”**

- 当进行**无符号数**运算时，如果最高位向更高位产生了进位（加法），或者向更高位借了位（减法），`CF = 1`。
- **例子：** 8 位寄存器最大能装 255（`FFH`）。如果你算 `255 + 1`，结果变成了 `0`，此时进位丢失了。CPU 就会把 `CF` 置为 `1`，提醒你：“大哥，无符号数装不下了，溢出啦！”

**5. OF (Overflow Flag - 溢出标志位，第 11 位) —— “有符号数溢出报警”**

- 这是最容易和 CF 搞混的！`OF` 专门针对**有符号数**（带有正负号的数）。
- **例子：** 8 位有符号数的范围是 -128 到 +127。如果你算 `100 + 100`（正数加正数），按理说是 200，但 8 位有符号数装不下 200，结果会变成一个负数！这种“正+正=负”或“负+负=正”的荒谬现象，就是有符号数溢出，此时 `OF = 1`。

*(注：第 4 位的 AF 是辅助进位标志，主要针对 BCD 码运算，现代编程中极少涉及，作为了解即可。)*

### 第二阵营：控制标志（程序员发出的指令）

这 3 个标志位是主动的。它们不会因为普通的数学运算而改变，而是需要程序员用特定的指令去设置它们，从而改变 CPU 的工作模式。

**1. DF (Direction Flag - 方向标志位，第 10 位) —— “内存搬运方向盘”** 还记得我们上一节讨论 `SI` 和 `DI` 双指针搬运字符串的代码吗？当时我们每搬一次，都要手动写 `add si, 2` 和 `add di, 2`。

- 如果结合串传送指令（如 `MOVSB`），CPU 会自动帮我们增加或减少 `SI` 和 `DI`。到底增还是减，就是 `DF` 决定的！
- 通过 `CLD` 指令让 `DF = 0`：指针自动**递增**（从低地址往高地址搬运，最常用）。
- 通过 `STD` 指令让 `DF = 1`：指针自动**递减**（从高地址往低地址倒着搬运）。

**2. IF (Interrupt Enable Flag - 中断允许标志位，第 9 位)**

- 当 `IF = 1` 时，CPU 允许响应外部的“可屏蔽中断”（比如你敲击了键盘）。
- 当 `IF = 0` 时，CPU 就会“屏蔽”这些外部打扰，专心执行当前代码。可以使用 `STI` （1）和 `CLI` （0）指令来开关它。

**3. TF (Trap Flag - 陷阱标志位，第 8 位)**

- 专门为调试程序（Debug）设计的。如果 `TF = 1`，CPU 每执行完一条指令，就会自动停下来，触发一个单步中断。这就是你用 Debug 工具可以“单步执行（Trace）”查看寄存器状态的底层硬件原理。

| **标志位**                                         | **功能**                                                     |
| -------------------------------------------------- | ------------------------------------------------------------ |
| ZF（zero）零标志位                                 | 结果为0是zf为1；结果为1时zf为0                               |
| PF（parity）奇偶标志位                             | 结果中所有二进制位中1的个数为偶数时，pf为1；为奇数时，pf为0  |
| SF（symbol）符号标志位（对**有符号**数来说有意义） | 结果为负时，sf为1；结果为正时，sf为0                         |
| CF（cin）进位标志位（对**无符号**数运算有意义）    | 运算结果的最高有效位向更高位的进位，或从更高位的借位时，CF为1<br />8位有符号数的范围为0-255<br />16位为0~65535（数据最好记住） |
| OF（overflow）溢出标志位（对**有符号数**有意义）   | 当运算结果上溢或者下溢时，OF为1<br />8位有符号数的范围为-128-127<br />16位为-32768~32767 |
| DF（direction）方向标志位                          | 控制串处理指令的方向；<br />DF=0，每次操作后si和di递增<br />DF=1，每次操作后si和di递减<br />cld指令设置DF为0<br />std指令设置DF为1 |

## 各类常用指令

| **指令**    | **功能**                                                     |
| ----------- | ------------------------------------------------------------ |
| adc指令     | 和add指令搭配使用，用于数据位数超过寄存器的位数，用add做低位加法，adc做高位加法 |
| sbb指令     | 带借位的减法指令，利用CF上记录的借位值。可进行高位减法运算。sbb执行后对CF进行设置。 |

| pushf指令   | 将标志寄存器中的所有值入栈                                   |
| popf指令    | 将栈中16位数据弹出，送入标志寄存器中                         |

**关于cmp指令：**

利用标志位看**无符号数**cmp的结果：

| Cmp ax，bx | 标志位                         |
| ---------- | ------------------------------ |
| ax=bx      | ZF=1                           |
| ax>bx      | ZF=0&&CF=0                     |
| ax<bx      | ZF=0&&CF=1                     |
| ax>=bx     | ZF=0或1&&CF=0                  |
| ax<=bx     | （ZF=1&&CF=0）或者(ZF=0&&CF=1) |

实战中的应用：无符号条件跳转指令这套纯粹基于 CF 和 ZF 的逻辑，直接对应了 8086 汇编中处理无符号数的条件跳转指令（Jump Instructions）：

- JE (Jump if Equal): 底层探测 ZF == 1 $\rightarrow$ 对应你的 ax=bx
- JB (Jump if Below): 底层探测 CF == 1 $\rightarrow$ 对应你的 ax<bx
- JA (Jump if Above): 底层探测 CF == 0 且 ZF == 0 $\rightarrow$ 对应你的 ax>bx
- JAE (Jump if Above or Equal): 底层探测 CF == 0 $\rightarrow$ 对应你的 ax>=bx
- JBE (Jump if Below or Equal): 底层探测 CF == 1 或 ZF == 1 $\rightarrow$ 对应你的 ax<=bx

利用标志位看**有符号数**cmp的结果：

判断有符号数cmp的结果主要是看ZF，OF，SF位

利用标志位看**有符号数**cmp的结果：

| **Cmp ax, bx** | **标志位 (结合 SF, OF, ZF)** |
| -------------- | ---------------------------- |
| ax = bx        | ZF = 1                       |
| ax > bx        | ZF = 0 且 SF = OF            |
| ax < bx        | SF != OF                     |
| ax >= bx       | SF = OF                      |
| ax <= bx       | ZF = 1 或 SF != OF           |

### 实战中的应用：有符号条件跳转指令

这套基于 SF、OF 和 ZF 的逻辑，直接对应了 8086 汇编中专门用于处理**有符号数**的条件跳转指令（Jump Instructions）。注意英文单词的变化，无符号用 Above/Below，**有符号用 Greater/Less**：

- **JE (Jump if Equal):** 底层探测 ZF == 1 $\rightarrow$ 对应你的 ax = bx （这个指令无符号和有符号通用，因为判断相等只看结果是不是0）
- **JL (Jump if Less):** 底层探测 SF != OF $\rightarrow$ 对应你的 ax < bx
- **JG (Jump if Greater):** 底层探测 ZF == 0 且 SF == OF $\rightarrow$ 对应你的 ax > bx
- **JGE (Jump if Greater or Equal):** 底层探测 SF == OF $\rightarrow$ 对应你的 ax >= bx
- **JLE (Jump if Less or Equal):** 底层探测 ZF == 1 或 SF != OF $\rightarrow$ 对应你的 ax <= bx

### 💡 核心记忆心法：

- **无符号看 AC (Above/CF)：** 只要是无符号数，就找带 `A` (Above) 和 `B` (Below) 的指令，底层盯死 **CF**。
- **有符号看 GS (Greater/SF/OF)：** 只要是有符号数，就找带 `G` (Greater) 和 `L` (Less) 的指令，底层盯死 **SF 和 OF 的异或关系**。

| OF为0（没有发生溢出） | SF记录的是实际结果的正负         |
| --------------------- | -------------------------------- |
| OF为1（发生了溢出）   | SF记录的结果和实际结果的正负相反 |

### 第一步：先看 ZF（零标志位）—— 判断“是否相等”

这是最简单的一步。

- 如果 **`ZF = 1`**：说明减法结果为 0，也就是 **`AX == BX`**。判断直接结束。
- 如果 **`ZF = 0`**：说明两者不相等，我们接着往下看。

------

### 第二步：深挖 SF 与 OF —— 判断“谁大谁小”

因为我们在做有符号数减法（AX - BX），我们真正关心的是：**减法出来的结果到底是正的还是负的？**

- 如果真实结果是正数，说明 **`AX > BX`**。
- 如果真实结果是负数，说明 **`AX < BX`**。

问题在于，**运算出来的表面符号（SF）并不总是真实的！** 这就是溢出标志位（OF）必须出场当“测谎仪”的原因。

#### 情况 A：没有发生溢出 (`OF = 0`) —— 天下太平

这时候 CPU 算出来的结果是可靠的，`SF` 说什么就是什么。

- 如果 `SF = 0`（表面结果是正数），说明真实结果也是正数。结论：**`AX > BX`**。
- 如果 `SF = 1`（表面结果是负数），说明真实结果也是负数。结论：**`AX < BX`**。

#### 情况 B：发生了溢出 (`OF = 1`) —— 【核心高能区】

什么时候会发生有符号数溢出？只有两种极端情况：**“正减负”** 和 **“负减正”**。 溢出会导致一个可怕的后果：**最高位（符号位）被进位/借位冲刷掉了，导致正数变成了负数，负数变成了正数。**

这就是你笔记里写的：**“SF 记录的结果和实际结果的正负相反”**。我们来推演一下这两种案发现场：

**1. 假设 `OF = 1`，且 `SF = 1`（表面结果为负）**

- *案情还原：* 既然溢出了，表面是负数，说明它溢出前肯定是个极大的**正数**。
- *推理逻辑：* 只有“巨大的正数”减去“巨大的负数”（正 - 负 = 正 + 正）才会变成极大的正数，并最终冲破 16 位的界限，导致符号位变成 1（假负数）。
- *结论：* 既然是正减负，那毫无疑问 **`AX > BX`**！你看，表面是负（SF=1），但结论却是大于。

**2. 假设 `OF = 1`，且 `SF = 0`（表面结果为正）**

- *案情还原：* 溢出后变成了正数，说明它原本是个极深的**负数**。
- *推理逻辑：* 只有“极深的负数”减去“巨大的正数”（负 - 正 = 负 + 负）才会往下穿透底线，溢出翻转为正号（假正数）。
- *结论：* 既然是负减正，那毫无疑问 **`AX < BX`**！表面是正（SF=0），结论却是小于。

## DF标志和串传送指令

flag的第10位是DF,方向标志位。

在串处理指令中,控制每次操作后si,di的增减。

- DF=0:每次操作后si,di递增;
- DF=1:每次操作后si,di递减。

1.串传送字节

格式1:movsb
功能:(以字节为单位传送)

- (1)((es) × 16 +(di)) =  ((ds) × 16 + ((si))

- (2)如果DF = 0则:(si)=(si) +1

  ​    			(di) = (di) + 1

- 如果DF=0则:(si)=(si)-1

  ​    		  (di) = (di) - 1

2.串传送字movsw

格式2:movsw
功能:(以字为单位传送)
将ds:si指向的内存字单元中word送入es:di中,然后根据标志寄存器DF位的值,将si和di递增2或递减2。

movsb和movsw进行的是串传送操作中的个步骤,一般来说,movsb和movsw都和rep配合使用,格式如下:

- rep movsb

  用汇编语法来描述repmovsb的功能就是:

  s:movsb

  loop s

- rep movsw

  用汇编语法来描述repmovsw的功能就是:

  s: movsw

  loop s

- 可见,rep的作用是根据cx的值,重复执行后面的串传送指令。

- 由于每执行一次movsb指令si和di都会递增或递减指向后一个单元或前个单元,则repmovsb就可以循环实现(cx)个字符的传送。书

使用串传送指令进行数据的传送,需要给它提供一些必要的信息,它们是:

1. 传送的原始位置：ds:si
2. 传送的目的位置：es:di
3. 传送的长度:cx
4. 传送的方向:DF。

### 🎯 实战任务：

假设我们有一个包含 16 个字符的字符串 `'Welcome to masm!'`，存放在 `data` 数据段中。我们现在的任务是：**用最高效的方式，把它完整地备份到 `extra` 附加段中。**

### 💻 完整汇编代码：

代码段

```asm
assume cs:code, ds:data, es:extra

; --- 1. 定义源数据区 ---
data segment
    db 'Welcome to masm!'  ; 刚好 16 个字节的源数据
data ends

; --- 2. 定义目标数据区 ---
extra segment
    db 16 dup (0)          ; 预留 16 个字节的空白空间，全填 0
extra ends

; --- 3. 代码区 ---
code segment
start:
    ; 【清单项 1：设置传送的原始位置 DS:SI】
    mov ax, data
    mov ds, ax
    mov si, 0              ; ds:si 指向 'W' 的物理地址
    
    ; 【清单项 2：设置传送的目的位置 ES:DI】
    mov ax, extra
    mov es, ax
    mov di, 0              ; es:di 指向空白区域的首地址
    
    ; 【清单项 3：设置传送的长度 CX】
    mov cx, 8              ; 注意！这里是 8 而不是 16！因为我们要用 movsw (字传送)
    
    ; 【清单项 4：设置传送的方向 DF】
    cld                    ; cld 指令让 DF=0。表示指针每次递增 (正向搬运)
    
    ; 【终极启动指令：一键连发】
    rep movsw              ; 机器开始自动循环 8 次，每次搬运 2 个字节
    
    ; 程序正常退出
    mov ax, 4c00h
    int 21h
    
code ends
end start
```
