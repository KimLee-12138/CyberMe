# 文件与异常处理

## 一句话解释
`with open() as f:`是Python文件操作的标准方式(上下文管理器自动关闭)。读(r)/写(w覆盖)/追加(a)/二进制(b)。JSON用json模块(load/dump)而非eval(安全风险)。`try-except-else-finally`处理异常——else在try成功时执行(不用包在try中)，finally无论如何都执行(清理资源)。

## 核心原理
**文件模式**：r(读/文件必须存在)、w(写/清空已有文件/创建新文件)、a(追加/文件不存在则创建)、r+(读写)、b(二进制)。encoding参数指定编码(默认系统编码，建议显式指定utf-8)。

**上下文管理器(with)**：`__enter__`返回资源，`__exit__`释放资源。即使发生异常也保证__exit__执行。可以自定义：`class MyCtx: def __enter__(self):... def __exit__(self,*a):...`

**JSON**：json.dump(obj,fp)写入文件；json.load(fp)读取。json.dumps(obj)→字符串。注意：Python的tuple→JSON的array(反序列化回来是list)。datetime等非标准类型需要自定义encoder。

**异常层次**：BaseException→Exception→(ValueError/TypeError/KeyError/IndexError/IOError/...)。`except Exception as e:`捕获大多数异常。空except(裸except)会捕获KeyboardInterrupt和SystemExit→不推荐。

## 完整例子
```python
import json
with open('data.json','r',encoding='utf-8') as f:
    data = json.load(f)
try: result = process(data)
except ValueError as e: print(f"Invalid: {e}")
except KeyError: print("Missing key")
else: print("Success") # 没异常才执行
finally: cleanup() # 一定执行
```

## 容易出错的地方
- w模式会清空文件→追加用a

- CSV用csv模块处理(逗号转义、引号)
- json.loads vs json.dumps(带s的是字符串)
- 不要用eval处理外部输入(json.load是安全的替代)
