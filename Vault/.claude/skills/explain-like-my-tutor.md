# explain-like-my-tutor：个性化解释

## 目标

按照 `10_Self/03_学习偏好.md` 中记录的用户偏好，解释一个数据库概念或解答一个问题。每次解释必须注明内容来源和置信度。

## 适用场景

- 用户问「XX 是什么意思」
- 用户要求解释某个数据库概念
- 考试前的概念复习

## 不适用场景

- 需要写代码的问题（用 `write-like-me`）
- SQL 执行/调试（给出可运行代码即可）
- 资料中没有的概念（直接说明未找到）

## 输入

- 必填：用户的问题或要解释的概念
- 可选：解释深度（概述/详细/考前冲刺）

## 前置检查

1. 读取 `10_Self/03_学习偏好.md` 了解用户偏好
2. 读取 `20_University/DB_数据库/00_数据库课程MOC.md` 定位相关笔记
3. 找到对应的知识笔记和章节概览

## 执行步骤

1. **定位**：在知识笔记中搜索相关概念
2. **读取来源**：检查知识笔记引用的原始资料
3. **按偏好解释**：
   - 结构化：一句话解释 → 核心原理 → 例子 → 易错点 → 关系
   - 有例子：每个抽象概念配具体例子
   - 有易错点：指出常见错误
4. **标注来源**：每段标明来自哪个文件/笔记
5. **标注置信度**：
   - 「根据课件」→ 来自原始资料
   - 「根据知识笔记」→ 来自蒸馏笔记（待核对）
   - 「以下为推断」→ AI 推理

## 输出

- 文本：按偏好格式的解释
- 末尾：依据文件清单 + 置信度说明

## 允许操作

- 读取知识笔记、提取版、MOC、学习偏好
- 引用笔记内容

## 禁止操作

- 编造资料中没有的定义或例子
- 把 AI 推断说成资料内容
- 不注明来源直接给结论

## 需要人工确认的操作

- 资料之间存在矛盾时（列出冲突，不自行选择）

## 失败处理

- 概念在知识库中找不到 → 明确说「当前资料中未找到」，不补写
- 学习偏好文件为空 → 使用默认结构化格式

## 验收标准

- 每个论断都有来源标注
- 区分了「资料事实」和「AI 推断」
- 符合用户偏好的解释结构
