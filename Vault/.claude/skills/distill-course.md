# distill-course：知识蒸馏

## 目标

从 `status: verified` 的提取版材料中，按统一模板生成正式知识笔记，并更新课程 MOC。

## 适用场景

- 提取版已完成人工核对（`status: verified`）
- 需要为新章节批量生成知识笔记

## 不适用场景

- 提取版仍为 `extracted` 或 `cleaned`（未经人工核对）
- 没有提取版材料的全新主题

## 输入

- 必填：提取版 Markdown 文件路径（`status` 必须为 `verified`）
- 必填：目标课程（如 DB）
- 可选：建议的知识点拆分方案

## 前置检查

1. 提取版的 `status` 是否为 `verified`
2. 是否已有同主题的正式笔记（避免重复）
3. 提取版中的 `review_reason` 是否已解决

## 执行步骤

1. **读取**：加载提取版内容，识别其中的概念、定义、公式、例子
2. **拆分**：将提取版内容拆分为独立的知识单元（一个概念一篇笔记）
3. **生成草稿**：按 `80_Templates/提取版Markdown模板.md` 的「正式知识点模板」格式生成每篇笔记
4. **填写元数据**：每篇设置 `status: cleaned`、`needs_review: true`、`verification: inferred`
5. **建立链接**：每篇新笔记先添加一条指向所属课程 `00_*_课程MOC.md` 的直接链接，再添加前置知识、对比概念、用于解决等关系链接
6. **更新 MOC**：在课程 MOC 中添加新笔记的索引和链接

## 输出

- 新增文件：正式知识笔记（`02_知识点/`）+ 更新后的 MOC
- 报告：新增笔记清单、链接关系图、待人工核对项

## 允许操作

- 读取 `verified` 状态的提取版
- 创建新的知识笔记
- 更新 MOC

## 禁止操作

- 从未经核对的提取版生成正式笔记
- 将 AI 生成内容标为 `verified`
- 覆盖已有笔记（只新增，冲突时标记并报告）

## 需要人工确认的操作

- 知识点拆分方案（生成前先展示计划）
- 超过 10 篇的批量生成
- 与已有笔记存在主题重叠时

## 失败处理

- 提取版不是 `verified` → 停止，提示用户先核对
- 提取版内容不足以拆分 → 生成一篇综合笔记并标记内容不足
- 公式/表格无法确认 → 保留原文并标记 `review_reason`

## 验收标准

- 每篇笔记来源字段完整
- 每篇新笔记都直接链接所属课程 MOC，课程 MOC 已被知识库总地图收录
- 所有笔记 `needs_review: true`
- 无未经核对的提取版被蒸馏
