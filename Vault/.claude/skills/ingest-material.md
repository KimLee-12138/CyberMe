# ingest-material：资料导入

## 目标

将一份原始资料走完「登记 → 复制到归档区 → 提取文本 → 标记待核对」的完整流程。

## 适用场景

- 用户提供一个新的课程文件（PDF/PPT/DOCX/MD）
- 阶段三的渐进归档批次处理

## 不适用场景

- ZIP 压缩包（需先用 `list-zip` 列清单）
- 已确认包含敏感信息的文件
- 已经在 `91_Raw-Archive` 中的文件（先检查哈希去重）

## 输入

- 必填：原始文件绝对路径
- 可选：课程编号（如 DB）、主题标签、优先级（默认 P0）

## 前置检查

1. 文件是否存在且可读
2. 哈希是否已存在于 `91_Raw-Archive/Registry/`（避免重复导入）
3. 文件名是否包含敏感关键词（密码/账号/身份证等）
4. 文件类型是否有对应的提取工具

## 执行步骤

1. **登记**：计算 SHA-256，检查是否与已有文件重复
2. **判断**：确定优先级（P0-P3）、敏感性（normal/sensitive/unknown）、文本可提取性
3. **复制**：将文件复制到 `91_Raw-Archive/{类型}/`，按命名规范重命名
4. **验证**：复制后重新计算哈希，与源文件比对
5. **提取**：根据文件类型提取文本，保存到 `90_课程提取资料/`
6. **标记**：提取版设置 `status: extracted`、`needs_review: true`
7. **记录**：更新 `变更日志.md` 和归档登记卡

## 输出

- 新增文件：归档副本（`91_Raw-Archive/`）、提取版 Markdown（`90_课程提取资料/`）、登记卡条目
- 修改文件：`01_Home/变更日志.md`
- 报告：操作摘要（文件名、哈希、大小、是否重复、提取字数）

## 允许操作

- 读取原始文件
- 复制到 `91_Raw-Archive/`
- 创建提取版 Markdown
- 更新 Registry 和变更日志

## 禁止操作

- 移动、重命名或删除原始文件
- 覆盖已有归档文件
- 在提取版中标 `verified`（只能标 `extracted`）
- 解压 ZIP 到归档区

## 需要人工确认的操作

- 标记为 `sensitive` 的文件
- 旧版 Office 格式（`.doc`、`.ppt`）提取失败时
- OCR 需求判断

## 失败处理

- 文件读取失败 → 停止，报告错误
- 提取工具不支持 → 标记为「需手动提取」，不生成空提取版
- 哈希冲突 → 标记为「完全重复」，只更新登记卡（记录原始路径），不重复归档
- 磁盘空间不足 → 停止，不部分执行

## 验收标准

- 归档副本哈希与源文件一致
- 提取版包含页码或幻灯片定位
- 源文件未被改动
- 变更日志已更新
