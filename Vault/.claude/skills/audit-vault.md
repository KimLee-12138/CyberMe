# audit-vault：Vault 审计

## 目标

全面检查 Vault 的完整性、安全性和合规性，输出问题清单和修复建议。

## 适用场景

- 每月评测前
- 怀疑知识库有问题时
- 备份恢复演练后验证

## 不适用场景

- 日常小范围检查（手动抽查即可）

## 输入

- 必填：审计范围（全部 / 仅数据库课程 / 仅个人模型）
- 可选：重点关注项（来源完整性/链接/安全/推断混入）

## 前置检查

1. 确认 Vault 可正常打开
2. 确认 Git 工作区干净

## 执行步骤

1. **来源完整性检查**：
   - 扫描所有正式知识笔记的 `source` 字段
   - 标记空来源或「待补充来源」
2. **断链检查**：
   - 扫描所有 `[[链接]]`，检查目标文件是否存在
   - 标记死链接
3. **孤立笔记检查**：
   - 找出未被 MOC 或任何其他笔记链接的笔记
   - 检查每篇课程笔记是否直接链接所属课程 MOC
   - 检查每个课程 MOC 是否被 `01_Home/知识库总地图.md` 收录
4. **重复 ID 检查**：
   - 扫描所有笔记的 `id` 字段，找出重复
5. **推断混入检查**：
   - 扫描 `verification: explicit` 的文件，检查是否包含「推断」「可能」「似乎」等不确定表述
6. **敏感信息检查**：
   - 扫描所有文件（排除 .gitignore 中的类型），查找密码/Token/身份证号/手机号模式
7. **Git 检查**：
   - 确认 Git 未跟踪 PDF/ZIP/密钥等排除类型
8. **生成报告**：问题按严重程度分级，给出修复建议

## 输出

- 新增文件：`70_Evaluation/Error-Logs/audit-YYYYMMDD.md`
- 报告内容：检查项、通过数、问题清单、修复建议

## 允许操作

- 读取 Vault 中所有 Markdown 文件
- 读取 Git 状态
- 生成审计报告

## 禁止操作

- 自动修复任何问题（只报告，不修改）
- 删除或移动文件

## 需要人工确认的操作

- 所有修复操作
- 敏感信息的判断（可能误报）

## 失败处理

- 某检查项无法完成 → 标记为「未检查」并说明原因

## 验收标准

- 覆盖全部 7 个检查维度
- 每个问题有文件位置和具体描述
- 修复建议具体可执行
