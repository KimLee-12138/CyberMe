# 代理权限矩阵

> 适用于 Claude Code 和 Codex。操作按风险分为四级。

## 一级：只读操作（无需确认）

| 操作 | 说明 |
|------|------|
| 读取 Vault 中的 Markdown 文件 | 知识笔记、MOC、配置、模板 |
| 读取 `AGENTS.md`、`CLAUDE.md` | 规则文件 |
| 读取 `10_Self/` | 个人模型（学习偏好、写作风格等） |
| 搜索文件内容（Grep/Glob） | 检索操作 |
| 查看 Git 状态和日志 | `git status`、`git log` |

## 二级：低风险编辑（需确认）

| 操作 | 说明 |
|------|------|
| 创建新的知识笔记 | `02_知识点/` 中新增文件 |
| 更新 MOC 链接 | 添加新笔记的索引 |
| 更新变更日志 | `01_Home/变更日志.md` |
| 更新当前任务 | `01_Home/当前任务.md` |
| 创建提取版 Markdown | `90_课程提取资料/` 中新增 |
| 更新个人模型中的待确认文件 | `10_Self/` 中标记 `inferred` 的文件 |
| 创建测试题 | `70_Evaluation/` 中新增 |

## 三级：需确认的批量操作（先计划后执行）

| 操作 | 说明 |
|------|------|
| 批量创建 >3 篇知识笔记 | 先展示计划 |
| 批量修改 >10 个文件 | 先展示修改清单 |
| 归档知识笔记到 `99_Archive/` | 需说明原因 |
| 更新 `AGENTS.md` 或 `CLAUDE.md` | 影响代理行为 |
| 创建或修改 Skill 文件 | `.claude/skills/` |
| 修改 YAML frontmatter 元数据 | 可能影响检索和链接 |

## 四级：禁止操作（绝对不允许）

| 操作 | 说明 |
|------|------|
| 覆盖或删除 `91_Raw-Archive/` 中的文件 | 原始证据不可变 |
| 移动或重命名 `学习资料/` 中的文件 | 源目录不可触碰 |
| 将 AI 推断标为 `verification: explicit` | 事实与推断必须分离 |
| 将未核对内容标为 `verified` | 只能人工确认 |
| 保存密码、密钥、Token、证件号 | 敏感信息不入库 |
| 绕过权限检查 | `dangerouslyDisableSandbox` 等 |
| 编造实验数据、结果、引用、个人经历 | 严禁虚构 |
| 未经确认删除知识文件 | 移到 `99_Archive/` 而非直接删除 |

## 紧急例外

如果操作符合以上禁止项但确实是用户明确要求的：
1. 先警告用户该操作违反 AGENTS.md 规则
2. 等待用户二次确认
3. 在变更日志中记录例外情况和用户确认
