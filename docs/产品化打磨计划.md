# CyberMe OS 产品化打磨计划

> 目标：从"能用"打磨到"好用"——面向中文使用者的产品级个人学习操作系统。

---

## 总览

项目目前功能骨架完整，但存在六类问题：

| 类别 | 严重程度 | 问题数 |
|------|----------|--------|
| 英文文案残留 | 高 | 15+ |
| 零操作反馈 | 高 | 所有 mutation |
| 加载/错误/空状态粗糙 | 高 | 12 个页面 |
| 功能残缺 | 中 | 5 处死按钮/占位符 |
| 交互细节缺失 | 中 | 10+ 处 |
| 视觉样式不统一 | 低 | 全局 |

### 1.1 统一枚举映射表

**文件**：新建 `apps/web/src/lib/enums.ts`

所有 API 返回的英文枚举值统一在此映射为中文，通过 `xxxLabel(key)` 函数调用。

```ts
// 掌握程度

// 重要性

// 文档状态
DOC_STATUS: extracted→已提取, cleaned→已清洗, active→活跃, resolved→已解决, archived→已归档

// 文档类型
DOC_TYPE: concept→概念, extract→摘录, formula→公式, procedure→流程, reflection→反思

// 设备平台

// 复习评分
```

**使用规范**：页面不再写裸 `{value}`，改用 `{xxxLabel(value)}`。找不到映射时回退显示原文。

### 1.2 Toast 通知系统

**文件**：新建 `apps/web/src/stores/toastStore.ts` + `apps/web/src/components/ui/Toast.tsx`

**目的**：创建、更新、删除等操作完毕后给用户明确反馈。目前零反馈，操作完了用户不知道成没成功。

**设计**：

```
┌─────────────────────────────────┐
│  ✓  保存成功                      │  ← 绿色，3秒后自动消失
│  ✗  删除失败，请重试               │  ← 红色
│  ℹ  正在生成提案...               │  ← 蓝色
└─────────────────────────────────┘
     右上角固定，最多堆叠 3 条
```

**API（在 React 组件内）**：
```tsx
import { useToastStore } from '@/stores/toastStore'
const addToast = useToastStore((s) => s.add)
addToast('success', '保存成功')
```

**API（在非 React 环境，如 mutation callbacks）**：
```ts
import { toast } from '@/stores/toastStore'
toast.success('创建成功')
toast.error('操作失败，请重试')
toast.info('正在处理...')
```

**特性**：
- 右上角 `fixed` 定位，`z-50`
- 自带进度条动画（3 秒）
- 点击可提前关闭
- success 绿色、error 红色、info 蓝色
- 使用 Tailwind 主题色，自动适配暗色模式

**集成**：`App.tsx` 根节点挂载 `<ToastContainer />`

### 1.3 统一状态组件

**文件**：新建 `apps/web/src/components/ui/LoadingState.tsx`、`ErrorState.tsx`、`EmptyState.tsx`

**目的**：目前 12 个页面的加载、错误、空状态各自实现，有的只写三个字"加载中..."，有的完全没有。统一后所有页面用同一套组件。

#### LoadingState

```
┌──────────────────────────────┐
│                              │
│     ████████████████         │  ← 骨架屏（灰色脉冲条）
│     ██████████               │
│     ████████████████████     │
│                              │
│        加载中...              │  ← 可选文字
│                              │
└──────────────────────────────┘
```

Props：`message?: string`（默认"加载中..."）、`lines?: number`（骨架条数，默认 3）

#### ErrorState

```
┌──────────────────────────────┐
│                              │
│           ⚠️                 │  ← 大图标
│     加载失败，请重试           │  ← 错误描述
│     ┌──────────┐             │
│     │  重新加载  │             │  ← 重试按钮
│     └──────────┘             │
└──────────────────────────────┘
```

Props：`message?: string`、`onRetry?: () => void`

#### EmptyState

```
┌──────────────────────────────┐
│                              │
│           📭                 │
│       暂无数据                │
│     点击上方按钮创建第一条      │  ← 引导文字
│                              │
└──────────────────────────────┘
```

Props：`icon?: string`（默认 📭）、`title?: string`、`description?: string`、`action?: { label: string; onClick: () => void }`

```ts
import { useEffect } from 'react'
export function useTitle(title: string) {
  useEffect(() => {
    const prev = document.title
    document.title = title ? `${title} — CyberMe OS` : 'CyberMe OS — 个人学习操作系统'

}
```

### 4.2 ChatPage 引用实时存储

**问题**：`ChatPage.tsx:136-140`，SSE `citations` 事件收到后注释写着 `// Save to the current streaming message context` 但没有实际存储逻辑，引用数据在流式过程中被丢弃。

**修复方案**：
```ts
// 当前（空壳）
es.addEventListener('citations', (_e) => {
  try { /* Save to the current streaming message context */ }
  catch { /* ignore */ }
})

// 改为（实际存储）
es.addEventListener('citations', (e) => {
  try {
    const data = JSON.parse((e as MessageEvent).data)
    setStreamingCitations((prev) => [...prev, ...data.citations])
  } catch { /* ignore */ }
})
```

并在消息显示时实时渲染引用标记。
